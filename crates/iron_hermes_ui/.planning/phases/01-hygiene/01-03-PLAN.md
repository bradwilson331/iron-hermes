| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | Cargo.lock (supply chain) | accept | Cargo.lock pins exact transitive dependency versions and checksums. Any tampering with Cargo.lock is caught by `cargo build` checksum verification. The `=0.7.1` pin in Cargo.toml (from Plan 01) ensures the lock file resolves to the known-good version. This is the standard Rust supply chain mitigation. No further action required for v1. |
| T-01-03-02 | — | git stage operation | accept | `git add Cargo.lock` is a local VCS operation with no network exposure. No security attack surface beyond standard file system access controls. |
