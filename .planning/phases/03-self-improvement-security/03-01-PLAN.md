    2. **Add `write_file_atomic` helper** — reusable atomic write function for context files (SELF-06, matching RESEARCH.md Open Question 3 resolution):
       ```rust
       /// Atomic write for context files: tempfile + fsync + rename.
       /// Prevents partial writes that could corrupt identity/memory files.
       /// Same pattern as MemoryStore (D-08) and ironhermes-cron, with fsync added.
       fn write_file_atomic(path: &std::path::Path, content: &[u8]) -> anyhow::Result<()> {
           use std::io::Write;
           let tmp_path = path.with_extension("tmp");
           {
               let mut f = std::fs::File::create(&tmp_path)?;
               f.write_all(content)?;
               f.flush()?;
               f.sync_all()?;  // fsync before rename for durability
           }
           std::fs::rename(&tmp_path, path)?;
           Ok(())
       }
       ```

| LLM -> file tools | Agent-generated content crosses into filesystem writes |
| context files -> system prompt | Files on disk are loaded into the LLM prompt next session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | WriteFileTool | mitigate | scan_context_content() blocks writes containing 10 threat patterns + invisible unicode (D-01) |
| T-03-02 | Tampering | PatchFileTool | mitigate | Scan full post-patch content, not just replacement string — prevents split-payload injection |
| T-03-03 | Elevation | context files | mitigate | Only SOUL.md/AGENTS.md/MEMORY.md/USER.md are scanned (D-02) — these are the files that become prompt content |
| T-03-04 | Spoofing | context_scanner | accept | RegexSet can be bypassed with novel injection patterns not in the 10-pattern set; acceptable because patterns match known attack vectors and can be extended |
| T-03-14 | Tampering | context file partial write | mitigate | Atomic writes (tempfile+fsync+rename) for all context files prevent partial-write corruption of SOUL.md/AGENTS.md (SELF-06) |
</threat_model>
