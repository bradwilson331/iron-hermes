       ```rust
       use std::collections::HashMap;
       use std::io::Write;
       use std::path::{Path, PathBuf};
       use fs2::FileExt;
       use tracing::warn;
       use crate::constants::*;
       use crate::context_scanner::scan_context_content;
       ```

       **`MemoryTarget` enum**: `Memory` and `User` variants, each mapping to filename and char limit.

       **`MemoryResult` type**: `Result<String, String>` — Ok contains JSON success message, Err contains JSON error message. Both are JSON strings for the tool to return directly.

       **`MemoryStore` struct**:
       - `entries: HashMap<MemoryTarget, Vec<String>>` — live entries (disk-authoritative for mutations)
       - `snapshot: HashMap<MemoryTarget, String>` — frozen at `load_from_disk()`, never mutated (D-12)
       - `memory_dir: PathBuf`

       **`MemoryStore::new(memory_dir: PathBuf) -> Self`**: Creates empty store. Creates `memory_dir` if it doesn't exist.

       **`MemoryStore::load_from_disk(&mut self) -> anyhow::Result<()>`**: Reads MEMORY.md and USER.md from `memory_dir`, splits by `ENTRY_DELIMITER`, stores in `entries`, captures formatted snapshot in `snapshot` for prompt injection (D-12). If files don't exist, entries are empty.

       **`MemoryStore::add(&mut self, target: MemoryTarget, content: &str) -> MemoryResult`**:
       - Re-read from disk under file lock (D-07, per Python reference lines 198-200)
       - Scan content with `scan_context_content(content, target.filename())` — if blocked, return error (D-13)
       - Check for exact duplicate — return error if found (D-14)
       - Check char capacity: sum of all entry chars + delimiter chars + new content chars > limit — return error with current usage and entry list (D-15)
       - Append entry, write atomically (D-08), release lock
       - Return JSON: `{"status": "added", "target": "memory", "entries": N, "chars_used": M, "chars_limit": L}`

       **`MemoryStore::replace(&mut self, target: MemoryTarget, old_text: &str, new_content: &str) -> MemoryResult`**:
       - Re-read from disk under file lock
       - Scan new_content with `scan_context_content` (D-13)
       - Find entries containing `old_text` — if 0 matches: error "no entry found"; if 2+ matches: error "multiple entries match, use more specific text" (D-10)
       - Replace the matched entry with `new_content`
       - Check capacity after replacement
       - Write atomically, release lock
       - Return JSON success

       **`MemoryStore::remove(&mut self, target: MemoryTarget, old_text: &str) -> MemoryResult`**:
       - Re-read from disk under file lock
       - Find entries containing `old_text` — same uniqueness check as replace (D-10)
       - Remove the matched entry
       - Write atomically, release lock
       - Return JSON: `{"status": "removed", ...}`

       **`MemoryStore::format_for_system_prompt(&self, target: MemoryTarget) -> Option<String>`**:
       - Returns the frozen `snapshot` value (D-12), not live entries
       - Format: `"## Memory\n\n{entries joined by newlines}"` for Memory, `"## User Profile\n\n{entries}"` for User
       - Returns `None` if snapshot is empty

       **Private helpers**:
       - `fn reload_target(&mut self, target: MemoryTarget) -> anyhow::Result<()>` — reads file, splits by delimiter, updates `self.entries[target]`
       - `fn write_target_atomic(&self, target: MemoryTarget) -> anyhow::Result<()>` — joins entries with `ENTRY_DELIMITER`, writes to temp file, fsync, rename (D-08, with fsync matching Python hermes-agent)
       - `fn with_file_lock<F, R>(path: &Path, f: F) -> anyhow::Result<R>` — opens `.md.lock` sidecar file, acquires exclusive flock via `fs2::FileExt::lock_exclusive()`, runs closure, unlocks (D-07)
       - `fn char_count(entries: &[String], delimiter: &str) -> usize` — total chars including delimiters between entries

         ```json
         {
           "type": "object",
           "properties": {
             "action": {
               "type": "string",
         ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM -> memory tool | Agent-generated content enters persistent memory store |
| memory files -> system prompt | MEMORY.md/USER.md loaded into prompt at session start |
| concurrent sessions -> memory file | Multiple sessions may write to same memory file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | MemoryStore.add | mitigate | scan_context_content() blocks injection patterns before persisting (D-13) |
| T-03-06 | Tampering | MemoryStore.replace | mitigate | Replacement content scanned before persisting |
| T-03-07 | Denial | MemoryStore capacity | mitigate | Bounded char limits (2200/1375) prevent unbounded growth (D-15) |
| T-03-08 | Tampering | concurrent writes | mitigate | fs2 advisory flock on .lock sidecar file prevents concurrent corruption (D-07) |
| T-03-09 | Information Disclosure | memory snapshot | accept | Memory is injected into prompt — by design the agent sees its own memory. No cross-user risk in single-operator deployment. |
</threat_model>
