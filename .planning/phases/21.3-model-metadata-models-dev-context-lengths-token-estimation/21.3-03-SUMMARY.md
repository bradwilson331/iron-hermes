## Decisions Made
- Pure parse functions take serde_json::Value for testability without requiring network access -- all 14 tests use sample JSON fixtures
- OpenRouter entries override models.dev entries for the same canonical key because OpenRouter provides richer tokenizer data (models.dev has no tokenizer field)
- strip_date_suffix detects -YYYYMMDD pattern at end of string; both versioned and unversioned keys stored from OpenRouter responses so alias lookups work
- load_from/save_to path-parameterized variants added alongside load/save for testable disk I/O without touching real ~/.ironhermes/ directory
- map_openrouter_tokenizer checks model ID for "4o", "o3", "o4", "4.1" to select o200k_base for newer GPT models; all others get cl100k_base per D-08 fallback

## Threat Model Compliance

All three threat mitigations from the plan's threat model are implemented:

- **T-21.3-05 (Tampering - cache file):** Cache loaded via serde_json into typed ModelsCacheEntry struct with unwrap_or_default on malformed JSON. Only declared fields populated. Test confirms malformed JSON returns empty cache.
- **T-21.3-06 (Spoofing - API responses):** Responses deserialized via serde_json::Value with typed field extraction (.as_u64(), .as_bool(), .as_str()). Unknown fields ignored. URLs are hardcoded constants (MODELS_DEV_URL, OPENROUTER_MODELS_URL). reqwest uses HTTPS.
- **T-21.3-07 (Info Disclosure - API key):** OPENROUTER_API_KEY sent only to hardcoded OPENROUTER_MODELS_URL via reqwest bearer_auth. Key not stored in FetchResult or ModelsCache.
