## Model metadata source

| Option | Description | Selected |
|--------|-------------|----------|
| Static lookup table (Recommended) | Hardcoded HashMap with known models. Fast, no network. Updated each release. User config overrides. | |
| API fetch at startup | Query OpenRouter /models or models.dev at startup, cache. Always current but adds latency. | |
| Config-only | Users must specify context_length in config.yaml. Simplest but worst UX. | |
| Static lookup table with manual API fetch | (User's own option) Static table + optional manual API fetch on user request. | ✓ |

**User's choice:** Static lookup table with manual API fetch at user request
**Notes:** Combines reliability of static data with ability to update via API when needed.
