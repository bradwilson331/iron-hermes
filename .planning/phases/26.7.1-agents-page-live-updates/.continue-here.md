# BLOCKING CONSTRAINTS — Read Before Anything Else

> These were discovered through failure during the Wave 1 UAT iteration cycle (4 hotfix rounds).
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: **WarpHermes is dead code** — `iron_hermes_ui/src/components/warp_hermes.rs` only mounts under `--features legacy-shell` (opt-in). The active root is `HermesApp` (`#[cfg(not(feature = "legacy-shell"))]`). Targeting WarpHermes for any provider/ws integration causes SSR to panic with `Could not find context dioxus_signals::signal::Signal<bool>`. The first attempt of this phase shipped 10 commits doing exactly that and had to be fully reverted. **Structural mitigation:** every plan file in this phase has been authored to target `hermes_app/mod.rs` directly; PATTERNS.md marks `warp_hermes.rs` as a forbidden target. Do not edit `warp_hermes.rs`.

- [ ] CONSTRAINT: **UAT must run from worktree path, not main repo** — `cd /Users/twilson/code/ironhermes/.claude/worktrees/agent-<id>/crates/iron_hermes_ui && RUSTC_WRAPPER= dx serve`. The main repo's checkout does NOT have the worktree's commits until they merge. Running `dx serve` from the main repo tests the unmerged base branch and "UAT failure" reports become noise. The first user UAT on Wave 1 wasted a cycle on this.

- [ ] CONSTRAINT: **wasm32 + native must both compile after every Rust task** — `cargo check -p iron_hermes_ui --features web` (native) AND `cargo check -p iron_hermes_ui --features web --target wasm32-unknown-unknown` (wasm) gate every commit. Native-only checks miss `JsCast`/`unchecked_ref` scope bugs; wasm-only checks miss `use_drop` import bugs that resolve via different paths.

- [ ] CONSTRAINT: **`std::time::Instant` panics on wasm32** — "time not implemented on this platform" at `library/std/src/sys/pal/wasm/.../time.rs:13`. Use `web_time::Instant` instead. `web-time = "1.1"` is already in `iron_hermes_ui/Cargo.toml`. It's `pub use std::time::Instant` on native and a `Performance.now()`-backed shim on wasm32 — same API.

- [ ] CONSTRAINT: **`use_drop` is in `dioxus::core`, not `dioxus::prelude`** — `use dioxus::core::use_drop;`. The prelude does not re-export it on the native target. Project convention in `warp_hermes.rs:2` and `wheel.rs:32`.

- [ ] CONSTRAINT: **`JsCast` trait must be in scope for `.unchecked_ref()` on `&JsValue`** — `use wasm_bindgen::JsCast as _;` (often gated `#[cfg(target_arch = "wasm32")]`). Without it, native build rejects with E0599 even though wasm32 compiles.

- [ ] CONSTRAINT: **Dioxus signal writes during render cause infinite loops if the value doesn't change** — `signal.set(new)` schedules a re-render whether or not `new` equals the current value. Any signal write in a component render body must be gated on actual change: `if *signal.read() != new { signal.set(new) }`. The current `prev_live.set()` in `ScreenAgents` is gated this way (commit `5d2f51ca`).

- [ ] CONSTRAINT: **`use_server_future(...)?` early-returns on suspended state, killing hook registrations** — every render in suspended state short-circuits before reaching `use_signal`/`use_future`/`use_effect` below. After `agents_resource.restart()` puts the resource back into `None`, the next render returns early, and the poll loop's `use_future` never gets registered again. Use `use_resource` instead (no suspension). `agents.rs:37` already uses `use_resource`.

- [ ] CONSTRAINT: **Signal-borrow discipline (clippy.toml)** — never hold `.read()`/`.write()` guards across `.await`. Read into Copy locals before any `.await`, drop the borrow with `;`. clippy enforces this on every commit.

**Do not proceed until all boxes are checked.**
