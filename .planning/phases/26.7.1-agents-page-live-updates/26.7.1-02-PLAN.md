<objective>
Plan 02 layers push-driven sub-second updates on top of Plan 01's polling baseline by reusing the existing chat WebSocket, AND wires the dynamic poll cadence (5 s while ws is up, 1500 ms while down).

Three integration moves, in order:

1. **Protocol** — add a payload-free `ChatStreamEvent::SubagentEvent {}` variant in `protocol.rs` plus a serde-round-trip unit test (Wave 0 of Plan 02).
2. **Server (Path A — global slot per researcher's Open Question 2 recommendation)** — `DelegateTaskWiring.progress_callback` is baked into the singleton `AppRuntimeBundle` at `AppState::init`. We cannot swap it per-turn. Instead, install a `subagent_callback_slot: Arc<tokio::sync::Mutex<Option<mpsc::UnboundedSender<ChatStreamEvent>>>>` on `AppState`. The bundle's `progress_callback` (constructed once at init) locks the slot, reads the current sender (if any), and forwards. `ws.rs` installs a per-turn sender into the slot RIGHT BEFORE `run_web_turn` and clears it after — using a RAII guard so the slot is cleared even if `run_web_turn` panics. For the v1 single-session web UI this acceptably broadcasts events from any session to the connected ws client; multi-session isolation is explicitly out of scope per CONTEXT.md.
3. **Client** — extend HermesApp's recv loop (active root — never WarpHermes) with a `SubagentEvent` match arm that increments the `subagent_events` signal Plan 01 created; add `is_ws_connected.set(true)` / `.set(false)` calls in the existing connect/close/err arms. In `ScreenAgents`, add a `use_effect` over `subagent_events()` that calls `agents_resource.restart()`. The poll loop's cadence read of `is_ws_connected` (already wired in Plan 01) now sees true/false transitions and the cadence automatically toggles between 5000 ms and 1500 ms.

Purpose: drop subagent-appearance latency from "next poll boundary" (up to 1.5 s) to <1 s, regardless of how short-lived the subagent is. The poll loop becomes the safety net for missed/dropped events rather than the primary refresh mechanism.

Output: protocol variant + test, server-side Path A plumbing, HermesApp recv-loop extension, ScreenAgents use_effect watcher. Plan 01's HOLD/decay/CSS/poll-loop are untouched.
</objective>

```
#[cfg(feature = "server")]
struct SubagentCallbackSlotGuard {
    slot: std::sync::Arc<
        tokio::sync::Mutex<
            Option<tokio::sync::mpsc::UnboundedSender<crate::protocol::ChatStreamEvent>>,
        >,
    >,
}

#[cfg(feature = "server")]
impl Drop for SubagentCallbackSlotGuard {
    fn drop(&mut self) {
        // Best-effort clear. Use blocking_lock since Drop cannot await.
        // The slot is held only across very short windows; contention is
        // not expected outside of pathological cases.
        if let Ok(mut guard) = self.slot.try_lock() {
            *guard = None;
        }
        // If try_lock fails (extremely unlikely — only the callback's
        // try_lock contends, and it doesn't hold the lock across .send),
        // we leak a stale Some(tx) until the next turn overwrites it. The
        // closed channel makes any further send a silent no-op. Acceptable.
    }
}
```

```
    use std::sync::Arc;

    /// Phase 26.7.1 Plan 02 (Wave 0): D-06 callback wiring shape.
    /// Mirrors the callback constructed in state.rs Task 2: lock the slot,
    /// read Some(tx), send ChatStreamEvent::SubagentEvent {}.
    #[tokio::test(flavor = "multi_thread", worker_threads = 2)]
    async fn test_subagent_callback_emits_event() {
        let (tx, mut rx) = mpsc::unbounded_channel::<ChatStreamEvent>();
        let slot: Arc<Mutex<Option<mpsc::UnboundedSender<ChatStreamEvent>>>> =
            Arc::new(Mutex::new(Some(tx)));
        let cb_slot = slot.clone();
        let cb: SubagentProgressCallback = Arc::new(move |_index: usize, _event: SubagentProgress| {
            if let Ok(guard) = cb_slot.try_lock() {
                if let Some(s) = guard.as_ref() {
                    let _ = s.send(ChatStreamEvent::SubagentEvent {});
                }
            }
        });

        // Invoke the callback as the delegate-task runner would.
        cb(0, SubagentProgress::Completed);

        let received = rx.recv().await.expect("expected SubagentEvent");
        assert!(

            "callback must send the SubagentEvent variant"
        );

        // After clearing the slot, the callback becomes a silent no-op.
        {
            let mut g = slot.lock().await;
            *g = None;
        }
        cb(1, SubagentProgress::Completed);
        // Nothing should arrive — give the runtime a moment to surface anything.
        let timed = tokio::time::timeout(std::time::Duration::from_millis(50), rx.recv()).await;
        assert!(timed.is_err(), "no events should be received after slot is cleared");

}
```
