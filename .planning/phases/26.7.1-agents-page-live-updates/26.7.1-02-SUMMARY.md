## Path A Design Confirmed

`AppState::subagent_callback_slot` is the central routing hub:

```
SubagentProgress event (any variant)
  → progress_callback (Arc<dyn Fn>, baked into AppRuntimeBundle at init)
    → try_lock(subagent_callback_slot)
      → if Some(tx): tx.send(SubagentEvent {})
        → rx in tokio::spawn turn block → ws send_raw(Text) → browser
          → HermesApp recv loop SubagentEvent arm
            → subagent_events.set(cur + 1)
              → ScreenAgents use_effect fires
                → agents_resource.restart()
```

vs.

```
ws.rs tokio::spawn: lock().await → *guard = Some(tx.clone())
                    _slot_guard dropped at end of block → None
```

No changes to `run_web_turn` signature. Single-session broadcast acceptable per CONTEXT.md.

## Known Limitations (Carried Forward per CONTEXT.md)

- **Single-session broadcast (T-26.7.1-02-01):** All subagent events from any in-flight turn route to whichever ws client currently holds the slot. Multi-session isolation explicitly deferred.
- **try_lock dropped events (T-26.7.1-02-02):** Under extreme contention, `try_lock` may drop an event. The poll loop's 5s safety net ensures eventual consistency. No telemetry on drops (T-26.7.1-02-04 — deferred).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] test_subagent_callback_emits_event: timed.is_err() insufficient assertion**
- **Found during:** Task 4 test execution
- **Issue:** Plan's test code used `assert!(timed.is_err(), ...)` but `tokio::time::timeout(50ms, rx.recv())` returns `Ok(None)` when the channel is closed (all senders dropped), not `Err(Elapsed)`. Clearing the slot drops the tx from the slot, closing the channel — so `rx.recv()` returns `None` immediately, making `timed` be `Ok(None)` (not an error), falsely failing the assertion.
- **Fix:** Changed assertion to `match timed { Err(_) => true, Ok(None) => true, Ok(Some(_)) => false }` — both timeout and channel-closed are valid "no spurious event" outcomes.
- **Files modified:** `crates/iron_hermes_ui/src/server/ws.rs`
- **Commit:** `16f8b8ec`

**2. [Pattern Decision] Tasks 1 and 5 combined to avoid wildcard match arms**
- **Found during:** Task 1 build gate
- **Issue:** Adding `ChatStreamEvent::SubagentEvent {}` to protocol.rs immediately caused non-exhaustive match errors in `hermes_app/mod.rs` (line 183). Plan acknowledged this: "Prefer to land Tasks 1 + 5 in the same compile unit — but if you ship Task 1 alone, the wildcard arms are the cleanest tooling."
- **Fix:** Added SubagentEvent arm + is_ws_connected lifecycle sets in mod.rs in the same commit as protocol.rs + warp_hermes.rs. No wildcard arms needed.
- **Commit:** `34f7c625`
