**Core poll loop pattern** (synthesis from RESEARCH.md Pattern 2):
```rust
// Source pattern: agents.rs:171,199 (TimeoutFuture) + warp_hermes.rs:145 (use_future loop)
const POLL_INTERVAL_BASELINE_MS: u64 = 1_500;
const POLL_INTERVAL_WS_CONNECTED_MS: u64 = 5_000;
const HOLD_DURATION: std::time::Duration = std::time::Duration::from_secs(5);

use_future(move || async move {
    loop {
        // All reads happen BEFORE .await — signal borrows must not span await (clippy.toml).
        let hidden = tab_hidden();                 // Copy bool — call-syntax avoids GenerationalRef
        let busy = killing_busy() || interrupting_busy(); // Copy bool from screen-level aggregates
        let connected = is_ws_connected();         // Copy bool from context (Plan 02 wires increment)
        let interval_ms = if connected { POLL_INTERVAL_WS_CONNECTED_MS } else { POLL_INTERVAL_BASELINE_MS };

        if !hidden && !busy {
            agents_resource.restart();
        }
        // No signal borrow held across this await — clippy.toml satisfied.
        gloo_timers::future::TimeoutFuture::new(interval_ms as u32).await;
    }
});
```

The planner picks; option (b) is the lower LoC change.

**Source:** `agents.rs:33-37` + `crates/iron_hermes_ui/clippy.toml`
**Apply to:** All new `use_future` and `use_effect` bodies in `agents.rs` and `warp_hermes.rs`

### Event-Listener Lifecycle (applies to visibilitychange and any future DOM listeners)

**Source:** `crates/iron_hermes_ui/src/components/warp_hermes.rs:500-551`
**Apply to:** Page Visibility listener in `agents.rs` (Plan 01 D-04)

Rule: Store closure in `Signal<Option<Closure<_>>>`. Install in `use_effect`. Remove and drop in `use_drop`. Never call `.forget()`.

### use_effect Cross-Signal Reactivity (applies to Plan 02 counter watcher)

### Callback Construction + tx.clone() (applies to SubagentProgressCallback in ws.rs)

**Source:** `crates/iron_hermes_ui/src/server/ws.rs:194-210` (ToolProgressCallback + ToolResultCallback)
**Apply to:** SubagentProgressCallback construction in Plan 02

Rule: `tx.clone()` before entering the `Arc::new(move |...|)` closure. One clone per callback. The callback lifetime is bounded to the `tokio::spawn` block — do not hoist it to session scope.
