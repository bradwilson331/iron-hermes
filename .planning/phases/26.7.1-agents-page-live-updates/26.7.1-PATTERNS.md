## CRITICAL TARGETING CORRECTION

**The active root is `HermesApp`, NOT `WarpHermes`.**

`warp_hermes.rs` compiles in every build but is **never mounted** in default builds
(`Cargo.toml` line 61: `default = ["web"]` — `legacy-shell` absent). Any context
provider added to `WarpHermes` is dead code at runtime; downstream `use_context` calls
panic with "Could not find context ...". This was the root cause of the prior phase
failure.

### Forbidden analog targets (DO NOT copy integration patterns from these)

| File | Reason |
|------|--------|
| `crates/iron_hermes_ui/src/components/warp_hermes.rs` lines 79-420 (component body) | `WarpHermes` only mounts when `--features legacy-shell` is passed. Never mounts in `dx serve` or `cargo build --features web`. Adding context providers here causes runtime context-not-found panics. |
| `app.rs` lines 93-95 (`#[cfg(feature = "legacy-shell")]` branch) | Dead in default build. |

`warp_hermes.rs` **may be read as a structural reference** for Dioxus idioms
(e.g. `use_drop` import placement, `Closure<dyn FnMut>` pattern) but must **never be
the integration target** — all new providers and ws arms go in `hermes_app/mod.rs`.

### Signal-Borrow Discipline (ALL new use_future, use_effect, and spawn closures)

**Source:** `hermes_app/mod.rs` + `agents.rs:33-37` + `clippy.toml:1-8`
**Apply to:** Every new `use_future` and `use_effect` body in `mod.rs` and `agents.rs`

`clippy.toml` enforces three types as invalid across await:

```toml
# Source: crates/iron_hermes_ui/clippy.toml lines 1-8
await-holding-invalid-types = [
  "generational_box::GenerationalRef",
  "generational_box::GenerationalRefMut",
  "dioxus_signals::WriteLock",
]
```

Correct pattern (from `mod.rs:173-175`, `mod.rs:219-222`):
```rust
// CORRECT — Copy value read, ref dropped at ;
let sid = *streaming_id.read();    // u64 — Copy
let cur = *subagent_events.read(); // u64 — Copy
// ...
some_future().await;               // No borrow held — clippy satisfied

// CORRECT — owned clone, ref dropped at }
let interval_ms: u32 = { *ws_connected.read() }.then_some(5_000).unwrap_or(1_500);
// ...
gloo_timers::future::TimeoutFuture::new(interval_ms).await;

// CORRECT — HashMap: clone into owned, ref dropped at }
let prev: Vec<AgentInfo> = { recently_terminated.read().values()
    .map(|(info, _)| info.clone()).collect() };
```

### Context Provider — Consumer Pairing

**Provider source:** `hermes_app/mod.rs:302-306` (and the two new lines added after 306)
**Consumer pattern:** `screen_router.rs:24` — `use_context::<Signal<T>>()`

```rust
// Provider (mod.rs — add after line 306):
use_context_provider(|| subagent_events);   // Signal<u64>
use_context_provider(|| is_ws_connected);   // Signal<bool>

// Consumer (agents.rs — add at top of ScreenAgents body):
let subagent_events = use_context::<Signal<u64>>();

```

### `use_effect` Subscription Pattern

**Source:** `hermes_app/mod.rs:71-85` (hydration gate) and `mod.rs:92-113` (persist effects)
**Apply to:** Plan 02 counter-driven restart in `agents.rs`

The key rule: the signal must be **read inside the closure body** to create the
subscription. Call-syntax (`signal()`) is the Dioxus 0.7 idiom for subscribing without
holding a borrow:
```rust
// Source: mod.rs lines 92-98 — use_effect subscription idiom
use_effect(move || {
    if !*hydrated.read() { return; }   // signal read → effect subscribes to hydrated
    let p = prefs.read().clone();      // second signal read → also subscribes to prefs
    ui_prefs::write_json(...);
});

// New (agents.rs Plan 02):
use_effect(move || {
    subagent_events();          // call-syntax — subscribes and returns Copy u64; no borrow held
    agents_resource.restart();
});
```

### Callback Construction + `tx.clone()` (ws.rs)

**Source:** `ws.rs:180-210` (all three existing callbacks)
**Apply to:** SubagentProgressCallback construction

Rule: one `tx.clone()` per callback, captured by move into the closure. The clone must
happen **inside the `tokio::spawn` block** — `tx` only exists there.

### `#[cfg(target_arch = "wasm32")]` Gate for DOM APIs

**Source:** `warp_hermes.rs:1` (use_drop import gate), `warp_hermes.rs:4-5` (JsCast gate)
**Apply to:** `web_sys::Document::hidden()` call in agents.rs poll loop

```rust
// Source: warp_hermes.rs lines 1-5 — import gating pattern
#[cfg(target_arch = "wasm32")]
use dioxus::core::use_drop;
#[cfg(target_arch = "wasm32")]
use wasm_bindgen::JsCast;
```

`Document::hidden()` is synchronous and returns `bool` — it does not require `JsCast`
directly, only the `"Document"` web-sys feature (already present in `Cargo.toml`).
Gate the call with `#[cfg(target_arch = "wasm32")]` or inside a `cfg` block.

## Anti-Pattern Callouts

### FORBIDDEN: Adding context providers to `warp_hermes.rs`

Lines `warp_hermes.rs:79-420` (the `WarpHermes` component body) are dead code in all
default builds. The prior phase attempt added providers here and caused runtime panics.

**Specific lines that caused the failure (DO NOT repeat):**
- Any `use_context_provider` call inside `WarpHermes() -> Element` body
- Any signal declaration inside `WarpHermes()` intended to be consumed by `ScreenAgents`
- The `#[cfg(feature = "legacy-shell")]` `root_shell()` branch at `app.rs:93-95`

**Correct location for all new providers:** `hermes_app/mod.rs` inside `HermesApp()`,
after line 306.

**Correct location for the SubagentEvent recv arm:** `hermes_app/mod.rs` inside the
`use_future` recv loop match block, after the `Error` arm (~line 231).

### FORBIDDEN: `AppState::init()` scope for per-turn callback

`DelegateTaskWiring.progress_callback` at `state.rs:124` is baked into the
`Arc<AppRuntimeBundle>` at init time. Constructing a `SubagentProgressCallback` there
that captures a per-turn `tx` sender is impossible — the sender doesn't exist until
`ws.rs:180`. Use Path A (global channel on `AppState`) or construct per-turn inside
`tokio::spawn` as shown in the ws.rs pattern above.

## Pitfalls

### Pitfall 1: WarpHermes dead-code trap (cfg-gated at call site, not definition)

Both `WarpHermes` and `HermesApp` are compiled in every build. The `#[cfg]` gate is at
`app.rs:root_shell()`, not at the component definition. Grep hits on `warp_hermes.rs`
look like live code but never execute at runtime. Always verify the active root by
reading `app.rs:root_shell()` AND `Cargo.toml:default`.

**Warning sign:** Any plan or diff that edits `warp_hermes.rs` as the integration target
for Phase 26.7.1. Compile-time success + runtime context panic is the failure mode.

### Pitfall 2: Signal borrow across `.await` (clippy.toml enforced as error)

`clippy.toml` treats `GenerationalRef`, `GenerationalRefMut`, and `WriteLock` held across
`.await` as errors via `-D warnings`. The poll loop's `ws_connected.read()` and
`rpc_in_flight.read()` must be read into Copy values and the statement ended before the
`TimeoutFuture::new(...).await`.

```rust
// WRONG — GenerationalRef from .read() held across await
let interval = if *ws_connected.read() { 5_000 } else { 1_500 };
gloo_timers::future::TimeoutFuture::new(interval).await;   // clippy error here

// CORRECT — Copy value, ref dropped at ;
let interval: u32 = if *ws_connected.read() { 5_000 } else { 1_500 };
                    // ↑ ref dropped at ; ↑
gloo_timers::future::TimeoutFuture::new(interval).await;   // no borrow held
```

**Lint gate:** `cargo clippy -p iron_hermes_ui --features web -- -D warnings`

### Pitfall 3: D-05 — per-card signals are component-local

`killing` and `interrupting` at `agents.rs:115-117` are declared with `use_signal` inside
`AgentCard` — they are invisible to `ScreenAgents`. The poll loop cannot observe them
directly. Solution: `rpc_in_flight: Signal<u32>` counter at `ScreenAgents` scope, passed
as a prop to `AgentCard`. AgentCard increments on RPC start, decrements on RPC end.

### Pitfall 4: `JsCast` trait not in prelude

`wasm_bindgen::JsCast` provides `.unchecked_ref()` and `.dyn_into()`. It is NOT
re-exported by `dioxus::prelude::*`. Import explicitly (copy from `warp_hermes.rs:4-5`):
```rust
#[cfg(target_arch = "wasm32")]
use wasm_bindgen::JsCast;
```

### Pitfall 5: `use_drop` import path

`use_drop` lives at `dioxus::core::use_drop`, NOT in `dioxus::prelude::*`. Copy from
`warp_hermes.rs:1-2`:
```rust
#[cfg(target_arch = "wasm32")]
use dioxus::core::use_drop;
```
Compiles on WASM but fails on native builds if the wrong path is used.

### Pitfall 6: Cargo.lock / dx version drift

`dioxus = "0.7"` in `Cargo.toml` resolves to the latest 0.7.x patch. If `dx` CLI was
built at a different 0.7.x patch, API signatures may silently diverge. If mysterious
"method not found" errors appear, run:
```
cargo update -p dioxus --precise <exact-version>
```

### Pitfall 7: `SubagentProgressCallback` is `Arc<dyn Fn>`, not `Box<dyn Fn>`

All three existing callbacks in `ws.rs` use `Box::new(...)`. The `SubagentProgressCallback`
type alias in `delegate_task.rs:38` is `Arc<dyn Fn(usize, SubagentProgress) + Send + Sync>`.
Use `std::sync::Arc::new(...)` — `Box::new(...)` will fail to coerce.
