Gray (neutral) is the correct semantic choice for "no longer active, no action required."

## New CSS Classes — Design Contract

These are the two new modifier classes. The executor adds them to `crates/iron_hermes_ui/assets/screens.css`.

### `.card.is-ended`

**Placement in file:** Immediately after `.card.is-active` (currently `screens.css:118-122`).

```css
.card.is-ended {
  opacity: 0.55;
  transition: opacity 200ms ease-out;
  pointer-events: none;
}
.card.is-ended .card-footer {
  display: none;
}
```

**Opacity value: 0.55.** Rationale: The design system's `-dim` token family consistently uses `0.55` alpha (e.g. `--teal-dim: rgba(57, 197, 207, 0.55)` at `site.css:23`). The corner chrome uses `opacity: 0.65` (less dim). A disabled/ended state should be dimmer than a visible chrome element. 0.55 is the system's established "dim" level, and it is the value D-10 proposes. **Do not tune to 0.5 or 0.6.**

**Transition: 200ms ease-out.** The existing `.card` rule already has `transition: border-color 0.18s, background 0.18s, transform 0.18s` (`screens.css:111`). 200ms matches this cadence. The opacity transition applies when the card gains `.is-ended` — this gives a brief fade-to-dim on HOLD start. When the card is removed from the render tree at HOLD expiry (5s), it disappears cleanly; no exit animation is needed.

**`pointer-events: none`** prevents hover states from activating on ended cards (no `.card:hover` background shift, no inadvertent click targets).

**Button hiding mechanism:** Pure CSS via `.card.is-ended .card-footer { display: none; }`. This is preferred over a conditional in RSX because it is a single source of truth and does not require signal reads in the card render. The entire `.card-footer` is hidden (which contains CHAT, INTERRUPT, and KILL). This satisfies D-10: all three buttons removed from view.

### `.pill.ended`

**Placement in file:** Immediately after `.pill.red` (currently `screens.css:211`).

```css
.pill.ended {
  color: var(--gray);
  border-color: var(--gray-faint);
  background: rgba(110, 118, 129, 0.06);
}
```

**Color: solid `var(--gray)` text.** Not `--gray-faint`. The `.pill.green` uses solid `var(--green)` text (not a faint/alpha variant). Visual weight parity demands solid `--gray` for text — it must be legible against the dimmed card. `--gray-faint` (rgba 0.45) would be unreadable at 0.55 card opacity.

**Background: `rgba(110, 118, 129, 0.06)`.** Exactly mirrors the pattern: `.pill.green` uses `rgba(63,185,80,0.06)`, `.pill.amber` uses `rgba(210,153,34,0.06)`, `.pill.red` uses `rgba(248,81,73,0.06)`. Same 0.06 alpha, gray RGB values (110, 118, 129).

**Border: `var(--gray-faint)` = `rgba(110, 118, 129, 0.45)`.** Mirrors `.pill.green` at `rgba(63,185,80,0.45)`, `.pill.amber` at `rgba(210,153,34,0.45)`, `.pill.red` at `rgba(248,81,73,0.45)`. Same 0.45 alpha.

## Interaction Contract

### Poll loop (Plan 01)

| Condition | Poll behavior |
|-----------|--------------|
| Tab visible, no kill/interrupt in flight | Fire every 1500ms (D-03) |
| Tab hidden (`document.hidden === true`) | Pause. Fire one immediate fetch on visibility restore (D-04) |
| Any card's `killing()` or `interrupting()` signal is true | Pause until both are false (D-05) |
| WS connected (Plan 02) | Widen to 5000ms — WS events drive updates; poll is safety net (D-08) |
| WS disconnected (Plan 02) | Tighten back to 1500ms baseline (D-08) |

### Recently-terminated diff (Plan 01)

| Event | Behavior |
|-------|----------|
| Agent id present in prev list, absent in new list | Add to `recently_terminated` HashMap with snapshot `AgentInfo` and `Instant::now()` |
| Agent id appears in `recently_terminated` AND in new live list | Remove from `recently_terminated` immediately (D-13 re-running id wins) |
| `recently_terminated` entry age > 5s | Remove from HashMap; card disappears from render tree (no animation) |
| User clicks PRUNE | Call `api_agents_prune(None)` AND call `recently_terminated.write().clear()` (D-12) |

### HOLD card rendering

| Element | RUNNING card | ENDED (HOLD) card |
|---------|-------------|------------------|
| CSS class on `.card` | `"card is-active"` | `"card is-ended"` |
| CSS class on status pill | `"pill green"` | `"pill ended"` |
| Pill text | `"{agent.status.to_uppercase()}"` (e.g. `"RUNNING"`) | `"ENDED"` (literal, D-09) |
| `card-meta` uptime | `"{agent.uptime_secs}s"` (live from poll) | `"{snapshot.uptime_secs}s"` (frozen, D-11) |
| `card-footer` | visible | hidden via CSS (`display: none`) |
