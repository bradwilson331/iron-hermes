# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: **Worktree-merge race with pre-existing uncommitted changes** — The orchestrator's post-wave cleanup script runs `git merge <worktree-branch>` after backing up only STATE.md / ROADMAP.md. If the main working tree has *other* uncommitted changes (plan files, todo files, .omc cache) that overlap with files the worktree modified, merge aborts — and because the script used `set -e` loosely, it still proceeded to `git branch -D <worktree-branch>`, destroying the branch pointer before recovery. The commits remained reachable via `git cat-file -t <sha>` and were recovered by `git merge <sha>` directly. **Structural mitigation:** before spawning Wave 2, run `git status --short` and EITHER commit/stash ALL pre-existing modifications to `.planning/` and `.omc/` OR verify they do not overlap with files the upcoming wave's `files_modified` lists.
