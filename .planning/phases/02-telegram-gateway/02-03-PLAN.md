**Constructor:** `pub fn new(config: Config, tool_registry: Arc<ToolRegistry>) -> Self` — creates Arc<RwLock<SessionStore>>::default(), CancellationToken::new().

**`pub async fn start(&self) -> Result<()>`:**
1. Resolve Telegram token from config (same `resolve_env_var` pattern as current runner.rs).
2. Create `Arc<TelegramAdapter>` from token.
3. Call `adapter.get_me()` to verify token, log bot name/username. Store bot_username for @mention filtering.
4. Register slash commands via `adapter.set_my_commands` (D-17):
   ```rust
   let commands = vec![
       TgBotCommand { command: "start".into(), description: "Start the bot".into() },
       TgBotCommand { command: "new".into(), description: "New conversation".into() },
       TgBotCommand { command: "clear".into(), description: "Clear history".into() },
       TgBotCommand { command: "help".into(), description: "Show help".into() },
   ];
   ```
5. Create mpsc channel: `let (msg_tx, msg_rx) = mpsc::channel::<TgUpdate>(256)`.
6. Create `Arc<Semaphore>` from `config.gateway.platforms["telegram"].max_concurrent_runs`.
7. Create handler: `Arc<GatewayMessageHandler>`.
8. Create `UserQueueManager`.
9. Spawn **poll_loop** task (CancellationToken clone):
   - Loop with `tokio::select!` on cancel.cancelled() vs `getUpdates` call.
   - On success: send each update through msg_tx. Record backoff success.
   - On error: check if 409 conflict (error message contains "Conflict" or "409"). If conflict: `backoff.record_conflict()`, check `is_fatal_conflict()` — if true, log fatal, cancel token, break. Else: `backoff.record_failure()`, sleep `backoff.next_delay()`.
   - Track `offset: Option<i64>`, increment on each processed update.

10. Spawn **dispatch_loop** task (CancellationToken clone):
    - `while let Some(update) = rx.recv().await` (also select on cancel):
    - Extract TgMessage from update. Skip if no message.
    - Convert to MessageEvent via `tg_message_to_event`.
    - **Whitelist check** (D-10/D-11/D-12): parse sender_id as i64, check if in config whitelist. If whitelist is empty or sender not in list: silently ignore (no response per D-11).
    - **Group @mention check** (D-09): if chat_type is "group"/"supergroup", check if content contains `@{bot_username}`. If not mentioned, skip.
    - Dispatch via `user_queue.dispatch(event)`. If returns `Some(rx)`: spawn a **per-chat worker** in the JoinSet.

11. **Per-chat worker** (spawned in JoinSet):
    - `while let Some(queued_msg) = chat_rx.recv().await`:
    - Acquire semaphore permit (blocks if at max concurrency per TG-06).
    - Call `handler.handle(&queued_msg.event, adapter.clone(), cancel.child_token())`.
    - On error, log and continue (don't crash the worker).
    - Drop permit when done.
    - After loop: `user_queue.remove(&chat_id)`.

12. Spawn **session_cleanup** task: every 5 minutes, acquire write lock on session_store, call `expire_stale(timeout_hours)`. Select on cancel.

13. **Wait for shutdown:** `tokio::signal::ctrl_c().await`. Then:
    - `self.cancel.cancel()` — propagates to all subtasks.
    - Drop msg_tx to close dispatch channel.
    - `while join_set.join_next().await.is_some() {}` — drain all in-flight agent runs per TG-05.
    - Log "Gateway shut down cleanly".
