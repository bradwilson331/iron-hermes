## Summary

All UAT gaps resolved. The root cause was that `TypedWebsocket::recv()` (Stream path)
tried to JSON-decode incoming text frames as Rust `String` type, which fails for raw JSON
object payloads like `{"session_id":"...","message":"..."}`. The fix switches the entire
server loop to `recv_raw()` / `send_raw(Message::Text(...))` to bypass the typed
serialization layer entirely. Server-side env loading (`dotenvy`) and a 5-second keepalive
Ping were also added.
