| Websocket event stream → client transcript state | Disconnect/reconnect faults can corrupt user-visible stream continuity |

|-----------|----------|-----------|-------------|-----------------|

| T-26.1-03-03 | I | `warp_hermes.rs` transcript error path | mitigate | Keep disconnect UI message generic (no low-level transport details), while logging detailed cause only in telemetry. |
| T-26.1-03-04 | T | reconnect-state handling in `warp_hermes.rs` | mitigate | Reset only transient stream pointers on disconnect and keep protocol parsing/sending tied to `crate::protocol` types. |
</threat_model>
