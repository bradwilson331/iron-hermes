Create `providers/memory-duckdb/src/bridge.rs` implementing the dedicated thread pattern per D-03:

```rust
use std::collections::HashMap;
use std::path::Path;
use std::sync::mpsc;
use ironhermes_core::memory_store::MemoryTarget;

pub enum DuckDbCommand {
    Add { target: String, content: String, respond: oneshot::Sender<Result<String, String>> },
    Replace { target: String, old_text: String, new_content: String, respond: oneshot::Sender<Result<String, String>> },
    Remove { target: String, old_text: String, respond: oneshot::Sender<Result<String, String>> },
    LoadAll { respond: oneshot::Sender<anyhow::Result<HashMap<String, Vec<String>>>> },
    Shutdown,
}

// Use std::sync::mpsc for oneshot responses (or tokio::sync::oneshot if preferred)
// The key: DuckDbBridge holds only Sender<DuckDbCommand> which IS Send+Sync
// Connection lives only on the worker thread

pub struct DuckDbBridge {
    tx: mpsc::Sender<DuckDbCommand>,
    thread: Option<std::thread::JoinHandle<()>>,
}

impl DuckDbBridge {
    pub fn new(db_path: &Path) -> anyhow::Result<Self> {
        let (tx, rx) = mpsc::channel();
        let db_path = db_path.to_owned();
        let thread = std::thread::spawn(move || {
            let conn = duckdb::Connection::open(&db_path)
                .expect("Failed to open DuckDB database");
            conn.execute_batch(schema::CREATE_SCHEMA)
                .expect("Failed to initialize DuckDB schema");
            
            for cmd in rx {
                match cmd {
                    DuckDbCommand::Add { target, content, respond } => {
                        let result = Self::handle_add(&conn, &target, &content);
                        let _ = respond.send(result);
                    }
                    DuckDbCommand::Replace { target, old_text, new_content, respond } => {
                        let result = Self::handle_replace(&conn, &target, &old_text, &new_content);
                        let _ = respond.send(result);
                    }
                    DuckDbCommand::Remove { target, old_text, respond } => {
                        let result = Self::handle_remove(&conn, &target, &old_text);
                        let _ = respond.send(result);
                    }
                    DuckDbCommand::LoadAll { respond } => {
                        let result = Self::handle_load_all(&conn);
                        let _ = respond.send(result);
                    }
                    DuckDbCommand::Shutdown => break,
                }
            }
        });
        Ok(Self { tx, thread: Some(thread) })
    }
    
    pub fn send(&self, cmd: DuckDbCommand) -> anyhow::Result<()> {
        self.tx.send(cmd).map_err(|_| anyhow::anyhow!("DuckDB worker thread has terminated"))
    }
    
    pub fn shutdown(&mut self) {
        let _ = self.tx.send(DuckDbCommand::Shutdown);
        if let Some(thread) = self.thread.take() {
            let _ = thread.join();
        }
    }
}

impl Drop for DuckDbBridge {
    fn drop(&mut self) {
        self.shutdown();
    }
}
```

The handle_add/handle_replace/handle_remove/handle_load_all are private static methods that operate on `&duckdb::Connection` directly. They implement the same logic as SqliteMemoryProvider but using DuckDB's API:
- Parameterized queries (prevent SQL injection)
- Capacity checking (sum content lengths + delimiter lengths)
- Duplicate detection
- Substring matching for replace/remove (SELECT WHERE content LIKE '%pattern%')

Create `providers/memory-duckdb/src/lib.rs`:

```rust
pub struct DuckDbMemoryProvider {
    bridge: DuckDbBridge,
    snapshot: HashMap<MemoryTarget, Vec<String>>,
}
```

The provider delegates all storage operations to the bridge:
- `new(db_path)`: Create DuckDbBridge
- `load_from_disk()`: Send LoadAll command, populate snapshot cache
- `add(target, content)`: Security scan via `scan_context_content()` FIRST (on caller thread, before sending to bridge). Then send Add command, await response via oneshot.
- `replace(target, old_text, new_content)`: Security scan new_content. Send Replace. Await response.
- `remove(target, old_text)`: Send Remove. Await response.
- `format_for_system_prompt(target)`: Read from frozen snapshot cache with capacity header per D-13.
- `to_memory_entries()`: Return snapshot cache.
- `shutdown()`: Send Shutdown command, join thread.

IMPORTANT: Security scanning happens on the caller's thread BEFORE sending to the bridge. This avoids passing `scan_context_content` into the worker thread and keeps the bridge focused on storage.

For oneshot channels: use `std::sync::mpsc::channel()` creating a (Sender, Receiver) pair. The Sender goes into the command enum, Receiver is held by the caller and `.recv()` blocks until the response arrives. This is simpler than tokio::sync::oneshot since the MemoryProvider trait methods are sync (`fn add`, not `async fn add`).
