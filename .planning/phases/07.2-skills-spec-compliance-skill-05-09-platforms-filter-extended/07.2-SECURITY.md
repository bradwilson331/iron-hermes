## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| SKILL.md file → `SkillFrontmatter` parse | User-authored YAML crosses into typed structs via `serde_yaml` | YAML strings |
| `metadata: serde_yaml::Value` opaque blob | Arbitrary YAML shapes accepted without schema validation | Nested YAML |
| `platforms: Vec<String>` → OS dispatch | User-authored strings drive platform matching logic | Platform names |
| SKILL.md name field → registry lookup | User-authored string drives case-insensitive registry lookup | Skill names |
| Regex engine input | User-authored name string fed to `regex::Regex::is_match` | Name strings ≤64 chars |
| `config.yaml` → `SkillsConfig.extra_paths` | User-controlled filesystem paths fed to read-only directory scans | PathBuf list |
| User config → `enabled` kill switch | User can disable skills subsystem entirely | Boolean |
