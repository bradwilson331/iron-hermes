# Phase 07.2 Plan 04: SKILL-08 SkillsConfig Summary

**One-liner:** Added `SkillsConfig` struct to `config.yaml` with `enabled` kill switch and `extra_paths` appended after the 3 hardcoded defaults (D-19), wired via `SkillRegistry::load_with_config` at all CLI call sites, with 9 new tests and zero regressions.

## What Was Built

Extended `crates/ironhermes-core/src/config.rs` with a new `SkillsConfig` struct (SKILL-08), added it as `pub skills: SkillsConfig` on `Config`, and added `SkillRegistry::load_with_config` + `build_skill_search_paths` to `crates/ironhermes-core/src/skills.rs`. All three CLI call sites were rewired. The existing `SkillRegistry::load(cwd)` is preserved as a thin wrapper.

## SkillsConfig Location

**File:** `crates/ironhermes-core/src/config.rs`

Lines after `RateLimitConfig` block (before `impl Config`):
- `SkillsConfig` struct defined with doc comment: `pub struct SkillsConfig`
- Fields: `pub enabled: bool`, `pub extra_paths: Vec<PathBuf>`
- `impl Default for SkillsConfig`: returns `enabled: true, extra_paths: Vec::new()`
- `Config` struct: `pub skills: SkillsConfig` added as last field

## load_with_config Location

**File:** `crates/ironhermes-core/src/skills.rs`

`SkillRegistry::load_with_config(cwd: &Path, config: &SkillsConfig) -> Self`:
- If `!config.enabled` → returns `Self { skills: Vec::new() }` immediately (D-20 kill switch)
- Otherwise calls `build_skill_search_paths(cwd, config)` to get paths, then `Self::load_with_paths(&search_paths)`

`build_skill_search_paths(cwd: &Path, config: &SkillsConfig) -> Vec<PathBuf>` (pub(crate)):
- Builds 3 defaults: `cwd/.ironhermes/skills`, `get_hermes_home()/skills`, `home/.agents/skills`
- Extends with `config.extra_paths.iter().cloned()` — extras appended AFTER defaults (D-19)

`SkillRegistry::load(cwd: &Path)`:
- Rewired to `Self::load_with_config(cwd, &SkillsConfig::default())` — thin wrapper preserved for backward compat (D-21)
