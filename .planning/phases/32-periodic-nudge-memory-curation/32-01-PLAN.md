<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| nudge AgentLoop -> MemoryManager | Nudge agent tool calls pass through MemoryTool -> MemoryStore security scan (scan_content, Phase 17) |
| messages_snapshot | Read-only clone of conversation; nudge cannot mutate the active messages vec in run_chat |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Tampering | nudge tool registry | mitigate | Build narrow ToolRegistry with only MemoryTool; session_search, web_read, execute_code deliberately excluded — nudge cannot exfiltrate data or run code |
| T-32-02 | Denial of Service | memory cap violation | accept | MemoryStore enforces 2,200 + 1,375 char caps; capacity_exceeded returned to agent, logged as warn, nudge terminates cleanly |
| T-32-03 | Denial of Service | recursive nudge amplification | accept | turns_since_nudge is a local u32 in run_chat; nudge's internal AgentLoop does not touch it — recursion structurally impossible |
| T-32-04 | Tampering | prompt injection via memory review | mitigate | All nudge writes go through MemoryManager::handle_tool_call -> MemoryStore::scan_content (existing Phase 17 scanner); no bypass path |
| T-32-05 | Availability | nudge blocking REPL | mitigate | tokio::spawn makes call fire-and-forget; REPL shows next prompt immediately regardless of nudge duration |
| T-32-SC | Tampering | cargo installs | accept | No new external packages in this phase; pure workspace changes only |
</threat_model>
