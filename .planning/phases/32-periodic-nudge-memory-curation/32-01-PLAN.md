| Boundary | Description |

| nudge prompt → MemoryManager | Untrusted LLM output in memory write calls goes through existing MemoryManager::handle_tool_call which calls scan_content security scanner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Tampering | nudge ToolRegistry | mitigate | Register only MemoryTool — session_search, execute_code, web_read absent from nudge registry by construction; no code path allows other tools to be injected |
| T-32-02 | Denial of Service | nudge AgentLoop iteration | mitigate | max_iterations=8 hard-coded in AgentLoop::new call; nudge cannot loop indefinitely |
| T-32-03 | Tampering | MEMORY_REVIEW_PROMPT prompt injection | accept | All writes route through existing MemoryManager::handle_tool_call → scan_content scanner (V5 ASVS — already mitigated at the write boundary) |
| T-32-04 | Denial of Service | nudge writes exceeding char cap | accept | MemoryStore enforces 2,200/1,375 char caps and returns capacity_exceeded error; nudge agent receives the error and stops |
</threat_model>
