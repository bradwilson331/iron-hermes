<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| REPL loop → tokio::spawn nudge | Cloned messages snapshot crosses async boundary; the snapshot is read-only and already persisted — no mutation of outer loop state |
| Gateway handler per-session counter | HashMap<SessionKey, u32> under std::sync::Mutex; only lock is held for the counter increment operation, released before tokio::spawn |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-05 | Denial of Service | tokio::spawn nudge blocking REPL | mitigate | Nudge is fire-and-forget (tokio::spawn, no .await at call site); REPL continues to next prompt immediately |
| T-32-06 | Denial of Service | Recursive nudge (nudge spawning nudge) | accept | Counter is in outer REPL loop / GatewayMessageHandler state; run_memory_nudge's internal AgentLoop run does not touch the outer counter — structurally impossible |
| T-32-07 | Tampering | Active system prompt mutated post-nudge | accept | Frozen-snapshot invariant (PRMT-06 / MEM-06) implemented in Phase 17/20; nudge writes to disk only; active session prompt unchanged |
| T-32-08 | Tampering | Gateway nudge client credential exposure | accept | Client clone reuses the same AnyClient already constructed for the main turn; no new credential loading |
</threat_model>
