**SQLite provider fully functional with FTS5 memory_recall (bm25 ranked search with snippets) plus 4 advanced hooks: sync_turn, on_pre_compress, system_prompt_block, queue_prefetch**

## Accomplishments
- memory_recall tool schema returned by get_tool_schemas() with query (required) and limit (optional, default 5) parameters
- handle_tool_call dispatches memory_recall to FTS5 search and delegates add/replace/remove to provider methods
- recall() executes FTS5 MATCH with bm25() ranking and snippet() generation, returns JSON array of {content, target, relevance_score, snippet}
- sanitize_fts_query() prevents FTS5 syntax injection by tokenizing and double-quoting each term (T-21.5-03)
- sync_turn fires tokio::spawn with spawn_blocking to rebuild FTS5 index asynchronously (D-07)
- on_pre_compress indexes discarded messages into conversation_extracts table with FTS5 for future recall (D-08)
- system_prompt_block surfaces 5 most recent memory entries with [SQLite Memory] header (D-10)
- queue_prefetch warms FTS5 cache with lightweight MATCH query (D-09)
- 12 new tests added (7 for Task 1, 5 for Task 2), all 28 unit tests + 3 integration tests pass

## Decisions Made
- sanitize_fts_query tokenizes on whitespace and wraps each term in double quotes -- prevents FTS5 boolean operator injection while preserving multi-term AND semantics
- conversation_extracts table created lazily inside on_pre_compress (not in schema.rs CREATE_SCHEMA) because it only matters when context compression fires
- system_prompt_block reads from live DB rather than frozen snapshot -- it provides contextual awareness independent of the frozen-snapshot pattern used by format_for_system_prompt
- RecallResult derives both Serialize and Deserialize -- Serialize for production JSON output, Deserialize for test assertions
