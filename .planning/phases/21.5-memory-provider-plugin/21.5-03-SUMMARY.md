## Decisions Made
- extract_entity_triples uses heuristic pattern matching with 7 relation keywords (is, has, uses, likes, prefers, works with, knows) -- avoids regex dependency while covering common natural language patterns
- find_or_create_entity and store_triples take &self (not &mut self) because GrafeoDB uses interior mutability -- this is critical for on_pre_compress and sync_turn which take &self per the MemoryProvider trait
- DuckDB fire-and-forget commands have no respond channel per D-07 pattern -- errors are logged via tracing::warn but never propagated to callers
- DuckDB recall uses ILIKE with wildcard escaping (% -> \%, _ -> \_) before parameterized $1 binding for T-21.5-06 injection prevention
- MemoryTarget HashMap converted to string-keyed HashMap for serialization in DuckDB sync_turn since MemoryTarget lacks Serialize derive

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed ArcStr type annotation ambiguity in find_or_create_entity**
- **Found during:** Task 1 (Grafeo provider implementation)
- **Issue:** `n.as_ref() == name` in find_or_create_entity was ambiguous because ArcStr implements both AsRef<str> and AsRef<[u8]>
- **Fix:** Changed to `AsRef::<str>::as_ref(n) == name` for explicit type disambiguation
- **Files modified:** providers/memory-grafeo/src/lib.rs
- **Verification:** cargo test -p memory-grafeo passes
- **Committed in:** 099f14b (Task 1 commit)

**2. [Rule 1 - Bug] Fixed MemoryTarget serialization in DuckDB sync_turn**
- **Found during:** Task 2 (DuckDB provider implementation)
- **Issue:** `serde_json::to_string(&entries.entries)` failed because MemoryTarget (HashMap key) does not implement Serialize
- **Fix:** Convert to string-keyed HashMap via `entries.entries.iter().map(|(k, v)| (k.label().to_string(), v)).collect()` before serialization
- **Files modified:** providers/memory-duckdb/src/lib.rs
- **Verification:** cargo test -p memory-duckdb passes
- **Committed in:** cd625cc (Task 2 commit)
