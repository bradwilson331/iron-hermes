## Decisions Made
- memory_provider_tool_names is a HashSet populated once at run() start rather than per-call lookup -- avoids repeated locking of memory_manager on every tool dispatch
- Intercept placement after tool_start ensures duration_ms is tracked; after tool_path_arg ensures subdirectory discovery state is saved before args are consumed
- Used async lock pattern (not spawn_blocking) because MemoryManager wraps tokio::sync::Mutex, unlike StateStore which uses std::sync::Mutex
