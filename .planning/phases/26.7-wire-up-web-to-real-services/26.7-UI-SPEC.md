## Typography

All type uses the token system from `assets/tokens.css`. **No new type styles introduced in Phase 26.7.** Use existing utility classes for all new or modified RSX.

**Two declared weights for this phase: 400 (regular) and 700 (bold).** The 36px screen-title size provides sufficient visual hierarchy without requiring a third weight. All semibold/bold elements use 700.

| Role        | Class / Token             | Size (px) | Weight | Line Height | Font family  |
|-------------|---------------------------|-----------|--------|-------------|--------------|
| Screen title | `.screen-title`           | 36px      | 700    | 1.05        | font-mono-cond |
| Body / description | `.t-body`, `.screen-sub` | 14px (12px for sub) | 400 | 1.5–1.55 | font-sans |
| Card title  | `.card-title`             | 14px      | 700    | 1.3         | font-sans    |
| Card meta / label | `.card-meta`, `.t-label` | 11–12px | 400 or 700 | 1.4–1.45 | font-mono |
| Error inline text | `.t-small` or inline `color: var(--danger)` | 12px | 400 | 1.45 | font-sans |

**Note on card meta / label weight:** Elements marked `400 or 700` use 400 for supplementary metadata (timestamps, secondary info) and 700 for uppercase identifiers and labels. No intermediate weight (600) is used anywhere in this phase.

**Source:** assets/tokens.css `.t-*` utilities; assets/screens.css `.screen-title`, `.screen-sub`

## Color

Token values shown for default theme `slate-dark`. All references MUST use CSS custom properties — never hardcoded hex. All five themes (`slate-dark`, `slate-light`, `iron-dark`, `terminal-dark`, `parchment-light`) are live via `[data-theme]` on `<html>`.

| Role             | CSS Variable         | Approx hex (slate-dark) | Usage                                     |
|------------------|----------------------|--------------------------|-------------------------------------------|
| Dominant (60%)   | `--bg-canvas`        | oklch(17% 0.012 260)     | Screen background, overall app surface    |
| Secondary (30%)  | `--bg-surface` / `--bg-elevated` | oklch(20–23%) | Cards, panels, `.card` backgrounds      |
| Accent (10%)     | `--accent`           | oklch(76% 0.14 210) — cyan | Reserved-for list below                  |
| Destructive      | `--danger`           | oklch(68% 0.18 25) — red | Kill / interrupt agent buttons only       |
| Success semantic | `--success`          | oklch(74% 0.17 150) — green | ACTIVE agent status badge, ENABLED pill |
| Warning semantic | `--warning`          | oklch(78% 0.14 85)       | Not used in Phase 26.7 screens            |

**Accent (`--accent`) is reserved for:**
- Active tab underline border (`.ih-tabs button[aria-selected="true"]`)
- Toggle control on-state (`.ih-toggle[aria-checked="true"]`)
- Screen tag text (`// MODULE NN` — currently uses legacy `var(--teal)`)
- Focus rings on interactive elements (`:focus-visible` outline)
- Section label diamond glyph (`.section-label::before`)
- The `.count` span inside section labels and panel-title counts
- Loading state rail animation (`.ih-cmd[data-state="run"] .rail`)

**Destructive (`--danger`) is reserved for:**
- Kill Agent button text color and hover background tint
- Interrupt Agent button text color and hover background tint
- Error message text rendered when `use_server_future` returns `Some(Err(...))`

**Source:** assets/tokens.css `:root` / theme blocks; assets/components.css `.ih-btn.danger`

## Interaction Contract — Per Screen

### Tools Screen (ScreenTools)

**Before (stub):** `stub_data::tools()` — static `Vec<ToolStub>` with `name`, `summary`, `status`, `group`.

**After (live):** `use_server_future(api::list_tools)?` returning `Vec<ToolInfo>` with `name: String`, `description: String`.

**Loading state:** `match resource() { Some(Ok(v)) => v, _ => Vec::new() }` — renders an empty `.grid` (zero tool cards). No skeleton, no spinner. (D-13)

**Error state:** When `resource()` is `Some(Err(_))` — render one `.tool-card` (or a plain `div` inside `.grid`) containing inline text: `"Could not load tools — check server connection."` in `color: var(--danger)` at `font-size: var(--fs-12)`. No modal, no toast.

**Toggle buttons (ENABLE ALL / DISABLE ALL):** Keep as ghost buttons in `.screen-actions`. They are non-functional in Phase 26.7 (no write endpoint). They remain as visual affordance only — no `onclick` handler required this phase.

**ToolCard:** The `status` field from `ToolStub` is not present on `ToolInfo`. The card `.tgl` toggle element stays in the RSX but renders with a static class (`tgl` — off state default) since enable/disable is out of scope for Phase 26.7.

---

### Agents Screen (ScreenAgents)

**Before (stub):** `stub_data::agents()` — static `Vec<AgentStub>`.

**After (live):** `use_server_future(api::get_agents_status)?` returning `Vec<AgentInfo>` (new type — exact fields per Claude's Discretion, but must include at minimum `name: String`, `status: String`).

**Loading state:** Empty `.grid.wide` — no cards rendered while `resource()` is `None`.

**Error state:** Inline error row inside `.grid.wide`: `"Could not load agents — check server connection."` in `color: var(--danger)`, `font-size: var(--fs-12)`.

**Kill button interaction:**
- Rendered with class `btn btn--sm btn--ghost` plus inline `style="color:var(--danger)"` to use the danger semantic.
- `onclick` calls `api::kill_agent(agent_id)` via a `use_server_future` or direct async spawn.
- After successful response: the row re-fetches (call `sessions_resource.restart()` pattern) or removes the agent from the displayed list optimistically.
- Confirmation: **inline confirmation only** — no modal. Button label becomes `"KILL?"` on first click, executes on second click within 3 seconds; resets to `"KILL"` on timeout or blur. This prevents accidental kills without introducing a modal overlay.

**Interrupt button interaction:**
- Rendered with class `btn btn--sm btn--ghost` plus inline `style="color:var(--warning)"`.
- `onclick` calls `api::interrupt_agent(agent_id)`.
- No confirmation required (interrupt is recoverable — agent resumes on next message).
- After call: button briefly shows `"..."` (disabled, 500ms) then reverts.

**Prune Agents button (screen-level action):**
- Rendered in `.screen-actions` as `btn btn--ghost btn--sm`.
- `onclick` calls `api::prune_agents()`.
- Label: `"PRUNE ENDED"` — specific verb + noun as required.
- No confirmation required. After call: resource restarts to refresh the list.

**Chat button per card:** Remains as `btn btn--sm` (active agent) or `btn btn--ghost btn--sm` (idle). Non-functional in Phase 26.7 (navigates to Chat screen with agent context — deferred).

### Memory Screen (ScreenMemory)

**Before (stub):** `stub_data::memory_entries()` — static `Vec<MemoryEntryStub>`.

**After (live):** `use_server_future(api::get_memory)?` returning `MemoryInfo` (new type — see Claude's Discretion in CONTEXT.md; must expose at minimum entries per store with `scope/store: String`, `key: String`, `value: String`, `updated_at: String`).

**Read-only phase:** No `+ ENTRY` button click handler. The `+ ENTRY` and `EXPORT` buttons remain in `.screen-actions` as visual affordance with no `onclick` handler. (D-07)

**Loading state:** `.panel` renders with header present but empty `.row-list` (no `MemoryEntryRow` items).

**Error state:** Inside the `.panel` > `.row-list`: a single `div` with text `"Could not load memory — check server connection."` in `color: var(--danger)`, `font-size: var(--fs-12)`.

**Two-store layout:** If `MemoryManager` exposes MEMORY.md and USER.md as distinct stores, render them as two separate `.panel` blocks (the existing split layout already accommodates this). Left panel: agent memory (MEMORY.md entries). Right column top panel: USER.md entries (replacing the static `User Profile` dl). Right column bottom panel: the static `Provider` dl remains stubbed (out of scope D-07).

**Filter input:** The `⌕ Filter…` search input inside the panel header remains wired to client-side string filtering only (no server call). Filter on `entry.key` + `entry.value` prefix match, case-insensitive.

### Skills Screen (ScreenSkills)

**Before (stub):** `stub_data::skills()` — static `Vec<SkillStub>`.

**After (live):** `use_server_future(api::list_skills)?` returning `Vec<SkillInfo>` (new type — fields from `SkillRegistry`; must include at minimum `name: String`, `description: String`, `category: String`, `version: String`, `enabled: bool`).

**Loading state:** Empty `.grid` — no SkillCard items.

**Error state:** One `div` inside `.grid` with `"Could not load skills — check server connection."` in `color: var(--danger)`, `font-size: var(--fs-12)`.

**Tab strip:** The 5 tabs (`ALL`, `BUNDLED`, `INSTALLED`, `ENABLED`, `UPDATES`) remain as visual affordance only. In Phase 26.7 all tabs render the full list (no filtering per tab). Counts in tab labels become dynamic only for `ALL · {n}` where `n` is the live list length. Other counts remain as placeholder text.

**Toggle (tgl element):** Driven by `skill.enabled: bool`. `class: if skill.enabled { "tgl on" } else { "tgl" }`. No click handler — enable/disable writes are out of scope.

**Screen sub copy:** Update to use actual count from live data: `"{count} loaded · {enabled_count} enabled for ` + `<code>` + `{active_profile_name}` + `</code>."` — or fall back to static copy when those counts are unavailable.

---

### Models Screen (ScreenModels)

**Before (stub):** `stub_data::models()` — static `Vec<ModelStub>` grouped by `family`.

**After (live):** `use_server_future(api::list_models)?` returning `Vec<ModelInfo>` (new type — fields from `ModelRegistry`; must include at minimum `id: String`, `family: String`, `context_window: String`, `status: String`).

**Loading state:** No family groups rendered — empty screen body below header.

**Error state:** One `div` below the screen header with `"Could not load models — check server connection."` in `color: var(--danger)`, `font-size: var(--fs-12)`.

**Family grouping:** Preserved exactly — same `families` dedup loop pattern, rendered as `.model-family-group` with `.section-label` and `.grid.wide`. Source order from `ModelRegistry` preserved.

**EDIT button per card:** Remains as `btn btn--ghost btn--sm` with no `onclick` handler (config editing is out of scope).

## Copywriting Contract

| Element                          | Copy                                                                                    |
|----------------------------------|-----------------------------------------------------------------------------------------|
| Tools — Primary CTA              | `"ENABLE ALL"` / `"DISABLE ALL"` (existing, non-functional this phase)                |
| Tools — empty state              | (empty grid, no copy — zero cards is valid when tools list returns empty)               |
| Tools — error state              | `"Could not load tools — check server connection."`                                     |
| Agents — Primary CTA             | `"PRUNE ENDED"` (new screen-level action button)                                        |
| Agents — empty state             | (empty grid, no copy — zero cards is valid)                                             |
| Agents — error state             | `"Could not load agents — check server connection."`                                    |
| Agents — kill confirmation       | First click: button label `"KILL?"` — second click executes. Timeout 3s resets to `"KILL"` |
| Memory — empty state             | (empty row-list, no copy — valid when memory store is empty)                            |
| Memory — error state             | `"Could not load memory — check server connection."`                                    |
| Skills — empty state             | (empty grid, no copy)                                                                   |
| Skills — error state             | `"Could not load skills — check server connection."`                                    |
| Models — empty state             | (empty body, no copy)                                                                   |
| Models — error state             | `"Could not load models — check server connection."`                                    |

**Error copy pattern:** `"Could not load {resource} — check server connection."` — consistent across all 5 screens. Problem stated first, action path provided second.

**Destructive action — Kill Agent:**
- First-click label: `"KILL?"` (uppercase, question mark as soft confirmation signal, no modal)
- Second-click executes; no further confirmation
- Interrupt Agent: no confirmation — label stays `"INTERRUPT"`, brief `"..."` feedback during in-flight call

**Source:** D-13 (CONTEXT.md) for error state guidance; D-06 for agents write fns
