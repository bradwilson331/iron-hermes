## What Was Built

### Task 1 ‚Äî api.rs additions

Added two new types and one server fn to `crates/iron_hermes_ui/src/server/api.rs`:

- `MemoryEntry { store: String, body: String }` ‚Äî R-3 shape (no key, no updated_at)
- `MemoryInfo { entries: Vec<MemoryEntry> }` ‚Äî derives Default for memory-disabled path
- `#[get("/api/memory")] pub async fn get_memory()` ‚Äî acquires tokio Mutex, calls
  `mgr.lock().await.to_memory_entries().await`, maps `MemoryTarget::Memory` ‚Üí `"agent"` and
  `MemoryTarget::User` ‚Üí `"user"`, returns one `MemoryEntry` per raw text block.
  `state.memory_manager == None` short-circuits with `Ok(MemoryInfo::default())`.

### Task 2 ‚Äî memory.rs rewrite

Replaced `stub_data::memory_entries()` with the live server fn:

- `use_server_future(crate::server::api::get_memory)?` suspension pattern
- `filter_text` signal for client-side case-insensitive substring filter on `body`
- `needle` extracted to owned `String` via `.to_lowercase()` before `rsx!` ‚Äî no GenerationalRef held across RSX
- `agent_rows` / `user_rows` partitioned and filtered before `rsx!` (PATTERNS.md Gotchas)
- Left panel: agent rows + filter input (`oninput` wired to `filter_text.set()`)
- Right-top panel: user rows (replaces hardcoded "ALPHA-7" User Profile dl)
- Right-bottom panel: Provider dl preserved as static stub per D-07
- Error copy in both panels: `"Could not load memory ‚Äî check server connection."`
- `MemoryEntryRow` rewritten to take `crate::server::api::MemoryEntry`; `mem-ts` shows `‚Äî` (U+2014)
- `+ ENTRY` / `EXPORT` buttons: static affordance, no onclick handlers
