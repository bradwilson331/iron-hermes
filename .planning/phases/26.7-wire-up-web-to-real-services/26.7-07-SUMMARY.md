## Deviation from PLAN.md

**This deviation is non-negotiable to document — it changes the runtime semantics of Drop versus what the plan prescribed.**

### Plan-as-written (Option B — REJECTED at execution time)

The plan's `<action>` block (PLAN.md lines 153–209) specified:

```rust
impl Drop for RegistrationGuard {
    fn drop(&mut self) {
        if let Some(arc) = self.registry.upgrade() {
            if let Ok(handle) = tokio::runtime::Handle::try_current() {
                let id = self.id.clone();
                handle.spawn(async move {
                    arc.write().await.unregister_internal(&id);
                });
            }
        }
    }
}
```

Pattern: `Handle::try_current().spawn(async)` — fire-and-forget async task on the multi-thread runtime. **Eventual consistency:** cleanup runs on a future runtime tick, not before Drop returns.

### As-built (Option C — APPROVED deviation)

```rust
impl Drop for RegistrationGuard {
    fn drop(&mut self) {
        if let Some(arc) = self.registry.upgrade() {
            if let Ok(handle) = tokio::runtime::Handle::try_current() {
                let id = self.id.clone();
                if let Ok(thread) = std::thread::Builder::new()
                    .name("registration-guard-drop".into())
                    .spawn(move || {
                        handle.block_on(async move {
                            arc.write().await.unregister_internal(&id);
                        });
                    })
                {
                    let _ = thread.join();
                }
            }
        }
    }
}
```

Pattern: `std::thread::Builder::spawn` (fresh OS thread, no tokio runtime affiliation) → `handle.block_on(async)` (legal on a non-runtime thread) → `thread.join()` (preserves **synchronous** Drop semantics — registry write completes before Drop returns). See subagent_registry.rs:85–121 for the as-built code with full rationale comment.

### Why the deviation

Option B (plan-as-written) introduced eventual consistency that broke two pre-existing tests inside `subagent_registry.rs`'s own `#[cfg(test)]` block:

1. `register_then_drop_guard_leaves_count_zero`
2. `list_preserves_all_entries`

Both tests assume **synchronous** Drop semantics with a single `tokio::task::yield_now().await` between `drop(guard)` and the post-drop assertion. Under Option B the spawned cleanup task had not yet completed when the post-drop read fired, so the assertions failed.

The PLAN.md `<action>` block had an explicit tripwire for exactly this situation (lines 236–240):

> "if a test breaks under cargo test, FAIL the build and ask the orchestrator — do NOT silently add `yield_now` calls into tests."

The executor honored that tripwire — halted, surfaced the failures, and the user authorized Option C as the resolution. Option C uses an OS thread (no tokio runtime affiliation, so `handle.block_on` cannot panic on a LocalSet) and `thread.join()` (so Drop blocks until cleanup completes, preserving the contract existing tests rely on). Tests pass **unchanged** under Option C.

### Authorization and safety evidence

- **Who authorized:** User, during execution, after executor halted at the tripwire.
- **Test evidence:** `cargo test -p ironhermes-agent --all-features` exits 0; 305+ tests pass without modification.
- **LocalSet escape evidence:** `std::thread::spawn` creates an OS thread with no tokio runtime affiliation. `handle.block_on` on such a thread is not subject to the `block_in_place` "multi-threaded runtime" check (that check inspects the current thread's tokio worker context, which an OS-spawned thread does not have). The Dioxus per-connection LocalSet that triggered the 26.7-06 panic cannot reach the spawned thread.
- **Cost:** One short-lived OS thread per Drop. Drops occur at subagent lifecycle boundaries (not hot-path), so the overhead is acceptable.

The struct-level doc comment (subagent_registry.rs:70–79) was updated to reflect Option C — it documents the OS-thread + join pattern and the synchronous-Drop guarantee, replacing the plan's eventual-consistency / yield-required language.
