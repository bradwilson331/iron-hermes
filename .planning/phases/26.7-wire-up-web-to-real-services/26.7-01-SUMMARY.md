# Phase 26.7 Plan 01: Tools Screen Wire-Up Summary

Replace `stub_data::tools()` with `use_server_future(api::list_tools)?` — live tool catalog from `state.runtime_bundle.registry`, loading/error states per UI-SPEC.

## What Was Built

Wired `ScreenTools` to the live `api::list_tools()` server function using the canonical Sessions screen pattern. The component now fetches the real tool catalog from the running runtime on mount and renders `Vec<ToolInfo>` into the existing `.tool-card` grid.

### Key changes in `tools.rs`

- **Removed:** `use crate::mocks::stub_data::{tools, ToolStub}` import
- **Added:** `use_server_future(crate::server::api::list_tools)?` — the `?` suspends until first resolution (D-03)
- **Loading guard:** `match tools_resource() { Some(Ok(v)) => v, _ => Vec::new() }` — empty grid during loading satisfies D-13
- **Error state:** `load_error` boolean pre-computed before `rsx!`; renders `div { style: "color:var(--danger);font-size:var(--fs-12);", "Could not load tools — check server connection." }` — verbatim UI-SPEC copy with em dash
- **ToolCard:** Updated from `ToolStub` to `crate::server::api::ToolInfo`. Field changes:
  - `tool.summary` → `tool.description`
  - `tool.status` removed — `.tgl` rendered with static class `"tgl"` (off-state default)
  - `tool.group` removed — `data-tool-group` attribute eliminated entirely
  - `class: if on { "on" }` on `.tool-card` removed (no status field)
- **ENABLE ALL / DISABLE ALL:** Static affordance buttons retained without `onclick` (D-12)
