## What Was Built

**Task 1 (api.rs):**
- `SkillInfo { name, description, category, enabled }` struct with `Serialize, Deserialize, Clone, Debug, PartialEq` derives. No `version` field (R-4).
- `#[get("/api/skills")] pub async fn list_skills()` server fn:
  - Reads `state.runtime_bundle.skill_registry.list()` synchronously (no `.await`)
  - Locks `state.runtime_bundle.active_skills` (std Mutex) in a scoped block, collects `active_names: HashSet<String>`, drops guard before registry iteration
  - Maps `SkillSource` enum to category string: Builtin→"bundled", Official→"official", Trusted→"trusted", Community→"installed", SelfCreated→"self-created"
  - Poisoned mutex produces `ServerFnError` (T-26.7-03-03 mitigated)

**Task 2 (skills.rs):**
- Removed `use crate::mocks::stub_data::{skills, SkillStub}` entirely
- `use_server_future(crate::server::api::list_skills)?` suspension pattern
- `count`, `enabled_count`, `load_error` extracted before `rsx!` (clippy.toml borrow discipline)
- Sub-copy: `"{count} loaded · {enabled_count} enabled for <code>default</code>."`
- ALL tab: `"ALL · {count}"` dynamic; BUNDLED/INSTALLED/ENABLED/UPDATES tabs static placeholders
- Error state: `"Could not load skills — check server connection."` in danger color inside `.grid`
- `SkillCard` accepts `crate::server::api::SkillInfo`; `.tgl on` class set per `skill.enabled`; `.card-meta` shows `{skill.category}` only (R-4: no version pill); no `onclick` handler (write ops deferred)
