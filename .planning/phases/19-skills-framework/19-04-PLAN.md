    1. In `crates/ironhermes-tools/src/skills_tool.rs`, add a free function:
    ```rust
    pub(crate) fn build_skill_config_header(
        skills_config: &HashMap<String, HashMap<String, serde_yaml::Value>>,
        skill_name: &str,
    ) -> Option<String> {
        let entry = skills_config.get(skill_name)?;
        if entry.is_empty() { return None; }
        let mut pairs: Vec<(String, String)> = entry
            .iter()
            .map(|(k, v)| (k.clone(), format_yaml_value_inline(v)))
            .collect();
        pairs.sort_by(|a, b| a.0.cmp(&b.0));  // deterministic ordering
        let body = pairs.iter().map(|(k, v)| format!("{} = {}", k, v)).collect::<Vec<_>>().join(", ");
        Some(format!("[Skill config: {}]", body))
    }

    fn format_yaml_value_inline(v: &serde_yaml::Value) -> String {
        match v {
            serde_yaml::Value::String(s) => s.clone(),
            serde_yaml::Value::Bool(b) => b.to_string(),
            serde_yaml::Value::Number(n) => n.to_string(),
            serde_yaml::Value::Null => "null".to_string(),
            other => serde_yaml::to_string(other).unwrap_or_default().trim().to_string(),
        }
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config.yaml skills.config.<name> → injected prompt header | user-owned config; user trusts their own values |
| Skill-author-declared config schema (HermesMetadata.config) → Phase 23 CLI consumer | schema text is part of frontmatter, scanned by Plan 05 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04-config-inject-instructions | Tampering | injected `[Skill config: ...]` header | mitigate | Plan 05's registry-load scan runs `scan_skill_content` over frontmatter AND body (D-14); this catches skills that embed injection text inside their declared config defaults. Config VALUES (from config.yaml) are user-owned, not attacker-owned; user controls their own config. If an attacker can write config.yaml they already have host access. Accept for config values; mitigate for schema strings via Plan 05 scan scope. |
| T-19-04-config-bleed | InfoDisclosure | cross-skill config values | mitigate | `skills_config.get(skill_name)` scopes lookup to the activated skill only; never returns other skills' maps (Task 2 step 1) |
| T-19-04-nondeterministic-output | Denial | cache invalidation from HashMap iteration order | mitigate | `pairs.sort_by(|a, b| a.0.cmp(&b.0))` makes header deterministic — stable across runs, safe for prompt caching (Task 2 step 1 + test_activate_config_key_ordering_stable) |
