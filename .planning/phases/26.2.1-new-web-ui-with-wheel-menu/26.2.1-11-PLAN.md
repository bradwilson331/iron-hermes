<objective>
Add client-side slash-command dispatch to the `ChatSendHandler` send fn in `crates/iron_hermes_ui/src/components/hermes_app/mod.rs` so that `/clear` (at minimum) clears the local bubble list without forwarding to the WebSocket, and unknown slash commands append a single 'unknown command' system-style bubble locally.

Purpose: UAT Test 8 caught the assistant returning a natural-language reply (`*Context cleared.* I have reset the conversation state…`) when the user typed `/clear` + Enter. That's the LLM's verbal interpretation of the literal string `/clear` — the slash command flowed straight through to inference because neither the client nor the server intercepts the `/` prefix on the message path. D-02 forbids server-side fixes in 26.2.1, and the canonical web slash commands (`/clear`, `/help`, `/status`, `/personality`) are pure UI-state operations with no server-side effect requirement, so client-side dispatch is architecturally correct.

Output: One edited file (`mod.rs`). One new private helper fn `dispatch_slash` (≈20-30 lines). One small early-return branch added at the top of the existing `send` EventHandler closure (≈4 lines). Zero CSS, zero server, zero protocol changes. D-02 preserved.
</objective>
