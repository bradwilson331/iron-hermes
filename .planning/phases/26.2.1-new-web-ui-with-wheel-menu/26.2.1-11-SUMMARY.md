# Phase 26.2.1 Plan 11: Client-side /clear slash-command dispatch (GAP-26.2.1-04) Summary

Closed GAP-26.2.1-04 by adding a `dispatch_slash` helper and an early-return slash-prefix branch to the `ChatSendHandler` `send` EventHandler in `hermes_app/mod.rs`. Typing `/clear` + Enter now empties the bubble list locally with zero WebSocket frames; unknown `/` commands append a single error-styled bubble locally; plain (non-slash) messages still flow through `ws.send_raw(ChatRequest)` exactly as before. D-02 preserved (zero diff in `src/server/` and `src/protocol.rs`). Legacy shell untouched (zero diff in `components/warp_hermes.rs`).

## What Was Built

- `dispatch_slash(cmd: &str, bubbles: &mut Signal<Vec<ChatBubble>>, next_id: &mut Signal<u64>)` — private fn at the bottom of `hermes_app/mod.rs` that splits `cmd` on the first whitespace, matches the `head` token, and dispatches:
  - `"/clear"` → `bubbles.write().clear()` (empties the local bubble list)
  - `_` (anything else) → appends a single `ChatBubble::error(id, "unknown command: {head}")` with a freshly-incremented `next_id`
- Early-return slash-prefix branch inserted into the existing `send` EventHandler closure (after the empty-trim check, BEFORE `let sid = session_id.read().clone();`):
  ```rust
  if trimmed.starts_with('/') {
      dispatch_slash(&trimmed, &mut bubbles, &mut next_id);
      return;
  }
  ```
- Inline comment block in the `send` EventHandler explaining the GAP-26.2.1-04 closure and the D-20 contract amendment.
- 5 static-grep regression tests in `#[cfg(test)] mod tests` at the bottom of `mod.rs`:
  - `dispatch_slash_helper_exists` — asserts `fn dispatch_slash` is present.
  - `send_eventhandler_intercepts_slash_prefix_before_websocket` — asserts `trimmed.starts_with('/')` and the `dispatch_slash(&trimmed, &mut bubbles, &mut next_id)` call.
  - `clear_arm_empties_bubble_list_locally` — asserts the literal `"/clear"` arm and `bubbles.write().clear()`.
  - `unknown_command_arm_appends_error_bubble_locally` — asserts `ChatBubble::error(id, msg)`.
  - `slash_branch_precedes_websocket_send_path` — asserts the slash branch's `starts_with('/')` source-text index is less than the WS path's `let sid = session_id.read().clone();` source-text index, so a future refactor cannot accidentally move the slash check below the WS send.

## Decisions Made

### Decision 1: `dispatch_slash` signature — `&mut Signal<...>` references over by-value Copy

**Chosen:** `fn dispatch_slash(cmd: &str, bubbles: &mut Signal<Vec<ChatBubble>>, next_id: &mut Signal<u64>)`

**Rationale:** The plan listed two acceptable forms — by-mut-ref OR by-value (since `Signal<T>` is `Copy` in Dioxus 0.7). Picked `&mut`-ref form because:
1. The call site reads `dispatch_slash(&trimmed, &mut bubbles, &mut next_id)` which makes the mutation intent explicit at the call site — a reader scanning the `send` closure sees the `&mut` and immediately knows the helper writes to those signals.
2. `Signal<T>`'s `.write()` and `.set()` take `&self` internally, so the `&mut` is just a marker — there's no borrow-checker cost.
3. Compiled cleanly under both default and `legacy-shell` features at first try; no need to fall back to by-value.

### Decision 2: Static-grep regression tests over functional unit tests

**Chosen:** Static-grep tests via `include_str!("mod.rs")`.

**Rationale:** Dioxus 0.7 `Signal<T>::new` requires a live `dioxus::prelude::ScopeId` and a `VirtualDom` context. The `iron_hermes_ui` crate's existing test bootstrap (see `wheel.rs`, `state.rs`, `ui_prefs.rs` tests) does not construct signals outside a component context — every existing test exercises pure data types (`Personality`, `WheelWedge`, `UiPrefs`) or pure DOM-string output. Adding a VirtualDom-rooted test would substantially expand this crate's test footprint for a single 30-line helper. The plan explicitly blessed this fallback: _"If the existing test bootstrap pattern doesn't easily support signals outside a component … skip the unit test and rely on a static-grep regression test."_

The 5 static-grep tests give better regression coverage than a single functional test would have anyway — they lock the call-site spelling, the precedence ordering (slash branch BEFORE WS path), AND the unknown-command bubble constructor. A future "refactor away the slash dispatch back to server-side" PR will fail at least 3 of the 5 tests.

### Decision 3: `ChatBubble::error` for the unknown-command arm (not `::system`)

**Chosen:** `ChatBubble::error(id, format!("unknown command: {head}"))`

**Rationale:** Inspection of `screens/chat.rs` (lines 61-86) confirms only three constructors exist: `ChatBubble::user`, `ChatBubble::assistant`, `ChatBubble::error`. No `::system` constructor exists. The plan explicitly contemplated this: _"if it does not, use `ChatBubble::error(id, text)` for the 'unknown command' bubble (visually distinct enough — error bubbles render with a distinct style; this is acceptable for the stub of unknown commands per UAT-26.2.1.md missing[1] 'skeleton stubbing them is fine')."_

The `.is-error` bubble class renders with a distinct red accent stripe and the `!` avatar (`ChatBubbleKind::Error` → `<div class="avatar error">!</div>` per `screens/chat.rs:228`), which is visually distinct enough from a normal assistant bubble that the user will not confuse it with a real reply.
