# Phase 26.2.1 Plan 10: Restore Dropped Chrome CSS + Add Per-Theme Legacy-Var Blocks Summary

**One-liner:** Restored 70 lines of chrome positioning CSS plus added 5 [data-theme] blocks reassigning the legacy var family, so AppFooter renders and theme cycling visibly flips the wheel/screens/footer palette — closes GAP-02 + GAP-03 with zero Rust changes (D-02 preserved).

## What Was Built

Two new appended sections in `crates/iron_hermes_ui/assets/site.css`:

1. **Chrome positioning + body-class toggles** (84 lines) — `.breadcrumb`, `.sys-meta`, `.app-footer`, `.app-footer .v`, `.app-footer .sep`, `.app-footer-right`, `.scanlines { opacity: 0.35 }`, `.hud-grid { opacity: 0.35 }`, plus six `body.*` class toggles (`no-scanlines`, `no-breadcrumb`, `no-footer`, `has-rail` / `:not(.has-rail)` / `:not(.on-chat)` rail visibility cluster, `density-dense .screen/.screen-title/.card/.panel`, `has-rail.on-chat #screen-chat` layout shift, `@media (max-width: 1100px)` responsive override).

2. **Per-theme legacy-var overrides** (117 lines) — 5 `[data-theme="<slug>"]` blocks each reassigning the full 20-var legacy palette (`--bg`, `--bg-deep`, `--bg-card`, `--bg-card-hover`, `--bg-input`, `--border`, `--border-faint`, `--border-strong`, `--teal`, `--teal-bright`, `--teal-dim`, `--teal-faint`, `--teal-glow`, `--text`, `--text-dim`, `--gray`, `--gray-faint`, `--green`, `--red`, `--amber`).
