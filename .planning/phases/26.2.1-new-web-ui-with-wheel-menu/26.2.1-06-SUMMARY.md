## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Dioxus rsx `class:` attribute literal-merge compile error**
- **Found during:** Task 2, first `cargo check` after writing the full ScreenChat body.
- **Issue:** rsx! cannot merge two adjacent `class:` attributes when both are non-format-string literals — the plan's suggested shape (`class: "chat-progress-row", class: if tr.done { ... } else { "is-running" }`) compiled OK in isolation but the second `class:` value was a conditional whose arms are bare string literals, which Dioxus 0.7 rejects with `error: Cannot merge non-fmt literals`.
- **Fix:** Consolidated to a single `class:` attribute. Extracted the conditional into a `let state_class = if tr.done { ... } else { "is-running" };` binding above the `rsx!` macro, then used a single format-string attribute: `class: "chat-progress-row {state_class}"`. Same pattern for `icon_class` / `icon_glyph`. The acceptance literal `class: "chat-progress-row` still grep-hits as a prefix.
- **Files modified:** `crates/iron_hermes_ui/src/components/hermes_app/screens/chat.rs`
- **Verification:** `cargo check -p iron_hermes_ui` exit 0; `cargo check --target wasm32-unknown-unknown` exit 0.
- **Committed in:** `e783f497` (part of the Task 2 commit).
