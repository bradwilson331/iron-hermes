## SID Badge Fix — Exact Diff Shape

**Before (1 line, always renders `agent:ma`):**

```rust
let sid_short: String = sid_full.chars().take(8).collect();
```

**After (4 lines, renders trailing uuid tail):**

```rust
let sid_short: String = if sid_full.len() <= 8 {
    sid_full.clone()
} else {
    sid_full[sid_full.len() - 8..].to_string()
};
```

Root cause: every web session id is formatted server-side as `agent:main:web:dm:{uuid}` (53+ chars). `.chars().take(8)` was returning the leading constant prefix `agent:ma` for every session, hiding the fact that SessionIdContext was already updating correctly on row click. Switching to `[len-8..]` exposes the uuid tail, which is the per-session unique portion.

Safety:
- Web session-id format is ASCII (uuid hex+dashes + ASCII prefix), so `String::len()` bytes == chars and `&str[len-8..]` lands on a char boundary.
- The `len() <= 8` guard handles empty/short ids without `usize` underflow.

## Helper-vs-Inline Decision

Task 3 invited an optional `truncate_sid_tail` helper + `#[cfg(test)] mod tests` unit-test trio. Chose **inline if/else** instead:

- The static-grep regression test from Task 1's `<verify>` block (asserts `sid_full[sid_full.len() - 8..]` is present and the old `chars().take(8)` form is gone) locks the contract against future regressions without adding a test module.
- Extracting a 4-line helper for a single use-site costs more lines than it saves and adds a public-API surface that is overkill for a one-line bug fix.
- Plan `<output>` §3 documented this exact choice as acceptable.

If a future plan introduces a second use-site (e.g., wheel-menu SID display, session-list mini-card), promote the inline form to `fn truncate_sid_tail(&str) -> String` at that time.
