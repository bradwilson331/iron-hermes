The bug-site in `screens/chat.rs` (around line 163-164):

```rust
let sid_full = session_id.read().clone();
let sid_short: String = sid_full.chars().take(8).collect();
```

`sid_full` for every web session is the format `agent:main:web:dm:{uuid}` (53+ chars). `.chars().take(8)` always returns `agent:ma` — the constant 17-char prefix's first 8 chars.

Fix shape — take the LAST 8 chars instead (the uuid tail), with a short-id safety guard:

```rust
let sid_full = session_id.read().clone();
let sid_short: String = if sid_full.len() <= 8 {
    sid_full.clone()
} else {
    sid_full[sid_full.len() - 8..].to_string()
};
```

Note: `String::len()` returns bytes, not chars. For an ASCII-only session-id format (`agent:main:web:dm:{uuid}`), bytes == chars and `&sid_full[sid_full.len() - 8..]` is safe. UUIDs are ASCII (hex digits + `-`). Use the byte-slice form for clarity and correctness against the known format.
