## Legacy-Shell Disposition

13 existing Warp-style components had to either be removed, gated, or kept side-by-side.

| Option | Description | Selected |
|--------|-------------|----------|
| Delete now | Remove `src/components/shell/` once new shell renders the Chat screen end-to-end. Cleanest tree but loses fallback. | |
| Feature-gate (recommended) | Keep Warp shell behind `legacy-shell` Cargo feature, default to new wheel UI. Allows rollback if the port misses something. Delete in a follow-up phase once UAT verifies. | ✓ |
| Delete after port verified | Don't gate — delete in the SAME phase but only after a `checkpoint:human-verify` UAT confirms the new shell + Chat round-trip works. | |

**User's choice:** Feature-gate (recommended).
**Notes:** Locked as CONTEXT.md D-25 + D-26. Planner has discretion on whether to rename directories (`shell/` → `shell_legacy/`) or use module-level `#[cfg(feature = "legacy-shell")]` declarations. Shared types in `src/state.rs` are NOT gated — both shells use them. Deletion of the legacy module is captured as deferred idea 26.2.14 (post-UAT).
