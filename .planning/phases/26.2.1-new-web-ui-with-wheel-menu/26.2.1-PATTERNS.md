### `src/components/hermes_app/wheel.rs` (NEW — SVG wheel + pointer drag/resize)

**Analog:** **NONE.** Closest existing primitives in the crate:
- `src/components/shell/command_palette.rs:69-92` — keyboard event handling pattern
- `src/components/shell/agent_panel.rs:23-39` — paginated local state pattern

**Pattern to use:** RESEARCH.md Pattern 2 (SVG generation) + Pattern 3 (pointer drag) + Pattern 8 (`WheelWedge::from_index` enum mapping). See RESEARCH.md lines 361-668 for the full skeleton.

**Why no analog:**
- The existing crate has zero SVG-in-RSX code beyond `<img src="...svg">` references; `wordmark.svg` (referenced in `title_bar.rs:11`) and `header.svg` are loaded as `Asset` constants, not generated.
- The existing crate has zero `pointermove`/`pointerup` at window scope; the closest is `warp_hermes.rs:505-552`'s `keydown` listener installed via `use_effect` + cleaned up in `use_drop`.

**Window listener idiom to mirror** (`warp_hermes.rs:505-552`):

```rust
#[cfg(target_arch = "wasm32")]
{
    use web_sys::KeyboardEvent as WebKeyboardEvent;

    let mut listener_slot: Signal<
        Option<wasm_bindgen::closure::Closure<dyn FnMut(WebKeyboardEvent)>>,
    > = use_signal(|| None);

    use_effect(move || {
        let Some(window) = web_sys::window() else { return; };
        let cb = wasm_bindgen::closure::Closure::<dyn FnMut(WebKeyboardEvent)>::new(
            move |ev: WebKeyboardEvent| { /* handler */ },
        );
        let _ = window.add_event_listener_with_callback("keydown", cb.as_ref().unchecked_ref());
        listener_slot.set(Some(cb));
    });

    use_drop(move || {
        if let Some(cb) = listener_slot.write().take() {
            if let Some(window) = web_sys::window() {
                let _ = window.remove_event_listener_with_callback(
                    "keydown",
                    cb.as_ref().unchecked_ref(),
                );
            }
        }
    });
}
```

**Caveats:**
- Reuse this exact closure-slot + use_drop teardown for `pointermove` and `pointerup` on the window. Do NOT `cb.forget()` (leaks + prevents cleanup). RESEARCH Pattern 3's `Box::leak` is wrong for production — copy the `listener_slot` approach instead.
- SVG hyphenated attributes MUST be quoted strings: `"stroke-width": "1"`, `"text-anchor": "middle"`, `"data-i": "{i}"`. See RESEARCH Pitfall 1.
- Geometry helpers (`polar`, `wedge_path`) at module scope, not inside `rsx!` body — RESEARCH Anti-Pattern: "Compute wheel geometry inside `rsx!` body".
- The signal-borrow safety rule (`clippy.toml`) applies: `let pos = wheel.read().position; drop(state);` BEFORE any `.write()`. The analog at `warp_hermes.rs:415-419` shows the same `drop(state);` discipline.

### Cross-cutting: Signal-borrow safety across async

**Source:** `src/components/warp_hermes.rs:585-614` (on_rerun handler) + `crates/iron_hermes_ui/clippy.toml`

**Apply to:** every async closure / spawn / use_future / use_effect that touches signals.

```rust
let mut on_rerun = move |id: u64| {
    // Read once into local, drop borrow BEFORE spawn.
    let cmd_text: Option<String> = {
        let bs = blocks.read();
        bs.iter().find(|e| e.id == id).and_then(|e| match &e.block {
            Block::Cmd { command } => Some(/* ... clone ... */),
            _ => None,
        })
    };  // <-- borrow dropped here
    if let Some(text) = cmd_text {
        let sid = session_id();  // call-as-fn clones the Copy String value
        let req = crate::protocol::ChatRequest { session_id: sid, message: text };
        let json = serde_json::to_string(&req).unwrap_or_default();
        spawn(async move {
            let _ = ws.send_raw(dioxus_fullstack::Message::Text(json)).await;
        });
    }
};
```

**Rule:** NEVER hold `GenerationalRef` / `GenerationalRefMut` / `WriteLock` across `.await`. Clippy enforces this. Pattern: read into local → drop borrow at `;` or `}` → then `.await`.

## Notes for the Planner

1. **`warp_hermes.rs` is the goldmine.** Nearly every live-wiring concern (WebSocket lifecycle, session bootstrap, slash command dispatch, signal hub composition, async-borrow discipline) is already solved there. The new Chat/Sessions/Settings screens should be near-mechanical extractions of sub-sections of that file.

2. **The legacy shell rename is mechanical.** `src/components/shell/` → `src/components/shell_legacy/`; `pub mod shell;` → `#[cfg(feature = "legacy-shell")] pub mod shell_legacy;` in `components/mod.rs`; `use crate::components::shell::*` → `use crate::components::shell_legacy::*` in `warp_hermes.rs`. Add `#[cfg(feature = "legacy-shell")]` to `pub mod warp_hermes;` and `pub use warp_hermes::WarpHermes;` lines in `components/mod.rs:2, 4`. Run `cargo clean -p iron_hermes_ui` after the rename per RESEARCH "Runtime State Inventory" row.

3. **The wheel is the highest-risk new construction.** No analog exists for window-scoped pointer events with proper closure-lifetime management. Use the `listener_slot: Signal<Option<Closure>>` + `use_drop` cleanup pattern from `warp_hermes.rs:497-552` — NOT `Box::leak` (which RESEARCH Pattern 3 suggests but is wrong for production).

4. **`use_websocket` must be hoisted.** Per RESEARCH Pitfall 5, if the chat screen unmounts (any time the user switches screens), the WebSocket connection drops. Either (a) keep all 13 screens mounted and `class:"is-active"`-toggle (RESEARCH Pattern 7 — recommended, matches `app.html`), or (b) hoist `use_websocket` to `HermesApp` and pass `ws` via context.

5. **Stub data is production code for this phase.** Drop the `#[cfg(any(test, feature = "demo"))]` gate that wraps `demo_block_entries()` in `state.rs:320`. New stub functions in `mocks/stub_data.rs` should be unconditional `pub fn`s.

6. **CSS load order matters.** RESEARCH Pattern 1 specifies `tokens → site → wheel → screens → components`. Get this wrong and accent variables, wheel positioning, or screen overrides will be wrong.

7. **The `use_context_provider` strategy in the new shell should split signals.** The legacy `ShellSettings` struct bundles `Signal<Personality>` into a `Copy` struct (`state.rs:278-282`). For the new shell, prefer three separate provides: `use_context_provider(|| Signal::new(WheelState::default()))`, `use_context_provider(|| Signal::new(Screen::default()))`, `use_context_provider(|| Signal::new(UiPrefs::default()))`. This way a wheel-position write doesn't invalidate consumers of the theme.
