**Before (dispatch_slash):**

```rust
fn dispatch_slash(
    cmd: &str,
    bubbles: &mut Signal<Vec<ChatBubble>>,
    next_id: &mut Signal<u64>,

    match head {
        "/clear" => { bubbles.write().clear(); }
        _ => {

            bubbles.write().push(ChatBubble::error(id, msg));
        }

```

**Before (call site):**
```rust

    dispatch_slash(&trimmed, &mut bubbles, &mut next_id);
```

**After (dispatch_slash):**
```rust
fn dispatch_slash(
    cmd: &str,
    bubbles: &mut Signal<Vec<ChatBubble>>,
    _next_id: &mut Signal<u64>,
) -> bool {
    match head {
        "/clear" => {
            bubbles.write().clear();
            true
        }
        // Per D-26.2.1-13-B: unknown slashes fall through to ws.send_raw
        _ => false,
    }
}
```

**After (call site):**
```rust
if trimmed.starts_with('/') && dispatch_slash(&trimmed, &mut bubbles, &mut next_id) {
    return;
}
```

## Decisions Made

1. **GAP-06 Strategy: z-index raise (Option A)** — BL/BR corners share same z-index 50 as footer; raising to 60 is 1 CSS rule, smaller than moving `bottom` offset which would require adjusting both corners individually and might shift them out of design position.

2. **GAP-07 Strategy: !important** — The plan's pre-analysis confirmed class-name parity and that the `body.no-scanlines` class IS applied correctly by `theme_effects.rs`. Adding `!important` is the smallest diff that reliably defeats any future animation override. Removing a hypothetical `display: block` override on `.scanlines` was not needed — no such rule exists.

3. **GAP-08 bool return + fall-through** — Plan 11 over-corrected by intercepting ALL slash commands client-side. The correct split: UI-state commands (`/clear`) return `true` (handled, stop); server-effecting commands (`/research`, future tool-routed commands) return `false` (unhandled, fall through to WS). The `_next_id` parameter is kept with underscore prefix to preserve call-site signature without touching callers.

4. **GAP-09 None filter (USER-APPROVED D-02)** — Exact change authorized by user in round-2 UAT review. Scope strictly bounded: ONE file (api.rs), ONE function (list_sessions), ONE argument (platform filter). No other server/* files touched.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Static-grep test self-reference in unknown_command_arm test**
- **Found during:** Task 3 verification (cargo test run)
- **Issue:** `!MOD_RS.contains("ChatBubble::error(id, msg)")` and subsequent `!MOD_RS.contains("unknown command:")` assertions were self-defeating — the assertion text itself contained the forbidden substrings, which `include_str!("mod.rs")` makes visible to the test.
- **Fix:** Changed the negative-assertion pattern to use runtime string construction: `["unknown", " command: {head}"].concat()` so the forbidden string never literally appears in the source file. Also simplified the assertions — the `format!("unknown command: {head}")` pattern is the unique discriminator; checking for `ChatBubble::error` would be a false positive because other send-handler paths legitimately use it.
- **Files modified:** `crates/iron_hermes_ui/src/components/hermes_app/mod.rs`
- **Verification:** `cargo test -p iron_hermes_ui --bin iron_hermes_ui` → 37 passed
- **Committed in:** `d6a8eae1` (Task 3 commit)
