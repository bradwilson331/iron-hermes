## CSS Asset Contract

The five CSS files from the bundle are copied **verbatim** into `crates/iron_hermes_ui/assets/` and linked from the root `App` component. No Tailwind conversion. No CSS-in-Rust. (CONTEXT D-07)

| File | Destination | Linked via |
|------|------------|-----------|
| `project/tokens.css` | `assets/tokens.css` | `document::Link { rel: "stylesheet", href: TOKENS_CSS }` |
| `project/site.css` | `assets/site.css` | `document::Link { rel: "stylesheet", href: SITE_CSS }` |
| `project/wheel.css` | `assets/wheel.css` | `document::Link { rel: "stylesheet", href: WHEEL_CSS }` |

| `project/components.css` | `assets/components.css` | `document::Link { rel: "stylesheet", href: COMPONENTS_CSS }` |

Asset constants declared at module top in `src/app.rs`:

```rust
const TOKENS_CSS:     Asset = asset!("/assets/tokens.css");
const SITE_CSS:       Asset = asset!("/assets/site.css");
const WHEEL_CSS:      Asset = asset!("/assets/wheel.css");
```

Load order: tokens.css → site.css → wheel.css → screens.css → components.css. Later files override earlier ones (screens.css has per-screen overrides that must beat site.css base rules).

If any bundle CSS references `assets/fonts/...` relative paths that resolve incorrectly after copy, the path string is rewritten (not the styles themselves). CONTEXT D-07 Claude's Discretion.

## Theme Switching Contract

Theme switching writes `data-theme` on `<html>` element (CONTEXT D-14):

```rust
// In Dioxus: via web_sys
let window = web_sys::window().unwrap();
let document = window.document().unwrap();
let html = document.document_element().unwrap();
html.set_attribute("data-theme", theme_name).unwrap();
```

On first paint, read from `localStorage` key `ih.ui.theme` (CONTEXT D-15). If absent, default to `"slate-dark"`. Apply in a `use_effect(|| ...)` that fires once on mount to avoid SSR/WASM hydration mismatch.

```rust
// Pseudocode — planner fills in Dioxus use_effect wiring
use_effect(move || {
    let theme = read_local_storage("ih.ui.theme").unwrap_or("slate-dark");
    set_html_data_theme(theme);
});
```

Theme persistence: write `ih.ui.theme` to localStorage whenever the user changes the theme in Settings or Tweaks panel.

### Wheel State Signal

```rust
#[derive(Clone, PartialEq, Debug)]
pub struct WheelState {
    /// CSS position (pixels from viewport top-left) of the .wheel-shell element
    pub position: (f64, f64),
    /// Current --wheel-size value (px), range 240..=640
    pub size: f64,
    /// Currently active (highlighted) wedge
    pub active_wedge: WheelWedge,
}

impl Default for WheelState {
    fn default() -> Self {
        Self {
            position: (24.0, 24.0),  // top: 24px, left: 24px per app.html
            size: 220.0,              // app.html default --wheel-size: 220px
            active_wedge: WheelWedge::Chat,
        }
    }
}
```

Provided at App root: `use_context_provider(|| Signal::new(WheelState::default()))`. Children access via `use_context::<Signal<WheelState>>()`.

**Persistence:** On every state change, write to `localStorage` key `ih.ui.wheel` as JSON. On mount, read and restore. JSON shape: `{"x": 24.0, "y": 24.0, "size": 220.0, "active": "chat"}`.

### Wheel Interaction Model (CONTEXT D-11)

| Interaction | Trigger | Effect |
|-------------|---------|--------|
| Hover wedge | `onmouseenter` on `.wheel-wedge[data-i]` | Set `active_wedge` to wedge index → hub label/sub/glyph update |
| Click wedge | `onclick` on `.wheel-wedge[data-i]` | Set `active_wedge` + launch screen |
| Click hub | `onclick` on `#wheel-hub` | Launch `active_wedge` screen (suppress if just dragged > 4px) |
| Drag rim | `pointerdown`/`pointermove`/`pointerup` on `.wheel-rim` | Translate `.wheel-shell` position; use `setPointerCapture` |
| Drag resize ring | `pointerdown`/`pointermove`/`pointerup` on `#resize-ring` | Update `--wheel-size` CSS var; delta = avg(dx, dy) |
| Arrow keys | `ArrowRight`/`ArrowDown`: next wedge; `ArrowLeft`/`ArrowUp`: prev wedge | When focus not in input/textarea |
| Enter | `keydown` Enter | Launch active wedge |
| `+`/`=` | Keydown | Increase wheel size by 24px |
| `-`/`_` | Keydown | Decrease wheel size by 24px |

**Launch action:** Sets `active_screen` signal to the matching `Screen` variant, which triggers `is-active` class on the corresponding `<section>`. Also updates body class (`on-chat` when Chat is active) and breadcrumb/footer labels.

**Drag suppression:** If pointer moves > 4px during rim drag, suppress the following `click` event.

## Accessibility Contract

### Focus Rings

Use `outline: 2px solid var(--accent)` on `:focus-visible`. Do NOT use `box-shadow` for focus rings (terminal aesthetic, plus box-shadow on dark terminal themes creates visual noise). (Consistent with `components.css` `.ih-btn:focus-visible` rule.)

```css
/* Override any user-agent focus ring */
:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 2px;
}
```

### Wheel Accessibility

- Wheel hub: `role="button"` + `aria-label="Launch {active_wedge_label}"` (updates dynamically)
- Each wedge path: `role="button"` + `aria-label="{wedge.label} — {wedge.sub}"` + `tabindex="0"`
- Tab order: hub first, then wedges in index order (0..11)
- Enter/Space on focused wedge: launch that wedge
- Arrow keys navigate wedges (see keyboard nav in Wheel Interaction Model above)

### Screen Sections

- Each `<section class="screen">`: `role="main"` when `is-active`; `role="region"` when inactive
- Active screen section: `aria-current="page"`
- Inactive screen sections: `aria-hidden="true"`

### Icon-Only Buttons

All icon-only buttons must have `aria-label`:

```rust
button {
    aria_label: "Close Tweaks panel",
    class: "tweaks-x",
    "×"
}
```

### Chat Input

- Textarea: `aria-label="Chat input"` + `aria-multiline="true"`
- Send button: `aria-label="Send message"`
- Chat stream: `aria-live="polite"` (announces new messages to screen readers)
- Tool progress rows: `aria-live="off"` (too noisy for screen readers during streaming)

### Chat Avatars

- Shield avatar image: `alt="Hermes"` (or `alt=""` if decorative — not informational)
- User avatar: no alt needed (decorative text glyph)

## Performance Contract

### CSS Loading

Single CSS load: 5 files linked in order at App root. No per-component CSS split in 26.2.1. The Dioxus asset pipeline (via `dx serve` / `dx build`) handles hashing and caching.

### Theme Flash Prevention

Default theme `slate-dark` is rendered by setting `data-theme="slate-dark"` on the server-rendered HTML root. On the client, `use_effect` reads localStorage and updates if different. This prevents FOUC on first load (user always sees slate-dark until client runs, then transitions if they had a different theme saved).

Alternatively, inject a tiny `<script>` in `<head>` before CSS load that reads localStorage and sets `data-theme` synchronously (pre-CSS execution). This eliminates all flash. Planner decides which approach to use.

### Scan-Bar and HUD Animations

All HUD animations use CSS keyframes only — no JavaScript `requestAnimationFrame`. (CONTEXT Claude's Discretion — web port explicitly permits CSS animations; the "no animation except knight-rider" rule applies to the TUI/CLI, not the web.)

```css
/* These are already in site.css — do not re-implement in JS */
@keyframes scan-bar-move { ... }
@keyframes blink { ... }
@keyframes hub-aura-pulse { ... }
@keyframes arrow-blink { ... }
@keyframes ring-pulse { ... }
@keyframes tweaks-in { ... }
@keyframes screen-in { ... }
```

### Signal Borrow Safety

All signal reads/writes must follow Dioxus 0.7 clippy rules (from `clippy.toml`):

- Do NOT hold `GenerationalRef`, `GenerationalRefMut`, or `dioxus_signals::WriteLock` across `.await` points.
- Pattern: read signal into local binding before any `await` call.

```rust
// CORRECT
let screen = active_screen.read().clone();
some_async_fn().await;

// WRONG — GenerationalRef held across await
let _ = some_async_fn().await;
let _ = active_screen.read(); // ReadLock must not span .await above
```

### WebSocket Reconnection

The WebSocket client should auto-reconnect on connection drop. Implement via `use_resource` or `use_effect` with a reconnect loop. Display `Connection lost · Reconnecting…` in the chat stream during reconnect attempts.
