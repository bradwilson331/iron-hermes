<objective>
Land the runtime theming + body-class effects + Tweaks panel — the three remaining pieces that complete the chrome surface area for the new shell.

1. Create `theme_effects.rs` with `#[component] pub fn ThemeEffects() -> Element`. This component RENDERS NOTHING (returns `rsx! {}`) — its sole job is to host three `use_effect` blocks that listen to context signals and write to the DOM. Render-state is driven by `use_signal(|| false)` for the open/closed flag (local; not in context). Each toggle row writes the matching `UiPrefs` field via `use_context::<Signal<UiPrefs>>()`.

   ```

   ```

Purpose: D-13/D-14/D-15 require all 5 themes to ship with persistent selection; D-16/D-17 require the Tweaks panel + the rail body-class plumbing. With Plan 04 wiring wheel state and Plan 05 wiring theme + tweaks, the visual UI is functionally complete (modulo Wave 3's chat/sessions/settings/stub-screen content).

| `no-footer` | hides `.app-footer` | `!prefs.footer` |
