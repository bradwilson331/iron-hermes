## Decisions Made

- **ThemeEffects returns `rsx! {}`** rather than rendering chrome — bundling three `use_effect` blocks in one no-DOM component is the cleanest way to keep `mod.rs` focused on layout and to ensure the effects subscribe to the same context graph as the other shell children.
- **Class-toggle, not conditional mount** for the panel — `class: if is_open { "is-open" }` matches the prototype's CSS animation hook (`app.html` line 131: `.tweaks.is-open { display: flex }`) and keeps signal subscriptions alive across open/close cycles.
- **Per-row state snapshot at component top** — read every signal field (`current_theme`, `current_accent`, `current_wheel_size`, `current_scanlines`, `current_breadcrumb`, `current_footer`, `current_rail`, `current_density`, `is_open`) into Copy/cloned locals BEFORE the rsx! body. This keeps signal borrows out of nested onclick closures (signal-borrow safety per `clippy.toml`).
- **Slider mirrors UiPrefs AND WheelState in one event** — single `oninput` writes both `prefs.with_mut(|p| p.wheel_size = val)` and `wheel_state.with_mut(|ws| ws.size = val)`. No extra watcher `use_effect` needed; the wheel SVG (Plan 04 consumer) reacts on the same render cycle.
- **JsCast imported at module scope under cfg(wasm32)** — `theme_effects.rs` top: `#[cfg(target_arch = "wasm32")] use wasm_bindgen::JsCast;`. The four wasm32 cfg-blocks include effect 3's `dyn_into::<HtmlElement>` call which requires `JsCast` in scope; importing once at module top is cleaner than per-block.
