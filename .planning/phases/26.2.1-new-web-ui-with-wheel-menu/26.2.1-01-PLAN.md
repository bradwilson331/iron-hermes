<objective>
Land the structural foundation for the new wheel-driven shell:

1. Copy the five bundle CSS files + Ioskeley Mono fonts + shield PNG into `crates/iron_hermes_ui/assets/`.
2. Add the `legacy-shell = []` Cargo feature (default OFF) and extend `web-sys` features for localStorage + DOM token list + CSS style declaration.
3. Rename `src/components/shell/` → `src/components/shell_legacy/` and gate it (plus `warp_hermes`) behind `#[cfg(feature = "legacy-shell")]`.
4. Introduce an empty `src/components/hermes_app/` module with a stub `HermesApp` component so the default build links cleanly.
5. Update `src/app.rs` to: (a) declare new asset constants and link all 5 CSS files via `document::Link`, in order tokens → site → wheel → screens → components; (b) branch the root child on the feature flag.

Purpose: Unlocks every downstream plan. Plan 02 lands state types in parallel; Plans 03–08 require this gated, asset-linked skeleton to mount their components against.

Output: A repository state where `cargo check -p iron_hermes_ui` and `cargo check -p iron_hermes_ui --features legacy-shell` both pass, the new shell is the default mount point (rendering an empty placeholder for now), and all five bundle CSS files plus assets are linked into the page head.
</objective>
