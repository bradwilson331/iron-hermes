## Deviations from Plan

### 1. [Rule 1 ‚Äî Bug] Dioxus 0.7 rsx! parser rejected consecutive `#[cfg]`-prefixed sibling elements

- **Found during:** Task 2, first `cargo check` after the initial app.rs rewrite.
- **Issue:** The first naive shape ‚Äî five sibling `#[cfg(feature = "legacy-shell")] document::Link { ... }` lines followed by two sibling `#[cfg(...)] crate::components::*::App {}` lines ‚Äî caused `error: expected identifier --> src/app.rs:62:9` at the first `#[cfg]`. The rsx! macro accepts `#[cfg(...)]` on a single child, but stacking multiple consecutive cfg-gated children at the top of the macro body produced an unrecoverable parse error.
- **Fix:** Refactored to two item-scope helper functions, `legacy_links() -> Element` and `root_shell() -> Element`, each defined twice with mirror `#[cfg(feature = "legacy-shell")]` / `#[cfg(not(feature = "legacy-shell"))]` attributes. The rsx! body calls them via `{legacy_links()}` and `{root_shell()}`. This keeps the branch compile-time (no `if cfg!(...)`) and preserves both acceptance criteria checks (cfg attributes still appear within 80 chars of `HermesApp`/`WarpHermes`).
- **Files modified:** `crates/iron_hermes_ui/src/app.rs`
- **Commit:** `cc477991`

### 2. [Decision Log ‚Äî D-25/D-26] Kept legacy CSS Link tags rather than deleting

Per CONTEXT D-25 ("legacy shell continues to compile") and D-26 ("kept as UAT fallback"), all five legacy CSS asset constants (`TAILWIND_CSS`, `MAIN_CSS`, `DESIGN_TOKENS_CSS`, `WARP_IH_CSS`, `SCANNER_ANIM_CSS`) and their `document::Link` emissions are retained under `#[cfg(feature = "legacy-shell")]`. The underlying `.css` files in `assets/` were untouched (Task 1 only copied new files in ‚Äî it did not delete legacy ones). This matches Plan 01's success criterion that `cargo check --features legacy-shell` continues to compile.

### 3. No deviations from CONTEXT decisions D-01, D-07, D-09

- D-01 (preserve legacy shell): WarpHermes composition root and shell_legacy primitives are unchanged in behaviour. Only `warp_hermes.rs`'s import path was rewritten ‚Äî every other byte is identical.
- D-07 (CSS load order): tokens ‚Üí site ‚Üí wheel ‚Üí screens ‚Üí components emitted in that exact sequence in `app.rs` (lines 56‚Äì60).
