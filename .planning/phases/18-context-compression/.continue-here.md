## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Engine mutates caller's `messages` Vec before running invariant check | SummarizingEngine and LocalPruningEngine pruned/inserted into `&mut Vec<ChatMessage>` then ran `check_orphan_invariant` at the end; on Err the corrupted Vec was forwarded to the LLM and Anthropic returned 400 "tool_use without tool_result". | blocking | Fixed in `883f8ce`: engines now snapshot pre-mutation and restore on any Err; `pre_chat_compress` does belt-and-suspenders snapshot-restore. Any new engine MUST either: (a) snapshot-restore, or (b) operate on a local Vec and only assign on success. |
| Worktree branches off stale base | Spawned executor worktrees repeatedly started 20+ commits behind develop and missed all Phase 18 files until they `git merge develop` first. | advisory | Every executor prompt this session included an explicit `git merge develop` bootstrap step. Keep doing this until worktree base-branch selection is fixed upstream. |
| No-op compression is silent | Prior to `e360b0e` there was no visible signal when compression was asked to run but short-circuited (e.g. `protect_last_tokens` protected everything). UAT looked broken but was structurally correct. | advisory | Observability landed: `summarizing_engine: no-op reason=...` and `local_pruning_engine: no-op reason=...` info logs now emit at each decision point. |
