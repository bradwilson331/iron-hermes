## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Coupling independent builders in a single `if let` guard | Tracker attachment was tied to hook attachment in engine_factory; the `None` case silently dropped BOTH. This hid the structural gap for multiple phases — the tracker was wired on AgentLoop, the pressure_warning.rs implementation existed with tests, but no production path actually fired it. | advisory | When attaching optional builders, attach each one independently: `if let Some(h) = hooks { e = e.with_hooks(h, &sid); }` and `if let Some(t) = tracker { e = e.with_pressure_tracker(t); }`. Separate concerns. |
| YAML config with mixed TOML syntax | `~/.ironhermes/config.yaml` silently falls back to defaults on malformed YAML without a visible parse-error log. User had `[memory] provider = "sqlite"` (TOML in YAML) and spent multiple test runs chasing ghost config values. | advisory | Config load path should log loud on parse failure rather than silently returning `Config::default()`. (Config::load_from currently propagates errors via anyhow, but the error surface to the user during `cargo run` is apparently suppressed.) |
