<objective>
Close 18-UAT.md Test 4 gap "PressureTracker fires on agent-side compression pressure band from CLI" by decoupling pressure tracker + session_id attachment from hook registry attachment in `engine_factory::build_context_engine`.

Live UAT 2026-04-14T03:36 ran with `agent.compression_threshold = 0.05`; session token ratio climbed into `[0.0425, 0.05)` (the 85%-of-threshold pressure band) and compression subsequently fired at `ratio=0.07`, but **no** `WARN ... context pressure warning` log appeared, **no** `ContextPressure` hook event was emitted, and **no** transient `[CONTEXT PRESSURE HIGH — earlier history may soon be summarized]` user-facing message was injected.

Root cause (verified, from 18-UAT.md "Gaps" section):

1. `crates/ironhermes-agent/src/engine_factory.rs:55` and `:104-106` gate **both** `with_hooks(h, &sid)` **and** `with_pressure_tracker(t)` behind a single combined guard:
   ```rust
   if let (Some(h), Some(t)) = (hooks, tracker) {
       e = e.with_hooks(h, &sid).with_pressure_tracker(t);
   }
   ```
2. CLI calls `attach_context_engine(..., hooks = None)` at `crates/ironhermes-cli/src/main.rs:303`, so the guard short-circuits and **neither** tracker **nor** session_id reaches the engine.
3. `context_engine.rs:144` (and the mirror at `:269` in `SummarizingEngine`) runs `check_and_maybe_emit` only when `(Some(tracker), Some(sid))` — both are absent → every pressure channel silently no-ops.
4. The agent-level `PressureTracker` attached on `AgentLoop` by `agent_wiring.rs:52` is a DIFFERENT tracker from the one the engine's `compress` loop consults; it is not used by the engine's pressure check.

Fix direction (minimal, surgical, preserves 18-10/18-11/18-12 behavior):

- Split `LocalPruningEngine::with_hooks(registry, session_id)` into two independent builders: `with_hooks(registry)` and `with_session_id(session_id)`. Mirror the split on `SummarizingEngine`.
- In `engine_factory.rs`, attach `session_id` unconditionally (it is always available — the factory takes `session_id: impl Into<String>`). Attach `tracker` when `Some`. Attach `hooks` when `Some`. Remove the combined `if let (Some, Some)` guard.

Purpose: make PressureTracker actually function under the CLI default wiring (no hook registry). Unblock UAT Test 4 for live pass.

Output:
- `context_engine.rs`: `with_hooks` signature narrowed to `(Arc<HookRegistry>) -> Self`; new `with_session_id(impl Into<String>) -> Self`.
- `summarizing_engine.rs`: mirror split on `SummarizingEngine`.
- `engine_factory.rs`: `build_local` closure + `summarizing` arm use three independent builder branches (`.with_session_id(&sid)`, `if let Some(t) = tracker`, `if let Some(h) = hooks`).
- RED test added to `context_engine.rs` (or a new integration test) that constructs a `LocalPruningEngine` with tracker + session_id but NO hooks, drives a message load into the 85% band, and asserts `check_and_maybe_emit` returned `true`. Currently (before fix) this test would compile but `pressure_warning_fired` would be `false`.
- GREEN test above passes after the fix.
- All existing tests in `agent_wiring.rs`, `context_engine.rs`, `engine_factory.rs` (if any) continue to pass unchanged or updated call-site for the signature change.
