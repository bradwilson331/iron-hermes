```rust
let mut skill_context_parts: Vec<String> = Vec::new();
if let Some(ref skill_reg) = skill_registry_tick {
    for skill_name in &job.skills {
        match skill_reg.read_content(skill_name) {
```

```rust
/// Resolve skill content for a cron job, prepending to the prompt.
/// Returns the full agent input with skill context.
/// Per D-08: skill content appears before the task prompt.
/// Per D-09: missing skills produce a warning and are skipped.
pub(crate) fn resolve_skill_context(
    registry: &ironhermes_core::SkillRegistry,
    skill_names: &[String],
) -> String {
    let mut parts = Vec::new();
    for name in skill_names {
        match registry.read_content(name) {
            Some(content) => parts.push(format!("## Skill: {}\n\n{}", name, content)),
            None => tracing::warn!(skill = %name, "Skill not found at tick time - skipping"),
        }
    }
    parts.join("\n\n---\n\n")
}
```

Then use this function in the tick loop instead of inline code.

| SkillRegistry -> PromptBuilder | Catalog text injected into system prompt (trusted, operator-controlled) |
| SkillRegistry -> CronJob | Skill content prepended to cron job prompt (trusted, operator-controlled) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | PromptBuilder catalog injection | accept | Skill catalog text comes from SkillRegistry which reads operator-installed SKILL.md files. Same trust boundary as SOUL.md and AGENTS.md which are already injected into the system prompt without scanning. |
| T-07-06 | Tampering | Cron skill resolution | accept | Skill content at cron tick time comes from the same SkillRegistry. The `job.skills` field is set by the agent via the cronjob tool, which already has prompt injection scanning (scan_cron_prompt). Skill content itself is operator-installed. |
| T-07-07 | Denial of Service | Large skill catalog | accept | Catalog is one line per skill (name + description). Even with 100 skills, this is under 10KB of prompt tokens. Operator controls how many skills are installed. |
</threat_model>
