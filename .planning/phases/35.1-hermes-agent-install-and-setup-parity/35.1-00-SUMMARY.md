## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] doctor.rs lib.rs re-export causes circular reference**
- **Found during:** Task 1 build (first attempt)
- **Issue:** Plan said to add `pub mod doctor;` to lib.rs so integration tests can reach it. When compiled as part of the lib crate, `ironhermes_cli::status_cmd::current_profile()` in doctor.rs is a self-reference — the lib crate cannot reference itself as `ironhermes_cli`.
- **Fix:** Removed `pub mod doctor;` from lib.rs. The `mod doctor;` declaration in main.rs is sufficient — doctor.rs compiles as part of the binary crate where `ironhermes_cli` refers to the lib crate (no cycle). Integration tests for doctor behavior already exist in `tests/doctor_integration.rs` using binary subprocess invocation, which doesn't require lib re-export.
- **Files modified:** crates/ironhermes-cli/src/lib.rs (no change from original), crates/ironhermes-cli/src/doctor.rs (fixed reference)
- **Commit:** 94bb35a1

**2. [Rule 2 - Scope] setup_wizard.rs already exists with real Phase 23 tests**
- **Found during:** Task 2 start
- **Issue:** Plan described creating a new `tests/setup_wizard.rs`, but the file already existed with 14 real tests from Phase 23.
- **Fix:** Appended the 6 Wave 0 stubs after the existing Task 3 section rather than creating/replacing the file. All existing tests preserved.
- **Files modified:** crates/ironhermes-cli/tests/setup_wizard.rs (append only)
