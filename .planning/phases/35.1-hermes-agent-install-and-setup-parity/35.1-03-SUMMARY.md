## has_runnable_llm Signature and Check Ordering

```rust
fn has_runnable_llm(config: &Config, hermes_home: &std::path::Path) -> bool
```

Check ordering (RESEARCH Pitfall 6 — env vars FIRST because dotenvy loaded before preflight):
1. `std::env::var("OPENROUTER_API_KEY"|"ANTHROPIC_API_KEY"|"OPENAI_API_KEY")` — post-dotenvy process env
2. Raw `.env` file scan: `l.starts_with(key) && l.len() > key.len()` — T-35.1-01 empty-value bypass guard
3. `config.model.base_url` lowercase contains `"localhost"` or `"127.0.0.1"` — Ollama escape hatch
4. `config.model.api_key` non-empty — deprecated but still accepted by `validate()` (backward compat)

## Integration Test Setup Nuances

**Minimal valid config.yaml for tests:** Uses `providers.openrouter.api_key_env: OPENROUTER_API_KEY` — this is a structural field that satisfies `config.validate()` (validate only checks the field is non-empty, not that the env var resolves). `has_runnable_llm` then checks if the env var is *actually set*, which it is not in the trigger test.

**Why not use `model.api_key: placeholder`:** Would have triggered Check 4 (backward-compat path) in `has_runnable_llm`, causing the test to not trigger the wizard. The `providers.openrouter.api_key_env` approach correctly separates validate() (structural) from has_runnable_llm (runtime).

## Decisions Made

1. **Added Check 4 (deprecated model.api_key):** The pre-existing `preflight_banner_appears_for_required_missing_prereq` toolset integration test sets `model.api_key: test_key_for_preflight_test` to pass `validate()`. Without Check 4, the D-08 detection would fire for this test (the key exists in config.yaml but not in env vars or .env), causing the wizard to launch and breaking the D-17 test. Check 4 preserves backward compatibility for users still on the deprecated config format.

2. **env_lock placement:** The `env_lock()` helper was placed between the two new d07_d08 tests (after the trigger test, before the negative test) since Rust compiles the file linearly — placement between callers is idiomatic and avoids forward-reference warnings.

3. **Integration test config.yaml design:** Used `providers.openrouter.api_key_env` (structural field) rather than `model.api_key` (deprecated) to cleanly separate validate() compliance from has_runnable_llm signal detection.
