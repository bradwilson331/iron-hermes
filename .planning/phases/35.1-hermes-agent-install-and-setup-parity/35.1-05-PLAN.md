<objective>
Gap B closure — `run_skills_section` should proactively install skills from the project repo into HERMES_HOME before scanning for prereqs.

Manual smoke testing revealed: `/Users/twilson/test/skills/` is empty. The current `run_skills_section` only configures prereqs for already-installed skills — if the dir is empty, it prints "No skills installed" and returns immediately. A full setup should install the skill library first.

Fix: at the top of `run_skills_section`, detect the project's skills source directory and copy skills into HERMES_HOME/skills/. Source detection order:
1. `IRONHERMES_SOURCE` env var (explicit path to project root, user-settable)
2. Walk up from `std::env::current_exe()` until a dir containing both `Cargo.toml` and `skills/` is found (dev-build detection)
3. If neither found, skip silently — graceful degradation for production installs

After installation, the existing SkillRegistry prereq loop runs normally against the newly-populated skills dir.

Project skill dirs (confirmed at `/Users/twilson/code/ironhermes/`):
- `skills/`: apple, autonomous-ai-agents, creative, data-science, devops, diagramming, dogfood, domain, email, feeds, ...
- `optional-skills/`: autonomous-ai-agents, blockchain, communication, creative, devops, dogfood, email, health, mcp, ...

Both are copied into `HERMES_HOME/skills/`. Existing files not overwritten (skip-if-exists).
