# Phase 35.1 Plan 05: Skills Source Installation Summary

**One-liner:** Gap-B closure — `run_skills_section` now auto-installs skills from the project repo (via `IRONHERMES_SOURCE` or `current_exe()` walk) into `HERMES_HOME/skills/` before scanning for prereqs.

## What Was Built

Two new helper functions added to `crates/ironhermes-cli/src/setup.rs` and integrated into `run_skills_section`:

**`fn find_project_skills_source() -> Option<PathBuf>`**
- Checks `IRONHERMES_SOURCE` env var first (explicit override)
- Falls back to `current_exe()` parent walk (up to 10 levels) looking for a dir containing both `Cargo.toml` and `skills/` subdir
- Returns `None` for graceful degradation (production installs where binary ships without source)

**`fn install_skills_from_source(source_root, hermes_home) -> Result<()>`**
- Iterates `skills/` and `optional-skills/` inside `source_root`
- For each category subdirectory: creates `hermes_home/skills/{category}/`, copies each file (skip-if-exists — preserves user-installed or newer versions)
- Silently skips missing source subdirs, non-dir entries at category level, non-file entries inside category dirs

**`run_skills_section` changes:**
- `create_dir_all(hermes_home/skills/)` ensures dir exists before SkillRegistry scan
- Calls `find_project_skills_source()` + `install_skills_from_source()` before any registry load
- Removed the early-return-on-missing-dir guard (no longer needed after create_dir_all)
