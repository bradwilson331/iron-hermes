### Human Verification Required

The following behaviors require interactive testing that cannot be verified programmatically. These were pre-declared in `35.1-VALIDATION.md` as Manual-Only Verifications.

### 1. First-run auto-launch (D-07)

**Test:** Set `HERMES_HOME=/tmp/test-home-fresh` (no config.yaml, no .env), then run `ironhermes` bare.
**Expected:** Wizard prompt appears immediately — the "Full setup (all sections)?" choice is presented before any REPL starts.
**Why human:** Requires full binary invocation in a fresh temp home directory. The d07_d08 integration test covers the binary-level routing, but the interactive TTY experience needs manual confirmation.

### 2. Setup wizard doctor output and completion summary (D-03/D-12)

**Test:** With a valid API key in `.env`, run `ironhermes setup` and complete the wizard.
**Expected:** After wizard sections, "IronHermes Doctor" header appears with health check lines, followed by "Setup complete!" with Provider, Model, and a next-step hint.
**Why human:** Interactive terminal with readline — test seams bypass rustyline and cannot observe the full printed output sequence end-to-end.

### 3. Fast-path skips skills and terminal sections (D-11)

**Test:** Run `ironhermes setup`, press Enter at the "Full setup?" prompt (choosing quick/default).
**Expected:** No skills prompts, no terminal/cwd prompts appear. Wizard completes with provider+model only, then doctor + summary.
**Why human:** The `if full_setup` branch is source-text verified, but the interactive flow needs human confirmation that pressing Enter correctly skips to the fast path.

### 4. Ollama users NOT prompted for setup (D-08)

**Test:** Set `model.base_url=http://localhost:11434` in config.yaml, clear all API keys from .env, run `ironhermes`.
**Expected:** Wizard does NOT launch — agent starts normally (or gives an Ollama-specific error, not a setup wizard prompt).
**Why human:** Requires either a running local Ollama instance or a controlled `HERMES_HOME` environment. The unit test `has_runnable_llm_returns_true_when_local_base_url_configured` covers the logic path, but the binary-level routing with a real localhost config needs confirmation.
