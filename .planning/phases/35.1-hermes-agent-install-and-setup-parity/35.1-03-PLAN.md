Wave 2 (parallel with Plan 02) — close the final remaining gap from RESEARCH § "Key Architecture Insight": D-07/D-08 first-run LLM detection in `preflight.rs`. Implement the `has_runnable_llm` helper and call it inside the existing valid-config branch of `run_preflight_check`. The detection signal: any non-empty API key (OPENROUTER/ANTHROPIC/OPENAI) in process env OR in raw .env file, OR a localhost/127.0.0.1 base_url in config (Ollama escape hatch) — both absent triggers setup.

**Critical constraint (RESEARCH Pitfall 3):** The Phase 23 preflight outer gate condition `matches!(cli.command, Some(Commands::Chat {..}) | Some(Commands::Gateway {..}) | None)` is byte-for-byte LOCKED. This plan adds a new check INSIDE the existing `Ok(config)` arm after `config.validate()`, NOT by widening that outer match. The D-08 check is co-located with the existing config-validity check; it cannot reach commands that the Phase 23 gate excludes.

**Parallel-safe with Plan 02:** This plan modifies `preflight.rs` and `tests/doctor_integration.rs`. Plan 02 modifies `setup.rs` and `tests/setup_wizard.rs`. Zero file overlap — both plans can execute concurrently in Wave 2.

Purpose: Without this, a user with a valid config.yaml but no API key configured gets a cryptic "model_provider unauthorized" error on first chat turn instead of being walked through setup. D-07/D-08 closes the loop on the "fresh install → working agent" north-star user journey.

Output: New `has_runnable_llm` helper in preflight.rs, integrated call site, one passing integration test covering both trigger and non-trigger cases.

| T-35.1-01 | Tampering | D-08 detection bypass via empty-value key (e.g., `OPENROUTER_API_KEY=` in .env) | mitigate | The raw .env scan requires `l.len() > key.len()` for every line — empty-value lines are rejected. Acceptance: unit test `has_runnable_llm_returns_false_when_dotenv_has_empty_key_value` asserts the bypass is blocked. |
| T-35.1-04 | Spoofing | A user-controlled config.yaml with `model.base_url: "https://attacker.example.com/localhost"` could trick the contains() check | accept | The substring check is intentionally permissive for the Ollama escape hatch. Risk is low: the only consequence of false-positive is suppressing the setup wizard for a user who manually configured their own config.yaml — this is a self-harm scenario, not an external attack vector. Network egress to the attacker URL is gated by the LLM provider's own URL handling downstream, not preflight. Adding a strict URL parser here would gold-plate the helper beyond CONTEXT D-08 scope. |
| T-35.1-05 | Information Disclosure | preflight could log the API key when checking it | accept | The implementation reads the env var but discards the value (only checks `!v.is_empty()`). | T-35.1-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies in Wave 2 Plan 03; existing workspace crates only (verified in RESEARCH.md § Package Legitimacy Audit). |
