- **Issue:** The plan's action block uses `matches("print_banner()")` and asserts `count == 1`. In main.rs, `print_banner()` (without semicolon) appears 4 times: 1 actual call (`print_banner();`), 1 function definition (`fn print_banner()`), and 2 doc-comment references. The assertion would have failed on a correct codebase.
- **Fix:** Changed the search string to `"print_banner();"` (with semicolon). This matches only the actual call site (count = 1), not the fn definition or doc comments.
- **Files modified:** `crates/ironhermes-cli/tests/invariants_22_3.rs`
