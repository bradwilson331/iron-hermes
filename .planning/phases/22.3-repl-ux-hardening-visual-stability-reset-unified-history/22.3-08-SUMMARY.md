# Phase 22.3 Plan 08: Flush stdout before scrollback erase in reset_terminal_visual (WR-02) Summary

**One-line stdout flush prepended to `reset_terminal_visual` body so buffered streaming token bytes drain to the terminal before `/clear` erases scrollback — closes code-review warning WR-02 with zero behavior change to the downstream ANSI write sequence.**

## Accomplishments

- **`reset_terminal_visual` now flushes stdout first.** The first executable line of the function body is `let _ = std::io::stdout().flush();`, inserted at `crates/ironhermes-cli/src/tui/render.rs:409` (between the existing `use std::io::Write as _;` trait import at line 408 and `let mut out = stderr();` which moved to line 418 after the 9-line insertion).
- **Unconditional flush drains stdout even in piped configurations.** The flush fires BEFORE the `if !out.is_tty() { return; }` guard, so when stderr is piped to a log file but stdout is the user's TTY, pending streaming tokens still land visibly on the pre-clear screen.
- **Zero downstream behavior change.** The CLR-1 scrollback erase (`\x1b[3J\x1b[H\x1b[2J`), the CLR-2 DECSTBM scroll-region command under the `scroll_end > 0` guard, the CLR-3 `prompt_position_ansi` cursor re-anchor, and the trailing stderr flush are all byte-for-byte identical to the pre-fix function.
- **WR-02 closed.** The race described in `22.3-REVIEW.md` §WR-02 — buffered streaming tokens being erased by `/clear` before ever being displayed — is now defensively eliminated at the helper boundary; the fix is independent of every upstream stdout writer's buffering discipline.
