### GAP-22.3-01 — streaming output clobbers prompt; prompt must be pinned with output streaming above
severity: blocking
source_test: 1
linked_defects: [D-1 ticker/output clobber, D-4 banner bleed (likely same root cause), D-6 /clear observability]
symptoms:
  - Agent output not visible during streaming
  - Streaming releases only on Ctrl+C — indicates a polling/render ordering issue in the main turn loop
  - Streamed tokens overwrite the input prompt row — indicates no reserved/scroll region is active outside of /clear
  - Slash commands unusable while streaming in progress

required_behavior:
  - The input prompt is pinned to the bottom row and remains visible continuously while any streaming or background agent output flows above it
  - Streaming output scrolls above the prompt, never onto or through it

  - Slash commands remain usable (typable, dispatchable) during streaming
recommended_approach_options:
  - Persistent bottom-row prompt: set DECSTBM=1..N-1 at REPL init, render prompt on row N with cursor-save/restore around every streaming write
  - Alternate screen buffer (like htop/less/fzf): enter on REPL init, exit on clean shutdown
  - Full TUI redraw loop (ratatui or raw crossterm): split the terminal into a scrollable content pane and a fixed input pane, with the streaming writer pushing into the content pane's ring buffer
