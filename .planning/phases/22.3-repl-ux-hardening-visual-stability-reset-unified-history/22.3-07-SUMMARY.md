## Decisions Made

- **#[tokio::test] form:** chose bare `#[tokio::test]` (single-thread default) rather than `#[tokio::test(flavor = "multi_thread")]`. Rationale: a single `tokio::fs::OpenOptions::open().await` call does not stress concurrency, and the rest of the agent crate (`transcript_roundtrip.rs`, `sqlite_mirror_fixture.rs`, `memory_flush_handler.rs`) uses the bare form. Only `budget_concurrency_stress.rs` uses `flavor = "multi_thread"` and only because it intentionally exercises multi-core scheduling.
- **Async fs API used:** `tokio::fs::OpenOptions::new().create(true).append(true).open(&self.path).await` — exact mirror of the pattern already used by `append()` 5 lines below in the same impl block.
- **Ordering preserved:** verified via `awk` that `TranscriptWriter::open(path)` (line 154) precedes `writer.touch().await` (line 160) precedes `reg.write().await.register(info)` (line 183) in `subagent_runner.rs`. This is the Plan 22.3-02 invariant that closes UAT D-3 (alias→transcript race).
- **cargo build --workspace and cargo test -p ironhermes-cli --test invariants_22_3 are GREEN** — the WR-01 fix does not regress any Phase 22.3 structural invariant.
- **GAP-22.3-01 (streaming clobber) is NOT closed by this plan** — that is Plan 22.3-11's scope. This plan addresses only code-review WR-01.
