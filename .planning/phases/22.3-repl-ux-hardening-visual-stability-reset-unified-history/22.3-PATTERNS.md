```rust
/// Returns a suggestion string if any candidate is within Levenshtein
/// distance 2 of `input`. Case-insensitive.
/// Returns None if 0 or more than 3 candidates match within threshold.
pub fn suggest_typo(input: &str, candidates: &[&str]) -> Option<String> {
    let input_lc = input.to_lowercase();
    let mut matches: Vec<(&str, usize)> = candidates
        .iter()
        .filter_map(|&c| {
            let d = levenshtein(&input_lc, &c.to_lowercase());
            if d <= 2 { Some((c, d)) } else { None }
        })
        .collect();
    matches.sort_by_key(|(_, d)| *d);
    match matches.len() {
        0 | 4.. => None,
        1 => Some(format!("— did you mean /{}?", matches[0].0)),
        2 | 3 => {
            let names: Vec<String> = matches.iter().map(|(c, _)| format!("/{}", c)).collect();
            Some(format!("— did you mean one of: {}?", names.join(", ")))
        }
    }
}

fn levenshtein(a: &str, b: &str) -> usize {
    let a: Vec<char> = a.chars().collect();
    let b: Vec<char> = b.chars().collect();
    let m = a.len(); let n = b.len();
    let mut dp = vec![vec![0usize; n + 1]; m + 1];
    for i in 0..=m { dp[i][0] = i; }
    for j in 0..=n { dp[0][j] = j; }
    for i in 1..=m {
        for j in 1..=n {
            dp[i][j] = if a[i-1] == b[j-1] { dp[i-1][j-1] }
                       else { 1 + dp[i-1][j].min(dp[i][j-1]).min(dp[i-1][j-1]) };
        }
    }
    dp[m][n]
}
```

**`mod.rs` wiring:** Add `pub mod typo;` to `crates/ironhermes-core/src/commands/mod.rs`. The `suggest_typo` function is then importable as `use crate::commands::typo::suggest_typo;` in handlers.rs and (via the core crate path) in tui/commands.rs.

**New function to add** (place immediately after `prompt_position_ansi`, before `prepare_prompt_with_reserve`):
```rust
/// Clear the terminal scrollback and screen, re-establish the DECSTBM
/// scroll region, and re-anchor the prompt at its reserved row.
/// Called by the `CommandResult::ResetTerminal` arm in run_chat (D-11).
/// No-op on non-TTY stderr.
pub fn reset_terminal_visual(reserved: u16) {
    use std::io::Write as _;
    let mut out = std::io::stderr();
    if !out.is_tty() {
        return;
    }
    // CLR-1: Erase scrollback + screen
    let _ = write!(out, "\x1b[3J\x1b[H\x1b[2J");
    // CLR-2 + CLR-3: Re-establish DECSTBM and re-anchor prompt
    if let Ok((_cols, rows)) = crossterm::terminal::size() {
        let scroll_end = rows.saturating_sub(reserved);
        if scroll_end > 0 {
            let _ = write!(out, "\x1b[1;{}r", scroll_end);
        }
        if let Some(bytes) = prompt_position_ansi(rows, reserved) {
            let _ = out.write_all(&bytes);
        }
    }
    let _ = out.flush();
}
```

**Import note:** `crossterm::terminal::size` is already imported in this file (used by `prepare_prompt_with_reserve`). `std::io::Write` is imported inline with `use std::io::Write as _;` matching the existing `prompt_position_ansi` inline import style.

## Shared Patterns

### Two-Enum CommandResult Discipline

**Source:** `crates/ironhermes-core/src/commands/mod.rs:131-151` (core enum) + `crates/ironhermes-cli/src/tui/extension.rs:113-129` (TUI enum) + `crates/ironhermes-cli/src/tui/commands.rs:110-132` (mapper)

**Apply to:** Any plan that adds the `ResetTerminal` variant

**The rule:** Every new variant added to the core enum MUST simultaneously be added to the TUI enum AND to the `map_core_to_tui` match in `tui/commands.rs`. The Rust compiler will refuse to compile if the mapper match is non-exhaustive. This is the built-in safety net — use it as the verification signal.

**Existing variant shapes to copy:**
- Unit variant (no payload): `ClearSession` in core, `McpReload` in both
- Variant with message: `ClearSession(String)` in TUI (note: TUI version carries the display string; core version is unit)
- `ResetTerminal` should be unit in BOTH enums (no display text — `/clear` produces no output)

### Error Output Pattern

**Source:** `crates/ironhermes-cli/src/main.rs:1158-1160`

**Apply to:** All new `CommandResult::Error` arms

```rust
CommandResult::Error(msg) => {
    eprintln!("{}", format!("Error: {}", msg).red());
    continue;
}
```

The `msg` string from `CommandResult::Error(msg)` is the inner message (without the `Error:` prefix). The prefix is added at the print site in `main.rs`. Handlers return `CommandResult::Error("Unknown /agents subcommand: log — did you mean /logs?".to_string())` — the `Error:` prefix is NOT included in the handler's string.

### Stderr Write Pattern (TTY check + is_tty guard)

**Source:** `crates/ironhermes-cli/src/tui/render.rs:390-403` (`prepare_prompt_with_reserve`)

**Apply to:** `reset_terminal_visual`

```rust
let mut out = stderr();
if !out.is_tty() {
    return;
}
let Ok((_cols, rows)) = size() else { return };
// ... writes ...
let _ = out.flush();
```

All raw ANSI escape writes go to stderr (not stdout). Always guard with `is_tty()`. Always `flush()` at the end.

### Static-Grep INV Test Pattern

**Source:** `crates/ironhermes-cli/tests/invariants_21_7.rs:1-320` (full file)

**Apply to:** All 6 INV-22.3-* tests

The three-part shape:
1. `const SRC: &str = include_str!("relative/path/to/file.rs");` at file top
2. `#[test] fn invariant_22_3_NN_description() { ... }` — one test per invariant
3. Assertion message format: `"INV-22.3-NN: human rule. Found {}."`

Cross-crate reads use `include_str!("../../other-crate/src/file.rs")` — see `invariants_21_7.rs:101` for the exact pattern.

### Fire-and-Forget Transcript Writes

**Source:** `crates/ironhermes-agent/src/transcript.rs:130-154`

**Apply to:** Any new transcript writes (not touch — touch is sync)

`append()` uses `tokio::spawn` + `tokio::fs::OpenOptions` with `create(true).append(true)`. Errors go to `tracing::warn!`, never bubble. `touch()` is different — it is sync `std::fs::OpenOptions` called at registration time (before the tokio worker runs).
