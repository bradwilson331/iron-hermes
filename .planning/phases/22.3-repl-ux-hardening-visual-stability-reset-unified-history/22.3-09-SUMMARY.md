# Phase 22.3 Plan 09: Join worker thread in ReplInputChannel::shutdown (WR-03) Summary

**`ReplInputChannel::shutdown` now joins the rustyline worker thread after sending `Command::Shutdown`, guaranteeing `rl.save_history(path)` (the worker's last act before `break`) completes before `shutdown()` returns — closes code-review warning WR-03 and strengthens the Plan 22.3-03 history-persistence contract (HIST-4) against fast-shutdown / emergency-exit scheduling races.**

## Accomplishments

- **`shutdown(self)` became `shutdown(mut self)` and now joins the worker.** The method body gained three semantic lines:
  1. A leading `// Phase 22.3 WR-03` comment tagging the fix.
  2. `if let Some(handle) = self.worker.take() { let _ = handle.join(); }` after the existing `send_command_only(Command::Shutdown)` call.
  3. An explanatory trailing comment noting that dropping `self` after the join is moot because the worker has already exited.
- **Doc comment updated** with a multi-paragraph rationale that records WR-03, the scheduling race under `std::process::exit(130)`, and the mandatory send-then-join ordering (send unblocks `cmd_rx.blocking_recv()`, join waits for the Shutdown arm's `save_history` to return).
- **Zero call-site edits.** The signature change from `self` to `mut self` is a binding-mode-only change; all existing callers pass an owned `ReplInputChannel`, so `cargo build --workspace` completes without any further edits.
