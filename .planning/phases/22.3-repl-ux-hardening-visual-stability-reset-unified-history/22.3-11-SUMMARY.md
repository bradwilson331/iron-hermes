# Phase 22.3 Plan 11: Streaming Scroll-Region Write (GAP-22.3-01 Closure) Summary

**One-liner:** DECSC/CUP/DECRC-wrapped `write_into_scroll_region` helper routes streaming tokens and the post-turn Hermes: label into the DECSTBM scroll region, preventing the rustyline prompt row from being clobbered during live agent turns.

## Objective

Close `GAP-22.3-01` from `22.3-HUMAN-UAT.md` — the BLOCKING live-TTY defect where streaming tokens from the agent turn wrote from the rustyline-owned prompt row (cursor's last resting position), overwriting/fragmenting user input visibility and making slash commands unusable mid-stream.

## Approach

Option A (from GAP-22.3-01 recommended_approach_options / CONTEXT D-15): per-write cursor discipline. Every streaming write site now:

1. Saves the cursor (`\x1b7` / DECSC).
2. Issues absolute CUP to the bottom row of the scroll region (`\x1b[{scroll_end};1H`).
3. Writes the payload + flushes stdout.
4. Restores the cursor (`\x1b8` / DECRC) to the pre-write position (typically the prompt row).

This is encapsulated in a single `pub fn tui::render::write_into_scroll_region(bytes, reserved)`, used from both the streaming-token callback (high-frequency, per-delta) and the post-turn `Hermes:` label (once per turn).
