<objective>
Make `config.delegation.max_iterations` a HARD CEILING on the caller/model-supplied
per-call and per-task `max_iterations` override in the `delegate_task` tool (D-03,
RESOLVED to Option B — CLAMP TO CEILING). A request at or below the ceiling is
honored verbatim; a request exceeding the ceiling is clamped down to the ceiling
with a `tracing::warn!`. This caps the DoS exposure of a model inflating its own
per-child iteration budget (V5 input validation) while preserving the Phase 32.2
feature of requesting a *smaller* budget.

Purpose: Enforcement MUST live UPSTREAM in `delegate_task.rs` because by the time the
value reaches `run_child` (where Plan 35-02 sizes the fresh per-child `BudgetHandle`),
the override has already been resolved — and `run_child` has no access to
`config.delegation`. Sizing the budget downstream faithfully reflects whatever this
plan clamps here.

Output: Clamp logic at both resolution sites + a rewritten regression test asserting
clamp semantics (request > ceiling clamped; request <= ceiling honored).
</objective>
