Close out Phase 35 with the cron-layer acceptance test (D-07.2 / the original
T-28.1-16 acceptance criterion) and the security-relevant documentation update (D-05).

(1) Add a subagent-LAYER cron independence test that proves a cron job delegating to
exhaustion leaves the interactive budget at full headroom. The existing
`cron_budget_is_independent_from_interactive_budget` (runner.rs:638) only proves
TOP-LEVEL cron independence (28.1-06); this plan adds a sibling that drains at the
subagent layer, confirming the shared `ToolRegistry` delegate runner is no longer a
contamination vector after Plan 35-02's fresh-per-child budget.

delegate runner / own AgentRuntime" fix sketch (lines ~253-287) is SUPERSEDED by the
global per-subagent model — replace it. Document the explicit PROV-10 retirement, the
D-03 clamp-to-ceiling policy, and the new runaway-delegation bound
`max_spawn_depth × max_concurrent_children × max_iterations` (no tree-wide ceiling).

(3) Clean up the residual PROV-10 comments in main.rs (:755/:1301/:2419) so they no
longer claim parent↔child shared-counter semantics (comment-only; kept out of Plan
35-02 to avoid cross-file overlap).

agent-crate change in 35-02, so the only ordering dependency is that 35-02's behavior must
be in place for the cron test and the doc claims to be true. Hence Wave 2.

the main.rs comment cleanup.
