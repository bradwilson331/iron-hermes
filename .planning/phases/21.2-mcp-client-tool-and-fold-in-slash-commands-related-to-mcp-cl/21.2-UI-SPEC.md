# Phase 21.2 — UI Design Contract

> CLI output and interaction contract for the `hermes mcp` subcommands, `/reload-mcp` slash command,
> and MCP connection status messages. This is a Rust CLI/backend phase — no web frontend exists.
> All "UI" is terminal output rendered via the `colored` crate following established cron.rs patterns.

---

## Design System

| Property       | Value                                                         |
|----------------|---------------------------------------------------------------|
| Tool           | none (Rust CLI — no shadcn, no web framework)                 |
| Preset         | not applicable                                                |
| Component library | none — plain `println!` / `eprintln!` with `colored` crate |
| Icon library   | none — Unicode box-drawing chars + ASCII status glyphs         |
| Font           | terminal monospace (user's terminal font)                     |

**shadcn Gate result:** Not applicable. Project is a pure Rust CLI with no React/Next.js/Vite stack.
No `components.json` exists and none is needed.

## Spacing Scale

Terminal output uses character-column indentation rather than pixel spacing. The following column
conventions are applied uniformly, matching the existing `cron.rs` pattern:

| Token | Value         | Usage                                               |
|-------|---------------|-----------------------------------------------------|
| xs    | 2 spaces      | Row content indent (all list/detail rows)           |
| sm    | 14 chars      | Label column width in detail views (`{:<14}`)       |
| md    | 20 chars      | Name column width in list tables (`{:<20}`)         |
| lg    | 70 chars      | Horizontal rule width (`"─".repeat(70)`)            |
| xl    | blank line    | Section separator between logical groups            |

Source: cron.rs established pattern (`{:<20}` columns, `"─".repeat(70)` rules, 2-space indent).

Exceptions: none.

---

## Typography

Terminal output has no font size or weight system. The following ANSI style roles map to `colored`
crate methods and must be applied consistently:

| Role        | `colored` Method     | Usage                                                          |
|-------------|----------------------|----------------------------------------------------------------|
| Heading     | `.bold().cyan()`     | Section titles: "MCP Servers", "MCP Server", "MCP Reloaded"   |
| Label       | `.dimmed()`          | Field labels in detail views: "Transport:", "Status:", etc.    |
| Name/ID     | `.yellow()`          | Server names and tool names (primary identifiers)              |
| Error label | `.red().bold()`      | "Error:" prefix on `eprintln!` lines                           |
| Body        | (no style)           | Descriptions, URLs, command strings, timestamps                |
| Muted       | `.dimmed()`          | Counts, metadata, hints, "none" placeholders                   |

Line height: not applicable (terminal line output is always 1 line per `println!`).

## Color

Terminal ANSI color roles following the established `colored` crate usage in `cron.rs`:

| Role            | `colored` Value  | Usage (exact elements)                                          |
|-----------------|------------------|-----------------------------------------------------------------|
| Dominant        | default terminal | Background, undecorated body text, descriptions, field values   |
| Secondary       | `.dimmed()`      | Labels, counts, metadata, timestamps, hints, "none" text        |
| Accent — cyan   | `.bold().cyan()` | Section headings only: "MCP Servers", "MCP Server", status summary heading |
| Accent — yellow | `.yellow()`      | Primary identifiers only: server name, tool names in list rows  |
| Status — green  | `.green()`       | Connected status: `"connected".green()`                         |
| Status — yellow | `.yellow()`      | Degraded/partial status: `"degraded".yellow()`, `"disabled".yellow()` |
| Status — red    | `.red().bold()`  | Error label prefix: `"Error".red().bold()` on eprintln lines    |

Accent cyan is reserved for: section headings and success confirmation verbs ("MCP server added",
"MCP reloaded") — never applied to data values or body text.

Accent yellow is reserved for: server names and tool names in tabular output — never applied to
labels, descriptions, or status words.

## Interaction Patterns

### Destructive action confirmation

`hermes mcp remove <name>` requires explicit confirmation before deletion.

Pattern (matches `cron.rs` `cmd_remove`):
```
Remove server 'github'? [y/N]
```
- Default is **N** (must opt in to delete).
- Non-`y`/`Y` input → print `Cancelled.` (dimmed) and exit 0.
- `y`/`Y` input → execute removal, print success line.

No secondary confirmation required (single-step, config-only change, reversible via `hermes mcp add`).

### Wizard prompts (`hermes mcp add`)

Pattern (matches `memory_setup.rs` `prompt_line`):
```
{Question} [{default}]: 
```
- Empty input → accepts default shown in brackets.
- Password fields (API keys) → no echo (use `rpassword` or equivalent).
- URL/command inputs → validate non-empty before proceeding.
- Wizard is linear (no backtracking); on validation failure, re-prompt the same field once,
  then abort with error message.

### `hermes mcp test` output width

Tool list from test connection must truncate description to fit 80-column terminals:
- Tool name column: 30 chars (`{:<30}`)
- Description: truncated at 47 chars with `…` if longer
- Total: 2 indent + 30 name + 3 separator + 47 desc = 82 chars max

## Detail View Layout

`hermes mcp test` and `hermes mcp configure` use the key-value detail pattern from `cron.rs`
`render_job_details()`:

```
MCP Server: github                         ← bold().cyan() heading
  Transport:     stdio                     ← {:<14} label dimmed, value plain
  Command:       npx @modelcontextprotocol/server-github
  Status:        connected                 ← status color applied
  Tools:         12 (3 filtered)           ← plain
  Timeout:       120s                      ← plain
```

Label column width: 14 chars (`{:<14}`), matching `cron.rs` detail view.

---

## List Table Layout

`hermes mcp list` uses the columnar pattern from `cron.rs` `cmd_list()`:

```
MCP Servers                                ← bold().cyan() heading
──────────────────────────────────────────────────────────────────────  ← "─".repeat(70)
  NAME                 TRANSPORT            STATUS       TOOLS           ← all .bold()
  github               stdio                connected    12              ← name .yellow(), status colored
  filesystem           stdio                failed       0               ← status .red() (no bold — just red for data)
  my-http-server       http                 disabled     5
──────────────────────────────────────────────────────────────────────
  3 server(s) total                         ← .dimmed()
```

Column widths:
- NAME: 20 chars (`{:<20}`)
- TRANSPORT: 20 chars (`{:<20}`)
- STATUS: 12 chars (`{:<20}` padded to account for ANSI escape sequences in color output)
- TOOLS: right-aligned count (no fixed width, last column)
