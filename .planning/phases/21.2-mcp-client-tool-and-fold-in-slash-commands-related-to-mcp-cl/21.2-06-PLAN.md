<objective>
Close three UAT gaps in the `hermes mcp add` / `hermes mcp test` interactive wizard, all living in `crates/ironhermes-cli/src/mcp_config.rs`:

- **GAP-1 (high):** `attempt_connect_and_list` has no timeout wrapper. When `npx` (or any child process) is alive but not speaking MCP JSON-RPC on stdout/stdin, the wizard hangs indefinitely and the only escape is Ctrl-C. Fix: wrap every call site of `attempt_connect_and_list` in `tokio::time::timeout(Duration::from_secs(config.connect_timeout), ...)` and emit a clear timeout error.
- **GAP-2 (medium):** `cmd_add` silently persists broken entries — on connection failure it falls through to `(vec![], 0)` and calls `save_mcp_server_config` with the broken command. Fix: after a failed connection attempt, prompt `Connection failed. [R]etry / [S]ave anyway / [A]bort?` (default Abort). Retry re-runs `attempt_connect_and_list`; Abort exits without writing config.yaml; Save anyway preserves the existing "user can fix later" escape hatch but now requires explicit consent.
- **GAP-3 (low):** The `Arguments (space-separated, or empty)` prompt offers no example. Users typed typos (`i`) or empty strings that lead directly into GAP-1. Fix: when command == `npx`, print the hint `Example: -y @modelcontextprotocol/server-filesystem /tmp` on the line above the args prompt.

Purpose: Eliminate the indefinite-hang failure mode and the silent-persist failure mode that blocked UAT Test 1. Users must get a deterministic error within `connect_timeout` seconds and an explicit three-way choice before a broken entry lands in config.yaml.

Output: Modified `crates/ironhermes-cli/src/mcp_config.rs` with new timeout wrapper, a `prompt_retry_save_abort` helper, an npx-hint branch in the args prompt, and a corresponding unit-test suite.
</objective>

    /// Default is Abort (safest — do not persist a broken entry without consent).
    /// Accepted inputs (case-insensitive): `r`/`retry` → Retry; `s`/`save` → SaveAnyway;
    /// `a`/`abort` or empty → Abort.
    fn prompt_retry_save_abort() -> RetrySaveAbort {
        print!("Connection failed. [R]etry / [S]ave anyway / [A]bort? [A]: ");
        io::stdout().flush().ok();
        let mut line = String::new();
        io::stdin().lock().read_line(&mut line).ok();
        match line.trim().to_lowercase().as_str() {
            "r" | "retry" => RetrySaveAbort::Retry,
            "s" | "save" | "save anyway" => RetrySaveAbort::SaveAnyway,
            _ => RetrySaveAbort::Abort,
        }
    }
