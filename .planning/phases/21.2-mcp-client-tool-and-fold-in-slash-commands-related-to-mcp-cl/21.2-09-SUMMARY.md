# Phase 21.2 Plan 09: GAP-6 Close — Clean Interactive Startup Summary

Closed GAP-6 (medium): interactive `ironhermes` / `hermes chat` now starts without the WARN flood buries the REPL prompt, and stdio MCP child processes no longer clobber the parent terminal with their stderr (e.g. `npx @modelcontextprotocol/server-filesystem` printing its Usage line on startup failure).

## What Shipped

### Task 1 — Context-aware tracing init (commit d9c51b2)

- `crates/ironhermes-cli/src/main.rs`: moved `dotenvy::from_path`, `ensure_home_dirs()?`, and `let cli = Cli::parse();` above `tracing_subscriber::fmt().init()`.
- Inserted a 3-branch `env_filter` match:
  - `RUST_LOG` set → `EnvFilter::from_default_env()` (operator intent always wins).
  - `RUST_LOG` unset AND interactive REPL → `EnvFilter::new("error")` (clean prompt).
  - `RUST_LOG` unset AND non-interactive → legacy `from_default_env().add_directive("ironhermes=info"...)` (preserves today's operator diagnostics for `hermes gateway` / `hermes agent` et al.).
- `is_interactive_repl = matches!(cli.command, Some(Commands::Chat { .. }) | None) && cli.execute.is_none();` — correctly excludes `hermes -e "prompt"` (batch mode via `run_single`).

### Task 2 — Stdio child stderr piping + regression tests (commit d8eb5f0)

- `crates/ironhermes-mcp/src/transport.rs::connect_stdio`: added one line inside the existing `configure(move |cmd| { ... })` closure, below `cmd.env_clear()`:
  ```rust
  cmd.stderr(std::process::Stdio::piped());
  ```
- `connect_http` untouched (no child process, no stderr concern).
- Added `#[cfg(test)] mod tests { ... }` at end of `transport.rs` (file previously had no tests):
  - `connect_stdio_pipes_child_stderr` — static-grep via `include_str!("transport.rs")` asserts the `cmd.stderr(std::process::Stdio::piped());` line is present in the configure closure.
  - `stdio_child_stderr_does_not_inherit_parent_fd` — runtime proof: spawns `sh -c "printf 'usage: ...' 1>&2"` (or `cmd /C` on Windows) with `.stderr(Stdio::piped()).stdout(Stdio::null())`, drains the piped handle, asserts the bytes arrive on the captured handle. Proves the `Stdio::piped()` contract independently of rmcp.

## GAP-6 Closure Evidence

**GAP-6a (WARN flood):**
- ✓ Interactive `hermes chat` / bare `hermes` (no `-e`) default EnvFilter = `error` when `RUST_LOG` is unset — WARN-level tracing from SkillRegistry / Cron / provider diagnostics is filtered out at init.
- ✓ `hermes gateway` / `hermes agent` / `hermes cron ...` et al. keep today's `ironhermes=info` default — operator visibility preserved.
- ✓ `RUST_LOG=<anything>` always wins via `EnvFilter::from_default_env()` first-branch arm.

**GAP-6b (child stderr leak):**
- ✓ stdio MCP child spawned via `connect_stdio` now has `.stderr(Stdio::piped())` configured. A child printing `Usage: mcp-server-filesystem [allowed-directory] ...` can no longer land on the parent's `You:` prompt line.
- ✓ Static-grep test `connect_stdio_pipes_child_stderr` locks the line in source against refactor drift.
- ✓ Runtime test `stdio_child_stderr_does_not_inherit_parent_fd` proves Stdio::piped() produces a captured handle and the child's bytes arrive there — not on the parent fd.

## Deviations from Plan

None — plan executed exactly as written. All acceptance criteria met, all greps pass, full regression gate green.

## Key Decisions

- **Reorder the init block.** `dotenvy::from_path`, `ensure_home_dirs()`, and `Cli::parse()` moved above `tracing_subscriber::fmt().init()` so the filter branch can read `cli.command` / `cli.execute`. Clap's derive parse is pure/idempotent — reorder is safe.
- **Interactive exclusion rule.** `hermes -e "prompt"` enters the `None` arm and dispatches to `run_single` — that's batch, not interactive. The `&& cli.execute.is_none()` guard correctly keeps the `ironhermes=info` default for `-e` invocations so CI and scripting use cases retain diagnostics.
- **Runtime test does not go through rmcp.** Spawning a plain `sh -c` child via `std::process::Command` with the exact same `.stderr(Stdio::piped())` configuration isolates the contract under test and avoids taking a dev-dependency on a live MCP server. `TokioChildProcess::new` wraps the same `tokio::process::Command` whose `stderr()` method delegates straight to `std::process::Command::stderr` — identical contract.
- **Inline spelling of `std::process::Stdio::piped()`.** No `use` added at top of `transport.rs`. Matches the grep acceptance criterion verbatim and keeps the diff minimal.
