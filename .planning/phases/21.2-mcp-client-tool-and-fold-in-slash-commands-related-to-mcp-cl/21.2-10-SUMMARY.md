# Phase 21.2 Plan 10: GAP-7 Close — Connected-Server Reporting Authority Summary

Fixed `McpManager::connected_server_names()` so a server whose child exits before the rmcp `initialize` handshake completes is NOT reported as connected. The authoritative signal is now a per-server `Arc<AtomicBool>` flag flipped to `true` ONLY after `client.list_all_tools().await?` resolves, replacing the buggy `tasks.keys().cloned().collect()` that reported every spawned task regardless of handshake completion.

## What Shipped

### Task 1 — thread connected AtomicBool through server_task (commit 9da1c05)

- Added `use std::sync::atomic::{AtomicBool, Ordering};` to `crates/ironhermes-mcp/src/server_task.rs`.
- Added `connected: Arc<AtomicBool>` parameter to `run_server_task` (line 48).
- Added `connected: &Arc<AtomicBool>` parameter to `connect_and_serve` (line 123).
- Inserted `connected.store(true, Ordering::SeqCst);` at line 144 — **immediately after** `client.list_all_tools().await?` at line 143 and BEFORE the `tracing::info!` log. Source-order locked: the flip-true site can only fire when both the rmcp `initialize` handshake and tool discovery have succeeded.
- Added `connected.store(false, Ordering::SeqCst);` on retry exhaustion (inside the `retries > MAX_RETRIES` branch) AND on post-loop cleanup (after the `loop { }` ends, before constructing `ServerTaskResult`) — two defensive guards against stale-true reads.
- Preserved the existing backoff `tracing::warn!` call body verbatim — minimal diff to the retry loop itself.
- No existing tests required changes (none of the four `#[cfg(test)] mod tests` tests invoke `run_server_task` directly).

### Task 2 — McpManager connected_flags map + regression tests (commit 780ac35)

- Added `use std::sync::atomic::{AtomicBool, Ordering};` to `crates/ironhermes-mcp/src/manager.rs` (line 6).
- Added new struct field `connected_flags: Mutex<HashMap<String, Arc<AtomicBool>>>` at `manager.rs:40`.
- Initialized the field in `McpManager::new` at `manager.rs:48`.
- **`start_all`** (lines 55-79): allocate `Arc::new(AtomicBool::new(false))` per server, pass as 5th arg to `run_server_task`, store in `connected_flags` alongside `tasks` and `configs`.
- **`start_all_and_wait`** (lines 85-253): same allocation pattern; the post-sleep heuristic block now consults the flag first (`if flag_true → push connected`), falling back to the existing `has_tools` check for the failed-task branch. Belt-and-braces: if the flag is true but `has_tools` is false, the server is still reported connected (the flag is authoritative).
- **`shutdown_all`** (lines 173-197): added `let mut flags = self.connected_flags.lock().await;` and `flags.remove(&name);` inside the drain loop so no stale entries survive shutdown.
- **`connected_server_names`** (lines 224-240 at HEAD) — the authoritative fix: replaced `tasks.keys().cloned().collect()` with an iteration over `connected_flags` that keeps only names whose flag loads `true` under `Ordering::SeqCst`. Still uses `try_lock` to preserve the existing sync-call-context invariant.
- Two new regression tests inside the existing `#[cfg(test)] mod tests` block:
  - **`connected_server_names_excludes_server_that_exited_before_initialize`** (line 463): spawns a server whose command is `false` on unix / `cmd /C exit 1` on windows — a process that exits immediately without speaking MCP. After a 500ms observation window, asserts `manager.connected_server_names()` is empty (the flag never flipped, the process exited, the old code would have over-reported). Cleans up via `shutdown_all` to avoid leaked tasks.
  - **`connected_server_names_includes_server_whose_flag_is_true`** (line 513): directly inserts a flag with `AtomicBool::new(true)` via `manager.connected_flags.lock().await`, then asserts `connected_server_names()` returns `vec!["ok_server"]`. Proves the new lookup path is not trivially empty — it actually reads the flag.

## Before / After at the Bug Sites

**Before (`manager.rs:190-196` at HEAD):**

```rust
pub fn connected_server_names(&self) -> Vec<String> {
    if let Ok(tasks) = self.tasks.try_lock() {
        tasks.keys().cloned().collect()  // <-- over-reports every spawned task
    } else {
        Vec::new()
    }
}
```

**After:**

```rust
pub fn connected_server_names(&self) -> Vec<String> {
    if let Ok(flags) = self.connected_flags.try_lock() {
        flags
            .iter()
            .filter_map(|(name, flag)| {
                if flag.load(Ordering::SeqCst) {
                    Some(name.clone())
                } else {
                    None
                }
            })
            .collect()
    } else {
        Vec::new()
    }
}
```

**Flag-flip site — `server_task.rs:143-144`:**

```rust
let mcp_tools = client.list_all_tools().await?;   // line 143 — rmcp initialize + discover gate
connected.store(true, Ordering::SeqCst);           // line 144 — GAP-7 flip-true site
```

**Downstream effect at `crates/ironhermes-cli/src/main.rs:1414`:** the `else if names.len() < n` branch now fires correctly when a misconfigured child crashed before initialize — the user sees `MCP: 0 tool(s) ready. 1 server(s) failed.` instead of the false-positive `MCP: 0 tool(s) ready from 1 server(s).`. GAP-7 closed.
