# Phase 21.2 Plan 08: Flush banner before rl.readline (GAP-5) Summary

**One-liner:** Close GAP-5 — insert explicit `io::stdout()/io::stderr().flush().ok()` pairs in `run_chat` so the banner + first `You:` prompt paint the moment `ironhermes` starts (even with `mcp_servers` configured), and lock the ordering into source with two static-grep regression tests.

## What changed

- `crates/ironhermes-cli/src/main.rs`:
  - **Insertion A (line 553-558):** `io::stdout().flush().ok();` + `io::stderr().flush().ok();` immediately after `print_banner();` in `run_chat`. Prevents the MCP background task's first `eprintln!` from racing the line-buffered banner paint.
  - **Insertion B (line 823-830):** Belt-and-braces `io::stdout().flush().ok();` + `io::stderr().flush().ok();` pair placed between `let mut exit_cleanly = false;` and `loop {` so any synchronous post-banner output (context injection, token estimator init, rustyline setup) also hits the terminal before `rl.readline` blocks.
  - **Task 2 (tests mod `tui_extension_wiring_tests`):** two new tests:
    - `initial_prompt_flush_precedes_readline` — asserts `io::stdout().flush().ok();` exists AND its first occurrence has a byte offset lower than the first `let readline = rl.readline(` occurrence. Pure static grep, no REPL spawn.
    - `initial_prompt_flushes_stderr_too` — asserts `io::stderr().flush().ok();` is present (pre-Task 1 count was 0, so any match proves the insertion).

## Key Decisions

- **Two flush pairs rather than one:** Plan mandated both positions and this executor honored both (belt-and-braces). If a later refactor inserts a `println!`/`eprintln!` between the banner and the REPL `loop`, the second pair catches it.
- **`print_banner()` left untouched:** Flush is the caller's responsibility; `print_banner` stays reusable from `run_single` / future entry points without forcing a flush contract into every call site.
- **Ordering test uses byte-offset compare:** Cheapest assertion that proves the flush appears BEFORE `rl.readline` in source without actually running the REPL or mocking rustyline.
- **stderr flush added alongside stdout:** Pre-edit count was 0, so any post-edit match cleanly proves the insertion. Complements GAP-6/Plan 09 (which will contain the child's stderr and lower the default tracing level) — together they close the full D-07 contract.

## Pair with Plan 09 (GAP-6)

This plan closes the flush race on the main thread. Plan 09 (GAP-6) contains the MCP child process's stderr and lowers the default interactive tracing level so the async clobber path disappears entirely. Only once both land does the D-07 "non-blocking startup" contract hold end-to-end. GAP-5 alone already makes the banner paint reliable under the common TTY line-buffered case.
