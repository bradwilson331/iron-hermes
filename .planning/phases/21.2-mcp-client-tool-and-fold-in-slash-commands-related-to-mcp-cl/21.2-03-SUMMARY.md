# Phase 21.2 Plan 03: MCP Client Runtime â€” McpManager, Server Tasks, Tool Dispatch Summary

**One-liner:** McpManager orchestrates per-server tokio tasks that connect via rmcp (stdio/HTTP), register discovered tools as `Box<dyn McpTool>` with `server__tool` naming and `[MCP: server_name]` description prefix, dispatch calls via mpsc channel with timeout, reconnect with exponential backoff (5 retries, 60s cap), and report `ServerTaskResult.failure_reason` for D-12 partial failure summaries.

## What Was Built

### Task 1: McpTool, Transport, Sampling

**crates/ironhermes-mcp/src/tool.rs**
- `McpTool`: implements `Tool` trait; holds `prefixed_name` (`server__tool`), `original_name`, `description` (`[MCP: server_name] ...`), `ToolSchema`, and `mpsc::Sender<McpCallRequest>`
- `McpCallRequest`: request struct carrying tool_name, arguments, and oneshot response channel
- `make_prefixed_name(server, tool)`: D-06 sanitization replacing hyphens/dots with underscores before double-underscore join
- `Tool::execute()`: sends McpCallRequest via channel, awaits oneshot response, sanitizes errors
- 5 unit tests for make_prefixed_name covering basic, hyphen, dot, mixed, and no-sanitization cases

**crates/ironhermes-mcp/src/transport.rs**
- `connect_stdio(config)`: TokioChildProcess with env_clear + build_safe_env (D-19), returns RunningService<RoleClient, ()>
- `connect_http(config)`: StreamableHttpClientTransport::from_uri (reqwest-backed), returns same type

**crates/ironhermes-mcp/src/sampling.rs**
- `SamplingCallback`: type alias for Arc<dyn Fn(String, Option<String>) -> Pin<Box<dyn Future...>>>
- `SamplingHandler`: wraps callback with AtomicU32 rate counter + max_rpm guard (T-21.2-07 mitigation)
- 3 unit tests: callback invocation, rate limit enforcement, model_hint passthrough

### Task 2: McpManager and server_task

**crates/ironhermes-mcp/src/server_task.rs**
- `ServerTaskResult { server_name, tool_names, failure_reason: Option<String> }`: D-12 data contract
- `MAX_RETRIES = 5`, `MAX_BACKOFF = 60s`
- `run_server_task()`: calls `interpolate_config`, exponential backoff loop calling `connect_and_serve`
- `connect_and_serve()`: connects via stdio or HTTP, calls `list_all_tools()`, creates mpsc channel, registers tools under write lock (dropped before .await), serves calls via `client.call_tool()` with timeout, extracts text from `RawContent::Text`
- D-08 enabled_tools filter applied during registration
- 4 unit tests: backoff_progression, server_task_result_captures_failure, MAX_RETRIES constant, MAX_BACKOFF constant

**crates/ironhermes-mcp/src/manager.rs**
- `StartResult { connected, failed: Vec<(String, String)>, tool_count }`: aggregates D-12 failure pairs
- `McpManager { registry, tasks: Mutex<HashMap<...(JoinHandle, CancellationToken)>>, configs }`
- `start_all()`: fire-and-forget tokio::spawn per enabled server (D-07)
- `start_all_and_wait()`: spawns tasks, waits 5s, checks registry for tool presence heuristic
- `shutdown_all()`: cancel all tokens, await joins, unregister_by_prefix per server (D-09)
- `reload()`: shutdown_all + start_all (D-09 fire-and-forget)
- `reload_and_report()`: shutdown_all + start_all_and_wait (D-09/D-12)
- `Drop`: cancels all task tokens on manager drop
- 4 unit tests: StartResult fields, new manager, start_all skips disabled, shutdown_all empty

### Auto-fixed Issues

**1. [Rule 1 - Bug] Wrong rmcp HTTP feature flag in Cargo.toml**
- **Found during:** Task 1 (pre-implementation, while verifying rmcp source)
- **Issue:** Cargo.toml had `transport-streamable-http-client`; `StreamableHttpClientTransport::from_uri` is only defined when `transport-streamable-http-client-reqwest` feature is enabled (adds reqwest backend)
- **Fix:** Changed feature to `transport-streamable-http-client-reqwest` in Cargo.toml
- **Files modified:** `crates/ironhermes-mcp/Cargo.toml`
- **Commit:** 936d6d0

**2. [Rule 3 - Blocking Issue] async-trait missing from Cargo.toml**
- **Found during:** Task 1 first cargo test run (E0432 unresolved import)
- **Issue:** `async_trait::async_trait` used in tool.rs but dependency not declared

- **Files modified:** `crates/ironhermes-mcp/Cargo.toml`
- **Commit:** 936d6d0

**3. [Rule 3 - Blocking Issue] CallToolRequestParams is #[non_exhaustive]**

- **Files modified:** `crates/ironhermes-mcp/src/server_task.rs`
- **Commit:** ea6b869

**4. [Rule 3 - Blocking Issue] Tool trait not in scope in server_task.rs**
