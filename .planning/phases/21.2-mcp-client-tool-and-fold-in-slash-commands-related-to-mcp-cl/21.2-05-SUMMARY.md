## What Was Built

### Task 1: mcp_config.rs

**crates/ironhermes-cli/src/mcp_config.rs**
- `McpAction` enum: `Add { name, url, command, args }`, `Remove { name }`, `List`, `Test { name }`, `Configure { name }` — clap `Subcommand`
- `handle_mcp_command(action: McpAction)`: public async dispatcher matching all 5 arms
- `cmd_list()`: columnar table per UI-SPEC — `bold().cyan()` heading, `"─".repeat(70)` rules, `{:<20}` columns with yellow server names, colored status words, dimmed footer count
- `cmd_add()`: interactive wizard — transport type prompt (stdio/http), command/URL/args input, env var collection, auth token, connect-and-discover via `attempt_connect_and_list`, per-tool enable selection, round-trip save via `save_mcp_server_config`
- `cmd_remove()`: checks existence, `[y/N]` confirmation, round-trip remove via `remove_mcp_server_config`
- `cmd_test()` (D-16): detail view (transport/command/URL/status/timeout), connects via `connect_stdio` / `connect_http`, lists tools with 30-char name column + 47-char truncated description, sanitizes errors (T-21.2-14)
- `cmd_configure()`: connects, presents per-tool `[Y/n]` prompts, saves updated `enabled_tools`
- `attempt_connect_and_list()`: shared helper — connects, calls `client.list_all_tools()`, drops client, returns `Vec<(name, description)>`
- `save_mcp_server_config()`: loads full YAML doc as `serde_yaml::Value`, modifies only `mcp_servers.{name}`, writes back — preserves all other config sections (T-21.2-13)
- `remove_mcp_server_config()`: same round-trip pattern for deletion (T-21.2-13)
- `prompt_line()` / `prompt_yn()`: stdin helpers matching memory_setup.rs wizard pattern
- 4 unit tests: module reachability, `prompt_yn` compile check, save roundtrip preserves existing keys, remove roundtrip preserves unaffected servers

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] colored crate Vec<(String,String)> type mismatch**
- **Found during:** Task 1 first cargo build (E0308)
- **Issue:** `cmd_add` `selected_tools` collected `(tool_name, desc)` tuples but `McpServerConfig.enabled_tools` requires `Vec<String>`; iterated over `&discovered_tools` instead of `&discovered_tools` (name only)
- **Fix:** Changed loop binding from `for tool_name in &discovered_tools` to `for (tool_name, _desc) in &discovered_tools` to extract just the name
- **Files modified:** `crates/ironhermes-cli/src/mcp_config.rs`
- **Commit:** 798ea2d (fixed before initial commit)

**2. [Rule 1 - Bug] colored crate width padding on colorized strings**
- **Found during:** Task 1 first cargo build (E0277: Display not implemented for colored string with format width)
- **Issue:** `{:<30}` and `{:<20}` width format specifiers don't work with `.yellow()` colored strings — colored's `ColoredString` does not implement the `Display` width trait
- **Fix:** Pad the raw string first (`format!("{:<N}", raw)`) then colorize the padded string (`.yellow()`)
- **Files modified:** `crates/ironhermes-cli/src/mcp_config.rs`
- **Commit:** 798ea2d (fixed before initial commit)

**3. [Rule 1 - Bug] unreachable_code warning from ok_or_else + process::exit pattern**
- **Found during:** Task 1 cargo build warnings (unreachable expression on `anyhow::anyhow!` inside diverging closure)
- **Issue:** `ok_or_else(|| { process::exit(1); anyhow::anyhow!(...) })` produces unreachable_code warning because the macro expands into reachable code even after a diverging `process::exit`
- **Fix:** Replaced `ok_or_else` closures with explicit `match` expressions using `None => { eprintln!(...); process::exit(1); }`
- **Files modified:** `crates/ironhermes-cli/src/mcp_config.rs`
- **Commit:** 995d7b7 (fixed before second commit)
