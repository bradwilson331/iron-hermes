    **2. Implement McpReloader for McpManager in main.rs (or a local adapter):**

    Since McpReloader is defined in ironhermes-core and McpManager in ironhermes-mcp,
    the impl can live in ironhermes-mcp's manager.rs or in main.rs via a thin wrapper.

    Preferred: implement directly on McpManager in ironhermes-mcp/src/manager.rs:
    ```rust
    use ironhermes_core::commands::context::{McpReloader, McpReloadResult};

    #[async_trait::async_trait]
    impl McpReloader for McpManager {
        async fn reload(&self) -> McpReloadResult {
            // Re-read config
            let config = ironhermes_core::Config::load().unwrap_or_default();
            let new_configs: HashMap<String, McpServerConfig> = config.mcp_servers.iter()
                .filter_map(|(name, val)| {
                    serde_yaml::from_value::<McpServerConfig>(val.clone()).ok()
                        .map(|c| (name.clone(), c))
                })
                .collect();
            let old_names = self.connected_server_names();
            McpManager::reload(self, new_configs).await;
            let new_names = self.connected_server_names();
            let tool_count = self.registered_tool_count().await;
            McpReloadResult {
                connected: new_names,
                failed: Vec::new(), // TODO: track failures in server_task
                tool_count,
            }
        }

        fn connected_server_names(&self) -> Vec<String> {
            McpManager::connected_server_names(self)
        }

        async fn registered_tool_count(&self) -> usize {
            McpManager::registered_tool_count(self).await
        }
    }
    ```

    NOTE: ironhermes-mcp already depends on ironhermes-core, so this impl is legal
    (no circular dep). The McpReloader trait is in ironhermes-core (leaf), impl is in
    ironhermes-mcp (depends on core).
