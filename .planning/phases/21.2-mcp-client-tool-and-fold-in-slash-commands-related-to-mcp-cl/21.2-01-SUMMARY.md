## What Was Built

### Task 1: ironhermes-mcp crate

**crates/ironhermes-mcp/Cargo.toml**
- Package manifest with `rmcp = { version = "1.5", features = ["client", "transport-child-process", "transport-streamable-http-client"] }`
- Workspace dep inheritance for tokio, serde, serde_yaml, serde_json, anyhow, regex, tracing
- Local path deps on ironhermes-core and ironhermes-tools

**crates/ironhermes-mcp/src/config.rs**
- `McpServerConfig`: stdio transport (command/args/env), HTTP transport (url/headers), common options (timeout=120, connect_timeout=60, enabled=true, enabled_tools, auth, sampling)
- `SamplingConfig`: enabled=false, max_rpm=10, max_tool_rounds=5
- `interpolate_env(value: &str) -> String`: replaces `${VAR}` from std::env, leaves unresolvable vars as literals (D-18)
- `interpolate_config(config: &mut McpServerConfig)`: applies interpolation to all string fields
- 18 unit tests covering deserialization, defaults, round-trips, interpolation

**crates/ironhermes-mcp/src/security.rs**
- `SAFE_ENV_KEYS`: allowlist of 8 keys (PATH, HOME, USER, LANG, LC_ALL, TERM, SHELL, TMPDIR)
- `build_safe_env(user_env) -> HashMap`: host allowlist + XDG_* prefix + user overrides (D-19)
- `CREDENTIAL_PATTERN`: static LazyLock<Regex> matching ghp_*, sk-*, Bearer tokens, token=, key=, API_KEY=, password=, secret= (D-20)
- `sanitize_error(text) -> String`: replaces credential matches with [REDACTED]
- 11 unit tests covering env filtering, exclusions, credential stripping

**crates/ironhermes-mcp/src/lib.rs**
- Public module declarations and re-exports of all public types/functions

### Task 2: Config.mcp_servers field

- Added `pub mcp_servers: HashMap<String, serde_yaml::Value>` to `Config` struct with `#[serde(default)]`
- Avoids circular dependency: ironhermes-core stores raw YAML, ironhermes-mcp parses into typed McpServerConfig
- 3 unit tests: round-trip with 2 servers, backward-compat empty default, serialize/deserialize cycle

### Auto-fixed Issues

**1. [Rule 1 - Bug] rmcp feature flag name correction**
- **Found during:** Task 1 (before implementation)
- **Issue:** RESEARCH.md assumption A1 listed `transport-streamable-http` as the feature name; actual crates.io name is `transport-streamable-http-client`
- **Fix:** Queried crates.io API for rmcp 1.5.0 features before writing Cargo.toml; used correct `transport-streamable-http-client`
- **Files modified:** `crates/ironhermes-mcp/Cargo.toml`
- **Commit:** f82e3c1

**2. [Rule 1 - Bug] CREDENTIAL_PATTERN raw string literal escaping**
- **Found during:** Task 1 GREEN phase (compilation error)
- **Issue:** `concat!` with raw string literals (`r"..."`) cannot use `\"` escapes inside character classes for the regex patterns with `[^\s&,;\"']`
- **Fix:** Used regular string literals (not raw) for the 6 credential assignment patterns that require backslash escapes
- **Files modified:** `crates/ironhermes-mcp/src/security.rs`
- **Commit:** f82e3c1

**3. [Rule 1 - Bug] Rust 2024 unsafe env mutation in tests**
- **Found during:** Task 1 GREEN phase (compilation errors E0133)
- **Issue:** `std::env::set_var` and `std::env::remove_var` require `unsafe` blocks in Rust 2024 edition
- **Fix:** Wrapped all test env mutations in `unsafe { ... }` blocks with SAFETY comments (consistent with Phase 21.6 convention)
- **Files modified:** `crates/ironhermes-mcp/src/config.rs`, `crates/ironhermes-mcp/src/security.rs`
- **Commit:** f82e3c1
