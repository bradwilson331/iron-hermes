<objective>
Close GAP-7 (high): `McpManager::connected_server_names()` currently reports every server whose task was `tokio::spawn`ed, regardless of whether the rmcp `initialize` handshake ever completed. VERIFICATION.md evidence (line 110-133): running `ironhermes` with a misconfigured `mcp-server-filesystem` child causes the startup message `MCP: 0 tool(s) ready from 1 server(s).` — the `names.len() == n` branch at `crates/ironhermes-cli/src/main.rs:1423` fires because `connected_server_names()` returned the server name even though no tool ever registered AND the child process had already exited.

Root cause confirmed by source read at HEAD:
- `crates/ironhermes-mcp/src/manager.rs:190-196`:
  ```rust
  pub fn connected_server_names(&self) -> Vec<String> {
      if let Ok(tasks) = self.tasks.try_lock() {
          tasks.keys().cloned().collect()
      } else {
          Vec::new()
      }
  }
  ```
  This reports every server whose task handle is still in the map — i.e., every server whose task was spawned, INCLUDING failed/exited ones whose `failure_reason` is populated.
- `crates/ironhermes-mcp/src/server_task.rs:107-252`: `connect_and_serve` does NOT currently surface an "initialized" signal back to the manager. The successful path flows through `list_all_tools().await?` on line 126, then registers tools in the write-guard block on lines 138-159. The task only reports failure via `ServerTaskResult.failure_reason` after the whole retry loop exhausts — but the manager's `connected_server_names()` has already answered "yes, this server is connected" long before the task exits.

Fix: introduce a per-server `Arc<AtomicBool>` connected flag.
- `McpManager` gains a `connected_flags: Mutex<HashMap<String, Arc<AtomicBool>>>` (parallel to the existing `tasks` and `configs` maps).
- `start_all` and `start_all_and_wait` allocate `Arc::new(AtomicBool::new(false))` per server, store it in `connected_flags`, and pass a clone into `run_server_task`.
- `run_server_task` adopts a new parameter `connected: Arc<AtomicBool>` and threads it into `connect_and_serve`.
- `connect_and_serve` calls `connected.store(true, Ordering::SeqCst)` on the SUCCESS path only — AFTER `list_all_tools().await?` resolves on line 126, BEFORE any tool registration. If any earlier step errors (transport connect, initialize, list_all_tools), the flag stays `false` and the retry loop above it will either retry or surface the error via `failure_reason`.
- `McpManager::connected_server_names()` now reads the flags map instead of `tasks.keys()`.
- `McpManager::shutdown_all()` removes the flag alongside the task handle.
- The `start_all_and_wait` heuristic at `manager.rs:118-132` (which already checks `has_tools` via registry inspection) is now belt-and-braces — kept for defensive behavior but the authoritative signal is the AtomicBool flag.

Secondary effect: the `else if names.len() < n` branch at `main.rs:1414` fires correctly, producing `MCP: 0 tool(s) ready. 1 server(s) failed.` — the user sees the truthful startup summary.

Purpose: Eliminate the false-positive "connected" count. A server that crashed before handshake completion must be reported as FAILED, not as connected-but-with-zero-tools. This closes UAT Test 4's third sub-issue and aligns the startup UX with D-12 ("partial failure reporting via ServerTaskResult.failure_reason").

Output: modified `manager.rs` (new connected_flags map + updated `connected_server_names` + flag allocation + shutdown cleanup), modified `server_task.rs` (new parameter + flag flip on success), plus a regression test using a `false`/`cmd /C exit 1` dummy command.

                }

    /// `list_all_tools()` both succeeded (GAP-7 contract). A spawned task that

    ///
    /// Note: uses try_lock to avoid deadlock in sync call contexts. Returns

    pub fn connected_server_names(&self) -> Vec<String> {
        if let Ok(flags) = self.connected_flags.try_lock() {

                .iter()
                .filter_map(|(name, flag)| {
                    if flag.load(Ordering::SeqCst) {
                        Some(name.clone())
                    } else {
                        None
                    }
                })
                .collect()
        } else {
