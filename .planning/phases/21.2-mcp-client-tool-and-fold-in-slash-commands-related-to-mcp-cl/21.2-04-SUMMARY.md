## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking Issue] dyn McpReloader needed to disambiguate reload() method**
- **Found during:** Task 2 first cargo build
- **Issue:** `mgr.as_ref().reload()` resolved to `McpManager::reload(new_configs)` (concrete method, takes 1 arg) instead of `McpReloader::reload()` (trait method, takes 0 args) — Rust resolves inherent methods before trait methods
- **Fix:** Coerce to `let reloader: &dyn McpReloader = mgr.as_ref();` before calling `reloader.reload().await`
- **Files modified:** `crates/ironhermes-cli/src/main.rs`
- **Commit:** 1db24b6

**2. [Rule 2 - Missing Critical Functionality] McpReload variant needed in TUI CommandResult and gateway handler**
- **Found during:** Task 1 workspace build
- **Issue:** Adding `McpReload` to `CoreCommandResult` caused non-exhaustive match errors in `map_core_to_tui` (tui/commands.rs) and gateway handler.rs
- **Fix:** Added `McpReload` variant to TUI `CommandResult`, added mapping in `map_core_to_tui`, added dead-code arm in gateway handler
- **Files modified:** `crates/ironhermes-cli/src/tui/extension.rs`, `crates/ironhermes-cli/src/tui/commands.rs`, `crates/ironhermes-gateway/src/handler.rs`
- **Commit:** 9427c6a

**3. [Rule 2 - Design Improvement] build_mcp_manager() helper extracted**
- **Found during:** Task 2 implementation
- **Issue:** The MCP wiring block is identical in run_chat, run_single, and run_gateway — duplicating ~50 lines would be error-prone and unmaintainable
- **Fix:** Extracted `build_mcp_manager(config, registry)` async helper that encapsulates the pattern; each run path calls it in one line
- **Files modified:** `crates/ironhermes-cli/src/main.rs`
- **Commit:** 1db24b6
