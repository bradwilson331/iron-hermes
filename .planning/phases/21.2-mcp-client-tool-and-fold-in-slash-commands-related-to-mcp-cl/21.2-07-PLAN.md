<objective>
Close GAP-4 (high severity, duplicates CR-01 in REVIEW.md): the MCP manager's tool-discovery heuristic and `unregister_by_prefix` call pass the RAW server name, but `make_prefixed_name` sanitizes `-`/`.`/`@`/`/` → `_` before registration. Consequence today:

- Servers with special-char names (e.g. `@modelcontextprotocol/server-filesystem`, `my-server`, `github.com`) never match the `starts_with("{raw_name}__")` heuristic in `start_all_and_wait` (line 117 of `manager.rs`) — they are never reported as `connected` even when healthy.
- `shutdown_all` calls `unregister_by_prefix(&name)` with the RAW name (line 152) — it strips NOTHING, so every `/reload-mcp` leaks the previous generation's tools and the registry accumulates duplicates.

Both failures are latent today only because UAT Test 1 hung on GAP-1 before a real special-char server ever reached reload. Once Plan 06 fixes the hang, a user running `hermes mcp add @modelcontextprotocol/server-filesystem` will immediately hit this double-registration bug.

Fix:
1. In `crates/ironhermes-mcp/src/tool.rs`, expose a standalone `pub fn sanitize_server_name(name: &str) -> String` that applies the exact same character replacements as `make_prefixed_name`'s server half (covering `-`, `.`, `@`, `/` → `_`). Update `make_prefixed_name` to delegate to it so the two paths cannot drift.
2. Broaden the sanitization set to include `@` and `/` — the UAT evidence (`@modelcontextprotocol/server-filesystem`) proves these must be rewritten too. Today `make_prefixed_name` only handles `-` and `.`. This is a targeted fix for GAP-4, not a scope expansion: the whole purpose of the sanitizer is to produce tool-registry-safe names, and `@` / `/` both appear in real MCP package identifiers.
3. In `crates/ironhermes-mcp/src/manager.rs`, use `sanitize_server_name` at both offending sites: the `starts_with("{sanitized}__")` heuristic in `start_all_and_wait` and the `unregister_by_prefix(&sanitized)` call in `shutdown_all`.
4. Add a regression test asserting `reload_with_special_char_server_name_does_not_duplicate_tools` — uses a server name hitting all four characters.

Purpose: Unblock the UAT re-test after Plan 06 ships. Without this fix, any user who adds an MCP server whose name matches a real-world npm package identifier will see tools double-register on every `/reload-mcp`.

Output: A correct, symmetric sanitization contract between registration and unregistration, covered by a hyphen-and-dot-and-at-and-slash regression test.
</objective>
