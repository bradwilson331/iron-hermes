# Phase 21.2 Plan 07: GAP-4 Close — Sanitized Prefix Symmetry at Register/Unregister Summary

Fixed the asymmetric sanitization between tool registration and unregistration in `McpManager` so servers with special-character names (`@`, `/`, `-`, `.`) now report `connected` correctly and do not leak duplicate tools on `/reload-mcp`.

## What Shipped

### Task 1 — sanitize_server_name helper + broadened sanitizer (commit 7cbc7f6)

- Added `pub fn sanitize_server_name(name: &str) -> String` to `crates/ironhermes-mcp/src/tool.rs` with `-`, `.`, `@`, `/` → `_` replacement.
- Rewrote `make_prefixed_name` to delegate to `sanitize_server_name` for both halves (server + tool).
- Re-exported `sanitize_server_name` from `crates/ironhermes-mcp/src/lib.rs`.
- Extended the `make_prefixed_name` doctest to cover the real-world `@modelcontextprotocol/server-filesystem` GAP-4 evidence string.
- Added 4 unit tests: `@`/`/` replacement, all-four-chars combined, npm-package regression, structural invariant.

### Task 2 — manager.rs bug fixes + reload round-trip regression (commit 9a17319)

- Imported `sanitize_server_name` at top of `manager.rs`.
- **Bug site 1 — `start_all_and_wait`** (was line 117): replaced `format!("{name}__")` with `format!("{}__", sanitize_server_name(name))`, so the `has_tools` heuristic now matches the sanitized registry keys. Special-char-named servers are now reportable as `connected`.
- **Bug site 2 — `shutdown_all`** (was line 152): replaced `unregister_by_prefix(&name)` with `unregister_by_prefix(&sanitize_server_name(&name))`, so the tools the server registered are actually removed on shutdown.
- Added `shutdown_all_unregisters_tools_for_server_with_special_char_name` — manually registers two tools under a `@scope/pkg-name.v1` prefix, seeds the task map, calls `shutdown_all`, asserts zero residue.
- Added `reload_with_special_char_server_name_does_not_duplicate_tools` — two back-to-back register+shutdown cycles; asserts the registry is empty (no accumulation drift).

## Before / After at the Bug Sites

**Before (`manager.rs` line 117):**

```rust
let has_tools = guard
    .get_definitions(None)
    .iter()
    .any(|t| t.function.name.starts_with(&format!("{name}__")));
```

**After:**

```rust
// GAP-4: registered tools use the SANITIZED prefix (make_prefixed_name
// replaces `-`/`.`/`@`/`/` with `_`), so the lookup must sanitize too.
let sanitized_prefix = format!("{}__", sanitize_server_name(name));
let has_tools = guard
    .get_definitions(None)
    .iter()
    .any(|t| t.function.name.starts_with(&sanitized_prefix));
```

**Before (`manager.rs` line 152):**

```rust
let removed = guard.unregister_by_prefix(&name);
```

**After:**

```rust
// GAP-4: unregister_by_prefix appends "__" to its argument; we must pass the
// already-sanitized server name so the match finds the tools we registered.
let removed = guard.unregister_by_prefix(&sanitize_server_name(&name));
```
