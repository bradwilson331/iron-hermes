# Phase 21.2 Plan 06: Gap-closure wizard hardening Summary

**Hermes MCP wizard now times out in `connect_timeout` seconds, never silently persists broken entries, and hints npx users toward canonical args — closing UAT gaps GAP-1/2/3 in `mcp_config.rs`.**

## Accomplishments

- **GAP-1 closed:** All three MCP CLI entry points (`hermes mcp add`, `hermes mcp test`, `hermes mcp configure`) now return within `config.connect_timeout` seconds with an actionable error (`"Timed out waiting for MCP initialize response after Ns. Check command and args."`) — no more indefinite hang when a child process is alive but not responding to MCP initialize.
- **GAP-2 closed:** `cmd_add` no longer silently writes broken entries. On connection failure the user is prompted `Connection failed. [R]etry / [S]ave anyway / [A]bort?` defaulting to Abort; Abort short-circuits with `Cancelled.` dimmed and returns `Ok(())` BEFORE `save_mcp_server_config`.
- **GAP-3 closed:** When the user selects `npx` as the stdio command, a dimmed hint line `Example: -y @modelcontextprotocol/server-filesystem /tmp` prints above the Arguments prompt — directly addressing the typo + empty-args path that fed into GAP-1.
- 4 new tests added (1 async + 3 sync); all 8 `mcp_config::tests` pass; workspace compiles clean.

## Decisions Made

- **Kept `attempt_connect_and_list` body untouched** — the new `_with_timeout` wrapper is the single layer all callers now go through. This preserves the existing transport-connect implementation as a pure helper.
- **Abort emits `"Cancelled.".dimmed()` and returns `Ok(())`** — matches the existing UI-SPEC cancellation copy used by cmd_remove and the cmd_add "already exists" branch (3 total call sites in the module).
- **SaveAnyway kept** — plan D-02 lists it as the explicit-consent variant of the legacy "user may fix server config later" escape hatch; the change converts an implicit default into an explicit opt-in.
- **Retry arm re-prints the "Connecting to..." cyan status** — the outer pre-loop print fires exactly once; subsequent Retry attempts get a fresh visible status line, matching the UX the user saw on the first attempt.
- **Literal-copy regression tests via `include_str!`** — 2 of the 3 new tests read this very module's source at compile time and assert substring presence, locking the exact user-facing copy against silent drift in future refactors.
