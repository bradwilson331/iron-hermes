## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] S-04 fire-and-forget write race under parallel test load**
- **Found during:** Task 10-03 (running full `cargo test -p ironhermes-cli` after Task 10-02)
- **Issue:** `TranscriptWriter::append` spawns a separate tokio task per call (fire-and-forget). The three appends in S-04 raced to open + write, and a fixed 300ms sleep was insufficient on a loaded parallel-test runner — the test observed `ToolCall` as the "last line" instead of `Cancelled`. Isolated run passed; parallel run failed.
- **Fix:** Replaced fixed sleep with a 5s poll-loop oracle that reads the file every 50ms and breaks as soon as (a) line count >= 3 AND (b) last line parses as `TranscriptLine::Cancelled`. Also inserted `yield_now() + 50ms` between appends so each spawned open() has a chance to start in sequence.
- **Files modified:** `crates/ironhermes-cli/tests/scenarios_cascade_cancel.rs`
- **Commit:** `0ff0b2d`

### Auto-added Missing Functionality

**2. [Rule 2 — Missing dev-deps] walkdir + sysinfo added to ironhermes-cli dev-deps**
- **Found during:** Task 10-01 / 10-02 compilation
- **Issue:** Plan referenced `walkdir` for S-08 source-tree walk and `sysinfo` for S-02 PID-liveness oracle, neither was in `ironhermes-cli`'s `[dev-dependencies]`.
- **Fix:** Added `walkdir = "2"` (fresh dep, not previously workspace-level) and `sysinfo = { workspace = true }` (surfaces existing workspace dep).
- **Files modified:** `crates/ironhermes-cli/Cargo.toml`
- **Commit:** `d3086da`

### Deferred Items (Out of Scope)

**Pre-existing parallel-test flake in `ironhermes-core`:** `provider::tests::provider_resolver_loads_disk_cache_at_build` panics under parallel test execution (test pollution from env-var mutation in sibling tests). Passes in isolation and under `--test-threads=1`. This is a pre-existing issue on `develop` unrelated to Plan 10 — no Plan 10 code touches `ironhermes-core`. Logged here for visibility; not fixed as out of scope per `<deviation_rules>` SCOPE BOUNDARY. `cargo test --workspace -- --test-threads=1` is the recommended workspace-gate command until a `serial_test` pass lands in a dedicated hygiene plan.
