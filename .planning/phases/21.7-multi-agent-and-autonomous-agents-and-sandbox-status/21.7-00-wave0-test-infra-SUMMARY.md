## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed self-tripping static-grep gate on BudgetHandle doc comment**
- **Found during:** Task 0-03 first test run of `budget_ordering_grep`.
- **Issue:** Initial `budget.rs` module doc started with "AI-SPEC Pitfall 9 (E-05): SeqCst ordering only — NEVER `Ordering::Relaxed` on this shared counter." The literal string `Ordering::Relaxed` inside the prose caused `budget_uses_only_seqcst_ordering` to fail on the shell file itself — the static-grep is a naive `.contains(...)` over the whole file.
- **Fix:** Rephrased the doc comment to "use SeqCst ordering only on this shared counter. The lax Relaxed variant is forbidden here because the pressure-tier transitions must be observed in a consistent total order across threads." — keeps the intent (Relaxed is forbidden) without embedding the exact forbidden substring.
- **Files modified:** `crates/ironhermes-agent/src/budget.rs`
- **Commit:** included in ee3d050 (the single Task 0-03 atomic commit)

**2. [Rule 1 - Bug] Plan referenced HERMES_HOME; live code uses IRONHERMES_HOME**
- **Found during:** Task 0-02 — writing the new test.
- **Issue:** Plan text showed `std::env::set_var("HERMES_HOME", tmp.path())` but `ironhermes_core::get_hermes_home()` reads `IRONHERMES_HOME` (constants.rs:31). The existing sibling test `test_ensure_home_dirs_creates_all_subdirs` already uses `IRONHERMES_HOME`.
