## Chosen `start_output_drain` API

**Option B** (from plan open_questions):

```rust
pub async fn start_output_drain(reg: Arc<RwLock<Self>>, id: &str) -> anyhow::Result<()>
```

Rationale (locked in plan Task 6-02 `<action>` recommendation):
- `&mut self` signature can't clone the registry Arc for drain tasks — it's the `Arc<RwLock<_>>` borrow that carries the shared-ownership semantics.
- The tokio `ChildStdout` / `ChildStderr` types are unstable across tokio versions and leaking them through the tool layer (`execute_code` / `terminal`) would force every downstream consumer to pin a compatible tokio.
- Keeping drain tasks INSIDE the registry module:
  - Preserves encapsulation of `ProcessSession.child` and `.cancel`.
  - Caller (tool layer) writes 2 lines: `spawn(...)` + `start_output_drain(reg.clone(), &id)`.
  - Tasks are fire-and-forget — shutdown happens naturally on EOF when `drain_and_kill` kills the child.

The alternative (Option A: `spawn()` returns `(id, Option<ChildStdout>, Option<ChildStderr>)`) was rejected.

## Gateway Budget Lifecycle

Chosen: **gateway-process-scoped** — matches the BudgetHandle Plan 05 precedent exactly.

At `run_gateway` startup:
```rust
let process_registry = Arc::new(RwLock::new(
    ProcessRegistry::new_for_session("gateway".to_string()),
));
```

Per-request handler `on_session_end`:
```rust
// session_id_str = format!("gw:{}:{}", event.chat_id, event.sender_id)
reg.write().await.drain_and_kill_session(&session_id_str).await
```

Since the registry's `task_id == "gateway"` and the per-request `session_id_str == "gw:..."` never matches, this call is a **deliberate no-op**. The call is still emitted so:
1. INV-21.7-07 grep-gate stays green and the wiring is audit-visible.
2. When Plan 09 lands per-session `ProcessRegistry` construction (or equivalent SessionStore plumbing), the per-request drain will start doing real work WITHOUT a call-site change.

Meanwhile, background-process cleanup in the gateway happens via:
- **LRU-over-64** — oldest running entry is killed before the 65th spawn is accepted.
- **FINISHED_TTL = 30min** — finished entries are dropped on the next spawn or prune call.

This is the same trade-off as Plan 05's BudgetHandle scope: gateway-process-scoped now, per-session when Plan 09 lands SessionStore plumbing.
