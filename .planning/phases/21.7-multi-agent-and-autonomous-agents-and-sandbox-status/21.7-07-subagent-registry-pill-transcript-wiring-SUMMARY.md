# Phase 21.7 Plan 07: Subagent Registry Pill + Transcript Wiring Summary

Wave 2 closeout that makes Plans 03/05/06 user-observable: every subagent spawn/complete/cancel now updates `SubagentRegistry`, writes a JSONL transcript, and refreshes the `agents: N/M` status-line pill via an off-render-path `watch::send_modify`. `CommandContext` grows four new trait-object handles (`process_registry`, `subagent_registry`, `budget`, `subagent_semaphore` + `max_subagents`) consumed by downstream Wave-3 plans for `/agents` and `hermes status`. D-07 cancellation audit is complete — every cancelled subagent's transcript ends with `TranscriptLine::Cancelled` BEFORE unregister drops the pill count. Three new static-grep invariants (INV-21.7-08/09/11) lock the regression surface, and Pitfall-8 / ISS-05 `.read().await` on the render path is statically forbidden. All existing Plan 03/05/06 tests stay green; new tests total 9 (5 trait-impl unit tests in context.rs, 4 pill unit tests in status_line.rs, 5 helper tests in subagent_runner.rs, + 3 new invariants).

**`run_child` new lifecycle (every code path):**

1. Generate `subagent_id = random_subagent_id()` (shape `sub_` + 12 hex from UUID v4 first 6 bytes).
2. Derive `task_summary = derive_task_summary(system_prompt)` — strips the DelegateTaskTool prefix + takes first 80 chars on a char boundary.
3. If scope is set, open a `TranscriptWriter::open(transcript_path_for(home, session, sub_id))`.
4. Register `SubagentInfo { id, task_summary, parent_id: None, started_at: Instant::now(), cancel: cancel_for_info, transcript_path }`. `cancel_for_info` is a clone of the passed `cancel_token` (or an orphan token if None).
5. Wrap the incoming `tool_progress` callback: every `tool_call` event appends a `TranscriptLine::now_tool_call(name, args_preview)` before calling the inner callback.
6. Build `AgentLoop`, wire cancel/progress/budget, run messages.
7. **Terminal marker (D-07 audit)**:
   - `cancel_token.is_cancelled()` → `writer.append(TranscriptLine::now_cancelled("cancelled via parent token"))`.
   - Ok(result) → `writer.append(TranscriptLine::now_done(preview))`.
   - Err → `writer.append(TranscriptLine::now_done("(error: run_child returned Err)"))` so every transcript terminates.
8. Unregister from SubagentRegistry **AFTER** the terminal line lands. This ordering ensures the pill drop to N-1 is always preceded by a durable transcript terminator.

## CommandContext Dep-Graph Decision

Chose trait-object form (plan Option B) with newtype wrappers to respect Rust's orphan rule:

```
ironhermes-core (leaf crate)
├─ trait BudgetSnapshot
├─ trait SubagentListSnapshot
└─ trait ProcessRegistrySnapshotHandle

ironhermes-agent (depends on core)
├─ impl BudgetSnapshot for BudgetHandle                  // owned type — no newtype
├─ struct SubagentRegistryHandle(Arc<RwLock<...>>)      // newtype for Arc<_>
└─ impl SubagentListSnapshot for SubagentRegistryHandle

ironhermes-exec (gains core dep — no cycle)
├─ struct ProcessRegistryHandle(Arc<RwLock<...>>)        // newtype
└─ impl ProcessRegistrySnapshotHandle for ProcessRegistryHandle
```

All three trait methods are SYNC (for /agents + hermes status handlers). The async-to-sync bridge uses `tokio::task::block_in_place` + `Handle::current().block_on` — the same pattern `ironhermes-core/src/commands/handlers.rs` uses for `/models refresh` at L248-254. Safe on the multi-thread runtime; locks uncontended in practice (single-session registry, single gateway registry).

## ISS-05 Resolution (Pitfall 8 Regression-Locked)

The pill refresh follows the pattern:

1. **Observe side** (async, off render path): a spawned task does ONE `registry.read().await.active_count()`. This runs on a tokio worker thread, NOT the render task.
2. **Emit side** (sync): `status_tx.send_modify(|s| s.active_subagents = n)`. send_modify is a sync method on `watch::Sender` — no `.await` involved.
3. **Render side** (pure): `render_status_line` reads `state.active_subagents: usize` — a copied value. Zero registry awaits, ever.

INV-21.7-11 locks this statically via 3 assertions:
- `grep -c 'registry.read().await' crates/ironhermes-cli/src/tui/status_line.rs` must be `0`.
- `grep -c 'registry.write().await' crates/ironhermes-cli/src/tui/status_line.rs` must be `0`.
- `grep -c 'status_tx.send_modify' crates/ironhermes-cli/src/main.rs` must be `≥ 1`.

All three pass today.
