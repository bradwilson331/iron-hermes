<objective>
Close GAP-21.7-01 by wiring a concurrent slash-command input path through the REPL during in-flight agent turns. The root cause is already diagnosed (`.planning/debug/agents-list-empty-while-live.md` — status `root_cause_found`): `rl.readline()` at `crates/ironhermes-cli/src/main.rs:994` is strictly sequential with `run_agent_turn` at `main.rs:1131-1196`, which in turn awaits every `delegate_task` spawn at `crates/ironhermes-tools/src/delegate_task.rs:326-332`. By the time the user can physically type `/agents list`, every subagent has already called `unregister()` at `crates/ironhermes-agent/src/subagent_runner.rs:252`. The registry is legitimately empty at query time.

**Chosen approach: Option B (concurrent input via tokio::select).** Move the blocking `rl.readline()` onto a dedicated `spawn_blocking` worker thread (a `ReplInputChannel` module in `ironhermes-cli`) that sends each line over a `tokio::sync::mpsc` channel. The existing mid-turn `tokio::select!` at `main.rs:1151-1193` gains a third arm that polls this channel. Slash-prefixed lines are dispatched through the existing `CommandRouter` + `CommandContext` machinery (same as the prompt-time branch at `main.rs:1046-1116`); non-slash lines are discarded with a dimmed notice ("turn in flight — slash commands only; input discarded"). The agent turn is never cancelled by mid-turn input.

**Why B over A and C:**
- Option A (make `detach=true` truly detach execute_batch early) breaks the `delegate_task` tool contract at `delegate_task.rs:326-340` — the agent consumes numbered batch results; silently turning that into "spawned N subagents" is a **scope reduction** of the locked D-02 contract (subagent runner surfaces results), not a gap closure.
- Option C (new `spawn: true` schema field + `/tail <id>` command) adds a new slash command + new schema surface — disproportionate for a gap plan. Contract break same as A.
- Option B reuses the existing `tokio::select!` pattern (Phase 21 Plan 03 Task 2), the existing `CommandRouter` + `CommandContext` (Plans 07/08), the existing `status_tx.send_modify` pill refresh (Plan 07 INV-21.7-11), and the existing `SubagentRegistryHandle::block_in_place` sync bridge. No tool-facing contract changes. No new schema. Fixes D-03, D-04 pill refresh observability, and unlocks `/stop` mid-turn drain as a side benefit.

**Arc identity** is already correct per Plan 07 Task 7-03 — debug session confirmed both the runner's `with_subagent_registry(subagent_registry.clone())` at `main.rs:737` and `CommandContext::with_subagent_registry(SubagentRegistryHandle::new(subagent_registry.clone()))` at `main.rs:1030-1033` clone the SAME `Arc`. No registry-wiring changes needed. The only defect is the absence of a concurrent input path — this plan adds it.
