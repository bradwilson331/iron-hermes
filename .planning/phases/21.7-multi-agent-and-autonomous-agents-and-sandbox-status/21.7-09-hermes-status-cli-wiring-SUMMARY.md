# Phase 21.7 Plan 09: hermes status CLI Wiring Summary

Closes Wave 3 CLI surface work by wiring the Plan 04 schema + DeepProbe
seam into a live `hermes status` command with all four section collectors,
a colored default text mode, a JSON mode, and real deep probes for
provider HEAD + FTS5 integrity (plus an honest-unreachable fallback for
MCP per ISS-11). Same plan also ships the D-08 / AI-SPEC Pitfall 5 fix:
both `DelegateTaskTool` timeout arms now hard-cancel the child
`CancellationToken` before returning `Err`, preventing subagents from
surviving past their wall-clock deadline despite the parent timing out.

## Exact `Commands::Status` Clap Shape

Binary (`main.rs`) — the live entry point:

```rust
Commands::Status(ironhermes_cli::status_cmd::StatusArgs)
```

Lib-reachable mirror (`cli_args.rs`) — used by INV-21.7-10 parse-level tests:

```rust
use crate::status_cmd::StatusArgs;
Status(StatusArgs),
```

Shared `StatusArgs`:

```rust
#[derive(clap::Args, Debug, Clone)]
pub struct StatusArgs {
    #[arg(long)] pub all: bool,
    #[arg(long)] pub deep: bool,
    #[arg(long)] pub json: bool,
}
```

Dispatch at `main.rs::main` replaces the old `cmd_status()` stub:

```rust
Some(Commands::Status(args)) => ironhermes_cli::status_cmd::run_status(args).await,
```

## D-08 Timeout Fix — Before / After

### Single path (`delegate_task.rs::execute`)

Before (broken):

```rust
let result = tokio::time::timeout(
    Duration::from_secs(self.config.timeout_secs),
    self.runner.run_child(/*...*/, child_cancel_token, /*...*/),
).await;
match result {
    // ...
    Err(_) => return Err(anyhow::anyhow!("Subagent timed out after {}s", self.config.timeout_secs)),
}
```

After (fixed):

```rust
let effective_timeout_secs = args.get("timeout_seconds")
    .and_then(|v| v.as_u64())
    .unwrap_or(self.config.timeout_secs);
let timeout_cancel = child_cancel_token.clone();

let result = tokio::time::timeout(
    Duration::from_secs(effective_timeout_secs),
    self.runner.run_child(/*...*/, child_cancel_token, /*...*/),
).await;
match result {
    // ...
    Err(_) => {
        if let Some(tok) = timeout_cancel.as_ref() { tok.cancel(); }
        tracing::info!(
            target: "ironhermes_tools::delegate_task",
            timeout_secs = effective_timeout_secs,
            "subagent cancelled via timeout"
        );
        return Err(anyhow::anyhow!("Subagent timed out after {}s", effective_timeout_secs));
    }
}
```

### Batch path (`delegate_task.rs::execute_batch`)

Same pattern: per-task `timeout_seconds` hoisted into
`per_task_timeout_secs` outside the spawned task; `timeout_cancel` clone
inside the spawn; cancel arm fires `.cancel()` and logs before returning
the timeout error string.

### Regression proof (`delegate_task_timeout_cancel.rs`)

- `DetachedSleepRunner` spawns a `tokio::spawn` task holding a clone of
  the cancel token. The task outlives the outer `run_child` drop, so
  the atomic flip only happens if `.cancel()` was called explicitly (not
  just via `Drop`).
- Test completes in ~1.1s (config timeout would be 60s+). Atomic is
  `true` after the cancel signal propagates (100ms grace to let the
  spawned watcher poll).

## LiveDeepProbe Implementation Choices

### Provider HEAD (`provider_health`)

- Uses a single `reqwest::Client` with a 5s outer-client safety belt.
- `tokio::time::timeout` enforces the per-call deadline (3s from
  `StatusReport::collect`).
- Status classification: 2xx / 3xx / 401 / 403 → `Ok(true)`. Rationale:
  401 / 403 mean the endpoint is serving — we only probe liveness, not
  auth.
- Unknown providers return `Ok(false)` with a `tracing::info` log at
  target `ironhermes_cli::status` (FM-5 — never falsely healthy).

Hardcoded probe URL table:

| Provider     | URL                                   |
| ------------ | ------------------------------------- |
| `anthropic`  | `https://api.anthropic.com/v1/messages` |
| `openai`     | `https://api.openai.com/v1/models`      |
| `openrouter` | `https://openrouter.ai/api/v1/models`   |

### FTS5 integrity (`fts5_integrity`)

- `rusqlite::Connection::open(path)` in `tokio::task::spawn_blocking`.
- `PRAGMA integrity_check` returns exactly `"ok"` on healthy DBs;
  any other string → `Ok(false)`.
- Missing file → `Ok(false)` (not `Err`). A missing state.db isn't
  "unhealthy" but it also isn't "ok-verified" — the deep probe reports
  it as such so operators see it.
- Wall-clock bound via `tokio::time::timeout` around the
  `JoinHandle`.

### MCP probe (`mcp_server`) — ISS-11 honest-unreachable fallback

Reports `McpProbeResult { reachable: false, tool_count: 0, latency_ms: 0 }`
with a `tracing::warn` at target `ironhermes_cli::status` explaining the
deferral. Rationale:

- `hermes status` is a one-shot CLI. Constructing an `McpManager` for a
  one-shot probe would duplicate ~300 lines of config parse + rmcp
  transport wiring.
- Reporting `reachable=true` would be a lie (FM-5 — false-negative
  masking outage); `reachable=false` is honest.
- A future phase threads the existing `McpManager` handle through
  `CommandContext` / `run_status` so the probe can do a real
  `list_all_tools` roundtrip. Tracked as ISS-11 in the plan text.

## rusqlite vs StateStore Seam Choice

Used `rusqlite::Connection::open` inline in probe.rs rather than
extending `StateStore` with a new `integrity_check()` method. Rationale:

- StateStore's constructor runs schema init + FTS5 setup migrations —
  we do NOT want those side effects during a one-shot health probe
  (the probe might fire against a DB a newer StateStore version would
  try to migrate).
- The direct `Connection::open` + `PRAGMA integrity_check` path is
  read-only on the DB — it never triggers migrations or DDL.
- rusqlite was already a workspace dep; adding it to ironhermes-cli's
  direct deps is a one-line change.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking] `live_probe` possibly-uninitialized in run_status**

- **Found during:** Task 9-02 first workspace build — the initial
  scaffolding structured `run_status` with a `live_probe` binding that
  was only initialized under `#[cfg(not(test))] args.deep=true`, so the
  `--deep=false` and `#[cfg(test)]` branches left it uninit.
- **Fix:** Switched to an `Option<LiveDeepProbe>` gated by
  `#[cfg(not(test))]`, with the probe_ref dispatch inside the same
  cfg-gated block and a `&NoopDeepProbe` fallback. The release binary
  path stays unchanged; the test build still compiles.
- **Files modified:** `crates/ironhermes-cli/src/status_cmd.rs`
- **Committed in:** `fcbd189`.

**2. [Rule 3 — Blocking] Test's MockDeepProbe type collision with published trait**

- **Found during:** Task 9-03 first test run — the initial integration
  test used `#[path = "../src/status_cmd/probe.rs"] mod probe;` to
  import `MockDeepProbe`, then tried to pass `&MockDeepProbe` into
  `StatusReport::collect` which takes `&dyn
  ironhermes_cli::status_cmd::probe::DeepProbe` from the LIB crate.
  Two crate identities → two distinct `DeepProbe` traits → compile
  error.
- **Fix:** Replaced the `#[path=...]` approach with a small `FaultyProbe`
  adapter living in the integration test that implements the published
  `DeepProbe` trait directly. `MockDeepProbe` stays inside the lib's
  own test scope where it was originally intended (Plan 04).
- **Files modified:** `crates/ironhermes-cli/tests/status_cmd_deep_probe.rs`
- **Committed in:** `7e5c02d`.

**3. [Rule 1 — Bug] Cancel test's single-select runner never observed cancel**

- **Found during:** Task 9-01 first GREEN cycle — initial test used a
  `tokio::select!` on `sleep(10s) OR cancel.cancelled()`. When
  `tokio::time::timeout` expired, it dropped the entire select (including
  the `cancelled()` branch) BEFORE our `.cancel()` call could set the
  token. The atomic was never flipped.
- **Fix:** Redesigned the mock runner to spawn a DETACHED
  `tokio::spawn` task holding a clone of the cancel token. The detached
  task outlives the outer future being dropped by tokio::time::timeout,
  so the only way it observes cancellation is via an explicit
  `.cancel()` — proving the fix rather than `Drop` semantics.
  Added a 100ms post-timeout sleep so the scheduler gives the watcher
  a chance to poll.
- **Files modified:** `crates/ironhermes-tools/tests/delegate_task_timeout_cancel.rs`
- **Committed in:** `ad57346`.
