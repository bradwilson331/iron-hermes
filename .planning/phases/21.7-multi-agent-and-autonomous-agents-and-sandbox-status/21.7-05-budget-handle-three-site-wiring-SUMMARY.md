fn check_budget_threshold(&mut self) -> Option<&'static str> {
    let budget = self.budget.as_ref()?;

## Gateway Budget Lifecycle

Chosen: **gateway-process-scoped** (single `BudgetHandle` constructed at `run_gateway` startup, cloned into both `AgentSubagentRunner` and `GatewayRunner` → `GatewayMessageHandler`, used by all per-request `AgentLoop` instances until process exit).

Rationale:
- A true per-request BudgetHandle would reset the counter on every message, decoupling it from `AgentSubagentRunner`'s registration lifetime and violating PROV-10 (parent + delegated children must share the same counter within a request, and across a multi-turn session).
- A true per-user-session BudgetHandle (one per `SessionKey`) requires plumbing the handle through `SessionStore` — that plumbing does not exist today and is explicitly scoped to Plan 09 (`hermes status`) where per-session budget readouts land.
- The gateway-process-scoped compromise gives:
  - ✅ PROV-10 parent + subagent shared decrement (same handle clones into both sites).
  - ✅ INV-21.7-04 three-site wiring parity satisfied without a lifecycle rewrite.
  - ⚠️  All Telegram users on the same gateway instance share the same budget pool. Once exhausted, every user sees clean-stops until the operator restarts the process OR until Plan 09 lands per-session plumbing.

Documented in `crates/ironhermes-cli/src/main.rs:1305-1316` as an inline block comment for future maintainers.
