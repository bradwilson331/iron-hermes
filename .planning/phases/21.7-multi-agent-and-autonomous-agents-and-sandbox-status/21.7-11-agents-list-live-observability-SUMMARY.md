# Phase 21.7 Plan 11: /agents list Live Observability Summary

Close GAP-21.7-01 — `/agents list` returned empty while delegate_task subagents were live — by adding a concurrent slash-command input path through the REPL so mid-turn slash dispatch works while the agent turn is in flight. Root cause (sequential `rl.readline()` blocking dispatch until every subagent had already unregistered) is cured by hosting rustyline's `DefaultEditor` on a dedicated OS thread behind a tokio mpsc bridge (`ReplInputChannel`) and adding a third arm to run_chat's `tokio::select!` loop.

## What Shipped

### Task 11-01 — ReplInputChannel module (commit `c6142fd`)

New module `crates/ironhermes-cli/src/repl_input.rs` (~385 LOC). Hosts the blocking `rustyline::DefaultEditor` on a dedicated `std::thread::spawn` worker thread and exposes a tokio-friendly mpsc-based API. Public surface:

```rust
pub struct ReplInputChannel { /* ... */ }
pub struct PromptRequest { pub prefix: String, pub in_turn: bool }
pub enum ReplLine { Line(String), Interrupted, Eof, Error(String) }

impl ReplInputChannel {
    pub fn spawn(history_path: Option<PathBuf>) -> anyhow::Result<Self>;
    pub fn request_prompt(&self, req: PromptRequest) -> anyhow::Result<()>;
    pub async fn recv_line(&mut self) -> Option<ReplLine>;
    pub fn drain_buffered(&mut self) -> usize;
    pub fn add_history(&self, line: &str);
    pub fn shutdown(self);
}
```

Internal shape is a command-channel pattern:

```text
   run_chat (tokio task)                     blocking std::thread worker
        |  request_prompt(req) --> cmd_tx.send(Prompt(req, oneshot_tx))
        |                                                          |
        |                                                   cmd_rx.blocking_recv()
        |                                                   rl.readline(&prefix)
        |                                                          |
        |  recv_line().await    <-- line_rx.recv()   <--   oneshot_tx.send(ReplLine)
        |                            (forwarder task demuxes per-prompt oneshots)
```

The `Command` enum serialises all editor ops (`Prompt(req, reply_tx) | AddHistory(String) | Shutdown`) under single ownership — the editor's `!Send` innards never cross a thread boundary. Per-prompt results ride a oneshot back to a forwarder tokio task that demuxes onto the public `line_rx` stream. Order is preserved: `request_prompt` writes the reply-receiver into the forwarder's queue and the command into the worker's queue in the same thread with no intervening await.

**Re-exported from `crates/ironhermes-cli/src/lib.rs`:** `ReplInputChannel`, `ReplLine`, `PromptRequest`.

Four unit tests, all green:
- `spawn_and_shutdown_cleanly`
- `shutdown_then_drain_is_fast`
- `add_history_after_shutdown_is_noop`
- `drain_buffered_is_nonblocking_and_zero_on_idle`

## Mid-Turn Select Loop — Final Shape

Three arms, `biased;` ordering preserved:

| Arm order | Trigger | Action | Affects turn? |
| --- | --- | --- | --- |
| 1 (biased-first) | `tokio::signal::ctrl_c()` | Double-ctrl-c state machine; emergency 3-press exit; `chat_cancel_token.cancel()` on CancelTurn/ExitCleanly. | YES — cancels the turn. |
| 2 (biased-second) | `repl_input.recv_line()` | Slash prefix → `build_cmd_ctx` → `dispatch_command` → route CommandResult. Non-slash → dimmed discard notice. `/clear` / `/mcp reload` ignored. `/quit` defers. | NO — turn continues. |
| 3 (biased-last) | `&mut run_fut` | `break 'turn r?;` | Natural completion. |

Biased ordering rationale:
- Ctrl-c wins races over slash (emergency-exit semantics).
- Slash input wins races against run_fut (so a burst of tool output doesn't starve the user's typed command).

## Deviations from Plan

### Rule 3 (blocking dep) — threading choice documented

Chose `std::thread::spawn` over `tokio::task::spawn_blocking` per the plan's Rule 3 anchor. Rustyline v15's `DefaultEditor` is `!Send` and holds an OS-level terminal handle; keeping it on a dedicated OS thread avoids pinning a tokio blocking-pool worker for the full session lifetime. `tokio::sync::mpsc::UnboundedSender` is `Send + Sync` from any thread and bridges cleanly. Documented at `crates/ironhermes-cli/src/repl_input.rs:~30` module-doc.

### Plan-checker advisory honored

Moved `prepare_prompt_with_reserve` / `finish_prompt_with_reserve` from around the worker-side readline to bracket the main-task `request_prompt` call. Row-reserve math now happens on the paint thread as required. Confirmed the inline `initial_prompt_flush_precedes_readline` test was updated to anchor on the new site.

### Inline test update — `rl.readline` anchor removed

The pre-plan-11 inline test `initial_prompt_flush_precedes_readline` used `.expect("run_chat must still call rl.readline for the REPL")` as the ordering anchor. With `rl.readline` removed, the test now anchors on `repl_input.request_prompt(` — semantically equivalent (still the first stdin-blocking site the GAP-5 banner must precede). Not a functional deviation; documented in the test docstring.

### `U-02 (drop-closes-worker)` unit test reshaped to `shutdown_then_drain_is_fast`

Original plan wording suggested a test that pattern-destructures the channel and asserts `recv_line` returns None. First iteration hung on `cargo test` — rustyline's Drop on a non-TTY test environment does not reliably release its terminal handle within the test timeout. Reshaped to an explicit `Shutdown` command + `drain_buffered` probe wrapped in a 500ms `tokio::time::timeout` — proves the worker exits and the line channel does not deadlock. Same invariant (shutdown is prompt), more robust test shape.
