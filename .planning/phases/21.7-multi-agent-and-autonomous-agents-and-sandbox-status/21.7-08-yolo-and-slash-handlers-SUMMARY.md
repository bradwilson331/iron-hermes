## Banner Text (Verbatim)

Emitted once per session on stderr by `print_yolo_banner_to_stderr(enabled)` in each of `run_single`, `run_chat`, `run_gateway`. With colored on, raw CSI sequences wrap the two text spans:

```
\x1b[1;31m--yolo enabled:\x1b[0m \x1b[31mall dangerous-command approvals are bypassed. Iteration budget, fatal error, and user interrupt are the only stops.\x1b[0m
```

With colored off: the same text minus ANSI. With `enabled == false`: zero bytes written. ANSI escapes are raw (`\x1b[1;31m` / `\x1b[31m` / `\x1b[0m`) rather than `colored::Colorize` so the bytes are deterministic under parallel tests — colored's `set_override` is process-global and the idempotent / disabled-writes-nothing / writes-bold-red tests raced against each other when they all changed that global.

## Final /agents and /stop Output Strings

### `/stop` (D-26, G-06)

| ProcessRegistry state | Output string |
|---|---|
| Wired, tracked = 0 before drain | `Stopped 0 background process(es).` |
| Wired, tracked = N before drain | `Stopped N background process(es).` |
| Not wired, agent idle             | `No agent is currently running. Use Ctrl-C to cancel an in-flight turn.` |
| Not wired, agent_running = true   | `Stopping agent... (note: cancellation token not yet wired — agent may continue)` |

The pre-drain tracked count is the killed number — `drain_and_kill` definitionally leaves 0 tracked, so the post-drain count is useless for operator feedback. The no-registry branch preserves the pre-Plan-08 advisory strings verbatim to keep `dispatch_stop_agent_idle_says_no_agent_running` + `dispatch_stop_agent_running_says_stopping` green.

### `/agents` (D-03)

| Invocation | Output |
|---|---|
| `/agents` (no args)  ⇒ same as `/agents list` | `No active subagents.` — or — `Active subagents:\n- sub_xxxxxxxx (summary…) — 3s\n- ...` |
| `/agents kill <id>` present    | `Cancelled subagent <id>.` |
| `/agents kill <id>` absent     | `No active subagent with id <id>.` |
| `/agents kill` (no id)         | `Error: /agents kill <id>: missing id` |
| `/agents logs <id>` present    | Bounded tail (last 200 lines) of the transcript file |
| `/agents logs <id>` absent     | `No transcript for id <id>.` |
| `/agents logs` (no id)         | `Error: /agents logs <id>: missing id` |
| No registry wired              | `Subagent registry not wired.` |
| Unknown subcommand             | `Error: Unknown /agents subcommand: <x>` |

Task-summary truncation is UTF-8-safe: `truncate_ellipsis(s, 80)` walks the `is_char_boundary` chain before appending `…`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Banner test race under parallel cargo test**

- **Found during:** Task 8-01 first green run — `maybe_print_yolo_banner_writes_bold_red_banner_when_enabled` failed because `colored::control::set_override(true)` in one test raced against `set_override(false)` in the parallel `writes_nothing_when_disabled` test. `colored` uses process-global state.
- **Fix:** Replaced `colored::Colorize` usage inside `maybe_print_yolo_banner` with raw ANSI CSI sequences (`\x1b[1;31m...\x1b[0m`). Deterministic bytes regardless of parallel-test state. The `colored` crate is still used elsewhere in main.rs (e.g. `cmd_version`) — only the banner bypasses it.
- **Files modified:** `crates/ironhermes-cli/src/yolo.rs`.
- **Committed in:** `87e373d` (Task 8-01, same commit as the feature).

**2. [Rule 1 — Bug] 2s semaphore warn test: tokio time driver ≠ std::Instant**

- **Found during:** Task 8-03 first RED/GREEN cycle — under `#[tokio::test(start_paused = true)]`, `tokio::time::advance(Duration::from_millis(2500))` jumps tokio's clock but `std::time::Instant::elapsed` uses the wall clock — actual elapsed ≈ microseconds, warn never fired.
- **Fix:** Harness uses `tokio::time::Instant` (time-driver-aware) inside the `run_instrumented_acquire` helper + `tokio::time::sleep(prewait_ms)` so paused time drives deterministic timing. Production code in `delegate_task.rs` still uses `std::time::Instant` — the warn-emission contract is about elapsed ≥ 2s, not about the specific Instant type.
- **Files modified:** `crates/ironhermes-tools/tests/delegate_task_semaphore_warn.rs`.
- **Committed in:** `14c305d`.

**3. [Rule 1 — Bug] tracing-test env-filter + span-scope dropped the warn**

- **Found during:** Task 8-03 second RED/GREEN cycle — even after fixing the timing, `logs_contain` returned false. Two root causes: (a) `tracing-test`'s default env-filter is `{test_crate_name}=trace`, dropping logs targeted at `ironhermes_tools::delegate_task` (different crate from the test binary `delegate_task_semaphore_warn`); (b) spawned tasks don't inherit the test's scope span, so `logs_contain`'s scope-filtered lookup missed warns emitted from `tokio::spawn` bodies.
- **Fix (a):** Enabled the `no-env-filter` feature on `tracing-test` in `ironhermes-tools/Cargo.toml` dev-deps. Makes the subscriber accept all targets.
- **Fix (b):** Redesigned the harness to drive `run_instrumented_acquire` inline in the test task (not via `tokio::spawn`) so the test's `info_span` wraps the acquire + warn.
- **Files modified:** `crates/ironhermes-tools/Cargo.toml`, `crates/ironhermes-tools/tests/delegate_task_semaphore_warn.rs`.
- **Committed in:** `14c305d`.

**4. [Rule 2 — Missing] main.rs three-arm dispatch to run_chat/run_single needed the yolo flag**

- **Found during:** Task 8-01 wiring — the `Commands::Chat` variant, the `None` batch arm (`cli.execute.is_some()`), and the bare-hermes arm all needed to pass the resolved yolo-flag into their respective `run_*` functions. The plan's `<action>` only said "at the top of run_chat / run_single / run_gateway, resolve yolo from flag + config" without specifying the caller plumbing.
- **Fix:** Added `cli_yolo_flag: bool` parameter to `run_chat` and `run_single`. Top-level `cli.yolo` OR'd with subcommand `Commands::Chat { yolo }` at the match arm. Gateway stays config-only (no CLI flag) per D-12.
- **Files modified:** `crates/ironhermes-cli/src/main.rs`.
- **Committed in:** `87e373d`.

**5. [Rule 3 — Blocking] ironhermes-tools dev-deps missed tracing-test + tokio[test-util]**

- **Found during:** Task 8-03 first build — the crate's `[dev-dependencies]` section was empty.
- **Fix:** Added `tracing-test = { workspace = true, features = ["no-env-filter"] }` and `tokio = { workspace = true, features = ["full", "test-util"] }` to `ironhermes-tools/Cargo.toml`. `test-util` enables `tokio::test(start_paused)` + `tokio::time::advance`.
