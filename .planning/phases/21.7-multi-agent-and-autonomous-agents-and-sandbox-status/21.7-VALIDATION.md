## Sampling Rate

- **After every task commit:** Run `cargo test -p <crate-under-test>` for the crate that owns the modified files. If the task touches multiple crates, run `cargo test --workspace -- <test_name_prefix>`.
- **After every plan wave:** Run `cargo test --workspace`.
- **Before `/gsd-verify-work`:** Full `cargo test --workspace` must be green, plus `cargo clippy --workspace -- -D warnings` and `cargo fmt --all -- --check`.
- **Max feedback latency:** ≤ 120s for full suite; ≤ 15s for incremental per-crate.

**Nyquist rationale:** Rust's incremental compile amortizes well; every commit gets a full per-crate suite run without crossing the 120s ceiling. Wave-boundary `cargo test --workspace` samples integration-level behavior (cross-crate trait usage, three-site parity, CLI subprocess snapshots) at a lower rate — acceptable because the static-grep three-site parity test (Wave 0 infra E-09) runs per-commit as a regression gate.

## Manual-Only Verifications

Some behaviors cannot be cleanly unit-tested and require manual / scripted smoke verification. Planner must surface these as explicit UAT items in PLAN.md `must_haves`.

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Double-ctrl-c cascade-cancel end-to-end | D-07, E-01 | Requires real terminal SIGINT delivery (cargo-test harness simulates but doesn't exercise real PTY signal path); flaky in CI unless we add a dedicated PTY harness, which is out of Phase 21.7 scope | Launch `hermes chat`, issue `delegate_task` with long-running shell, press ctrl-c twice within 500ms, assert: (a) parent prompt returns, (b) `ps` shows zero hermes children, (c) no zombies via `ps -A -o pid,stat`. Document in UAT. |
| `--yolo` blast-radius smoke | D-11, D-13, E-02 | Behavioral verification that the bypass actually fires in a live agent loop (not just that the code path exists) — requires real LLM call | Launch `hermes chat --yolo "run a safe shell command"`, assert no approval prompt; then `hermes chat --yolo "run: echo test"` with a fake-dangerous shim, assert execution (not blocked). Document exact UAT steps. |
| Status pill visibility under real subagent load | D-04, E-07 | Visual / TUI behavior — snapshot testing the pill string via unit test is possible, but "pill hides at 0" across a real terminal requires live observation | Launch `hermes chat`, dispatch 2 subagents, observe `agents: 2/N` pill in status line; wait for completion, observe pill disappears. |
| Status-line pill pressure color changes | D-04, E-07 | Same reasoning as above | Drive budget to 70% / 90% thresholds via scripted long-loop input, observe pill color shift (green → yellow → red). |
| `hermes status --deep` live probe against real backends | D-21, E-06 | The `#[cfg(test)]` mock seam covers the logic; a real end-to-end probe against a live Anthropic / local MCP / state.db is a production smoke test, not CI | Run `hermes status --deep` in a sandbox with real providers/MCPs; assert exit 0 and all probes report healthy. |
