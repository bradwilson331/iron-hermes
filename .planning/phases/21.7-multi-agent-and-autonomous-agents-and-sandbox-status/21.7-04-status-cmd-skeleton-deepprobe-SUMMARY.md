# Phase 21.7 Plan 04: status_cmd Skeleton + DeepProbe Seam Summary

Locked the `hermes status` v1 JSON schema (D-20) behind an insta baseline snapshot, stood up the `DeepProbe` trait seam (D-21/E-06) with `NoopDeepProbe` + cfg-gated `LiveDeepProbe`/`MockDeepProbe`, and verified via `nm` on the release rlib that the `MockDeepProbe` test-only mock does not link into the production binary. Plan 09 (Wave 3) now has a stable schema to collect into and three deterministic bail-string hooks to replace with live probes.

## What Shipped

### Task 4-01 — StatusReport v1 schema + sub-structs (commits 3d52978 RED, 4deb437 GREEN)

`crates/ironhermes-cli/src/status_cmd.rs` — replaced the 2-line Wave-0 placeholder with the ~200-line Wave-1 skeleton:

- `StatusArgs` (clap::Args) — three booleans: `--all`, `--deep`, `--json`.
- `StatusReport` — 7 top-level component sections (D-18 four sections expanded into the canonical seven-key shape from AI-SPEC §4).
- Sub-structs: `ProviderStatus`, `MemoryStatus`, `MemoryFile`, `GatewayStatus`, `SubagentStatus`, `BudgetView`, `ProcessesStatus`, `ProcessEntry`, `McpStatus`, `McpServer`, `YoloStatus`.
- Every Option field is `#[serde(skip_serializing_if = "Option::is_none")]` so Plan 09 can add richer fields (`--deep` outputs, `--all` per-server detail) without breaking the v1 schema for existing consumers.
- `StatusReport::fixture()` returns a deterministic byte-stable example used by the insta baseline.
- `run_status()` is an async stub that `anyhow::bail!`s — so a malformed early wiring attempt fails loud rather than silent-succeed.
- `print_styled()` is a no-op stub — Plan 09 fills it with `colored` styling matching `memory_cmd.rs:22-75`.
- `pub mod probe;` declaration added at the top of the file.

`crates/ironhermes-cli/tests/status_cmd_schema.rs` — three tests:
- `status_report_v1_json_schema_snapshot` — insta baseline of the serialized fixture.
- `status_report_round_trips` — serde_json round-trip equality.
- `top_level_keys_are_exactly_seven` — asserts the v1 shape lock (extra or missing top-level key is a failure).

`crates/ironhermes-cli/tests/snapshots/status_cmd_schema__status_report_v1.snap` — committed baseline. Future field renames MUST bump v2 or the snapshot rejects.

### Task 4-02 — DeepProbe trait seam + fault-injection harness (commits 4aa8fbe RED, b65431e GREEN)

`crates/ironhermes-cli/src/status_cmd/probe.rs` — new file (~200 lines) exporting:

- `McpProbeResult { name, reachable, tool_count, latency_ms }` — serialize-able probe outcome.
- `trait DeepProbe: Send + Sync` with three async methods:
  - `provider_health(provider_name, timeout) -> anyhow::Result<bool>`
  - `fts5_integrity(db_path, timeout) -> anyhow::Result<bool>`
  - `mcp_server(name, timeout) -> anyhow::Result<McpProbeResult>`
  - Every method takes an explicit `timeout: Duration` — T-21.7-04-02 DoS mitigation; implementers MUST wrap work in `tokio::time::timeout`.
- `NoopDeepProbe` — default when `--deep` is NOT set; returns healthy without side effects.
- `LiveDeepProbe` (`#[cfg(not(test))]`) — placeholder that bails with the exact strings:
  - `"LiveDeepProbe::provider_health not yet implemented — Plan 09"`
  - `"LiveDeepProbe::fts5_integrity not yet implemented — Plan 09"`
  - `"LiveDeepProbe::mcp_server not yet implemented — Plan 09"`
  (Plan 09 executor should `grep -rn "LiveDeepProbe::" crates/ironhermes-cli` to locate them.)
- `MockDeepProbe` (`#[cfg(test)]`) — builder-style fault injection:
  - `set_provider(Result<bool>)`, `set_fts5(Result<bool>)`, `set_mcp(name, Result<McpProbeResult>)`
  - Unset overrides default to healthy (keeps unrelated tests green).

`crates/ironhermes-cli/tests/status_cmd_probe_seam.rs` — 6 integration tests:
- `provider_probe_reports_unhealthy_on_mocked_500` (S-14)
- `provider_probe_surfaces_error_not_ok_false` (Err vs Ok(false) distinction — critical for UI coloring)
- `fts5_probe_reports_corrupt` (S-15)
- `mcp_probe_reports_timeout_unreachable` (S-16)
- `mcp_probe_unset_server_returns_default_healthy`
- `noop_probe_always_returns_healthy`

The test file uses `#[path = "../src/status_cmd/probe.rs"] mod probe;` to compile probe.rs directly into the integration test crate — required because `MockDeepProbe` lives behind `#[cfg(test)]` and is intentionally not part of the lib's public API (T-21.7-04-05).
