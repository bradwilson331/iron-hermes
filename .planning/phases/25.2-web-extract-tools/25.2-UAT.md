### 6. YouTube URL Dispatches to Skill
expected: Ask the agent "Extract https://www.youtube.com/watch?v=dQw4w9WgXcQ". Agent recognizes YouTube and either (a) returns content via the youtube-content skill, or (b) returns a clear error explaining the youtube-content skill needs setup (e.g., yt-dlp). It does NOT silently try to scrape the YouTube page as HTML.
result: pass
notes: web_extract did the right thing — YouTube was correctly classified and dispatched to the youtube-content skill, which returned an explicit "skill not installed" error (option b). The agent (LLM, not the tool) then chose to fall back to a separate web reader tool to satisfy the user's request — that's agent-level retry behavior, NOT web_extract silently HTML-scraping YouTube. Tool contract upheld. Follow-up: install the youtube-content skill (`hermes skills add youtube-content`) to get clean dispatch in option (a).

### 7. SSRF Protection Blocks Internal IPs
expected: Ask the agent "Extract content from http://192.168.1.1/admin". Agent receives a clear error stating the URL was blocked (private/internal IP), and does NOT actually attempt to fetch.
result: pass
notes: SSRF guard fired with a clear "private/internal IP address" error and explicitly listed the blocked ranges (192.168.x.x, 10.x.x.x, 172.16-31.x.x). Agent did NOT attempt the fetch. Confirms validate_url_async() pre-fetch SSRF check from Plan 01 + threat T-25.2-03 mitigation across all backends.

### 8. Long Content Gets Summarized

No raw HTML, no truncated mid-sentence dumps, content is coherent and referenced to the source. Confirms Plan 11's `route_tiers` + `summarize_chunked` pipeline (semaphore-bounded fan-out + final synthesis) works end-to-end on real-world content well above the Tier 3 threshold (500K chars).
