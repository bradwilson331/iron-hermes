## Accomplishments

- `pub enum UrlClass { YouTube, Pdf, Web }` and `pub enum Backend { Firecrawl, Exa, Tavily, Local }` exported from `web_extract::dispatch` — the shape Plans 25.2-06..09 and 25.2-13 will consume.
- `classify_url()` covers all six YouTube hosts (`youtube.com`, `www.youtube.com`, `youtu.be`, `www.youtu.be`, `m.youtube.com`, `music.youtube.com`) with case-insensitive lowercase compare against an exact-match allow-list — `evil-youtube.com` and `youtube.com.evil.example` both correctly classify as `Web`, mitigating T-25.2-host-spoof.
- PDF classification uses path suffix only (case-insensitive, ignores `?dl=1` and `#anchor`) — `https://example.com/article?file=foo.pdf` correctly classifies as `Web`, mitigating T-25.2-pdf-spoof.
- `select_backend()` follows the D-04 chain: `FIRECRAWL_API_KEY` > `EXA_API_KEY` > `TAVILY_API_KEY` > `Local`. Reads env vars at call time per the `web_read.rs:550` pattern — cheap and Plan-14-friendly.
- `reroute_for_pdf("application/pdf; charset=binary")` returns `true`, supporting D-03 mid-fetch reroute when the local backend's GET response surprises with a PDF Content-Type.
- 8 unit tests pass; clippy on `ironhermes-tools` produces zero new warnings on the touched file.

## Decisions Made

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Use `url::Url::parse` + `host_str()` (not regex) | Matches the `ironhermes-core::ssrf:16` workspace pattern; gives a fully-parsed URL we can also inspect for path. Regex on hosts is exactly the foot-gun T-25.2-host-spoof exists to prevent. | ✓ Good |
| Unparseable URLs → `UrlClass::Web` (not error) | D-02 partial-success contract: per-URL failures populate `ExtractionResult.error` later, not `Result::Err` from the classifier. The Web branch will surface the underlying parse failure when it tries to fetch. | ✓ Good |
| `select_backend()` reads env on every call | Matches `web_read.rs:550` precedent. No caching means Plan 14 integration tests can mutate env vars between cases via the existing `env_lock + EnvGuard` helpers from `tests/web_extract_integration.rs`. | ✓ Good |
| `reroute_for_pdf()` as a separate pure fn (not part of `classify_url`) | The local backend (Plan 09) needs to call it AFTER the GET response — coupling it to `classify_url(&str)` would force a string-based content-type round-trip. Separate fn keeps both signatures honest. | ✓ Good |
| Skip unit tests for `select_backend()` | Env-mutating tests race with parallel cargo test threads. Plan 14 owns env_lock-coordinated coverage. Documented as a comment in the test module. | ✓ Good |
| Add `url = { workspace = true }` to ironhermes-tools (Rule 3) | The plan called for `url::Url::parse` but the dep wasn't on the tools crate (only on `ironhermes-core` via ssrf). Required to compile. | ✓ Good (Rule 3 auto-fix) |
