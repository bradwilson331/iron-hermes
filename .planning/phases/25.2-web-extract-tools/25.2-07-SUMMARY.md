# Phase 25.2 Plan 07: D-04 / D-06 Exa + Tavily Backends Summary

**One-liner:** Implements the Exa (`x-api-key` header, plain-text response → wrapped with D-07 Markdown header) and Tavily (`Bearer` auth, no title → derived from URL path last segment) backends as hand-rolled reqwest modules — both mirror `firecrawl.rs`'s SSRF-guarded structural template and expose `*_ENDPOINT_OVERRIDE` env vars for Plan 14 wiremock hermetic tests, completing the 3-external-backend set for the Plan 13 D-04 chain.

## Accomplishments

- `pub async fn fetch_with_exa(url: &str) -> Result<ExtractionResult>` in `web_extract::backends::exa`:
  - **SSRF pre-validation FIRST** (line 1 of fn body) via `crate::web_local::validate_url_async` — T-25.2-03 mitigation enforced structurally.
  - **POST `https://api.exa.ai/contents`** with `.header("x-api-key", &api_key)` (NOT Bearer — verified against Exa API docs 2026-05-01) and `{ "ids": [url], "text": true }` body.
  - **Two-stage error surfacing**: HTTP-status check first, then `body.statuses[].error` per-URL inspection — handles Exa's documented partial-failure response shape where the HTTP layer returns 200 but individual URLs may carry inline errors.
  - **D-07 Option B Markdown wrapping**: Exa's `text` field is plain text (NOT Markdown — explicit RESEARCH.md Open Question §6 callout); we prepend `# {title}\nSource: {url}\n\n` to maintain the cross-backend D-07 contract.
  - **EXA_ENDPOINT_OVERRIDE** env var hook via private `resolve_endpoint()` helper — Plan 14 wiremock parity with the firecrawl.rs:118-120 pattern.
- `pub async fn fetch_with_tavily(url: &str) -> Result<ExtractionResult>` in `web_extract::backends::tavily`:
  - **SSRF pre-validation FIRST** (line 1 of fn body) via `crate::web_local::validate_url_async` — T-25.2-03 mitigation enforced structurally.
  - **POST `https://api.tavily.com/extract`** with `.bearer_auth(&api_key)` and `{ "urls": [url], "format": "markdown", "extract_depth": "basic" }` body.
  - **Failed-results-first error surfacing**: inspects `body.failed_results[0].error` before `body.results` — matches Tavily's documented response shape where partial-batch failures land in a sibling array.
  - **derive_title_from_url() helper**: Tavily returns no `title` field; helper takes URL path last non-empty segment, strips `.pdf` and `.html` suffixes, returns empty for invalid URLs / root paths. Exercised by 4 unit tests covering pdf-suffix, html-suffix, empty-path, and invalid-URL edge cases.
  - **D-07 Option B Markdown wrapping**: Tavily already returns Markdown via `format=markdown`, so wrapping is just header-prepend (`# {title}\nSource: {url}\n\n` or `Source: {url}\n\n` when title is empty).
  - **TAVILY_ENDPOINT_OVERRIDE** env var hook via private `resolve_endpoint()` helper — same shape as exa.rs and firecrawl.rs.
- 6 new unit tests pass (1 Exa endpoint helper + 1 Tavily endpoint helper + 4 Tavily derive_title cases); cargo build clean; clippy clean on the new module surface (zero warnings touching exa.rs or tavily.rs).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Critical] `#[allow(dead_code)]` added to `ExaStatus.status` field**

- **Found during:** Task 1 (after first cargo check)
- **Issue:** The plan's `<action>` block included the `status: Option<String>` field on `ExaStatus` for schema parity with Exa's documented response shape, but `fetch_with_exa` only reads `status.error`. Without `#[allow(dead_code)]`, this would have emitted a new compiler warning, regressing the warning baseline (currently 9 pre-existing warnings, all in `deferred-items.md`).
- **Fix:** Added `#[allow(dead_code)] // Reserved for future per-URL status surfacing (mirrors firecrawl status_code).` to the `status` field. Schema parity preserved without polluting baseline.
- **Files modified:** `crates/ironhermes-tools/src/web_extract/backends/exa.rs`
- **Verification:** `cargo check -p ironhermes-tools 2>&1 | grep -E 'exa|status'` returns zero matches; only the pre-existing `DEFAULT_SAFE_TOOLS` warning surfaces.
- **Committed in:** `4a7a3e1` (Task 1 commit, batched with the backend impl since the attribute is part of the same atomic struct definition).

**2. [Rule 2 - Critical] `#[allow(dead_code)]` added to `TavilyFailedResult.url` field**
