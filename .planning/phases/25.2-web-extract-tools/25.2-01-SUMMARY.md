## Decisions Made

- **Excluded `BOILERPLATE_SELECTORS` and `CONTENT_SELECTORS` from the web_read.rs import** even though they're now `pub` in web_local.rs. The plan explicitly allowed this (`Include only the items actually referenced elsewhere in web_read.rs`), and confirming via `grep` showed both names are only consumed inside the moved `extract_content_local` body. Importing dead names would trigger rust 2024 unused-import warnings.
- **Dropped `use scraper::{Html, Selector}` and `ssrf::is_safe_url` from web_read.rs.** Both were only referenced inside the moved bodies. Without removal, `cargo check -p ironhermes-tools` would fail under our `-D warnings` posture (dead `use` statements).
- **Smoke tests in web_local kept minimal** (3 tests) per plan's exact spec — the deeper algorithmic coverage (paragraph/sentence/word-break truncation, htmd conversion, header prepend, etc.) lives in `web_read::tests` (18 tests) and exercises the moved code through the new import path. No duplication.
