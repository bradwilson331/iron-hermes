# Phase 25.2 Plan 04: D-08 Base64 Strip + D-19 Secret-URL Detection Summary

**One-liner:** Pure-string sanitizer module landed: 9-entry const SECRET_URL_PATTERNS, regex-based base64 image stripper using OnceLock-cached compile, and a percent-decoding `contains_secret` predicate covering both raw + URL-encoded secret patterns — all behind a 3-function public surface validated by 11 unit tests.

## What Was Built

This plan replaces the Wave 0 stub at `crates/ironhermes-tools/src/web_extract/sanitize.rs`
with the full D-08 + D-19 implementation. The module is the first non-stub Wave 1 deliverable
in Phase 25.2's per-URL pipeline:

1. **D-08 base64 image stripping** — `pub fn strip_base64_images(content: &str) -> String`
   uses a OnceLock-cached `Regex` (compiled once on first call, reused thereafter) to scan for
   `data:image/<mime>;base64,<payload>` and replace each match with the literal `[image stripped]`.
   Bounded character classes (`[a-zA-Z0-9+.\-]+`, `[A-Za-z0-9+/=\n]+`) keep the pattern
   safe under the `regex` crate's RE2 NFA backend (no backtracking, per RESEARCH.md
   Pitfall 4). Newlines inside payloads are tolerated so wrapped MIME-style base64
   chunks still match.
2. **D-19 secret-URL pattern matching** — `pub const SECRET_URL_PATTERNS: &[&str]`
   exposes the 9-entry curated list (`token=`, `api_key=`, `api-key=`, `password=`,
   `secret=`, `bearer ` (with space), `auth=`, `access_token=`, `access-token=`).
   Plan 13 will pass operator-supplied extras via `&cfg.extract.redact_url_patterns`
   (Plan 02 ExtractConfig field).
3. **D-19 detection predicate** — `pub fn contains_secret(url, extra_patterns) -> bool`
   builds a case-insensitive haystack of the URL plus its percent-decoded form
   (so `?token%3Dabc` is caught the same as `?token=abc`), then short-circuits on
   first match against the const list OR the operator extension list.
4. **Hand-rolled percent decoder** — `fn percent_decode_lossy(s) -> String` (15 lines,
   private to module) handles `%xx` byte sequences with invalid-escape passthrough.
   Preserves the no-new-workspace-deps mandate (D-25): the `urlencoding` /
   `percent-encoding` crates are NOT added; only the doc comments mention them
   (explaining what we don't use).
5. **11 unit tests** locked into `mod tests` cover:
   - simple base64 image strip
   - multi-image strip with embedded newline (count=2 assert)
   - non-image data URLs untouched
   - token=` query-param match
   - every individual const pattern (loop test, 8 entries excluding bearer)
   - `Bearer ` (space) match via percent-decoded `%20`
   - case-insensitive (`TOKEN=ABC`, `Api_Key=Abc`)
   - percent-encoded `?token%3Dabc` decode
   - operator extension list (`x_custom_secret=`)
   - clean-URL negative cases (`?id=1234`, `arxiv.org/.../12345.pdf`)
   - const completeness + exact `len() == 9` lock
