# Phase 25.2 Plan 02: D-22 ExtractConfig in ironhermes-core Summary

**One-liner:** Added the `extract:` config block (`ExtractConfig` struct + `Config::extract` field + `lib.rs` re-export) and registered `summarization` as the second resolve_role consumer in `RESERVED_ROLE_NAMES`, all backward-compatible via `#[serde(default)]`.

## What Was Built

### Schema additions (CONTEXT D-22)

- New public struct `ExtractConfig` in `crates/ironhermes-core/src/config.rs` with 6 fields:
  - `max_parallel_summaries: usize` (default `4`) — D-15 single-semaphore permit count for both multi-URL parallelism AND per-chunk summarization.
  - `summary_chunk_chars: usize` (default `100_000`) — D-11 tier-3 chunk size.
  - `refuse_threshold_chars: usize` (default `2_000_000`) — D-11 tier-4 refusal cutoff.
  - `summary_tier2_threshold_chars: usize` (default `5_000`) — D-11 tier 1→2 boundary.
  - `summary_tier3_threshold_chars: usize` (default `500_000`) — D-11 tier 2→3 boundary.
  - `redact_url_patterns: Vec<String>` (default empty) — D-19 operator extension list appended to the const default `SECRET_URL_PATTERNS` (consumed by Plan 04 sanitize).
- `impl Default for ExtractConfig` matching the locked spec defaults.
- New `pub extract: ExtractConfig` field on `Config` with `#[serde(default)]` — pre-25.2 YAML configs (no `extract:` block) parse cleanly.
- Re-export added to `crates/ironhermes-core/src/lib.rs` (slotted into the existing `pub use config::{...}` block).

### Phase 26 cascade extension (CONTEXT D-13)
