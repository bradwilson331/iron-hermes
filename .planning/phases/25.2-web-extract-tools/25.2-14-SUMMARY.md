# Phase 25.2 Plan 14: D-13 Cascade Second Consumer + D-20 Production Binary Wireup Summary

Wires `web_extract` into the production binary end-to-end: adds the cascade-aware
`AnyClientSummarizationHandle` (Phase 26 second consumer of `resolve_role`), re-exports it
from `ironhermes-agent`, and calls `registry.register_web_extract_tool` in all three CLI
construction sites (`run_chat` / `run_single` / `run_gateway`). Two guards (a 3-site source-grep

silent regression. Phase 25.2 is now feature-complete — `web_extract` ships.

```rust
#[tokio::test]
async fn web_extract_tool_appears_in_definitions_after_wireup() {
    let config = Config::default();
    let resolver = ironhermes_core::ProviderResolver::build(&config).unwrap();
    let handle: Arc<dyn ironhermes_core::SummarizationClientHandle> =
        Arc::new(AnyClientSummarizationHandle::new(Arc::new(resolver)));
    let tmp = tempfile::tempdir().expect("tempdir");
    let skill_registry = Arc::new(SkillRegistry::load(tmp.path()));
    let mut registry = ToolRegistry::new();
    registry.register_web_extract_tool(handle, skill_registry);
    let defs = registry.get_definitions(None);
    let names: Vec<String> = defs.iter().map(|d| d.function.name.clone()).collect();
    assert!(names.iter().any(|n| n == "web_extract"), ...);
}
```

Test result: `12 passed; 0 failed` (10 pre-existing any_client tests + dyn-compat test +
new smoke test).
