## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parent agent -> Child agent | Task description crosses from parent LLM context to child system prompt |
| Child agent -> Parent tools | Child accesses parent tools via filtered registry |
| Child agent -> Filesystem | Child's TerminalTool operates in isolated temp CWD |
| Child agent -> Memory | Child reads parent memory but cannot write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Elevation of Privilege | delegate_task recursion | mitigate | delegate_task structurally excluded from child registry in build_child_registry — never registered (AGENT-05) |
| T-09-02 | Denial of Service | unlimited subagents | mitigate | tokio::sync::Semaphore with configurable max_subagents (default 3) + timeout (default 300s) per D-08, D-14, D-16 |
| T-09-03 | Tampering | child writes to parent memory | mitigate | MemoryTool::new_read_only blocks save/forget actions with error message per D-12 |
| T-09-04 | Information Disclosure | child reads parent terminal state | mitigate | TerminalTool::with_cwd(tempdir) isolates child to unique temp directory per D-10 |
| T-09-05 | Tampering | prompt injection via task description | accept | Task comes from parent LLM which is trusted; no additional scanning at this layer |
| T-09-06 | Elevation of Privilege | child accesses skills system | mitigate | SkillsTool silently skipped in build_child_registry per D-05 |
| T-09-07 | Elevation of Privilege | child accesses execute_code | mitigate | execute_code silently skipped in build_child_registry — subagents cannot spawn sandboxed code execution |
