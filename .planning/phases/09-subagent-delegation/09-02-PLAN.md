```rust
    config.resolve_base_url(),
    config.resolve_api_key().unwrap_or_default(),
    config.model.default.clone(),
);

    gateway_client,
    subagent_semaphore,
    Some(memory_store.clone()),
    None, // hook_registry not yet Arc'd — will be set after Arc wrapping
    config.subagent.clone(),
);
```

NOTE: The hook_registry is built AFTER the ToolRegistry is Arc-wrapped in the current code. Since `register_delegate_task_tool` needs to happen before Arc wrapping, pass `None` for hook_registry initially. The DelegateTaskTool will still function — child agent loops just won't fire hook events. If the executor finds a way to restructure the initialization order (build hook_registry before tool registration), they should pass `Some(hook_registry.clone())` instead. This is acceptable for initial delivery — hook events from child agents are a nice-to-have, not a requirement.

ALTERNATIVE: If the executor restructures initialization to build `Arc<HookRegistry>` before tool registration, pass `Some(hook_registry.clone())` to `register_delegate_task_tool`. This is preferred but not required.
