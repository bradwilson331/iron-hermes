## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parent CancellationToken -> Child token | Parent cancel propagates to children via token hierarchy |
| Detach flag -> Token isolation | detach:true creates independent token, breaking propagation |
| Progress callback -> stderr | Progress data written to stderr in CLI, not to stdout/LLM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-14 | Denial of Service | orphaned child after parent cancel | mitigate | Default propagate (D-21): child token derived from parent, cancellation cascades automatically via tokio_util::CancellationToken::child_token() |
| T-09-15 | Denial of Service | detached child runs indefinitely | accept | detach:true is opt-in by the LLM; timeout (D-08) still applies as backstop; semaphore slot held until completion |
| T-09-16 | Information Disclosure | progress callback leaks task content | mitigate | SubagentProgress::Started truncates task_summary to 50 chars; tool names are not sensitive (they are tool metadata, not arguments) |
</threat_model>
