<objective>
When `hermes cron create` runs without `--deliver`, compute the default delivery target from `~/.ironhermes/config.yaml::gateway.platforms.telegram` and route the new job back to its origin Telegram chat when the gateway whitelist has exactly one entry. This restores the v1.x ergonomic where CLI-created cron jobs auto-delivered to TG when a TG gateway was running. Phase 22.4.2.1 Plan 02 already wired `dispatch_delivery` to consume `deliver="origin"` jobs (per `22.4.2.1-02-SUMMARY.md`); this plan makes `cmd_create` produce such jobs.

Purpose: pure default-routing fix. Three coordinated edits — (1) add `OriginDecision` enum + `Config::telegram_default_origin()` method to `ironhermes-core::config` (RESEARCH §Q-OQ1 Option A: plain-data `Single { platform: String, chat_id: String }` to avoid circular crate dep), (2) extract `pub(crate) fn resolve_cron_deliver(deliver_flag: Option<String>, config: &Config) -> (String, Option<JobOrigin>)` in `cmd_create` and consult the helper for the None case (RESEARCH §Q-OQ2 chosen seam), (3) flip `Create.deliver` from `String` (default "local") to `Option<String>` per D-11. Tests + INV land in the same plan because the new test target needs the new helper to exist (Wave 0 dep).

```rust
pub fn telegram_default_origin(&self) -> OriginDecision {
    let Some(tg) = self.gateway.platforms.get("telegram") else {
        return OriginDecision::None;
    };
    if !tg.enabled {
        return OriginDecision::None;
    }
    match tg.whitelist.len() {
        0 => OriginDecision::None,
        1 => OriginDecision::Single {
            platform: "telegram".to_string(),
            chat_id: tg.whitelist[0].to_string(),
        },
        _ => OriginDecision::Multi {
            whitelist: tg.whitelist.iter().map(|id| id.to_string()).collect(),
        },
    }
}
```

NEW resolve_cron_deliver helper (this plan adds, in crates/ironhermes-cli/src/cron.rs — pub(crate) for direct unit-test invocation per RESEARCH §Q-OQ2):
```rust
/// Resolve the (deliver, origin) pair for `hermes cron create`.
/// - Some(flag) → respect explicitly per D-04, helper not consulted.
/// - None + OriginDecision::Single → ("origin", Some(JobOrigin{...}))
/// - None + OriginDecision::Multi → ("local", None) + caller eprintln hint
/// - None + OriginDecision::None → ("local", None) silently per D-05
pub(crate) fn resolve_cron_deliver(
    deliver_flag: Option<String>,
    config: &ironhermes_core::config::Config,
) -> (String, Option<ironhermes_cron::JobOrigin>) {
    use ironhermes_core::config::OriginDecision;
    match deliver_flag {
        Some(d) => (d, None),
        None => match config.telegram_default_origin() {
            OriginDecision::Single { platform, chat_id } => (
                "origin".to_string(),
                Some(ironhermes_cron::JobOrigin {
                    platform,
                    chat_id,
                    chat_name: None,
                    thread_id: None,
                }),
            ),
            OriginDecision::Multi { whitelist } => {
                eprintln!(
                    "hermes cron create: Telegram gateway has multiple authorized chats — defaulting to deliver=local."
                );
                eprintln!(
                    "                      Pass --deliver telegram:<chat_id> to route to a specific chat (whitelist: {:?}).",
                    whitelist
                );
                ("local".to_string(), None)
            }
            OriginDecision::None => ("local".to_string(), None),
        },
    }
}
```
