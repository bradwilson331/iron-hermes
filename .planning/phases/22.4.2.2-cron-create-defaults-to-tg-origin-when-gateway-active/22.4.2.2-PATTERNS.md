## Critical Architecture Note: Crate Dependency Boundary

`ironhermes-core/Cargo.toml` does NOT list `ironhermes-cron` as a dependency (confirmed). `ironhermes-cron` imports from `ironhermes-core` (`use ironhermes_core::get_hermes_home`). Adding `ironhermes-cron` to `ironhermes-core` dependencies would create a **circular dependency**.

**Resolution (per RESEARCH Q-OQ1):** Define `OriginDecision` in `ironhermes-cron` (where `JobOrigin` lives), not in `ironhermes-core`. Expose `telegram_default_origin(config: &Config) -> OriginDecision` as a free function in `ironhermes-cron` (e.g., in `crates/ironhermes-cron/src/config_ext.rs` or appended to `delivery.rs`). The CLI crate depends on both crates and assembles the result. Alternatively, define a local data-only struct in `ironhermes-core` and convert in the CLI — but the enum-in-cron approach is cleaner.

And add this method inside `impl Config` after `env_path()`:

```rust
/// Compute the default cron delivery origin from the TG gateway config.
/// Returns `OriginDecision::None` when TG is missing, disabled, or has empty whitelist.
/// Returns `OriginDecision::Single` when whitelist has exactly one entry.
/// Returns `OriginDecision::Multi` when whitelist has >1 entries (caller emits hint).
pub fn telegram_default_origin(&self) -> OriginDecision {
    let Some(tg) = self.gateway.platforms.get("telegram") else {
        return OriginDecision::None;
    };
    if !tg.enabled {
        return OriginDecision::None;
    }
    match tg.whitelist.len() {
        0 => OriginDecision::None,
        1 => OriginDecision::Single {
            platform: "telegram".to_string(),
            chat_id: tg.whitelist[0].to_string(),
        },
        _ => OriginDecision::Multi {
            whitelist: tg.whitelist.iter().map(|id| id.to_string()).collect(),
        },
    }
}
```

```rust
fn cmd_create(
    name: String,
    schedule: String,
    prompt: String,
    deliver: Option<String>,   // was: String with default_value="local"
    skills: Vec<String>,
) -> Result<()> {
    // Security scan on prompt (unchanged)
    if let Err(e) = scan_cron_prompt(&prompt) {
        eprintln!("{}: {}", "Error".red().bold(), e);
        return Err(anyhow!("Prompt blocked by security scanner"));
    }

    // Parse schedule (unchanged)
    let parsed = parse_schedule(&schedule)
        .with_context(|| format!("Invalid schedule: {:?}", schedule))?;
    let schedule_display = match &parsed {
        ScheduleParsed::Once { display, .. } => display.clone(),
        ScheduleParsed::Interval { display, .. } => display.clone(),
        ScheduleParsed::Cron { display, .. } => display.clone(),
    };

    // NEW: compute deliver target and origin (D-04: explicit flag wins, no helper)
    let (deliver_str, origin_opt) = match deliver {
        Some(d) => (d, None),
        None => match ironhermes_core::config::Config::load()
            .unwrap_or_default()
            .telegram_default_origin()
        {
            ironhermes_core::config::OriginDecision::Single { platform, chat_id } => (
                "origin".to_string(),
                Some(ironhermes_cron::JobOrigin {
                    platform,
                    chat_id,
                    chat_name: None,
                    thread_id: None,
                }),
            ),
            ironhermes_core::config::OriginDecision::Multi { whitelist } => {
                eprintln!(
                    "hermes cron create: Telegram gateway has multiple authorized chats \
                     — defaulting to deliver=local.\n\
                     \                      Pass --deliver telegram:<chat_id> to route \
                     to a specific chat (whitelist: {:?}).",
                    whitelist
                );
                ("local".to_string(), None)
            }
            ironhermes_core::config::OriginDecision::None => ("local".to_string(), None),
        },
    };

    let mut store = open_store()?;
    let job = store.add_job(
        name,
        prompt,
        parsed,
        schedule_display.clone(),
        deliver_str,   // was: deliver
        skills,
        origin_opt,    // was: None
    )?;

    // println! output (unchanged)
    println!("{}: {} ({})", "Job created".bold().cyan(), job.name.bold(), job.id.dimmed());
    println!("  {:<12} {}", "Schedule:".dimmed(), schedule_display);
    if let Some(next_run) = job.next_run_at {
        println!(
            "  {:<12} {}",
            "Next run:".dimmed(),
            next_run.format("%Y-%m-%d %H:%M UTC").to_string().dimmed()
        );
    }
    Ok(())
}
```

//! Phase 22.4.2.2 — behavioral tests for `cmd_create` TG default-routing.
//!
//! Tests drive `Config::telegram_default_origin()` directly (unit seam) and verify
//! the resulting `OriginDecision` maps to the correct (deliver_str, origin_opt) pair.
//! `IRONHERMES_HOME` is redirected to a TempDir for each test so `Config::load()`
//! reads the fixture config.yaml.
//!
//! Run: `cargo test -p ironhermes-cli --test cron_default_deliver`

use ironhermes_core::config::{Config, OriginDecision};
use ironhermes_cron::{JobOrigin, JobStore};
use tempfile::TempDir;

/// Process-global env lock — `IRONHERMES_HOME` is shared across the whole
/// test binary and tests can race under `cargo test --jobs N`.
fn env_lock() -> &'static std::sync::Mutex<()> {
    use std::sync::OnceLock;
    static LOCK: OnceLock<std::sync::Mutex<()>> = OnceLock::new();
    LOCK.get_or_init(|| std::sync::Mutex::new(()))
}

/// Write a fixture config.yaml for TG gateway config.
fn make_test_config(tmp: &TempDir, tg_enabled: bool, whitelist: &[i64]) {
    let whitelist_yaml = if whitelist.is_empty() {
        "      whitelist: []".to_string()
    } else {
        let entries = whitelist
            .iter()
            .map(|id| format!("        - {}", id))
            .collect::<Vec<_>>()
            .join("\n");
        format!("      whitelist:\n{}", entries)
    };
    let yaml = format!(
        "gateway:\n  platforms:\n    telegram:\n      enabled: {}\n{}\n",
        tg_enabled, whitelist_yaml
    );
    let config_path = tmp.path().join("config.yaml");
    std::fs::write(&config_path, yaml).unwrap();
}
