<objective>
When the LLM calls the `cronjob` tool's `create` action without supplying a `deliver` arg, default-route the new job exactly the way Plan 01's CLI `cmd_create` does — consult `Config::telegram_default_origin()`, route to `deliver="origin"` with a populated `JobOrigin` when whitelist len==1, fall back to `local` with a hint when whitelist len>1, fall back silently to `local` when TG is disabled or missing. This brings the LLM-tool entry point to parity with Plan 01's CLI entry point so cron jobs created by the agent itself ALSO auto-deliver back to Telegram by default.

Purpose: pure default-routing fix at a second consumer site. The helper (`Config::telegram_default_origin`) and enum (`OriginDecision`) already exist after Plan 01 lands — Plan 02 is mechanical. The KEY DELTA from Plan 01 is the surface for the multi-chat hint: from inside an LLM tool, `eprintln!` to stderr would pollute output the LLM reads back, so we use `tracing::warn!` instead. The tool's JSON response field `deliver` reflects the actual decision so the LLM sees the result; the warn is for operator-side observability.

- New `crates/ironhermes-tools/tests/cronjob_tool_default_deliver.rs` with 5 behavioral tests mirroring Plan 01's 5 cases + `env_lock()` + `make_test_config` fixture.

```rust
let deliver_arg: Option<String> = args
    .get("deliver")
    .and_then(|v| v.as_str())
    .map(|s| s.to_string());

let (deliver, origin_opt): (String, Option<ironhermes_cron::JobOrigin>) = match deliver_arg {
    Some(d) => (d, None),
    None => {
        let config = ironhermes_core::config::Config::load().unwrap_or_default();
        match config.telegram_default_origin() {
            ironhermes_core::config::OriginDecision::Single { platform, chat_id } => (
                "origin".to_string(),
                Some(ironhermes_cron::JobOrigin {
                    platform,
                    chat_id,
                    chat_name: None,
                    thread_id: None,
                }),
            ),
            ironhermes_core::config::OriginDecision::Multi { whitelist } => {
                tracing::warn!(
                    "cronjob tool create: Telegram gateway has multiple authorized chats — defaulting to deliver=local. Pass deliver=telegram:<chat_id> to route (whitelist: {:?})",
                    whitelist
                );
                ("local".to_string(), None)
            }
            ironhermes_core::config::OriginDecision::None => ("local".to_string(), None),
        }
    }
};
```
