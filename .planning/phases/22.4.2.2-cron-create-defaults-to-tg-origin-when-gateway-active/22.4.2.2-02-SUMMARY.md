# Phase 22.4.2.2 Plan 02: TG Default-Routing in CronjobTool handle_create Summary

**One-liner:** LLM-tool path now auto-routes cron jobs to Telegram via `Config::telegram_default_origin()` when `deliver` arg is absent — brings `CronjobTool::handle_create` to parity with Plan 01's `cmd_create` CLI path, using `tracing::warn` (not `eprintln`) for the multi-chat hint per D-03 LLM-tool discipline.

## What Was Built

Plan 01 established `OriginDecision` enum and `Config::telegram_default_origin()` in `ironhermes-core`. This plan is the second consumer: the LLM-tool entry point at `CronjobTool::execute({"action":"create",...})`.

Three coordinated edits:

1. **`crates/ironhermes-tools/src/cronjob_tool.rs`** — Three surgical edits inside `handle_create`:
   - Import: added `JobOrigin` to the `ironhermes_cron` use statement
   - Edit B: replaced the static `let deliver = args.get("deliver").unwrap_or("local")` block with a two-step decision: `deliver_arg: Option<String>` extraction, then a match on `Config::telegram_default_origin()` when `deliver_arg` is `None`
   - Edit C: `store.add_job` 7th argument flips from `None` to `origin_opt`
   - `CronjobTool::new` signature bit-identical to pre-plan state (D-02 verified)
   - Multi-chat hint uses `tracing::warn!` (NOT `eprintln!`) per D-03

2. **`crates/ironhermes-tools/tests/cronjob_tool_default_deliver.rs`** (new file) — 5 behavioral integration tests exercising `CronjobTool::execute()` with `IRONHERMES_HOME` fixture override:
   - `tg_enabled_single_chat_routes_to_origin_via_tool` — deliver=origin, origin.chat_id=12345
   - `tg_enabled_multi_chat_falls_back_to_local_via_tool` — deliver=local, origin null
   - `tg_disabled_falls_back_to_local_via_tool` — deliver=local, origin null
   - `tg_section_missing_falls_back_to_local_via_tool` — deliver=local, origin null
   - `explicit_deliver_arg_skips_helper_via_tool` — explicit arg wins despite eligible config (D-04)

3. **`crates/ironhermes-cli/tests/invariants_22_4.rs`** — Appended `invariant_22_4_2_2_02_cronjob_tool_consults_origin_helper` as function 64 (final INV ledger count).

## D-12 UAT Note (MANDATORY — verbatim from plan verification block)

> Newly-enabled TG configs auto-route subsequent `cron create` calls only — including those issued by the LLM via the `cronjob` tool. Existing jobs in `~/.ironhermes/cron/jobs.json` are NOT migrated — fix individually with `hermes cron edit <id> --deliver telegram:<chat_id>`.
>
> Test the LLM-tool default by:
> 1. Ensure `~/.ironhermes/config.yaml` has `gateway.platforms.telegram.enabled: true` and a single-entry whitelist.
> 2. Start `hermes chat` and ask the agent to schedule a job: "Create a cron job that runs every hour and prints hello."
> 3. Inspect `~/.ironhermes/cron/jobs.json`: the new job MUST have `"deliver": "origin"`, `"origin": {"platform": "telegram", "chat_id": "<your_chat_id>", ...}`.
> 4. Multi-chat operator check: temporarily add a second whitelist entry, ask the agent to create another job, observe `tracing::warn` in the gateway log (NOT in the chat transcript), verify the created job has `deliver="local"`, `origin=null`.
