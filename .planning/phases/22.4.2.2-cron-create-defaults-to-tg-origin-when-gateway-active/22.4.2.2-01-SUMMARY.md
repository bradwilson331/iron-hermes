# Phase 22.4.2.2 Plan 01: TG Default-Routing in cmd_create Summary

**One-liner:** JWT-free TG origin auto-routing via `OriginDecision` enum in ironhermes-core + `resolve_cron_deliver` helper wired into `cmd_create` — restores v1.x ergonomic where `hermes cron create` (no `--deliver`) auto-routes to Telegram when the gateway whitelist has exactly one entry.

## What Was Built

Phase 22.4.2.1 Plan 02 (previously completed) wired `dispatch_delivery` to consume `deliver="origin"` jobs by reading `job.origin.platform` and `job.origin.chat_id`. This plan makes `cmd_create` *produce* such jobs by default when the TG gateway is configured with a single authorized chat.

Three coordinated edits:

1. **`crates/ironhermes-core/src/config.rs`** — Added `pub enum OriginDecision { None, Single { platform, chat_id }, Multi { whitelist } }` (plain-data, no `JobOrigin` embedding, no circular crate dep) and `pub fn Config::telegram_default_origin(&self) -> OriginDecision` method that reads `gateway.platforms.telegram.{enabled, whitelist}`. Four unit tests appended to `mod tests`.

2. **`crates/ironhermes-cli/src/cron.rs`** — Four surgical edits:
   - `Create.deliver`: `String` (with `default_value = "local"`) → `Option<String>` (no `default_value`, per D-11)
   - `cmd_create` signature: `deliver: String` → `deliver: Option<String>`
   - Extracted `pub(crate) fn resolve_cron_deliver(deliver_flag: Option<String>, config: &Config) -> (String, Option<JobOrigin>)` helper above `cmd_create`
   - `cmd_create` body: inserts `Config::load().unwrap_or_default()` + `resolve_cron_deliver` call; passes `deliver_str` and `origin_opt` to `store.add_job` (was raw `deliver` and `None`)
   - `cmd_edit` is **bit-identical** to pre-plan state (D-06 honored)

3. **`crates/ironhermes-cli/tests/cron_default_deliver.rs`** (new file) — 5 behavioral tests exercising `Config::telegram_default_origin()` directly with IRONHERMES_HOME fixture override pattern.

4. **`crates/ironhermes-cli/tests/invariants_22_4.rs`** — Appended `invariant_22_4_2_2_01_cmd_create_consults_origin_helper` as function 63 (Plan 02 of this phase will append function 64).

## D-12 UAT Note (MANDATORY — verbatim from plan verification block)

Newly-enabled TG configs auto-route subsequent `cron create` calls only. Existing jobs in `~/.ironhermes/cron/jobs.json` are NOT migrated — fix individually with `hermes cron edit <id> --deliver telegram:<chat_id>`.

Test the default by:
1. Ensure `~/.ironhermes/config.yaml` has `gateway.platforms.telegram.enabled: true` and a single-entry whitelist.
2. Run `hermes cron create --name 'test' --schedule '*/5 * * * *' --prompt 'echo hi'` (no `--deliver` flag).
3. Inspect the resulting job in `~/.ironhermes/cron/jobs.json`: `deliver` MUST be `"origin"`, `origin.platform` MUST be `"telegram"`, `origin.chat_id` MUST match the whitelist entry.

Multi-chat fallback test: temporarily add a second whitelist entry, run the same `cron create`, observe stderr hint, verify created job has `deliver="local"`, `origin=null`.
