# Phase 07.3 Plan 01: Cron Tick AgentLoop + HookRegistry Wiring Summary

**One-liner:** Real AgentLoop construction in cron tick task with HookRegistry capture, skill-content wiring, and four Wave-0 tests — closes v1.1 audit gaps.integration[0] and gaps.integration[1].

## What Was Done

Replaced the 11-line placeholder stub in the cron tick task closure of `crates/ironhermes-gateway/src/runner.rs` with a real `AgentLoop` construction and execution path. The placeholder string `"[Tick runner: agent execution pending full integration]"` is gone. The actual LLM response now flows into `complete_job_run()`.

### Changes (all in `crates/ironhermes-gateway/src/runner.rs`)

**New imports (sub-step 4a):**
- `ironhermes_agent::{AgentLoop, LlmClient, PromptBuilder}`
- `ironhermes_core::{ChatMessage, MessageContent, Role}`

**New helper function (sub-step 4b):** `pub(crate) async fn execute_cron_job(...)` — 120-line helper that:
1. Resolves `full_input` (skill content + user prompt) — D-02
2. Derives `cron_chat_id` from `job.origin` or falls back to `job.id` — D-12
3. Fires `MessageReceived` event to the shared `HookRegistry` — D-06, D-07
4. Builds `PromptBuilder` with `platform="cron"` — D-10
5. Constructs fresh `AgentLoop` per job (no streaming/tool-progress callbacks) — D-09
6. Wires optional `HookRegistry` via `with_hook_registry()` — D-06, D-08
7. Calls `agent.run(messages).await` — D-01
8. On success: fires `ResponseSent`, calls `complete_job_run(..., &output, true)` — D-03, D-14
9. On error: sanitizes to `"[Agent error: {}]"`, fires `ResponseSent`, calls `complete_job_run(..., &error_output, false)` — T-07.3-03, D-14

**Tick task rewrite (sub-step 4c):** Added four new closure captures before `join_set.spawn`:
- `let hook_registry_tick = self.hook_registry.clone()` — D-04
- `let tool_registry_tick = self.tool_registry.clone()` — D-11
- `let memory_store_tick = self.memory_store.clone()` — D-11
- `let config_tick = self.config.clone()` — D-11

The per-job loop body now calls `execute_cron_job(job, ...)` and logs errors without panicking — D-14, T-07.3-04.

**New tests (Tasks 1-3):**
- `test_placeholder_string_absent` — D-17; checks production-code portion of runner.rs (stops at `#[cfg(test)]`) contains no placeholder
- `test_cron_skill_reaches_llm` — D-15; `#[ignore]`-gated on `IRONHERMES_TEST_LLM=1`
- `test_cron_hook_registry_receives_events` — D-16; fires MessageReceived + ResponseSent, asserts `platform="cron"` and `chat_id="test-job-42"`
- `test_complete_job_run_receives_real_output` — D-03; proves `complete_job_run` persists `output` argument in `last_status`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `ref` keyword incompatible with `&Option<Arc<...>>` parameters**
- **Found during:** Sub-step 4b implementation (build failure)
- **Issue:** The plan's template code used `if let Some(ref registry) = hook_registry` where `hook_registry: &Option<Arc<...>>`. Rust 2021 edition treats this as double-borrow (`ref` on a reference type is illegal in this context).
- **Fix:** Removed `ref` keyword from all seven pattern matches in `execute_cron_job` — `if let Some(registry) = hook_registry` (auto-dereferences the outer `&`).
- **Files modified:** `crates/ironhermes-gateway/src/runner.rs`

**2. [Rule 1 - Bug] `Config::load_or_default()` does not exist**
- **Found during:** Task 1 (test_cron_skill_reaches_llm) compile check
- **Issue:** Plan template used `ironhermes_core::Config::load_or_default()` but only `Config::load()` and `Config::load_from()` exist.
- **Fix:** Changed to `Config::load()` in the `#[ignore]`-gated test.
- **Files modified:** `crates/ironhermes-gateway/src/runner.rs`

**3. [Rule 1 - Bug] Self-referential false positive in test_placeholder_string_absent**
- **Found during:** Task 1 test execution (test failed when it should have passed)
- **Issue:** `include_str!("runner.rs")` reads the full source including test code. The assertion text contains the placeholder string, so `source.contains(placeholder)` was always true.
- **Fix:** Changed the check to operate only on the production code portion — slice source up to `#[cfg(test)]` marker, then check that slice.
- **Files modified:** `crates/ironhermes-gateway/src/runner.rs`

**4. [Rule 1 - Bug] `JobStore::load_or_create()` / `guard.insert()` / `guard.get()` / `stored.last_output` do not exist**
- **Found during:** Task 3 implementation (studying actual JobStore API)
- **Issue:** Plan template used non-existent API methods. Actual API: `JobStore::open(dir: PathBuf)`, `add_job(name, prompt, schedule, display, deliver, skills, origin)`, `get_job(id: &str)`, and output stored in `last_status` (not `last_output` which doesn't exist on `CronJob`).
- **Fix:** Updated both test_cron_skill_reaches_llm and test_complete_job_run_receives_real_output to use real API.
- **Files modified:** `crates/ironhermes-gateway/src/runner.rs`
