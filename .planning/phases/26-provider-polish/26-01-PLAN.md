    Append to `crates/ironhermes-core/src/config.rs` (place near the existing struct definitions, before the `#[cfg(test)]` block). Hand-rolled char loop per PATTERNS § "slug/identifier validator reuse" — avoids dragging the `regex` crate into ironhermes-core if it isn't already a direct dep:

    ```rust
    /// Phase 26 D-04 (PROV-04): validate that `value` is a legal POSIX-ish env var name
    /// matching `^[A-Z][A-Z0-9_]*$`. Rejects empty, lowercase, shell metachars, leading
    /// digit/underscore. Hand-rolled char loop — no regex dep needed in ironhermes-core.
    pub fn validate_api_key_env(value: &str) -> Result<(), anyhow::Error> {
        if value.is_empty() {
            anyhow::bail!("api_key_env must not be empty");
        }
        let mut chars = value.chars();
        let first = chars.next().unwrap();
        if !first.is_ascii_uppercase() {
            anyhow::bail!(
                "api_key_env '{}' is not a valid env var name — must match [A-Z][A-Z0-9_]* (first char must be uppercase letter)",
                value
            );
        }
        for c in chars {
            if !(c.is_ascii_uppercase() || c.is_ascii_digit() || c == '_') {
                anyhow::bail!(
                    "api_key_env '{}' is not a valid env var name — must match [A-Z][A-Z0-9_]* (invalid char {:?})",
                    value, c
                );
            }
        }
        Ok(())
    }

    /// Phase 26 D-05 (PROV-06): validate that `role` is one of the five reserved
    /// auxiliary task names. Unknown role names are rejected at config load
    /// (Pitfall 3 — fail fast, not deep in the agent loop on first LLM call).
    pub const RESERVED_AUXILIARY_ROLES: &[&str] = &[
        "vision",
        "compression",
        "session_search",
        "skills_hub",
        "mcp_helper",
    ];

    pub fn validate_role_name(role: &str) -> Result<(), anyhow::Error> {
        if RESERVED_AUXILIARY_ROLES.contains(&role) {
            Ok(())
        } else {
            anyhow::bail!(
                "role '{}' is not one of the reserved auxiliary roles ({})",
                role,
                RESERVED_AUXILIARY_ROLES.join(", ")
            )
        }
    }
    ```

    Add unit tests covering all behaviors. Per CONTEXT D-04 the regex is verbatim `^[A-Z][A-Z0-9_]*$`; the hand-rolled loop above implements the same predicate. Use `anyhow::Error` for return type — `anyhow` is already a workspace dep used pervasively in this crate (verified in PATTERNS.md / Standard Stack).
