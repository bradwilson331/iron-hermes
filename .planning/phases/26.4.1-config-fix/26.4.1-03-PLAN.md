<objective>
CFG-03: Extend the `run_preflight` gate condition in `crates/ironhermes-cli/src/main.rs` so `hermes gateway` triggers the same first-run/fix-mode wizard auto-launch that `hermes chat` and bare `hermes` already trigger. Today the gate only fires for `Commands::Chat` and `None` (bare invocation); `Commands::Gateway` is excluded, so a missing or invalid config.yaml causes gateway to crash with a low-level error instead of running the wizard.

Purpose: Operators following the canonical onboarding (run `ironhermes gateway` to start the Telegram bot) currently get a hard failure when config.yaml is missing. This plan widens the gate to one additional variant — a single-line condition change — bringing gateway in line with chat/bare paths.

Output: One-line edit to the `run_preflight` gate in `main.rs` + a sibling gate that also tracks the `is_interactive_repl` and `is_chat_or_bare` log-filter conditions need to remain UNTOUCHED (gateway is not interactive REPL, must keep `info` logging) + one regression test in `crates/ironhermes-cli/tests/`.
</objective>
