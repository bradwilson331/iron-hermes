<objective>
CFG-02: Stop the wizard from saving an auxiliary provider name that `ProviderResolver::build()` will later reject. Add a known-provider allow-list check in BOTH `run_minimum_viable_flow` and `run_agent_section`. If the operator types a non-empty value not on the list, print a warning and skip saving (the auxiliary stage becomes a no-op for that run).

Purpose: Today, typing `local` at the `Auxiliary provider (cheaper helper-task model — Enter to skip)` prompt persists `auxiliary.provider: "local"` to config.yaml. On the very next launch, `ProviderResolver::build()` panics with `auxiliary.provider 'local' is not a known provider — define it in providers: first` (provider.rs:386), bricking the install. The fix is to validate the freeform input at write-time so the bad value never enters config.yaml.

Output: Modified `crates/ironhermes-cli/src/setup.rs` adding a `KNOWN_AUX_PROVIDERS` constant + `is_known_aux_provider` helper + warning-and-skip guard at both call sites + four unit tests.
</objective>

           "openai",

           "mistral",

       /// Phase 26.4.1 (CFG-02): case-insensitive membership check against
       /// KNOWN_AUX_PROVIDERS. Empty/whitespace input returns false (caller
       /// already handles "Enter to skip" before reaching this guard).
       fn is_known_aux_provider(name: &str) -> bool {
           let trimmed = name.trim();
           if trimmed.is_empty() {
               return false;

           let lower = trimmed.to_lowercase();
           KNOWN_AUX_PROVIDERS.iter().any(|p| *p == lower.as_str())
