# Phase 26.4.1 Plan 03: Gateway Preflight Gate (CFG-03) Summary

**One-liner:** Widened `run_preflight` gate in main.rs to include `Commands::Gateway` so `hermes gateway` auto-launches the first-run/fix-mode wizard instead of crashing with a low-level error when config.yaml is missing or invalid.

## What Was Built

**Task 1 — Gate widening (`main.rs`):**

The `run_preflight` gate condition was a single-line change from:
```rust
let run_preflight =
    matches!(cli.command, Some(Commands::Chat { .. }) | None) && cli.execute.is_none();
```
to:
```rust
let run_preflight = matches!(
    cli.command,
    Some(Commands::Chat { .. }) | Some(Commands::Gateway { .. }) | None
) && cli.execute.is_none();
```

A Phase 26.4.1 (CFG-03) amendment doc comment was appended to the existing Phase 23/24 comment block, explaining:
- Why gateway needs the wizard (operator onboarding canonical path)
- Why Phase 23's invariant is still satisfied (non-interactive subcommands excluded; `-e` suppresses gate)
- Why the two sibling gates (`is_interactive_repl`, `is_chat_or_bare`) are NOT widened (gateway is a daemon, not a REPL; must keep `ironhermes=info` log filter)

**Task 2 — Regression test (`invariants_26_4_1_cfg_03.rs`):**

Three source-text invariant tests using `include_str!` pattern:
- `gate_widened_to_gateway_for_run_preflight` (INV-01): exactly one occurrence of the widened pattern, bound to `run_preflight`
- `sibling_gates_not_widened_to_gateway` (INV-02): exactly two un-widened sibling gate occurrences (covering both `cli.command` and `&cli.command` forms)
- `phase_amendment_doc_comment_present` (INV-03): "Phase 26.4.1" or "CFG-03" marker within 1500 chars before the widened gate

All 3 tests pass. Full crate suite: 188 passed, 1 pre-existing failure (see below).
