# Phase 26.4.1 Plan 02: Auxiliary Provider Allow-List Guard (CFG-02) Summary

**One-liner:** Added `KNOWN_AUX_PROVIDERS` const + `is_known_aux_provider` case-insensitive guard at both wizard auxiliary call sites to prevent `ProviderResolver::build()` panic when user types `local` (or any unknown name).

## What Was Built

The wizard previously persisted any freeform text typed at the auxiliary provider prompt verbatim into `config.yaml`. On next launch, `ProviderResolver::build()` would panic with `"auxiliary.provider 'local' is not a known provider — define it in providers: first"` (provider.rs:386), bricking the install.

This plan adds an upstream allow-list filter at both call sites so the bad value never reaches disk.

### Changes

**`crates/ironhermes-cli/src/setup.rs`**

1. Added `KNOWN_AUX_PROVIDERS: &[&str]` const with 6 lowercase built-in provider names: `openrouter`, `anthropic`, `openai`, `google`, `mistral`, `groq`

2. Added `is_known_aux_provider(name: &str) -> bool` helper — case-insensitive, empty/whitespace returns false

3. Modified `run_minimum_viable_flow` auxiliary stage (was 3 lines, now 10 lines): wraps `apply_auxiliary_answer` in `if is_known_aux_provider(...)` guard; unknown providers print warning to stderr and skip save

4. Modified `run_agent_section` auxiliary stage (was 12 lines, now 19 lines): same guard pattern, preserves EOF-tolerant model prompt and the "Auxiliary routing set to X/Y." confirmation message in the success branch

5. Added 4 unit tests:
   - `cfg_02_is_known_aux_provider_accepts_built_ins` — all 6 providers + case variants
   - `cfg_02_is_known_aux_provider_rejects_unknown` — `local`, empty, whitespace, `foo`, `openai-proxy`, `my-llm`
   - `cfg_02_known_aux_providers_const_locked` — exact array equality assertion
   - `cfg_02_both_aux_call_sites_have_guard` — source-text ordering invariant via `include_str!`
