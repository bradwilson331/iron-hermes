**One-liner:** First-run wizard API key now persisted to `$HERMES_HOME/.env` as `{PROVIDER}_API_KEY=value` with `providers.{provider}.api_key_env` set in `config.yaml`, eliminating the deprecated `model.api_key` write and the gateway deprecation warning.

## What Was Built

**`provider_env_var_name(provider: &str) -> String`** ‚Äî private helper added above `prompt_required`. Derives canonical env-var name: `provider.to_uppercase().replace('-', "_") + "_API_KEY"`. Tested by Test 3.

**`run_minimum_viable_flow` API key step rewritten** ‚Äî replaced `apply_api_key_answer(config, &api_key)` with:
1. `let env_var_name = provider_env_var_name(&provider)` ‚Äî derive canonical name
2. `write_env_var_to_dotenv(hermes_home, &env_var_name, &api_key)?` ‚Äî atomic 0600-mode .env upsert
3. `config_setter::config_set(hermes_home, &format!("providers.{}.api_key_env", provider), &env_var_name)?` ‚Äî set providers map in config.yaml

**`run_model_section`** ‚Äî unchanged, still calls `apply_api_key_answer` (backward compat, D-01 deferred per CONTEXT).

**`apply_minimum_viable_answers` seam** ‚Äî unchanged, still calls `apply_api_key_answer` (non-interactive test seam, not production path).
