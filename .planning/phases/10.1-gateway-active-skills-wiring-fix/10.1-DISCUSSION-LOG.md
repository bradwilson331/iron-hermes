## Arc sharing strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Add set_active_skills() setter | Same pattern as set_memory_store() / set_hook_registry(). main.rs passes the Arc it created, handler stops creating its own. | ✓ |
| Pass through constructor | Add active_skills as a required param to GatewayRunner::new() and GatewayMessageHandler::new(). More explicit but breaks existing pattern. | |
| Extract from ToolRegistry | Add a method on ToolRegistry to retrieve the active_skills Arc baked into SkillsTool. Avoids passing separately but creates tight coupling. | |

**User's choice:** Add set_active_skills() setter (Recommended)
**Notes:** Consistent with existing wiring pattern for shared state.

| Option | Description | Selected |
|--------|-------------|----------|
| Store on GatewayRunner too | GatewayRunner gets active_skills field + set_active_skills(). In start(), passes Arc to handler. Mirrors memory_store/hook_registry flow. | ✓ |
| Pass directly to handler only | Skip GatewayRunner storage. Breaks encapsulation since handler creation is inside runner. | |

**User's choice:** Store on GatewayRunner too (Recommended)
**Notes:** Mirrors how all other shared state flows through the runner.
