### Human Verification Required

#### 1. Cookie Persistence End-to-End Test

**Test:** Launch IronHermes. Use the browser tools to navigate to a site that sets a login cookie (any authenticated site). Log in. Call `browser_close`. Wait a moment. Invoke any browser_* tool (e.g., `browser_navigate`) to trigger re-spawn. Navigate to the same site again.

**Expected:** The session cookie is present — the site shows the user as still logged in, without requiring re-authentication. The `$HERMES_HOME/browser-profile` directory should exist on disk and contain a chromium profile after the first spawn.

**Why human:** Requires a live chromium binary and network access. Unit tests cannot invoke `Browser::launch()` in isolation (chromiumoxide spawns a real process). This verifies the end-to-end persistence behavior that is the sole motivation for phase 26.3 — all automated checks confirm the wiring is correct, but only a real browser spawn can confirm chromium actually honors the `--user-data-dir` flag and preserves cookies across process restarts.
