<objective>
Phase 26.3 — Persistent chromiumoxide user_data_dir.

Add `user_data_dir: Option<String>` to `BrowserConfig` and wire it into both copies of `BrowserSession::spawn()` (ironhermes-tools and ironagent-tools-api), defaulting to `$HERMES_HOME/browser-profile` when unset. Extend `ensure_home_dirs()` to scaffold the directory at CLI startup.

Purpose: Without this fix, every browser spawn falls back to chromiumoxide's process-scoped temp dir, destroying cookies / localStorage / IndexedDB / login state on every `browser_close`. This blocks all real-world authenticated browser automation — the core use case for the browser toolset.

Output: Surgical 4-file change. Zero new dependencies. Five new tests (UDD-01..UDD-05) covering field defaults, YAML round-trip, backward compat, directory scaffolding, and compile-time wiring.
</objective>
