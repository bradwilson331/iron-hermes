# Phase 26.3 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| operator → config.yaml → BrowserConfig | Operator-controlled YAML; `user_data_dir` is operator-only (NOT LLM-controlled) | Filesystem path string |
| BrowserConfig → CdpBrowserConfig builder | In-process Rust call; PathBuf passed by reference, no serialization | `&PathBuf` (safe, no shell involvement) |
| chromiumoxide → chromium child process | chromiumoxide passes path as `--user-data-dir=` argv arg; chromium owns the directory | Argv (process-boundary safe) |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-26.3-path | Tampering | BrowserConfig.user_data_dir → CdpBrowserConfig::builder | mitigate | Operator-only config field (NOT LLM-callable). Path passed as `&PathBuf` to chromiumoxide builder — no shell expansion, no string interpolation, no command-line parsing. Same threat surface as pre-existing `chromium_path` field. | closed |
| T-26.3-singleton | Denial of Service | chromium SingletonLock (two ironhermes processes contending on same profile) | accept | See Accepted Risks Log — AR-26.3-01 | closed |
| T-26.3-stale-cookies | Information Disclosure | $HERMES_HOME/browser-profile cookies persisting across operator sessions | accept | See Accepted Risks Log — AR-26.3-02 | closed |

*Status: open · closed*

## Accepted Risks Log

| Risk ID | Threat Ref | Rationale | Accepted By | Date |
|---------|------------|-----------|-------------|------|
| AR-26.3-01 | T-26.3-singleton | Typical deployment is single-process (gateway OR CLI, not both simultaneously). If concurrent contention becomes a real problem, a future phase can add per-session profile suffixes. Out of scope for 26.3 — would require schema change + lifecycle wiring. Documented in RESEARCH.md Pitfall 1. | operator (Brad Wilson) | 2026-05-05 |
| AR-26.3-02 | T-26.3-stale-cookies | Cookie persistence is the INTENDED behavior — Phase 26.3 exists to make cookies persist across `browser_close` calls. Operator is the only principal with read access to $HERMES_HOME (per Phase 24 profile isolation). Operators wanting ephemeral browsing can set `browser.user_data_dir: "/tmp/ephemeral-profile"` explicitly in config.yaml. | operator (Brad Wilson) | 2026-05-05 |

*Accepted risks do not resurface in future audit runs.*
