## Shared Patterns

### Optional String field on a serde config struct
**Source:** `crates/ironhermes-core/src/config.rs` — `chromium_path: Option<String>` in `BrowserConfig` (line 541) and `ProviderConfig.api_key_env: Option<String>` (line 163).
**Apply to:** `BrowserConfig.user_data_dir` field addition.

Key rules observed in both analogs:
- The struct itself carries `#[serde(default)]` — no per-field attribute needed for `Option<T>` (serde deserializes missing fields as `None` automatically when the struct has `#[serde(default)]`)
- The `Default` impl sets `Option` fields to `None` explicitly (not via `..Default::default()` shorthand)
- Doc comment references the phase decision number and semantics

### Resolving an optional config field to a computed path at call time
**Source:** `crates/ironhermes-tools/src/browser_session.rs` — `find_chromium_binary(config.chromium_path.as_deref())` (line 49) and its implementation at lines 154–242.
**Apply to:** `user_data_dir` resolution inside `spawn()`.

The `user_data_dir` resolution is simpler than `chromium_path` (two-level: config or computed default, no env-var tier). Use `.as_deref()` on the `Option<String>` to get `Option<&str>` before matching — this is the consistent project idiom for `Option<String>` config fields.

### Directory scaffolding with idempotent `create_dir_all`
**Source:** `crates/ironhermes-cli/src/main.rs` lines 521–537.
**Apply to:** `ensure_home_dirs()` extension.

`std::fs::create_dir_all` is idempotent — safe to call on existing directories. The `with_context` closure uses `home.display()` + the sub-path string for human-readable error messages. No existence pre-check is needed.

### Test isolation with `IRONHERMES_HOME` override + `tempfile::tempdir()`
**Source:** `crates/ironhermes-cli/src/main.rs` lines 2688–2741 (`ensure_home_dirs_tests` module).
**Apply to:** New `browser-profile` test in the same module.

Pattern: acquire `ENV_LOCK`, create temp dir, set `IRONHERMES_HOME`, call function, assert, remove env var. The `unsafe` block is required for `std::env::set_var` / `remove_var` (Rust edition policy). The existing `static ENV_LOCK: Mutex<()>` serializes env-mutating tests.
