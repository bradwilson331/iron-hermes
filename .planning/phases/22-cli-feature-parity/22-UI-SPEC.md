## Spacing Scale

Terminal "spacing" is expressed in character counts and newlines, not pixels. The established convention in `main.rs` follows this pattern:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1 space char | Inline label/value separator (e.g. `"Tool: " + name`) |
| sm | 2 space chars | Indented list items (`"  [icon] name"`) |
| md | 4 space chars | Double-indented key-value pairs (`"  Key:    value"`) |
| lg | 1 blank line (`println!()`) | Section separator within a command output block |
| xl | 40-char `─` rule | Major section heading separator |

Exceptions: none for Phase 22. The new output surfaces (guardrail error, JSONL log entry, hook lifecycle messages) follow the same `sm` indent convention as existing tool progress output.

## Typography

Terminal typography is weight (bold/normal) and color applied via the `colored` crate. There is no font size variation in a terminal.

| Role | Colored Modifier | Line Wrapping | Usage |
|------|-----------------|---------------|-------|
| Body / value text | none (plain) | natural terminal wrap | Tool args, event data, log payloads |
| Label / key | `.dimmed()` | none — single token | Prefix labels like `"Tool:"`, `"Event:"`, `"Blocked:"` |
| Heading | `.bold().cyan()` | none — single line | Section headers: `"IronHermes Status"`, `"IronHermes Doctor"` |
| Emphasis / name | `.yellow()` | none — single token | Tool name in progress output, active skill name |
| Success / actor | `.bold().green()` | none — single token | `"You:"` prompt prefix, success confirmations |
| Status pill (TUI) | cyan · magenta · green · yellow · dimmed | none — pill bar | Phase 21 TUI status line — unchanged in Phase 22 |

Source: established patterns in `crates/ironhermes-cli/src/main.rs` lines 166, 213, 327, 505.

## Color

Terminal colors via `colored` crate. No hex values — these are ANSI terminal color names.

| Role | Colored Value | Usage |
|------|--------------|-------|
| Dominant (60%) | plain (terminal default) | All body text, tool args, event payloads, JSONL content |
| Secondary (30%) | `.dimmed()` | Label prefixes, metadata, secondary context |
| Accent — names (10%) | `.yellow()` | Tool name in `"Tool: {name} {args}"` line, skill name on activation |
| Accent — actor | `.bold().green()` | `"You:"` prompt prefix only |
| Accent — section | `.bold().cyan()` | Section headings (`"IronHermes Status"`, `"IronHermes Doctor"`) |
| Destructive / error | `.bold().red()` (reserved) | Guardrail-blocked tool message, fatal errors — see Copywriting Contract |

Accent reserved for:
1. Tool name tokens in progress lines (`.yellow()`)
2. User prompt actor prefix `"You:"` (`.bold().green()`)
3. Section-level command output headings (`.bold().cyan()`)
4. Guardrail block notification prefix — `.bold().red()` (new in Phase 22, see below)
5. TUI pill bar color rotation — unchanged from Phase 21 D-04

Phase 22 introduces `.bold().red()` for the guardrail blocked message prefix. This is the only new color usage in this phase. All other color assignments are pre-existing and unchanged.

## Copywriting Contract

### Existing output patterns (unchanged in Phase 22)

| Element | Pattern | Example |
|---------|---------|---------|
| Tool progress | `"Tool:" label + name.yellow() + args.dimmed()` | `Tool: write_file {"path":"foo.rs",...}` |
| Fatal error | `"error: {e}"` → `eprintln`, then `std::process::exit(1)` | `error: config.yaml not found` |
| Session interrupt | `"\nInterrupted."` | (Phase 21 established) |

### New output surfaces introduced in Phase 22

| Element | Copy | Format |
|---------|------|--------|
| Guardrail blocked — stderr prefix | `"Blocked:"` | `.bold().red()` label, space, then tool name in `.yellow()` |
| Guardrail blocked — stderr body | `"Tool '{name}' is blocked by guardrail configuration."` | plain body text on next token |
| Guardrail blocked — with error_detail=full | Append: `" ({reason})"` where reason comes from `format_guardrail_error` | plain, parenthesized |
| Guardrail blocked — with error_detail=minimal | No reason appended — blocked message only | matches `ErrorDetailLevel::Minimal` semantics |
| JSONL event log entry | Machine-readable — no human copy contract. One JSON object per line. | Written by `create_jsonl_listener` — no new human-readable text |
| Hook lifecycle — session:start | No user-visible output. Fires internally, writes to JSONL only. | Silent to terminal |
| Hook lifecycle — agent:start/step/end | No user-visible output. Fires internally, writes to JSONL only. | Silent to terminal |
| Hook lifecycle — tool:called/completed | No user-visible output beyond existing Tool: progress line. | Silent to terminal |
| Webhook delivery failure (retry queued) | No user-visible output in Phase 22. Failures go to retry queue. | Silent to terminal |

### Empty state

Phase 22 has no empty state to display. The tools being wired (execute_code, skills_tool, cron_tool) emit their own messages when invoked with no data. The hook registry with no listeners is silent. No new empty state copy is needed.

### Destructive actions

Phase 22 introduces no destructive actions visible to the user. `BlocklistGuardrail` prevents tool dispatch but does not delete data. The blocked message (above) is the only new user-facing output. No confirmation dialogs or prompts are added.

## Terminal Interaction Contract

### New interactions introduced in Phase 22

| Interaction | Behavior |
|-------------|----------|
| Blocked tool attempt | Agent requests a blocked tool → guardrail intercepts → stderr line: `Blocked: {name}` + reason per error_detail level → agent receives guardrail error envelope → agent may retry or inform user |
| execute_code invocation | Agent calls execute_code → existing Tool: progress line fires → sandbox executes → result returned → ToolCompleted fires to JSONL. No new terminal output. |
| Skills tool activation | Agent calls skills_tool → existing Tool: progress line fires → active_skills Arc updated → SkillActivated hook fires to JSONL. No new terminal output. |
| Cron job registration | Agent calls cron_tool → existing Tool: progress line fires → job written to JobStore. No new terminal output. |
| JSONL event log | Written to configured path (default: `~/.ironhermes/events.jsonl` or as configured). Zero terminal output — file-only. |

### Unchanged interactions (Phase 21 — do not regress)

| Interaction | Phase 21 contract |
|-------------|-------------------|
| Knight Rider scanner | Animates during in-flight turns/tool calls on stderr |
| Pill status line | Persistent: mode · model · provider · tokens/limit · hint |
| Double ctrl-c | First press cancels turn, second press within 1.5s exits |
| Non-TTY no-op | TUI disabled silently when stderr is not a TTY |
