//! Phase 22 — static-grep regression tests for CLI tool + hook parity.
//!
//! These tests read main.rs source text and assert that both `run_chat`
//! and `run_single` contain the required wiring calls. They guard against
//! future refactors that might accidentally remove a tool registration or
//! hook wiring call.

/// Read main.rs and extract function bodies by finding `fn {name}` and
/// collecting until the next top-level `fn ` or end-of-file.
fn extract_function_body(source: &str, fn_name: &str) -> String {
    let marker = format!("fn {}(", fn_name);
    let start = source.find(&marker).unwrap_or_else(|| {
        panic!("Could not find function '{}' in main.rs", fn_name)
    });
    // Find the next top-level fn after the start (skip the current one)
    let rest = &source[start + marker.len()..];
    let end = rest.find("\nfn ").or_else(|| rest.find("\nasync fn "))
        .map(|pos| start + marker.len() + pos)
        .unwrap_or(source.len());
    source[start..end].to_string()
}

fn read_main_rs() -> String {
    std::fs::read_to_string(
        std::path::Path::new(env!("CARGO_MANIFEST_DIR")).join("src/main.rs")
    ).expect("Failed to read main.rs")
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HookRegistry -> JSONL file | Event data written to local filesystem |
| HookRegistry -> webhook endpoint | Event data sent to external HTTP endpoints with HMAC signing |
| RetryQueue -> disk | Failed webhook events persisted for retry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-05 | Information Disclosure | JSONL event log | accept | Event log written to user-controlled path under HERMES_HOME. Single-operator deployment. Log contains tool names and args — acceptable for audit trail. |
| T-22-06 | Tampering | Webhook HMAC signing | mitigate | Use existing `create_webhook_listener` which applies HMAC-SHA256 signing per webhook config. Do not hand-roll webhook delivery. |
| T-22-07 | Denial of Service | RetryQueue unbounded growth | mitigate | `drain_retry_queue` called on CLI startup with TTL expiry (default 24h). Stale events purged. Same mitigation as gateway. |
| T-22-08 | Information Disclosure | Hook events leaking sensitive tool args | accept | Tool args already logged in existing Tool: progress line on stderr. JSONL adds persistence but no new information exposure. Single-operator deployment. |
</threat_model>
