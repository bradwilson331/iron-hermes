<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent -> execute_code sandbox | Agent-requested code execution crosses into sandboxed subprocess |
| Agent -> tool dispatch | Agent tool calls pass through guardrail check before dispatch |
| Skill env vars -> sandbox | Skill-declared env vars bypass secret-strip via active_skills whitelist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Elevation of Privilege | execute_code RPC registry | mitigate | RPC registry excludes execute_code and terminal tools per D-04 — only file tools, web tools, memory tool registered. Prevents recursive sandbox execution. |
| T-22-02 | Information Disclosure | active_skills env var bypass | mitigate | Share single `active_skills` Arc between `register_skills_tool` and `register_execute_code_tool_with_active_skills` so only agent-activated skill env vars pass through. No extra env vars leak. |
| T-22-03 | Elevation of Privilege | BlocklistGuardrail bypass | mitigate | `add_guardrail` called on mutable registry BEFORE Arc wrap. Same guardrails apply to CLI and gateway. Subagent receives same registry with same guardrails. |
| T-22-04 | Tampering | HooksConfig loaded from disk | accept | Config file is user-controlled (single-operator deployment). No external attacker can modify it without filesystem access already granted. |
</threat_model>
