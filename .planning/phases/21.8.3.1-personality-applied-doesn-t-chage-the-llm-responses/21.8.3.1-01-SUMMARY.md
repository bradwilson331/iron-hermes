## Decisions Made

- D-02 rename: `next_turn_personality_overlay` → `active_personality_overlay` — the old name implied single-turn consumption; the bug was that it was only ever written, never read. The new name clarifies session-persistent intent.
- D-05 "clear" pre-check goes in `handle_subsystem_mutator` BEFORE the `match core_result` block, not in the core handler — core `cmd_personality` has no "clear" case and returns `Error("Unknown personality: clear")` when dispatched with that arg.
- D-06 raw overlay text: no "Active personality:" header prefix — raw preset text appended with `"\n\n"` separator, matching `PromptBuilder::set_overlay()` slot 8 behavior.
