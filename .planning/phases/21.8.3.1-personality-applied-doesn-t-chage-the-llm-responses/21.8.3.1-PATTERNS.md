## Shared Patterns

### Session-Persistent State (Option<String> field)

**Source:** `crates/ironhermes-gateway/src/handler.rs` — `skill_overlays`, `skills_config`, `gateway_engine` fields
**Also:** `crates/ironhermes-cli/src/tui_rata/app.rs` — `fast_enabled`, `verbose_enabled`

**Apply to:** `App.active_personality_overlay` (rename from `next_turn_personality_overlay`) and `GatewayMessageHandler.active_personality_overlay` (new field).

Pattern: field initialized to `None` in constructor; mutated by command dispatch; read each turn; never auto-cleared by the turn loop.

### Per-Turn Local Clone Mutation

**Source:** `crates/ironhermes-gateway/src/handler.rs:782-788` — skill overlay injection into PromptBuilder
**Apply to:** `event_loop.rs` overlay injection into `messages_snapshot`.

Both patterns: read session-persistent state, apply to local per-turn object (message clone or PromptBuilder), never mutate canonical history.

### Slash Command Pre-Check (surface intercepts before core dispatch)

**Source:** `crates/ironhermes-cli/src/tui_rata/commands.rs:1024-1043` — personality arm with Output heuristic filter
**Apply to:** D-05 "clear" pre-check in `commands.rs` and D-08 gateway pre-dispatch "clear" check.

Pattern: match the command name, check args before calling core dispatcher, return early with `SlashOutcome::Handled` or send Telegram message and `return Ok(())`.
