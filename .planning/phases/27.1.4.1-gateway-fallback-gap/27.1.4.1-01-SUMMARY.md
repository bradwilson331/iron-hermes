Shared `wire_fallback_if_configured` helper with dual `tracing::warn!` branches, wired at all 5 AgentLoop sites (cron runner, subagent runner, CLI batch runner, gateway handler, UI state), with 3 static invariant tests confirming coverage.

## What Was Built

### Task 1: Static invariant test files (RED gate)

Three `include_str!`-based regression tests created — one per crate — following the `invariants_22_4.rs` pattern. Each asserts `.with_fallback(` appears in the target source file.

- `crates/ironhermes-gateway/tests/invariants_27_1_4_1.rs` — asserts `runner.rs` contains `.with_fallback(`
- `crates/ironhermes-agent/tests/invariants_27_1_4_1.rs` — asserts `subagent_runner.rs` contains `.with_fallback(`
- `crates/ironhermes-cli/tests/invariants_27_1_4_1.rs` — asserts `batch/runner.rs` contains `.with_fallback(`

These tests were initially RED (the sources lacked `.with_fallback(`), turned GREEN after Task 3.

### Task 2: `wire_fallback_if_configured` helper

Added `pub fn wire_fallback_if_configured(mut agent: AgentLoop, resolver: &ProviderResolver) -> AgentLoop` to `crates/ironhermes-agent/src/any_client.rs` after `build_role_client`. Key behaviors:

- Empty `fallback_providers`: returns agent unchanged, no warning.
- `fallback_providers.first()` resolves to a configured provider: builds client with `fb_endpoint.default_model` (not the primary's model), chains `agent.with_fallback(fb_client)`.
- Provider name not found in resolver: emits `tracing::warn!("fallback provider '{}' not found in resolver — running without fallback", fb_name)` and returns agent unchanged.
- Client build fails: emits `tracing::warn!(error = ?e, "fallback provider '{}' client build failed — running without fallback", fb_name)` and returns agent unchanged.
- No panic, no credential leak (only `fb_name` is interpolated).

Re-exported from `crates/ironhermes-agent/src/lib.rs` as `pub use any_client::{..., wire_fallback_if_configured}`.

### Task 3: Wire at all 5 AgentLoop construction sites

**3 new sites (closing PROV-07 gap):**

1. `crates/ironhermes-gateway/src/runner.rs` (cron path): removed `D-12: cron uses fixed provider, NO fallback` comment, added helper call after `AgentLoop::new`. Added `wire_fallback_if_configured` to `use ironhermes_agent::{...}` import.

2. `crates/ironhermes-agent/src/subagent_runner.rs`: added helper call after `AgentLoop::new`, before cancel token forwarding. Extended `use crate::any_client::{AnyClient, wire_fallback_if_configured}`.
