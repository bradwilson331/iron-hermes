<objective>
Close the PROV-07 fallback coverage gap. Three production `AgentLoop` construction sites (gateway cron runner, agent subagent runner, CLI batch runner) build `AgentLoop` without `.with_fallback()` — when the primary provider is unavailable they fail hard with no retry. Two other sites (handler.rs main conversation, iron_hermes_ui/server/state.rs) have fallback wiring but a triple-nested `if let` that fails silently on misconfiguration.

This plan: (1) creates the 3 Wave-0 static invariant test files (initially RED), (2) extracts a shared `wire_fallback_if_configured` helper in `ironhermes-agent` with `tracing::warn!` on both misconfiguration branches and re-exports it, (3) wires the helper at all 5 sites — turning the invariant tests GREEN.

Purpose: provider fallback resilience parity across every agent execution path; eliminate silent fallback-misconfig failures.
Output: shared helper + 5 wired sites + 3 invariant test files, full workspace builds clean, all invariant tests pass.

Implements: D-01, D-02, D-03, D-04, D-05, D-06, D-07. (D-08..D-11 config/docs consistency is Plan 02.)
</objective>
