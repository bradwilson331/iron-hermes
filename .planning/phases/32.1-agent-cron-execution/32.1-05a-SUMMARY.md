### Task 2: script_runner.rs (TDD)

Implemented `crates/ironhermes-cron-runner/src/script_runner.rs` via TDD:

**RED commit:** 15 failing tests covering all specified behavior.

**GREEN commit:** Full implementation:

- `ScriptOutcome` struct (`ok`, `stdout`, `stderr`, `wake_agent`)
- `run_job_script(script_name: &str) -> Result<ScriptOutcome>`:
  - Two-phase sandbox: reject `..` path components (pre-canonicalize) + `canonicalize() + starts_with(scripts_dir)` (post-canonicalize, catches symlinks)
  - Absolute paths rejected immediately with "escapes scripts_dir" error
  - `scripts/` directory auto-created with `chmod 0700` on Unix
  - Interpreter selection: `.sh`/`.bash` → `$BASH_PATH` or `/bin/bash`; else → `$PYTHON_PATH` or `python3`
  - `IRONHERMES_CRON_SCRIPT_TIMEOUT` env var (default 120s) via `tokio::time::timeout`
  - Timeout returns `ScriptOutcome { ok: false, stderr: "script timed out after Xs" }`, not `Err`
  - `kill_on_drop(true)` ensures subprocess cleanup on timeout
  - Calls `parse_wake_gate` on stdout to populate `outcome.wake_agent`
- `parse_wake_gate(stdout: &str) -> bool`: finds last non-empty line, parses JSON, returns `false` only for `{"wakeAgent": false}`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Rust edition 2024: `std::env::set_var`/`remove_var` are unsafe**
- **Found during:** Task 2 GREEN phase (compilation failure)
- **Issue:** Workspace uses `edition = "2024"` where env mutation functions are marked `unsafe`. All test `set_var`/`remove_var` calls failed to compile.
- **Fix:** Wrapped all env mutation calls in `unsafe {}` blocks in the test module.
- **Files modified:** `crates/ironhermes-cron-runner/src/script_runner.rs`
- **Commit:** 2a89d1d3

**2. [Rule 1 - Bug] Path traversal test failed for non-existent target**
- **Found during:** Task 2 GREEN phase (test_path_traversal_rejected failed)
- **Issue:** `../etc/passwd` joined to tempdir doesn't exist on disk, so `canonicalize()` fails with "not found" before reaching the `starts_with()` check. The error message was "script not found or unreadable" instead of "escapes scripts_dir".
- **Fix:** Added pre-canonicalize `..` component scan using `Path::components()`. Any `ParentDir` component immediately returns `bail!("script path escapes scripts_dir: ...")`. The post-canonicalize `starts_with()` guard remains as a second layer for symlink escapes.
- **Files modified:** `crates/ironhermes-cron-runner/src/script_runner.rs`
- **Commit:** 2a89d1d3

**3. [Rule 1 - Bug] Poisoned mutex caused cascade test failures**
- **Found during:** Task 2 GREEN phase (tests 7 and 13 failing with PoisonError)
- **Issue:** `env_lock().lock().unwrap()` panics when a prior test panicked while holding the lock, poisoning it. Tests 7 and 13 ran after test 3 initially panicked.
- **Fix:** Changed `env_lock()` to return `MutexGuard` directly using `unwrap_or_else(|e| e.into_inner())` to recover poisoned locks. Updated all call sites from `env_lock().lock().unwrap()` to `env_lock()`.
- **Files modified:** `crates/ironhermes-cron-runner/src/script_runner.rs`
- **Commit:** 2a89d1d3
