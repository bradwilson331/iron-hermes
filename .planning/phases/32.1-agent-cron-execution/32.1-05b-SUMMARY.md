## What Was Built

### Task 1: `prompt_builder.rs`

Implements `build_job_prompt(job, script_output, skill_registry) -> Result<AssembledPrompt>` with the locked five-step assembly order from CONTEXT.md:

1. Cron-hint banner (`[IMPORTANT: You are running as a scheduled cron job...]`)
2. Skill content ā€” each found skill wrapped with invocation prefix; missing skills collected and prepended as skip-missing warning
3. `## Script Output` block (only when `script_output` is `Some` and non-empty after trim)
4. `## Output from job 'X'` blocks for each `context_from` entry ā€” UUID-guarded, truncated at 8000 bytes with `[... output truncated ...]` suffix
5. User-supplied `job.prompt`

After assembly, `ironhermes_cron::scan_cron_prompt` is called on the full assembled view. Non-`None` `blocked_reason` in `AssembledPrompt` signals callers (Plan 06) to emit a BLOCKED delivery doc.

A convenience `scan_assembled(assembled: &str) -> Option<String>` wrapper is also exported.

### Task 2: `timeout.rs`

Implements two primitives:

- `run_with_wall_clock(fut, wall_secs)`: wraps future in `tokio::time::timeout`; returns `Err("wall-clock timeout after Xs")` on deadline; `wall_secs == 0` disables.
- `run_with_inactivity_timeout(fut, activity_summary_fn, inactivity_secs)`: polls `activity_summary_fn()` every 5 seconds via `tokio::select!`; fires when `idle_secs > inactivity_secs`; returns `Err("inactivity timeout after Xs of no activity")`; `inactivity_secs == 0` disables.

Both implement the `0 => unlimited` convention from CONTEXT.md Ā§Timeout & resilience.
