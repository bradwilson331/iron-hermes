<objective>
Amend `crates/ironhermes-cron/PARITY.md` so §10 and §11 reflect that Phase 32.1
pulls the previously-out-of-scope agent-execution items in, then extend the
`CronJob` struct in `crates/ironhermes-cron/src/job.rs` with the nine per-job
runtime override fields the rest of the phase needs. The new fields must use
`#[serde(default)]` so existing `jobs.json` records on disk continue to load
unmodified. Harden `origin` deserialization so a malformed (non-object) origin
on disk produces `origin = None` instead of failing the entire job's serde.

Purpose: every downstream plan in this phase (runner crate, storage parity,
delivery routing, CLI commands, gateway migration) reads or writes these
fields. Without them, none of the runtime-override behavior locked in
CONTEXT.md §Per-job runtime overrides is reachable.

Output: a documentation update plus the extended `CronJob` model with
backward-compatible serde defaults and a defensive origin deserializer.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `jobs.json` on disk → in-process `CronJob` | An operator (or attacker with FS write) can craft arbitrary JSON; the new defensive deserialization is the input-validation gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32.1.01-01 | Tampering | `CronJob` deserialization | mitigate | `#[serde(default)]` on every new field + lenient `origin` deserializer prevent a single malformed record from disabling the whole job set. Affected job is loaded with `origin = None` and a `tracing::warn!` is emitted so operators see the coercion. |
| T-32.1.01-02 | Information Disclosure | `tracing::warn!` on bad origin | accept | Origin payload may contain a chat_id; logging it under `warn!` is acceptable since the same chat_id is already in `jobs.json` on disk. Future tightening can move to `debug!` if needed. |
| T-32.1.01-SC | Tampering | npm/pip/cargo installs | mitigate | This plan introduces zero new crates; no slopcheck required. Verified by `grep -nE "^[a-z0-9_-]+ *= *\"" crates/ironhermes-cron/Cargo.toml` showing no new dependency lines after Plan 01 lands. |
