## What Was Built

### Task 1: delivery.rs + TgSendApi relocation

`delivery.rs` implements the multi-target dispatch loop:

- `wrap_for_delivery(job, config, body)` ŌĆö applies wrap_response header/footer from config
- `extract_media_paths(body) -> (String, Vec<PathBuf>)` ŌĆö strips `MEDIA:` lines, returns cleaned text + paths
- `dispatch_all_targets(targets, output, job, config, tg_client)` ŌĆö iterates all `DeliveryTarget`s, routes to `tg_client.send_message` or stdout/file, collects per-target errors, returns `Vec<String>` of error messages

`TgSendApi` trait relocated from `ironhermes-gateway::telegram` to `ironhermes-cron::adapter`. Gateway re-exports it via `pub use ironhermes_cron::TgSendApi`. This breaks a dependency cycle that would occur when Plan 07 makes gateway depend on cron-runner.

11 tests: FakeTg mock implementation, media extraction, wrap_response formatting, error accumulation, stdout target.

### Task 2: runner.rs ŌĆö CronRunnerContext + run_cron_job

`CronRunnerContext` is the Arc-shared dependency bundle:

```rust
pub struct CronRunnerContext {
    pub job_store: Arc<std::sync::Mutex<JobStore>>,
    pub skill_registry: Option<Arc<SkillRegistry>>,
    pub tool_registry: Arc<tokio::sync::RwLock<ToolRegistry>>,
    pub memory_manager: Option<Arc<TokioMutex<MemoryManager>>>,
    pub hook_registry: Option<Arc<HookRegistry>>,
    pub config: Config,
    pub mcp_manager: Option<Arc<McpManager>>,
    pub tg_client: Option<Arc<dyn TgSendApi>>,
}
```

`run_cron_job` orchestration flow:
1. Wake-gate: run script if configured; skip if returns `wake_agent: false`
2. No-agent path: runs script directly, skips AgentLoop entirely
3. Toolset scoping: `tool_registry.scope_to(enabled_toolsets)` for per-job filtering
4. Prompt build via `build_prompt`
5. AgentLoop with per-job model/provider/base_url overrides
6. Dual timeout: `run_with_wall_clock` wraps `run_with_inactivity_timeout` wraps `agent.run`
7. Empty final_response ŌåÆ soft failure (last_status=error, no panic)
8. `complete_and_dispatch`: persist via `complete_job_run`, deliver via `dispatch_all_targets`
9. Hook events: `JobStarted` + `ResponseSent` per execution

`WorkdirGuard` RAII guard sets/restores `TERMINAL_CWD` for workdir-scoped jobs.

`with_first_target_locals` scopes `CRON_AUTO_DELIVER_PLATFORM`, `CRON_AUTO_DELIVER_CHAT_ID`, `CRON_AUTO_DELIVER_THREAD_ID` task-locals around the agent run.

22 tests covering all paths.

**Deviation (ToolRegistry):** `scope_to()` required cloning tools from the registry into a new `Self`. The `Tool` trait had no `clone_box`. Instead of modifying all ~13 `Tool` implementations, changed internal `HashMap` storage from `Box<dyn Tool>` to `Arc<dyn Tool>`. Public API (`register`, `register_dynamic`) still accepts `Box<dyn Tool>` ŌĆö converted via `Arc::from(tool)` on insert. This is a non-breaking change and `Arc<dyn Tool>` is cloneable.

### Task 3: tick_loop.rs ŌĆö 60s tick scaffold

`run_tick_loop(ctx, cancel)`:
- 60s `tokio::time::interval` with `MissedTickBehavior::Skip`
- First tick fires immediately at t=0 ŌĆö skipped as boot guard (gateway parity)
- Calls `run_one_tick` on each subsequent tick

`run_one_tick`:
1. Calls `prepare_mcp_for_tick` (non-fatal)
2. `run_tick_check` to collect due jobs (errors logged, loop continues)
3. Partitions `due_jobs` by `workdir.is_some()` into workdir-serial / parallel
4. Workdir jobs run sequentially (TERMINAL_CWD is process-global)
5. Parallel jobs via `JoinSet`, bounded by `IRONHERMES_CRON_MAX_PARALLEL` semaphore if set

`prepare_mcp_for_tick(_mcp: &McpManager)` ŌĆö public no-op stub. Body emits `tracing::debug!` message noting MCP discovery primitive is not yet implemented. Future phase replaces body.

5 tests: cancel exits cleanly, boot guard, failure isolation, env usize, workdir partition.
