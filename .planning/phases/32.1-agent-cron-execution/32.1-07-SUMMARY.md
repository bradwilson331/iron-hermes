## What Was Built

### Task 1: CLI Subcommands

- `hermes cron trigger <id-or-name>` — sets `next_run_at = Utc::now()` via `JobStore::trigger_job`; resolves by id or name; prints `Triggered job <id>`
- `hermes cron tick` — acquires tick lock, scans due jobs, calls `ironhermes_cron_runner::run_cron_job` for each, exits with summary
- `hermes cron daemon` — constructs `CronRunnerContext`, spawns ctrl-c watcher, calls `ironhermes_cron_runner::run_tick_loop` until cancelled
- `build_cron_runner_ctx` helper shared by tick and daemon; `tg_client: None` (CLI path is always standalone)
- 6 tests in `tests_phase_32_1` module (trigger by id, by name, nonexistent, tick exits, daemon cancel, ctx construction)

### Task 2: Gateway Migration

- Deleted `pub(crate) async fn execute_cron_job` and `pub async fn dispatch_delivery` from gateway/src/runner.rs
- Gateway cron tick task now calls `ironhermes_cron_runner::run_tick_loop(cron_ctx, tick_cancel).await` after constructing `CronRunnerContext` from existing gateway Arcs
- `fast_forward_backlog` preserved (gateway-specific boot guard, runs before entering tick loop)
- `dispatch_delivery` re-export removed from gateway lib.rs
- Added `ironhermes-cron-runner` dep to gateway Cargo.toml
- Static regression test `execute_cron_job_no_longer_exists_in_gateway` guards against re-introduction
- PROV-07 invariant test (invariants_27_1_4_1.rs) updated to check cron-runner's runner.rs (where `wire_fallback_if_configured` now lives)

### Task 3: MEDIA Tag Routing

- Added `send_voice`, `send_image_file`, `send_video`, `send_document` to `TgSendApi` trait in `ironhermes-cron/src/adapter.rs`
- Implemented all four on `TelegramAdapter` via `send_file_multipart` helper (reads file bytes, posts multipart form to Telegram Bot API)
- `MediaKind` enum + `classify_media_path` with closed extension allowlist:
  - Image: png, jpg, jpeg, gif, webp
  - Video: mp4, mov, webm, mkv
  - Voice: ogg, opus, mp3, wav, m4a
  - Document: everything else (including no extension)
- `route_media_payload` dispatches each extracted path after text body is sent
- Plan 06 deferred `warn!()` block replaced with `route_media_payload` call in Ok arm
- 14 MEDIA routing behavior tests (testm1-testm14) in delivery.rs
- All FakeTg/FakeTgClient implementations across crates updated with the four new methods
