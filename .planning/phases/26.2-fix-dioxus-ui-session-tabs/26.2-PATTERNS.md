### Signal Borrow Safety (applies to ALL closures in `warp_hermes.rs`)
**Source:** `warp_hermes.rs` patterns throughout (e.g., `on_rerun` lines 472-503, `submit` lines 506-590)
**Rule:** Never hold a `Signal::read()` or `Signal::write()` guard across a scope boundary or `.await`. Extract values into owned locals first.

```rust
// CORRECT — extract before the await / multi-signal mutation:
let sid = {
    let ts = tabs.read();
    ts.get(idx).map(|t| t.session_id.clone())
};  // read guard dropped here
active_tab.set(idx);
session_id.set(sid.unwrap_or_default());

// WRONG — would panic at runtime (RefCell semantics):
// active_tab.set(idx);
// session_id.set(tabs.read().get(idx).unwrap().session_id.clone());
//   ↑ read() borrow may overlap with prior write guard in same turn
```
