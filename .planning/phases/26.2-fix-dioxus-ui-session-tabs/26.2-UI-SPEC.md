| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon gaps, `wh-tab-dot` margins, close-button margin-left |
| sm | 8px (`--space-2`) | Tab gap, inline element padding, traffic-light gap |
| md | 16px (`--space-3`) | Default panel padding, stream scroll padding |

- Title bar height: 32px (non-4-multiple, inherited from `warp-ih.css` `.wh-titlebar` — do not change, Warp shell contract)

- Tab-dot indicator: 6px × 6px circle (fixed, Warp shell contract)
- Close button `×`: `margin-left: 4px; font-size: 11px` (already in `title_bar.rs` — preserve as-is)
- Disabled-tab container: no padding change; apply `pointer-events: none; opacity: 0.5` on `.wh-tabs` wrapper only when `scanner_active` is true (CONTEXT.md D-02 / §Specific Ideas)

## Interaction Contract

This section is specific to Phase 26.2 (interactive tab strip). Consumed by executor as the behavioral truth for RSX event wiring.

### Tab Click (on_tab_click: EventHandler<usize>)

| Condition | Behavior |
|-----------|----------|
| `scanner_active` is `false` | Set `active_tab` to clicked index; write `session_id` from `tabs[idx].session_id`; call `blocks.set(Vec::new())`; call `messages.write().clear()` |
| `scanner_active` is `true` | No-op. Tab click is swallowed. Visual: `.wh-tabs` container has `pointer-events: none; opacity: 0.5` |

RSX: `div { onclick: move |_| on_tab_click.call(i), ... }`

### Close Button (on_tab_close: EventHandler<usize>)

| Condition | Behavior |
|-----------|----------|
| Not the last tab | Remove `tabs[i]`; if `i == active_tab`, switch to `i.saturating_sub(1)` (or `min(i, tabs.len()-1)` after removal); if `i < active_tab`, decrement `active_tab` by 1 to stay on same session |
| Last remaining tab | Remove tab; auto-create new session (same as on_tab_new flow); ensures tabs vec is never empty |

RSX: `button { onclick: move |evt| { evt.stop_propagation(); on_tab_close.call(i); }, ... }`

**Event isolation:** `evt.stop_propagation()` on close button prevents parent `div`'s `on_tab_click` from firing. Do not rely on coordinate checks — use `stop_propagation()` exclusively (RESEARCH.md Pattern 1, verified `evt.stop_propagation()` in Dioxus 0.7).

### New Tab Button (on_tab_new: EventHandler<()>)

| Step | Action |
|------|--------|
| 1 | `spawn(async move { ... })` |
| 2 | Await `create_session()` server function |
| 3 (success) | Push `Tab { label: "New Session".to_string(), live: true, session_id: sid.clone() }` to `tabs` signal |
| 4 (success) | Set `active_tab` to `tabs.read().len() - 1` (read AFTER write guard dropped) |
| 5 (success) | Set `session_id` to returned `sid` |
| 6 (success) | `blocks.set(Vec::new())` |
| 7 (success) | `messages.write().clear()` |
| 8 (error) | Log error; leave tabs signal unchanged; no UI change |

RSX: `button { onclick: move |_| on_tab_new.call(()), ... }`

**Label:** New tab label is exactly `"New Session"` — no UUID display, no prompt (CONTEXT.md D-04).

### Disabled State (scanner_active = true)

Apply inline style to `.wh-tabs` container div:
```
style: if disabled { "pointer-events: none; opacity: 0.5;" } else { "" }
```
`disabled` is derived from `scanner_active()` read at the `WarpHermes` call site and passed as `disabled: bool` prop to `TitleBar` (CONTEXT.md §Specific Ideas).

## Copywriting Contract

| Element | Copy |
|---------|------|
| New tab label | `New Session` (exact — no quotes in UI, no UUID suffix) |
| "+" button | `+` (single character, Unicode U+002B) |
| Close button | `×` (multiplication sign, Unicode U+00D7 — not ASCII `x`) |
| Empty tab strip (never shown) | Not applicable — closing last tab auto-creates a new session; strip is never empty |
| Error: create_session fails | No user-visible error copy this phase — log error internally, leave tabs unchanged (silent fail per RESEARCH.md Pattern 4) |
| Disabled tabs tooltip | None — no tooltip; visual grey-out is the only affordance (CONTEXT.md D-02) |

**No destructive confirmations in this phase.** "×" close removes the tab from local signal only (session persists in StateStore per CONTEXT.md D-05). No confirmation dialog, no undo affordance.
