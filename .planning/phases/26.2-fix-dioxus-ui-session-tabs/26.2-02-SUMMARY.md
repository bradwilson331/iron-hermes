# Phase 26.2 Plan 02: Interactive Tab Strip (D-01..D-09) Summary

Fully interactive TitleBar session tab strip implemented via EventHandler callbacks, Signal<Vec<Tab>>, and three closures in WarpHermes. Tab clicks switch sessions (D-01), streaming greys out the tab strip (D-02), "+" creates new sessions (D-03/D-04), "×" removes tabs with auto-switch and last-tab guard (D-05/D-06), and tabs stay in sync with session lifecycle (D-07/D-08).

## What Was Built

### Tab struct extension (state.rs)

Added `pub session_id: String` field to `Tab` struct with doc comment. Updated `demo_tabs()` with `session_id: "demo-0"/"demo-1"/"demo-2"` values. The existing `label` and `live` fields are unchanged.

### TitleBar rewrite (title_bar.rs)

Replaced the 3-prop static component with a 7-prop interactive component:
- `disabled: bool` — greys out `.wh-tabs` container with `pointer-events: none; opacity: 0.5` when true (D-02)
- `on_tab_click: EventHandler<usize>` — fires on tab div click with index
- `on_tab_close: EventHandler<usize>` — fires on `×` button with `evt.stop_propagation()` to prevent parent click (D-05)
- `on_tab_new: EventHandler<()>` — fires on `+` button
- The `×` element changed from a bare `span` to a `button` with button-reset styles plus `background: none; border: none; cursor: pointer;`

### WarpHermes wiring (warp_hermes.rs)

- `active_tab` signal changed to `let mut`
- New `let mut tabs = use_signal(Vec::<Tab>::new)` signal
- Mount `use_effect` extended: on `create_session()` success, pushes Tab with `"New Session"` label if not already present (D-08 idempotent via `already_present` check), then sets `active_tab` to new last index
- New `use_effect` after `sessions = use_server_future(...)`: seeds `tabs` from `list_sessions()` result, skipping already-present session IDs (D-07 double-seed guard)
- Old `let tabs: Vec<Tab> = match sessions() { ... }` inline derivation removed entirely
- Three closures added after `on_rerun`:
  - `on_tab_click`: guards `scanner_active()`, extracts `session_id` into owned String, drops read guard, sets `active_tab` + `session_id` + clears blocks/messages
  - `on_tab_new`: `spawn(async move)` calling `create_session().await`, pushes tab, sets active, clears blocks/messages; error arm logs to `web_sys::console`
  - `on_tab_close`: removes tab, checks `is_empty()`, if empty spawns create_session (same as on_tab_new body), else computes new active index per D-06 formula, extracts session_id, clears blocks/messages
- TitleBar call site updated to 7-prop form with `disabled: scanner_active()`

### Regression tests (websocket_lifecycle_parity.rs)

Three new source-text tests added:
- `tab_click_clears_blocks_and_switches_session_id` — asserts on_tab_click closure, scanner_active() guard, blocks.set, messages.write().clear
- `tab_new_calls_create_session_and_appends_tab` — asserts on_tab_new, create_session().await, "New Session" label, tabs.write().push
- `tab_close_uses_stop_propagation` — asserts evt.stop_propagation(), 3 EventHandler prop declarations, disabled prop, pointer-events style

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] sessions-seeding use_effect placed at wrong position initially**
- **Found during:** Task 2 Step D — effect was added before `sessions` signal was declared
- **Issue:** `use_server_future(list_sessions)` is declared later in the function; effects referencing undeclared signals would not compile
- **Fix:** Moved the sessions-seeding `use_effect` to immediately after the `sessions = use_server_future(...)` declaration
- **Files modified:** `crates/iron_hermes_ui/src/components/warp_hermes.rs`
- **Commit:** b8ad730

**2. [Rule 1 - Bug] on_tab_click and on_tab_close closures required `mut`**
- **Found during:** Task 2 build verification
- **Issue:** Rust requires `let mut` when a closure is called via mutable call (capturing mutable signals)
- **Fix:** Changed `let on_tab_click` and `let on_tab_close` to `let mut on_tab_click` and `let mut on_tab_close`
- **Files modified:** `crates/iron_hermes_ui/src/components/warp_hermes.rs`
- **Commit:** b8ad730

**3. [Rule 1 - Bug] TitleBar call site had redundant closure wrappers**
- **Found during:** Task 2 clippy run — `redundant_closure` lint fired on `move |idx| on_tab_click(idx)`
- **Fix:** Changed to direct binding `on_tab_click: on_tab_click`
- **Files modified:** `crates/iron_hermes_ui/src/components/warp_hermes.rs`
- **Commit:** b8ad730

**4. [Rule 1 - Bug] Fix pre-existing parity test: tracing import assertion mismatch**
- **Found during:** Task 3 test run — `server_ws_runs_turn_in_spawned_task_and_streams_concurrently` was failing
- **Issue:** Test asserted `use tracing::warn;` but ws.rs has `use tracing::{info, warn};` (documented as pre-existing in Plan 01 SUMMARY)
- **Fix:** Added `|| ws.contains("#[cfg(feature = \"server\")]\nuse tracing::{info, warn};")` alternative
- **Files modified:** `crates/iron_hermes_ui/tests/websocket_lifecycle_parity.rs`
- **Commit:** 6b46aa6

**5. [Rule 1 - Bug] Fix pre-existing parity test: disconnect log message mismatch**
- **Found during:** Task 3 test run — `server_ws_disconnect_teardown_distinguishes_clean_recv_from_broken_send` was failing
- **Issue:** Test asserted `"websocket recv closed; exiting connection"` but ws.rs logs `"websocket close frame received; exiting connection"` (documented as pre-existing in Plan 01 SUMMARY)
- **Fix:** Added third alternative `|| ws.contains("websocket close frame received; exiting connection")`
- **Files modified:** `crates/iron_hermes_ui/tests/websocket_lifecycle_parity.rs`
- **Commit:** 6b46aa6
