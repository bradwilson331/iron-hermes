## Tab Switch Depth

| Option | Description | Selected |
|--------|-------------|----------|
| Session switch only | Change session_id for future messages, clear block stream. No history loading. | ✓ |
| Load session history | Fetch stored messages from StateStore, populate block stream. Requires new server function. | |

**User's choice:** Session switch only (Recommended)
**Notes:** Keeping it minimal — no history loading this phase.

---

### Tab switch during streaming

| Option | Description | Selected |
|--------|-------------|----------|
| Disable tab switching while streaming | Ignore tab clicks when scanner_active is true. No teardown complexity. | ✓ |
| Allow switch, leave stream running | Tab switches immediately; old session stream continues silently. | |

**User's choice:** Disable tab switching while streaming (Recommended)

## New Tab ("+" Button)

| Option | Description | Selected |
|--------|-------------|----------|
| Create + switch immediately | Call create_session(), add tab, switch active_tab, clear block stream. | ✓ |
| Create in background | Create session and tab without switching. User must click to activate. | |

**User's choice:** Create + switch immediately (Recommended)

## Close Tab ("×" Button)

| Option | Description | Selected |
|--------|-------------|----------|
| UI-only remove | Remove from tabs signal; session stays in StateStore. No delete API needed. | ✓ |
| Delete from StateStore | Call delete_session server function. Permanent removal. | |

**User's choice:** UI-only remove (Recommended)

---

### Active tab closed

| Option | Description | Selected |
|--------|-------------|----------|
| Switch to neighbor tab | Auto-switch to prev (or next if first). If last tab, auto-create new session. | ✓ |
| Switch to first tab | Always jump to index 0. | |

**User's choice:** Switch to neighbor tab (Recommended)

## Session List Sync

| Option | Description | Selected |
|--------|-------------|----------|
| Local signal, no re-fetch | Signal<Vec<TabEntry>> mutated directly on create/close. Server future only seeds initial list. | ✓ |
| Re-fetch from server on each change | Call list_sessions() after every create/close. Always server-consistent. | |

**User's choice:** Local signal, no re-fetch (Recommended)

---

### On-mount session in tabs

| Option | Description | Selected |
|--------|-------------|----------|
| Yes — add it after creation | After create_session() succeeds, push TabEntry into tabs signal and set active. | ✓ |
| No — just use list_sessions() result | Don't track on-mount session specially. | |

**User's choice:** Yes — add it to the tabs signal after creation (Recommended)

## Claude's Discretion

- `TabEntry` struct shape — extend existing `state::Tab` with `session_id: String` or introduce a wrapper
- "×" button event handling — `stopPropagation` approach to avoid triggering parent tab click
- CSS for greyed-out tabs during streaming — `opacity` / `pointer-events: none` inline style

## Deferred Ideas

- Load session message history on tab switch — requires `list_session_messages` server function + `ChatMessage → BlockEntry` mapping; deferred to future phase
- Delete session from StateStore via "×" — requires `delete_session` server function; deferred
- Tab renaming — new capability, separate phase
- Tab reordering (drag-and-drop) — new capability, separate phase
