## Accomplishments

- Implemented `SkillScanner` trait with three implementations: `CoreSkillScanner` (delegates to `scan_skill_content`), `AlwaysCleanScanner` (happy-path tests), `AlwaysBlockedScanner` (rejection tests). `ScanVerdict` carries per-file results with `has_blocks()` and `summary()` helpers. `enforce_trust_gate` applies D-15: Community + scan-hit = hard-reject, Builtin/Official/Trusted + scan-hit = WARN-BUT-LOAD.

- Implemented `install()` with the 5-step atomic pipeline porting `install_from_quarantine` from Python reference. Quarantine `TempDir` scoped under `.hub/quarantine/` (same FS, Pitfall 2). `bundle_content_hash` produces deterministic SHA-256 over sorted file paths + content. Category parsed from `metadata.hermes.category` frontmatter, defaulting to `"general"`. `AlreadyInstalled` error on duplicate installs.

- Implemented `update()` with hash-drift detection: fetch latest, compare content hashes, re-scan (resolved open question #1), atomically replace old version. Returns `AlreadyInstalled` error with hash prefix when content is unchanged. `uninstall()` removes manifest entry first (safe ordering), then directory, then cleans empty parent category dir.

## Install Pipeline Steps

| Step | Action | Failure behavior |
|------|--------|------------------|
| 1. Fetch | `source.fetch(identifier)` | Error propagated; no state created |
| 2. Quarantine | `tempdir_in(.hub/quarantine/)` + `write_bundle_to_dir` | TempDir auto-cleans on drop |
| 3. Scan | `scanner.scan_bundle` + `enforce_trust_gate` | Community+blocked = hard-reject; quarantine auto-cleans |
| 4. Atomic move | `rename` or `copy_dir_recursive`+remove | Falls back to copy on cross-FS |
| 5. Manifest | `HubManifest::load_or_default` + insert + `save` | Atomic .tmp+rename save |

## Decisions Made

- **SkillScanner operates on `&[BundleFile]`:** No filesystem dependency in the scan step. The scanner receives in-memory file content directly from the bundle, making tests fast and deterministic without writing to disk first.
- **`enforce_trust_gate` as free function:** Separates the D-15 policy decision from the scanner implementation. Any scanner can be paired with any trust level.
- **Category defaults to `"general"`:** When `metadata.hermes.category` is absent from SKILL.md frontmatter, the skill installs under `skills/general/<name>/`.
- **`update()` returns error on no-drift:** Rather than a silent no-op, returns `AlreadyInstalled` with the first 12 chars of the hash so the user gets explicit feedback.
- **Manifest-first removal:** `uninstall()` saves the manifest (with entry removed) before deleting the directory. If dir removal fails, the skill is at least de-registered.

## Security Threat Mitigations Applied

| Threat | Mitigation |
|--------|------------|
| D-11: quarantine isolation | Bundle never written directly to final location; TempDir under .hub/quarantine/ |
| D-15: trust-gated enforcement | `enforce_trust_gate` hard-rejects Community+scan-hit; WARN-BUT-LOAD for Trusted |
| Pitfall 2: cross-FS rename | Quarantine under .hub/ (same FS); `atomic_move` fallback to copy+remove |
| Content integrity | `bundle_content_hash` SHA-256 stored in manifest; update detects drift |
| Partial state | Quarantine TempDir auto-cleans on any error; manifest written only on success |
