## Fixed Issues

### CR-01: SSRF via unvalidated `tarball_url` in well-known adapter

**Files modified:** `crates/ironhermes-hub/src/well_known.rs`
**Commit:** 571861e
**Applied fix:** Added SSRF validation of `tarball_url` extracted from remote `index.json` before passing it to `fetch_tarball_bundle`. The validation enforces HTTPS scheme and rejects private/loopback hosts via `is_private_host`, matching the same guards applied to identifiers in `parse_identifier`. Validation is skipped in `test_mode` to allow wiremock (HTTP + loopback) integration tests.

### WR-01: Potential panic from string slice on short hashes

**Files modified:** `crates/ironhermes-cli/src/skills_cmd.rs`, `crates/ironhermes-hub/src/installer.rs`
**Commit:** 8a78c2e
**Applied fix:** Replaced all four instances of `[..12]` string slicing on hash values with safe `.get(..12).unwrap_or(...)` to prevent panics on unexpectedly short strings. Fixed in `cmd_install` (line 244), `cmd_update` (lines 403-404), and `installer::update` "already up to date" message (line 229).

### WR-02: `futures::executor::block_on` inside async-callable code path

**Files modified:** `crates/ironhermes-cli/src/skills_cmd.rs`, `crates/ironhermes-cli/Cargo.toml`
**Commit:** 1389f78
**Applied fix:** Made `build_sources` an `async fn` and replaced the `futures::executor::block_on(GitHubAuth::resolve(...))` call with a direct `.await`. Updated all three call sites (`cmd_install`, `cmd_search_impl`, `cmd_update`) to `.await` the now-async function. Removed the unused `futures` dependency from the crate's `Cargo.toml`.

### WR-03: `write_bundle_to_dir` does not validate file paths against traversal

**Files modified:** `crates/ironhermes-hub/src/installer.rs`
**Commit:** 41ee588
**Applied fix:** Added defense-in-depth path validation in `write_bundle_to_dir`: (1) re-validates each `file.path` via `crate::tarball::validate_bundle_rel_path` to reject `..`, absolute paths, etc., and (2) verifies that the resolved `dest` path is still under the target `dir` via `starts_with` check. This guards against `SkillBundle` structs constructed by future code paths that bypass tarball extraction.

### WR-04: Trust recomputation in `cmd_list_impl` duplicates logic and may drift

**Files modified:** `crates/ironhermes-cli/src/skills_cmd.rs`
**Commit:** 1dd5f27
**Applied fix:** Extracted the inline trust recomputation logic from `cmd_list_impl` into a dedicated `recompute_trust_str` helper function with a doc comment explicitly noting it must stay in sync with `resolve_source` in `ironhermes-core/src/skills.rs` and the adapters' `trust_level_for` methods. Updated `cmd_list_impl` to call the new helper.

### WR-05: `is_private_host` SSRF guard incomplete for IPv6

**Files modified:** `crates/ironhermes-hub/src/well_known.rs`
**Commit:** c9175ca
**Applied fix:** Enhanced `is_private_host` to: (1) strip bracket wrapping from IPv6 addresses (e.g., `[::1]` -> `::1`), (2) reject IPv6 link-local addresses (`fe80::`), (3) reject IPv6 unique-local addresses (`fc00::`/`fd00::`), with case-insensitive matching. Added comprehensive test coverage for all new IPv6 cases including bracket-wrapped variants and a public IPv6 address (2001:db8::1) as negative test.
