## Decisions Made

- **SkillSource variant ordering:** Trusted inserted between Official and Community (`Builtin, Official, Trusted, Community`). Existing match arms that merged Builtin/Official into one arm now merge all three; Community branch unchanged.
- **Pure-function auth:** `resolve_from_env(getenv: impl Fn(&str) -> Option<String>)` separates pure precedence logic from side-effecting subprocess and env reads. Tests exercise the pure function; `gh` subprocess is covered by the async `resolve()` path using `tokio::time::timeout(2s)`.
- **No serial_test dep:** Env-mutation tests in `manifest.rs` use a `Mutex<()> ENV_LOCK` guard instead of adding `serial_test = "3"`, keeping the dev-dep surface minimal.
- **HubConfig backward compat:** Both `SkillsConfig` and `HubConfig` carry `#[serde(default)]` at the struct level so any pre-19.1 `config.yaml` without a `hub:` key still deserializes cleanly.

## Manifest Atomic-Save Semantics

`HubManifest::save()` writes to `<manifest_path>.json.tmp` then `std::fs::rename()` to final path. Both files are under `$HERMES_HOME/skills/.hub/` so the rename stays within the same filesystem (Pitfall 2 â€” cross-FS atomic rename failure avoided).

Unknown manifest fields survive the round-trip via `#[serde(flatten, default)] extras: HashMap<String, serde_json::Value>` on both `HubManifest` and `ManifestEntry`.
