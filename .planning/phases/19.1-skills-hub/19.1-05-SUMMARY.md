## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] github.rs subtraction overflow on root-level SKILL.md**
- **Found during:** Task 2 (hub_search_response_shape test panic)
- **Issue:** `entry_path.len() - "/SKILL.md".len()` underflows when `entry_path == "SKILL.md"` (8 - 9 bytes). This occurs when a tap's top-level directory contains a bare `SKILL.md` with no subdirectory prefix.
- **Fix:** Replaced raw subtraction with `strip_suffix("/SKILL.md").unwrap_or_else(|| strip_suffix("SKILL.md")...)` and handled empty `skill_dir` â†’ use repo name as skill name, bare repo path as identifier.
- **Files modified:** `crates/ironhermes-hub/src/github.rs`
- **Commit:** `cd7cb8c`

**2. [Rule 3 - Blocking] ironhermes-cli integration tests cannot access binary-only crate**
- **Found during:** Task 1 (test compilation: `use of unresolved module ironhermes_cli`)
- **Issue:** `[[bin]]`-only crates don't expose a library for `use ironhermes_cli::...` in `tests/`.
- **Fix:** Added `[lib]` target `src/lib.rs` that re-exports `pub mod skills_cmd`. Both binary and test can reach the module.
- **Files modified:** `crates/ironhermes-cli/Cargo.toml`, `crates/ironhermes-cli/src/lib.rs` (new)
- **Commit:** `343db30`

**3. [Rule 1 - Bug] `#[tokio::test]` async tests calling `block_on` inside runtime**
- **Found during:** Task 1 (cli_search_json_format_returns_valid_json panic)
- **Issue:** Tests marked `#[tokio::test]` that called `rt.block_on()` inside the closure failed with "Cannot start runtime from within runtime".
- **Fix:** Changed `with_hermes_home` closure tests to either plain `#[test]` (sync), or restructured async test to call the async fn directly under `#[tokio::test]`.
- **Files modified:** `crates/ironhermes-cli/tests/skills_cmd_test.rs`
- **Commit:** `343db30`
