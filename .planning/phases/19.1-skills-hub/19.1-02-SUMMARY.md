## Accomplishments

- Implemented `tarball::validate_bundle_rel_path` (port of Python `_validate_bundle_rel_path` lines 121-148): rejects empty, NUL, absolute, `..`, Windows prefix, root components; returns cleaned `/`-joined relative path. `extract_tarball_prefix` enforces `MAX_EXTRACTED_BYTES` (50 MB) and `MAX_ENTRIES` (1000) caps with `saturating_add` overflow safety; skips symlinks/hardlinks outright (T-19.1-02-01/02).
- Implemented `GitHubSource` with exact `DEFAULT_TAPS` port (D-02), Trees API search, tarball download + extraction, `trust_level_for` using only `owner/repo` prefix (T-19.1-02-04), and rate-limit detection from `X-RateLimit-*` headers (T-19.1-02-06). 9 wiremock integration tests.
- Implemented `WellKnownSkillSource` with `https_only(true)` reqwest client (T-19.1-02-03), SSRF guard rejecting loopback/RFC-1918 (T-19.1-02-03), hardcoded `trust_level_for → Community` (D-07/T-19.1-02-05), and origin allowlist. 6 wiremock integration tests.
- Implemented `SkillsShSource` wrapping `Arc<GitHubSource>` via `registry.json` resolution; `fetch` delegates to `GitHubSource` then re-stamps `source_id = "skills-sh"` (D-06/D-09). 4 wiremock integration tests.

## Adapter Trait Conformance

All three adapters implement `HubSource: Send + Sync` from Plan 01:

| Adapter | `source_id()` | `trust_level_for()` | `search()` | `fetch()` |
|---------|---------------|---------------------|------------|-----------|
| `GitHubSource` | `"github"` | `Trusted` if `owner/repo` in `trusted_repos`; else `Community` | Trees API, substring match, per-tap fail-soft | Tarball download + `extract_tarball_prefix` |
| `WellKnownSkillSource` | `"well-known"` | Always `Community` (D-07, hardcoded) | Fetch `/.well-known/skills/index.json`, substring match | Fetch index → `tarball_url` → extract |
| `SkillsShSource` | `"skills-sh"` | `Community` (sync deferred; install pipeline re-resolves) | `registry.json` substring match | Resolve → `GitHubSource.fetch` + re-stamp |

## tarball.rs Invariants

| Invariant | Implementation |
|-----------|----------------|
| Path traversal guard | `validate_bundle_rel_path` rejects empty, NUL, absolute, `..`, `Prefix`, `RootDir` components |
| Size bomb | `MAX_EXTRACTED_BYTES = 50 MB`; `total.saturating_add(size) > MAX` → `HubErrorKind::Parse` |
| Entry count bomb | `MAX_ENTRIES = 1000`; `count > MAX` → `HubErrorKind::Parse` |
| Symlink/hardlink escape | `EntryType != Regular && != GNUSparse → continue` (skip non-regular) |

## Trust-Level Contracts

- **GitHubSource**: variable — reads `HashSet<String>` from `config.hub.trusted_repos` (D-04/D-08). `trust_level_for("owner/repo/any/path")` splits on first two `/` segments; exact match → `Trusted`, else `Community`.
- **WellKnownSkillSource**: hardcoded `Community` for all identifiers regardless of origin (D-07/T-19.1-02-05). No config path can override.
- **SkillsShSource**: synchronous method returns `Community` (deferred). The install pipeline calls `GitHubSource.trust_level_for` on the resolved GitHub identifier at install time (D-06/D-08).

## Security Threat Mitigations Applied

| Threat ID | Mitigation |
|-----------|------------|
| T-19.1-02-01 (path traversal) | `validate_bundle_rel_path` rejects `..`, absolute, prefix, root; symlinks skipped |
| T-19.1-02-02 (tarball bomb) | `MAX_EXTRACTED_BYTES` + `MAX_ENTRIES` hard caps with `saturating_add` |
| T-19.1-02-03 (SSRF) | `https_only(true)` client; `is_private_host` guard; allowlist enforcement |
| T-19.1-02-04 (trust spoofing) | `splitn(3, '/')` takes only first 2 segments for repo lookup; path ignored |
| T-19.1-02-05 (trust drift) | `WellKnownSkillSource::trust_level_for` hardcoded, no config path |
| T-19.1-02-06 (rate limiting) | 403 + `X-RateLimit-Remaining=0` → `RateLimited` with `retry_after_s` |
| T-19.1-02-07 (auth leak) | `tracing::instrument(skip(self))` on async fns; auth never stringified in errors |
| T-19.1-02-08 (MITM) | HTTPS-only client on SkillsShSource registry fetch; content hash at install time (Plan 03) |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] `with_api_base` gated behind `#[cfg(test)]` unreachable from integration tests**
- **Found during:** Task 1
- **Issue:** Integration test crates don't inherit `cfg(test)` from the lib, so `with_api_base` was inaccessible.
- **Fix:** Changed to `#[doc(hidden)] pub fn` accessible from any caller.
- **Files modified:** `crates/ironhermes-hub/src/github.rs`

**2. [Rule 1 - Bug] `tar` crate rejects `..` in path at construction time (fixture build)**
- **Found during:** Task 1 (`github_fetch_rejects_traversal` test)
- **Issue:** `header.set_path(".../../...")` panics — the crate validates paths.
- **Fix:** Constructed the traversal tarball using raw 512-byte ustar header bytes with the malicious path written directly into the name field, bypassing the crate check.
- **Files modified:** `crates/ironhermes-hub/tests/fixtures/mod.rs`

**3. [Rule 3 - Blocking] WellKnownSkillSource HTTPS-only + SSRF guard blocked wiremock (HTTP + loopback)**
- **Found during:** Task 2
- **Issue:** wiremock listens on `http://127.0.0.1:PORT` which hits both the HTTPS-only check and the SSRF loopback guard.
- **Fix:** Added `test_mode: bool` field + `new_http_for_tests()` constructor that bypasses both checks. Production `new()` always enforces both.
- **Files modified:** `crates/ironhermes-hub/src/well_known.rs`

**4. [Rule 1 - Bug] Allowlist check compared bare host vs host:port**
- **Found during:** Task 2 (`well_known_fetch_happy_path`)
- **Issue:** `allowed_origins` stored `"127.0.0.1:PORT"` but `url.host_str()` returns `"127.0.0.1"` without port.
- **Fix:** Compare against both `host` and `host:port` authority in the allowlist check.
- **Files modified:** `crates/ironhermes-hub/src/well_known.rs`

**5. [Rule 1 - Bug] SkillsShSource https_only client rejected HTTP wiremock registry URL**
- **Found during:** Task 3
- **Issue:** `with_registry_url` changed the URL string but not the reqwest client's https_only setting.
- **Fix:** `with_registry_url` detects `http://` prefix and rebuilds the client without `https_only`.
- **Files modified:** `crates/ironhermes-hub/src/skills_sh.rs`
