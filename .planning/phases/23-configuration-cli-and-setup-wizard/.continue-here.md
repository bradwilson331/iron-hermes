## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Hardcoded model default in wizard | Plan 02's `setup.rs` originally wired `model.default = "openrouter/qwen-2.5-coder-32b"` (invalid ID, 404'd on first chat call). Iterations through `qwen/qwen-2.5-coder-32b-instruct` (404 entitlement) and `qwen/qwen3-coder:free` (429 rate-limit) showed every fixed default has a fragile failure mode. | advisory | Wizard now uses `prompt_required(rl, "Default model (e.g. openai/gpt-4o-mini)")` — forces user to enter a model their account actually supports. Future wizard sections (gateway/tools, Phase 25/26) should follow the same "required prompt + hint" pattern instead of locking in a default. |
| Inverted preflight gate scope | Plan 02 §Task 7 wrote `skip_preflight = matches!(... Setup | Config)` then `if !skip_preflight { run_preflight }` — over-broad. Triggered the rustyline wizard for every non-setup/config subcommand (skills, mcp, cron, memory, doctor), which EOFd on stdin in scripted/test runs. | blocking | Preflight gate is now `matches!(cli.command, Some(Chat) | None) && cli.execute.is_none()` — only the two genuinely interactive entry points. When future phases add more CLI subcommands, do NOT touch this gate; new commands inherit the correct skip semantics automatically. |
