Define the `GuardrailHook` trait (per D-05 — complex logic layer):
```rust
pub trait GuardrailHook: Send + Sync {
    /// Inspect a tool call and decide whether to allow, warn, or block.
    /// This method is synchronous — no I/O should be performed here.
    fn check(&self, tool_name: &str, args: &serde_json::Value) -> GuardrailDecision;

    /// Human-readable name for this guardrail (used in log messages).
    fn name(&self) -> &str;
}
```

Implement `BlocklistGuardrail` (per D-05 — config-driven blocklist, checked first):
```rust
pub struct BlocklistGuardrail {
    blocked_tools: Vec<String>,
}

impl BlocklistGuardrail {
    pub fn new(blocked_tools: Vec<String>) -> Self {
        Self { blocked_tools }
    }

    pub fn from_config(config: &HooksConfig) -> Self {
        Self::new(config.blocked_tools.clone())
    }
}

impl GuardrailHook for BlocklistGuardrail {
    fn check(&self, tool_name: &str, _args: &serde_json::Value) -> GuardrailDecision {
        if self.blocked_tools.iter().any(|b| b == tool_name) {
            GuardrailDecision::Block {
                reason: format!("tool '{}' is on the blocklist", tool_name),
            }
        } else {
            GuardrailDecision::Allow
        }
    }

    fn name(&self) -> &str {
        "blocklist"
    }
}
```

Add helper function for formatting the error message returned to the agent (per D-06):

```rust
use crate::config::ErrorDetailLevel;

pub fn format_guardrail_error(
    tool_name: &str,
    reason: &str,
    guardrail_name: &str,
    detail_level: &ErrorDetailLevel,
) -> String {
    match detail_level {
        ErrorDetailLevel::Full => {
            format!("Tool '{}' blocked by guardrail '{}': {}", tool_name, guardrail_name, reason)
        }
        ErrorDetailLevel::Minimal => {
            "Tool call blocked by security policy".to_string()
        }
    }
}
```

```rust
// Guardrail intercept (HOOK-02): check all guardrails before dispatch.
// Per D-05: config blocklist is registered first, trait hooks second.
for guardrail in &self.guardrails {
    match guardrail.check(name, &args) {
        ironhermes_hooks::GuardrailDecision::Allow => {}
        ironhermes_hooks::GuardrailDecision::Warn { reason } => {
            tracing::warn!(
                tool = %name,
                guardrail = %guardrail.name(),
                reason = %reason,
                "Guardrail warning (proceeding)"

            // Continue to next guardrail — warn does not block

        ironhermes_hooks::GuardrailDecision::Block { reason } => {

                name,

            return Err(anyhow::anyhow!("{}", error_msg));

}
```

| T-06-04 | Tampering | Guardrail bypass via arg mutation | mitigate | Guardrail check receives the same `&args` reference passed to `tool.execute()` — no copy-after-check gap. Check and execute are in the same `dispatch()` call with no intermediate mutation point. |
