## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent loop -> JSONL log file | Hook events written to disk may contain user message previews |
| hooks.toml -> HooksConfig | Config file parsed from user-writable path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | log_writer.rs JSONL file | mitigate | Content previews truncated to 200 chars via floor_char_boundary (D-02); full content never logged. File written to user's home directory with default permissions. |
| T-06-02 | Tampering | config.rs hooks.toml parsing | accept | Config file is in user's home directory — same trust level as the binary itself. No elevation of privilege from malformed TOML (toml crate handles parsing safely). |
| T-06-03 | Denial of Service | registry.rs fire() spawning tasks | mitigate | Each fire() spawns lightweight tokio tasks. Listeners are registered at startup only (not unbounded). JSONL writer is synchronous file append — no unbounded queue growth. |
</threat_model>
