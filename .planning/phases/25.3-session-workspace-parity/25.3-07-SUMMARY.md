# Phase 25.3 Plan 07: PromptBuilder::with_workspace_root Summary

**One-liner:** D-W-2 builder method — `PromptBuilder::with_workspace_root(&Path)` appends `[Workspace: <root>]` to `PromptSlot::Identity` (durable slot 1) so Plan 8 can inject session-scoped workspace awareness without breaking the Anthropic prompt cache (Pitfall 2). Idempotent on duplicate calls; falls back to `DEFAULT_AGENT_IDENTITY` when called before SOUL.md load. 6 unit tests lock append semantics, slot placement, idempotency, and the durable-region invariant.

## What Shipped

### `crates/ironhermes-agent/src/prompt_builder.rs` (MODIFIED, +140 lines)

**New public method (32 lines incl. doc comment, lines ~189-220):**

```rust
pub fn with_workspace_root(mut self, root: &std::path::Path) -> Self {
    let workspace_line = format!("[Workspace: {}]", root.display());
    let current = self
        .slots
        .get(&PromptSlot::Identity)
        .cloned()
        .unwrap_or_else(|| DEFAULT_AGENT_IDENTITY.to_string());
    if current.contains(&workspace_line) {
        return self;  // idempotency guard
    }
    let with_workspace = format!("{}\n\n{}", current, workspace_line);
    self.set_slot(PromptSlot::Identity, with_workspace);
    self
}
```

Doc comment cross-references CONTEXT.md Discretion (slot recommendation), RESEARCH.md Pitfall 2 (cache breakage), and PATTERNS.md (### MOD: prompt_builder.rs section).

**Method placement:** inserted in the existing `impl PromptBuilder` block, immediately after `set_skill_registry` and before `with_system_message` — co-located with the other `with_*` builder methods so it's discoverable by Plan 8 and future contributors.

**Test placement:** appended to the existing `#[cfg(test)] mod tests` at the end of the file, under a banner header `Phase 25.3 D-W-2: with_workspace_root tests (Pitfall 2 cache stability)`.
