# Phase 25.3 Plan 09: AgentLoop trajectory callback wireup (D-T-1 + D-T-3) Summary

**One-liner:** AgentLoop now appends a TrajectoryEntry to per-session JSONL on EVERY tool call (success + failure) via the Arc<dyn TrajectoryWriterHandle> handle attached at all 5 construction sites; args are post-redact via Plan 5's Tool::redact_args; impact_level derived from a 3-tier name classifier; turn_index increments per agent turn; append failures log via tracing::warn! and never abort the turn (best-effort ledger per CONTEXT D-T-2).

## Why This Plan Existed

Plan 4 ships TrajectoryWriter (append-only fsync-per-line). Plan 5 ships Tool::redact_args (the redaction seam). Plan 6 ships the trait-object cycle-break and CommandContext fields. Plan 8 wires the writer through CommandContext at all 4 slash-dispatch sites. Plan 9 closes the loop: every tool execution in AgentLoop appends one TrajectoryEntry to disk, so Phase 25.4 Curator's heuristic gate (D-C-2) sees real session data.

Per RESEARCH.md Architectural Responsibility Map: "Trajectory write happens at AgentLoop tool-result callback; lifecycle controls open/close." Plan 9 is that callback.
