CR-01: build_app_deps opens a `StateStore` (lines 403-406) but NEVER calls `state_store.create_session(...)`. The REPL session UUID generated at line 143 is only used as the trajectory directory key. Result: every REPL session is invisible to `/sessions`, `/resume`, `/history`, `/export-session`, and `/sessions --workspace` (no row exists to match against). Same Pitfall-1 class regression that the original Plan 8 parity-guard missed.

CR-04: The same `build_app_deps` never constructs a PromptBuilder + calls `.with_workspace_root(&ws.root)`. The durable Identity-slot `[Workspace: <root>]` line that Plan 7's tests lock for cache stability is missing on the DEFAULT `hermes chat` surface (ratatui REPL has been default since Phase 22.4). The model has no in-context awareness of which workspace it is in.

Purpose: Unblock per-D-W-1 value persistence on the primary user-facing surface AND restore the Identity-slot prompt line for the same surface. Together these complete the workspace contract for the REPL.

Output: A modified `build_app_deps` that (a) calls `state_store.create_session(&session_id, "cli-repl", Some(client.model()), None, None, workspace_root_str)`, (b) constructs a PromptBuilder, calls `.with_workspace_root(&ws.root)` when workspace is Some, builds the system message, and seeds it into `app.history` so the per-turn AgentLoop sees it via `messages_snapshot`.
