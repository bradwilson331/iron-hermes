## Decisions Made

- **result_or_error encoding via flattened Option pair** — chose `result: Option<String>` + `error: Option<String>` with `skip_serializing_if` over a custom `Result<String, String>` serde impl. Rationale: zero custom serde code, simpler downstream consumer parsing (no enum variant dispatch), matches the wire shape downstream consumers (Phase 25.4 Curator, RL pipelines) actually want. Mutual exclusion enforced by `success`/`failure` constructors. PATTERNS.md flagged the trade-off; the golden-file test locks the shape against silent drift.
- **Stub-then-fill task ordering** — Task 1 lands a minimal `format.rs` (struct + enum shells, no methods) so `cargo build -p ironhermes-trajectory` succeeds at Task 1 commit. Task 2 RED targets the missing methods (`weight`, `success`, `failure`) — rustc E0599 errors satisfy the RED gate without needing flaky `assert!(false)` placeholders. Task 2 GREEN then drops in the impls. Cleaner than the plan's alternate "comment `pub mod format` until Task 2" suggestion because every commit on the branch builds cleanly.
- **ImpactLevel discriminants + snake_case dual encoding** — `#[repr(u8)] Read=0/Write=5/SystemChange=10` AND `#[serde(rename_all = "snake_case")]`. Wire is human-readable (`"read"` / `"write"` / `"system_change"`) for grep-ability and operator UX; Phase 25.4 Curator extracts numeric weight via `self as u8` with zero overhead. Both encodings locked in tests (`impact_level_wire_strings_locked` for strings, `impact_level_discriminants_are_0_5_10` for weights).
- **`tracing` as direct dep (not dev-dep)** — Plan 4's writer will `warn!` on append failures inside this crate; pre-declaring it keeps Plan 4's diff scoped to writer.rs only, avoiding a Cargo.toml ping-pong.
