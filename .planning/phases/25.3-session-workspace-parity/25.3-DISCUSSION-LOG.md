## 1. Reference architecture (user-provided)

User opened by clarifying that "session-workspace parity" means bringing IronHermes' session model to parity with the [hermes-agent](https://github.com/NousResearch/hermes-agent) Python reference. Key concepts captured:

### Sessions folder layout (parity target)

`~/.hermes/sessions/[session_id]/`:

| File | Purpose |
|---|---|
| `messages.json` | Raw conversation: User/Assistant/System messages including hidden tool outputs. |
| `metadata.json` | Session-specific state: active model, temperature, title, last-access timestamp. |
| `context.json` | Compressed conversation summary (used when full history exceeds context window). |
| `trajectories.jsonl` | Granular log of every tool execution (audit + RL training). |

### Python implementation layers

1. **Database layer** — `hermes_state.py::SessionDB` manages SQLite index mirroring the flat-file folders. `create_session()` generates UUID + creates folder.
2. **Trajectory manager** — `agent/trajectory.py::Trajectory` writes to `messages.json` immediately to prevent data loss on crash; `add_tool_call()` appends to the trajectory.
3. **Compressor** — `trajectory_compressor.py` monitors token count; on threshold, summarizes and writes to `context.json`.

### Built-in commands (already mostly present in IronHermes via Phase 21.1)

- `/new` / `/reset` — initialize new session directory
- `/undo` — truncate `messages.json`
- `/save [name]` — update `metadata.json` alias
- `/compress` — force trajectory_compressor

### Curator + Learning Loop (the bridge)

`tools/curator.py` evaluates session trajectories:

1. **Heuristic gate:** `tool_calls ≥ 3` + impactful tool used (`{terminal, write_file, patch_file, mcp_call}`) + non-trivial duration.
2. **LLM-based educational-value scoring:** trajectory summary → LLM with fixed prompt → 1–10 score; ≥ 7 promotes.
3. **Skill creation:** `impl From<Session> for Skill` writes new `.md` to `~/.hermes/skills/`.

Mapped to Rust concepts: `trait Curatable`, `struct ToolAction { name, impact_level }`, `Evaluator` trait.

### Learning Loop summary

> Sessions = Working RAM (short-term episodic). Skills = ROM (long-term procedural). The Curator is the flush operation.
