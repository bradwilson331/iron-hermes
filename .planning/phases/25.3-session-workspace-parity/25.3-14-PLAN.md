<objective>
Close the verifier-flagged BLOCKER + WR-02: `crates/ironhermes-gateway/src/session.rs::SessionStore::get_or_create` (lines 107-115) hardcodes `None` for the `workspace_root` argument with the explicit comment "Plan 0 placeholder — Plan 8 wires resolved workspace". Plan 8 wired `Workspace` into `GatewayRunner` + `GatewayMessageHandler` + per-message `CommandContext` + `PromptBuilder`, but NEVER threaded it down into `SessionStore::new` or `SessionStore::get_or_create`. As a result, every Telegram-originated session row has `workspace_root=NULL` even when the gateway was launched from a workspace-rooted cwd.

Why this matters (from VERIFICATION.md gap analysis):
1. **D-W-1 contract failure on the primary user-facing surface.** Telegram is documented as "the primary user-facing surface" (CONTEXT D-T-3) and currently 0% of Telegram sessions carry workspace metadata.
2. **`/sessions --workspace` filter silently excludes Telegram sessions** (`WHERE workspace_root = ?2` cannot match NULL).
3. **Phase 25.4 Curator's per-workspace skill scoping is starved** for the gateway surface.
4. **Existing parity-guard surface didn't catch this** — the guard greps for `.with_workspace(` and `set_workspace(`, not for the `state.create_session(..., workspace_root)` argument shape.

Purpose: Restore the D-W-1 value-persistence contract for the Telegram gateway surface. Plan 25.3-12 added INV-25.3-09 + INV-25.3-11 (RED) to lock the contract; this plan turns them GREEN.

Output: A modified `SessionStore` with an `Option<Arc<Workspace>>` field (or equivalent) that is populated by `GatewayRunner` from the same Arc that flows into `GatewayMessageHandler.workspace`, and a regression test that proves Telegram session rows carry the resolved workspace_root.

    //! workspace_root onto the SQLite sessions row when SessionStore was given
    //! a Workspace via set_workspace.
    //!
    //! Locks the contract that 25.3-VERIFICATION.md FAILED on at must-have #28.
    //! Without this test, a future refactor could re-introduce the hardcoded-None
    //! regression that shipped with Plan 8 and was caught by the verifier.
    
    use std::sync::{Arc, Mutex};
    use std::path::PathBuf;
    
    use ironhermes_core::workspace::Workspace;
    use ironhermes_core::Platform;
    use ironhermes_gateway::session::{SessionKey, SessionStore};
    use ironhermes_state::StateStore;
    
    fn fresh_state(dir: &std::path::Path) -> Arc<Mutex<StateStore>> {
        let db_path = dir.join("state.db");
        let store = StateStore::new(&db_path).expect("open fresh state.db");
        Arc::new(Mutex::new(store))
    }
    
    fn synthetic_workspace(root: PathBuf) -> Workspace {
        // Construct a minimal Workspace by hand — we don't need resolve_from_cwd
        // since this test is about the persistence path, not resolution.
        Workspace {
            soul_path: None,
            agents_chain: vec![],
            memory_dir: root.join(".ironhermes/memory"),
            skills_dir: root.join("skills"),
            tools_config: None,
            root,
        }
    }
    
    #[test]
    fn gateway_session_with_workspace_persists_workspace_root() {
        let tmp = tempfile::tempdir().expect("mk tempdir");
        let proj = tmp.path().join("myproj");
        std::fs::create_dir_all(&proj).expect("mkdir proj");
        std::fs::create_dir_all(proj.join(".ironhermes")).expect("mkdir .ironhermes");
    
        let state = fresh_state(tmp.path());
        let mut store = SessionStore::new(state.clone());
        let ws = Arc::new(synthetic_workspace(proj.clone()));
        store.set_workspace(ws.clone());
    
        let key = SessionKey::new(Platform::Telegram, "chat-1".to_string());
        let sess = store.get_or_create(key, "test-model", "telegram");
        let session_id = sess.session_id.clone();
    
        // Read back from SQLite — the row must carry workspace_root.
        let state_locked = state.lock().expect("lock state");
        let row = state_locked
            .get_session(&session_id)
            .expect("query session")
            .expect("session row exists");
        drop(state_locked);
    
        let expected = proj.to_str().expect("UTF-8 tempdir path").to_string();
        assert_eq!(
            row.workspace_root,
            Some(expected),
            "Phase 25.3-14: gateway sessions with a resolved Workspace MUST persist \
             workspace_root onto the sessions row. Without this, /sessions --workspace \
             and Phase 25.4 Curator are starved on the primary user-facing surface."
        );
    }
    
    #[test]
    fn gateway_session_without_workspace_persists_null() {
        let tmp = tempfile::tempdir().expect("mk tempdir");
        let state = fresh_state(tmp.path());
        let mut store = SessionStore::new(state.clone());

            "Phase 25.3-14: gateway sessions with no resolved Workspace MUST persist \
