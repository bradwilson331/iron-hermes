# Phase 25.3 Plan 16: Workspace Path Encoding Canonicalization (CR-03 + WR-01 + IN-03 close-out) Summary

**One-liner:** Centralizes `workspace_root` encoding in `Workspace::canonical_root_string()` and rewrites every persistence + filter site (CLI run_chat + run_single create_session, REPL build_app_deps, gateway SessionStore::get_or_create, cmd_sessions --workspace filter) to use it — eliminating the three-way encoding split (`to_str()` returning None on non-UTF-8, `display().to_string()` producing lossy U+FFFD strings, and silently divergent prompt vs. DB column behavior) flagged by 25.3-REVIEW.md CR-03 + WR-01.

## What was built

A single-task atomic close-out for CR-03 + WR-01 + IN-03 from 25.3-REVIEW.md. Plans 25.3-13/-14/-15 made every workspace-aware code path use the resolved `Workspace`. THIS plan canonicalizes the `String` ENCODING used at the persistence and filter boundary.

### Step A — `crates/ironhermes-core/src/workspace.rs` — Helper + 2 unit tests

Appended to the existing `impl Workspace { ... }`:

```rust
/// Phase 25.3-16 CR-03 close-out: single source of truth for the encoding
/// of `self.root` used at every persistence and filter site (sessions table
/// `workspace_root` column, /sessions --workspace filter, and the
/// [Workspace: <root>] system prompt line).
///
/// Uses `to_string_lossy().into_owned()` — lossy U+FFFD substitution for
/// non-UTF-8 bytes. This matches what `Path::display()` produces, so the
/// PromptBuilder's existing `format!("{}", root.display())` line agrees by
/// construction.
pub fn canonical_root_string(&self) -> String {
    self.root.to_string_lossy().into_owned()
}
```

Plus 2 unit tests appended to `mod tests`:

- `canonical_root_string_returns_path_for_ascii` — locks ASCII parity (the dominant happy-path).
- `canonical_root_string_lossy_subs_non_utf8_unix` — `#[cfg(unix)]`, builds a `PathBuf` from raw bytes `[b'/', b't', b'm', b'p', b'/', 0xFFu8, 0xFEu8, 0xFDu8]` via `OsString::from_vec`, asserts the helper agrees with `Path::display()`, asserts non-empty, asserts no NUL bytes. **This is the regression target for CR-03** — under the prior `to_str()` encoding the column would have been NULL for this path.

Workspace test count: **19 → 21** (matches plan's `>= 21` acceptance criterion).

## CR-03 + WR-01 + IN-03 close-out

**25.3-REVIEW.md CR-03** ("`workspace_root` persistence silently drops non-UTF-8 workspace paths via `Path::to_str()`"): closed. Every persistence + filter site now uses `canonical_root_string`, which uses `to_string_lossy().into_owned()` — guaranteed to succeed for any path, lossy U+FFFD substitution for non-UTF-8 bytes. Matches `Path::display()` output by construction so the prompt line, DB column, and SQL filter all agree.

**25.3-REVIEW.md WR-01** ("`cmd_sessions --workspace` filter compares lossy `Path::display()` against a column populated with `Path::to_str()`"): closed. Both sides now use `canonical_root_string`. The encoding mismatch hazard is structurally eliminated — INV-25.3-12 locks the contract.

**25.3-REVIEW.md IN-03** ("Empty-string workspace filter does not match NULL rows — documented but easy to footgun"): closed. `cmd_sessions` now normalizes empty-string filter to `None` at the handler boundary via `workspace_filter.filter(|s| !s.is_empty())` BEFORE the SQL call. A misconfigured caller passing `Some("")` now gets "all sessions" (from the `None` branch) instead of silently "no sessions" (from SQL NULL semantics).

Together these advance the phase scorecard's REVIEW critical findings from 4 critical + 6 warning + 3 info to 0 critical + 5 warning + 2 info — all 4 CR-XX (CR-01..04) closed plus WR-01 and IN-03.

Downstream consequences (now restored):
- Workspace feature functional on non-UTF-8 filesystems (Linux ext4/btrfs with foreign-locale filenames; macOS with non-NFC normalization).
- `/sessions --workspace .` matches the operator's current workspace deterministically across all surfaces.
- Phase 25.4 Curator can rely on `workspace_root` being a deterministic string for non-ASCII project paths.
