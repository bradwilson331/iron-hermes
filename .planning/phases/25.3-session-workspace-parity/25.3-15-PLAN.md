<objective>
Close CR-02: gateway opens ONE process-wide `TrajectoryWriter` at `<workspace>/.ironhermes/sessions/gateway-<random-uuid>/trajectories.jsonl` (main.rs:2309-2330). Per-message `GatewayMessageHandler::run_agent` builds a `session_id_str = format!("gw:{}:{}", chat_id, sender_id)` (handler.rs:679) and passes that to AgentLoop telemetry, but the trajectory writer it shares is on the `gateway-<uuid>` path — NOT on the canonical SQLite session UUID, NOT on `gw:<chat_id>:<sender_id>`, NOT on anything that downstream consumers can find.

Why this matters (from 25.3-REVIEW.md CR-02):
1. **All Telegram users' tool calls land in a single shared file** — chat_id boundaries that other layers maintain (rate-limiter, session store) are erased in the trajectory ledger.
2. **`session_cmd::resolve_trajectory_source` and `SessionDirectoryExport::write` both look up trajectories at `<root>/sessions/<session_id>/trajectories.jsonl`** where `<session_id>` is the canonical SQLite session UUID. For gateway sessions the file is at `gateway-<uuid>` — the per-session lookup fails silently and `hermes session export` produces a 4-file directory with NO `trajectories.jsonl`.
3. **Phase 25.4 Curator's per-session weighting is starved** — the Curator reads trajectories keyed by session UUID; gateway sessions have empty trajectory streams visible at the canonical path.

Per CR-02 recommendation (Option A — preferred): move `TrajectoryWriter::open(...)` into `run_agent` (handler.rs) using the canonical `session_id` retrieved from `SessionStore::get_or_create`. Cache writers in `SessionStore` so the same chat reuses one writer across messages instead of reopening per call.

Purpose: Restore D-T-3 cross-surface parity for the gateway. Without this fix, the trajectory ledger is unreachable from the canonical session-export path on the primary user-facing surface — defeating the whole point of D-T-3.

Output: Per-Telegram-session trajectory files at `<trajectory_root>/sessions/<canonical_session_uuid>/trajectories.jsonl`, lazily opened on first tool call per session, cached in SessionStore for subsequent messages.
</objective>
