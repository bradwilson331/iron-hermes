# Phase 25.3 Plan 14: SessionStore workspace threading (verifier BLOCKER close-out) Summary

**One-liner:** Threads the resolved `Workspace` from `GatewayRunner` into `SessionStore` so per-message Telegram sessions persist `workspace_root` onto SQLite, closing the verifier-flagged BLOCKER and turning INV-25.3-09 + INV-25.3-11 GREEN.

## What was built

A two-task atomic close-out for the verifier-flagged gateway gap (must-have #28 in 25.3-VERIFICATION.md, WR-02 in 25.3-REVIEW.md):

### Task 1 â€” Production fix (commit `0b2394c`)

**`crates/ironhermes-gateway/src/session.rs`**

- Added `workspace: Option<Arc<ironhermes_core::workspace::Workspace>>` field to `SessionStore`.
- Added `pub fn set_workspace(&mut self, workspace: Arc<Workspace>)` setter.
- Removed the literal `// workspace_root: Plan 0 placeholder â€” Plan 8 wires resolved workspace` inline trailing comment.
- `SessionStore::get_or_create` now reads `self.workspace.as_ref().and_then(|ws| ws.root.to_str())` into a local `workspace_root: Option<&str>` and passes it to `state.create_session(...)`.

**`crates/ironhermes-gateway/src/runner.rs`**

- `GatewayRunner::set_workspace` now also propagates the same `Arc<Workspace>` into the inner `SessionStore` via `self.session_store.try_write()`. On `try_write` failure (impossible during normal setup), emits `tracing::warn!` rather than panicking â€” surfaces a future contention bug without crashing the gateway.

**`crates/ironhermes-cli/tests/invariants_25_3.rs`**

- Removed `#[ignore]` from `invariant_25_3_09_gateway_session_store_passes_workspace_root` and `invariant_25_3_11_session_store_get_or_create_accepts_workspace_root`. Both are now GREEN on default `cargo test` runs.

### Task 2 â€” SQLite round-trip regression test (commit `29c6e41`)

**`crates/ironhermes-gateway/tests/session_workspace_root.rs`** (new file)

Two integration tests that exercise the actual SQLite round-trip:

| Test | Asserts |
|------|---------|
| `gateway_session_with_workspace_persists_workspace_root` | After `SessionStore::set_workspace(<some-arc>)` + `get_or_create(...)`, `state.get_session(<id>).workspace_root == Some(<root-as-str>)` |
| `gateway_session_without_workspace_persists_null` | Without `set_workspace`, the inserted row has `workspace_root == None` (global-mode behavior) |

Why this is needed in addition to INV-25.3-09 + INV-25.3-11:
- The parity-guards lock the SOURCE-TEXT SHAPE â€” they grep `gateway/session.rs` for the `workspace_root` token and the absence of `Plan 0 placeholder`. They cannot detect a behavioral regression where the code shape is correct but the value silently fails to round-trip into SQLite.
- This integration test exercises `SessionStore -> StateStore -> SQLite -> StateStore::get_session` end-to-end and asserts the actual column value.
