# Phase 25.3 Plan 13: REPL create_session + with_workspace_root (CR-01 + CR-04) Summary

**One-liner:** Wire the missing `state_store.create_session(...)` and `prompt_builder.with_workspace_root(&ws.root)` calls into `tui_rata/event_loop.rs::build_app_deps` so the default `hermes chat` REPL surface persists a sessions row with workspace_root AND injects the durable `[Workspace: <root>]` Identity-slot line — closing CR-01 and CR-04 from `25.3-REVIEW.md`.

## What was built

A single-task plan that lands two functionally-related fixes in the same `build_app_deps` function (CR-01 and CR-04 share state — the resolved `workspace`, the StateStore, the `session_id`), plus the small `AppDeps`/`App::new` plumbing required to deliver the pre-built system message to the per-turn AgentLoop.

### Three concrete code sites modified

#### 1. `crates/ironhermes-cli/src/tui_rata/event_loop.rs` — Edit 1: fail-loud StateStore + create_session (CR-01)

Replaced the prior silent-swallowing block:

```rust
let state_store = ironhermes_state::StateStore::open_default()
    .ok()
    .map(|s| Arc::new(std::sync::Mutex::new(s)));
```

with a `match` that calls `s.create_session(&session_id, "cli-repl", Some(client.model()), None, None, ws_root_str)` immediately after a successful open. Both error paths (StateStore::open failure AND create_session failure) now log via `tracing::warn` instead of being silently swallowed.

`ws_root_str` is computed as `workspace.as_ref().and_then(|ws| ws.root.to_str())` — verbatim shape required by the plan acceptance criterion and by INV-25.3-07's RED-state hint message.

#### 2. `crates/ironhermes-cli/src/tui_rata/event_loop.rs` — Edit 2: PromptBuilder + with_workspace_root (CR-04)

Inserted directly after Edit 1 (so `workspace`, `client`, `skill_registry`, `memory_manager`, and `config` are already in scope):

```rust
let system_message: Option<ironhermes_core::types::ChatMessage> = {
    let mut prompt_builder = ironhermes_agent::prompt_builder::PromptBuilder::new(
        client.model(),
        "cli-repl",
    );
    if let Some(ref ws) = workspace {
        prompt_builder = prompt_builder.with_workspace_root(&ws.root);
    }
    prompt_builder.set_skill_registry(skill_registry.clone());
    if let Some(ref mgr) = memory_manager {
        prompt_builder.set_memory_manager(mgr.clone());
    }
    prompt_builder.set_user_profile_enabled(config.memory.user_profile_enabled);
    prompt_builder.load_memory().await;
    prompt_builder.load_skills();
    Some(prompt_builder.build_system_message())
};
```

This mirrors `run_chat` field-for-field except for `source = "cli-repl"`. The `with_workspace_root` call is gated on `workspace.is_some()` so global/no-workspace sessions don't get a phantom `[Workspace: ]` line.

#### 3. `crates/ironhermes-cli/src/tui_rata/event_loop.rs` — Edit 3a: thread system_message into AppDeps return

Appended `system_message,` to the `AppDeps { ... }` literal at the end of `build_app_deps`.

#### 4. `crates/ironhermes-cli/src/tui_rata/app.rs` — Edit 3b: AppDeps field + App::new seed + test_deps default

Three sub-edits in the same file:

- Added `pub system_message: Option<ChatMessage>` to the `AppDeps` struct (with doc comment explaining the seed contract).
- Modified `App::new` to seed `history: Vec<ChatMessage>` from `deps.system_message` before the `Self { ... }` literal:

  ```rust
  let mut history: Vec<ChatMessage> = Vec::new();
  if let Some(sys) = deps.system_message {
      history.push(sys);
  }
  ```

  and replaced `history: Vec::new(),` with `history,` in the literal.

- Added `system_message: None` to the `#[cfg(feature = "test-support")] fn test_deps()` constructor so existing tests still compile.

#### 5. `crates/ironhermes-cli/tests/invariants_25_3.rs` — Edit 4: un-ignore INV-25.3-07 + INV-25.3-08

Removed the two `#[ignore = "Phase 25.3 Plan NN ..."]` attributes from the parity-guard tests that this plan turns GREEN. The remaining three `#[ignore]` tests (INV-25.3-09, -10, -11) stay gated for plans 14 and 16.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 — Critical correctness] Replaced silent `.ok()` swallow with tracing::warn**

- **Found during:** Edit 1 implementation.
- **Issue:** The plan's Edit 1 instruction matched the prior silent-swallow pattern but called for a fail-loud version. The original code at lines 402-406 swallowed both StateStore::open errors AND would have swallowed create_session errors had we just chained them. This would have hidden CR-01 (and any future CR-01-class regression) from operators.
- **Fix:** Used a `match` on `StateStore::open_default()` with explicit `Ok(mut s) => { ... }` and `Err(e) => { tracing::warn!(...); None }` arms, mirroring the plan's instruction verbatim. Both error paths log via `tracing::warn`. None state_store falls back gracefully (slash commands like /sessions report "session storage not configured" rather than panicking).
- **Files modified:** `crates/ironhermes-cli/src/tui_rata/event_loop.rs`
- **Why this is Rule 2 not Rule 4:** The fix is structurally identical to what the plan body specified; "fail-loud" is a correctness requirement (operators need observability into session-persistence failures), not an architectural choice. The change is local to the same code block the plan rewrites.

The plan executed exactly as written for the three edits, the AppDeps wireup, the App::new seeding, the test_deps backfill, and the un-ignore of INV-25.3-07 + INV-25.3-08.
