## Decisions Made

- **Testable-seam free function over `#[cfg(test)]` constructor**: The plan offered two paths for testing the WebExtractTool override — either build a test-only `WebExtractTool::for_redact_test()` constructor with stub deps, or extract the redaction logic into a `pub(crate)` free function. Chose the free function because (1) `redact_args` reads zero fields of `self`, so the dependency injection is pure ceremony; (2) the existing test mod in web_extract.rs deliberately avoids constructing `WebExtractTool` (it tests `process_one_url` behavior at the sub-function level — see `process_one_url_redacts_secret_in_error_envelope`); (3) the plan explicitly blessed this fallback as "acceptable per Phase 24 Plan 03's `run_memory_setup_with_io` precedent."
- **Trait method placement immediately after `prerequisites()` default**: Mirrors the existing pattern at registry.rs:47 — keeps the default-method block contiguous and parsing-stable for future extensions.
- **Doc-comment trace: "Phase 25.3 D-T-1 / Discretion D-2"**: Provenance line so future readers can find the CONTEXT.md decision without grep gymnastics.
