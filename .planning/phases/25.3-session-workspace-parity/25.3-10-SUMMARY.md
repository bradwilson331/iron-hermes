## Decisions Made

- **Single SQL with NULL-tolerant binds rather than 4 if/else branches.** The pattern `WHERE (?1 IS NULL OR col = ?1)` is exactly as injection-safe as a fixed query (rusqlite binds positionally) and reduces the prepared-statement cache pressure to one statement instead of four. Mirrors the dynamic-WHERE style already proven in `search_messages` (lib.rs:527+) but simpler because there are only two filter axes and both can be expressed as nullable binds.
- **list_sessions becomes a delegate, NOT a parallel implementation.** Approach A was plan-preferred and chosen verbatim. The alternative (rewrite both) would have duplicated the 5-line SELECT projection and increased the regression surface for the 24 existing call sites Plan 0 fixed up.
- **list_sessions_text on the adapter delegates to list_sessions_text_filtered too.** This keeps the format string ("Recent sessions:" / "No sessions found.") in ONE place. Two adapters (production + test) duplicate the body for cross-crate visibility reasons (the production adapter is private to ironhermes-cli) but each adapter's two methods share a single format path.
- **Helpful error message names both fixes.** When bare `--workspace` is used with `ctx.workspace=None`, the operator gets: `"No workspace resolved at cwd. Pass --workspace <path> explicitly, or run /sessions from a directory under a .ironhermes/ or .hermes/ marker."` Both escape hatches surfaced — operator never wonders why the filter resolved to nothing.
- **Test-side FakeStateStore records LAST call only.** Each test does exactly one dispatch, so `Mutex<Option<...>>` is sufficient. Avoided `Vec<...>` accumulator + assertion-on-length complexity. Mirrors FakeToolsetSession's `AtomicUsize` counters for the same locality reason.
