# Phase 25.3 Plan 02: Workspace newtype + resolve_from_cwd Summary

**One-liner:** Pure-logic D-W-1 landing — `Workspace { root, soul_path, agents_chain, memory_dir, skills_dir, tools_config }` newtype + cwd walk-up resolution helper in `ironhermes-core`, with 13 unit tests covering marker preference, soul-path precedence, skills_dir top-level placement, and filesystem-root termination. Plan 8 will wire the call sites; this plan ships only the type and the resolver.

## What Shipped

### `crates/ironhermes-core/src/workspace.rs` (NEW, 232 lines)

Public surface:
- `pub struct Workspace` with the six D-W-1 fields (`root`, `soul_path`, `agents_chain`, `memory_dir`, `skills_dir`, `tools_config`) — derives `Debug, Clone, PartialEq, Eq, Serialize, Deserialize`.
- `pub fn resolve_from_cwd(cwd: &Path) -> Option<Workspace>` — walk-up resolver.

Private surface:
- `Workspace::from_root_and_marker(root, marker_name)` constructor — used by `resolve_from_cwd` and indirectly exercised by every test.

Resolution algorithm:
1. Inspect `<current>/.ironhermes/` (preferred) and `<current>/.hermes/` (compat) at each ancestor.
2. First marker found wins; `.ironhermes/` precedence locked at the same node by `ironhermes_wins_when_both_markers_present`.
3. `current.pop()` walks to parent; loop terminates when `pop()` returns false at filesystem root → `None`.
4. Caller (Plan 8) interprets `None` as "use `~/.ironhermes/` global fallback".

### `crates/ironhermes-core/src/lib.rs` (MODIFIED, +9 lines)

- Added `pub mod workspace;` to module declarations (alphabetically appended after `pub mod types`).
- Added re-export block: `pub use workspace::{Workspace, resolve_from_cwd as resolve_workspace_from_cwd};` with doc comment explaining the alias rationale.
