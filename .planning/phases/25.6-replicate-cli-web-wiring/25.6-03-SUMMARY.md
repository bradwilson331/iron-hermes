# Phase 25.6 Plan 03 Summary

Rewired web runtime startup and handlers around a startup-scoped shared `AppState` backed by the shared runtime factory.

- `AppState::init` builds one shared `AppRuntimeBundle` and stores app-scoped runtime handles for server use.
- Server startup initializes and installs global app state once in `crates/iron_hermes_ui/src/main.rs`.
- WebSocket handler (`ws_chat`) uses injected global app state and no longer performs per-connection `AppState::init()`.
- API handlers now return real data from runtime/state-store sources: web sessions, command metadata, and tool definitions.
- Added `crates/iron_hermes_ui/tests/server_runtime_parity.rs` to lock startup-scope and parity invariants.

Verification passed with server runtime parity tests under the `server` feature.
