<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser WebSocket client → server turn execution | Untrusted user input reaches AgentLoop/tool dispatch path |
| HTTP API caller → session/tool listing | Untrusted requests access persistent state and runtime metadata |
| StateStore persistence → multi-surface session namespace | Namespace mixups can leak/merge data between surfaces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25.6-03-01 | S | web session identity | mitigate | Enforce Platform::Web session key creation and persistence path in API/WS flow (D-12) with tests. |
| T-25.6-03-02 | T | WS lifecycle wiring | mitigate | Remove per-connection AppState re-init and lock startup-injected state path via regression tests (D-07). |
| T-25.6-03-03 | I | API tool/session responses | mitigate | Source command/session/tool data from AppState-backed runtime/state store, not mock/stub placeholders. |
| T-25.6-03-04 | E | tool exposure drift | mitigate | Validate web runtime exposes CLI-parity high-level tool surface through factory-backed registry (D-14). |
</threat_model>
