## Fixed Issues

### WR-01: Cached `reserved` row count desyncs from runtime widget mutations

**Files modified:** `crates/ironhermes-cli/src/tui/render.rs`
**Commit:** b538dd4
**Applied fix:** Replaced the cached `u16` field `TuiHandle.reserved` with an `Arc<AtomicU16>` shared between the TuiHandle and the render loop. The render loop now stores the current `reserved_rows(&widgets)` value into the atomic on every tick (after processing widget add/remove events). All read sites (`reserved_row_count()`, `cleanup_on_exit()`, `shutdown()`) now load from the atomic with `Relaxed` ordering. This ensures callers of `prepare_prompt_with_reserve` / `finish_prompt_with_reserve` always see the current reserved row count, even after runtime `UpdateWidget` or `RemoveWidget` events change the widget set.

### WR-02: Pre-readline keybinding poll silently eats non-matching key events

**Files modified:** `crates/ironhermes-cli/src/main.rs`
**Commit:** d8fdb9e
**Applied fix:** Added a guard `let has_idle_bindings = !keybinding_registry.help_entries().is_empty()` before the `crossterm::event::poll` call. The poll (and potential event consumption) is now skipped entirely when no extensions have registered any keybindings. This eliminates the risk of consuming and losing user key events in the default zero-extension configuration. When extensions do register bindings, the existing behavior (poll + consume + match) is preserved.

### WR-03: `AboveStatus` widget rendering can underflow `row_cursor` to 0 and overwrite content

**Files modified:** `crates/ironhermes-cli/src/tui/render.rs`
**Commit:** 039eab9
**Applied fix:** Added two `row_cursor == 0` break guards in the AboveStatus widget rendering loop: one at the outer widget iteration level (breaks before processing a widget if no room remains) and one at the inner line iteration level (breaks before rendering a line that would land on row 0). This prevents AboveStatus widgets from overwriting the scrollable content area at the top of the terminal, regardless of how many widgets or how much content they contain. The previous code only prevented underflow past 0 but still allowed writing to row 0 itself.

### WR-04: `status.hint` style override is accepted but silently dropped

**Files modified:** `crates/ironhermes-cli/src/tui/render.rs`
**Commit:** 538f1c7
**Applied fix:** Replaced the silent `let _ = color;` drop with a `tracing::debug!` log message that explicitly states the `status.hint` style slot override was accepted but is not yet applied as a render-time color override. This gives extension authors diagnostic feedback (visible when running with `RUST_LOG=debug`) instead of silently discarding their configuration. The log message notes this is a future enhancement.
