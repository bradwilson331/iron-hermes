## Extension Architecture

| Option | Description | Selected |
|--------|-------------|----------|
| Trait object | Define TuiExtension trait with methods mirroring 5 Python hooks. Extensions implement trait; TuiHandle holds Vec<Box<dyn TuiExtension>>. | |
| Callback closures | Register individual closure callbacks for each hook point. More flexible for one-off customizations. | |
| Hybrid (user-proposed) | Trait objects for static contract + message bus (channels) for dynamic contract + command registry with extension-first priority. | ✓ |

**User's choice:** Hybrid architecture — user provided a detailed three-layer design before the question was presented.
**Notes:** User specified: (1) Trait objects for static contract (widgets, layout, help), (2) Message bus via channels for dynamic updates (TuiEvent::UpdateWidget), (3) Command registry ensuring extensions checked before core slash commands.

### Follow-up: Channel type

| Option | Description | Selected |
|--------|-------------|----------|
| Reuse existing pattern | Add TuiEvent enum + mpsc channel on TuiHandle, consistent with watch::channel pattern in render.rs | ✓ |
| Dedicated EventBus type | Standalone EventBus struct, more decoupled, supports fan-out | |

**User's choice:** Reuse existing pattern

## Widget & Layout Model

| Option | Description | Selected |
|--------|-------------|----------|
| Slot-based | Named slots (AboveStatus, BelowScanner, StatusRight). DECSTBM adjusts dynamically. | ✓ |
| Free-form row reservation | Extensions request N rows, stacked in registration order. | |
| Status line pills only | Fixed 3-row layout, extensions only add status pills. | |

**User's choice:** Slot-based layout with named slots

### Follow-up: Widget abstraction

| Option | Description | Selected |
|--------|-------------|----------|
| String-based | Widget { id, content (ANSI), height }. Extensions produce rendered strings. | ✓ |
| Render trait | Widget is a trait with fn render(&self, width) -> String. | |

**User's choice:** String-based Widget struct

---

## Keybinding Registration

| Option | Description | Selected |
|--------|-------------|----------|
| Idle-only dispatch | Pre-readline key check with KeyContext (Idle/InFlight/Always). No conflict with rustyline. | ✓ |
| Rustyline custom helper | Use Helper trait to intercept keys inside editor. Couples to rustyline. | |

**User's choice:** Idle-only dispatch with KeyContext enum

---

## Command Processing Scope

| Option | Description | Selected |
|--------|-------------|----------|
| Hook point only | Create process_command() hook + dispatch chain. Core fallback is stub until SKILL-12/13/14. | ✓ |
| Full command router | Build complete slash command system now (SKILL-12/13/14). Expands scope. | |

**User's choice:** Hook point only — extensions can register commands now, built-in commands come later.
