# Phase 22.1 Plan 02: TUI Extension Render + REPL Wiring Summary

**One-liner:** Extension type contracts wired into the render loop (TuiEvent mpsc channel, widget slot compositing, dynamic DECSTBM, merged style overrides) and REPL loop (extension-first command dispatch, pre-readline keybinding check), with zero-extension path identical to Phase 21.

## What Was Built

**render.rs** (major extension, +535 net lines):
- `TuiHandle::new_with_extensions(initial_status, extensions)`: collects initial widgets (prefixed IDs), merges style overrides (catch_unwind), creates mpsc event channel, sets dynamic DECSTBM, spawns `render_loop_with_extensions`
- `TuiHandle::new()` delegates to `new_with_extensions(_, Vec::new())` — zero-extension backward compat
- `event_sender() -> Option<mpsc::UnboundedSender<TuiEvent>>`: runtime widget update channel
- `reserved_row_count() -> u16`: cached reserved rows for prompt positioning
- `extensions() -> &[Box<dyn TuiExtension>]`: slice of registered extensions
- `reserved_rows(widgets)`: dynamic DECSTBM calculation (base 3 + widget heights for non-StatusRight slots)
- `merge_style_overrides(extensions)`: last-wins merge with catch_unwind per extension
- `prepare_prompt_with_reserve(reserved)` / `finish_prompt_with_reserve(reserved)`: dynamic prompt positioning
- Backward-compat wrappers `prepare_prompt()`, `finish_prompt()`, `reset_scroll_region()` call with `reserved=3`
- `render_loop`: accepts `UnboundedReceiver<TuiEvent>`, widget HashMap, StyleOverrides; drains events via `try_recv` loop each tick
- `TuiEvent::UpdateWidget` — update widget content/height in HashMap
- `TuiEvent::RemoveWidget` — remove widget, recalculate DECSTBM
- `TuiEvent::FlashHint` — store flash message with countdown ticker
- `redraw_with_extensions`: composites AboveStatus/BelowScanner/StatusRight widgets, applies style override slots (`scanner.lit`, `scanner.trail`, `scanner.bg`, `status.separator`, `status.hint`)
- `build_scanner_frame`: applies color overrides; delegates to `knight_rider::frame()` for default colors (exact Phase 21 fidelity)
- 14 tests: 5 existing (preserved) + 9 new (reserved_rows, merge_style_overrides, event_sender, construct_with_empty_extensions)

**main.rs** (REPL wiring, +121/-26 lines):
- `TuiHandle::new_with_extensions` replaces `TuiHandle::new` — empty extensions vec for now
- `KeybindingRegistry::register_from_extensions(tui.extensions())` built at REPL start
- `prepare_prompt_with_reserve(tui.reserved_row_count())` / `finish_prompt_with_reserve(tui.reserved_row_count())` replace static calls
- Pre-readline keybinding check: `crossterm::event::poll(Duration::ZERO)` + `match_key(&key_event, &KeyContext::Idle)`
- Extension-first command dispatch replaces inline match: `dispatch_command(tui.extensions(), cmd, args)` with `CommandResult::Handled/Silent/Error` arms
- Side effects preserved: quit breaks loop, clear truncates messages, status calls `cmd_status()`
- `print_help()` delegates to `crate::tui::commands::format_help(&[], None)`
- `tui_extension_wiring_tests` module: 5 static-grep invariant tests (INV-22.1-01..05)

**mod.rs**: Added `prepare_prompt_with_reserve` and `finish_prompt_with_reserve` to re-exports.
