1. **`dispatch_command` function** (per D-06):
   ```rust
   pub fn dispatch_command(
       extensions: &[Box<dyn TuiExtension>],
       cmd: &str,
       args: &[&str],
   ) -> CommandResult
   ```
   Extension-first dispatch chain: iterate extensions in registration order, call `ext.process_command(cmd, args)`. If any returns `Some(result)`, return that result immediately. If all return `None`, fall through to core stub:
   - `"quit" | "exit" | "q"` -> `CommandResult::Handled("Goodbye!".to_string())`
   - `"clear"` -> `CommandResult::Handled("Conversation cleared.".to_string())`
   - `"status"` -> `CommandResult::Handled("Status...".to_string())`
   - `"help"` -> `CommandResult::Handled(format_help(extensions, keybinding_registry))` (pass None for keybinding_registry in this stub; Plan 02 wires the real one)
   - `_` -> `CommandResult::Error(format!("Unknown command: /{}. Type /help for available commands.", cmd))`

   Wrap each `ext.process_command()` call in `std::panic::catch_unwind()` — if an extension panics, log a warning via `tracing::warn!("tui: extension '{}' panicked in process_command() -- skipping", ext.name())` and continue to the next extension (security: extension panic containment per Pitfall 3). Since catch_unwind requires UnwindSafe, use `std::panic::AssertUnwindSafe` wrapper around the closure.

2. **`format_help` function:**

   pub fn format_help(

   - If keybinding_registry is Some, append `"\nKeybindings:"` header followed by `"  {key_display}  {description}  ({context})"` for each registered binding

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extension -> TuiHandle | Extension code (compiled-in, not dynamically loaded) produces widget content and command results consumed by the render loop |
| User input -> command dispatch | User-typed `/` commands are parsed and dispatched to extensions then core |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22.1-01 | Denial of Service | Widget content rendering | mitigate | Cap widget height at MAX_WIDGET_HEIGHT=10; Widget::truncate_content() truncates lines to terminal width; implemented in extension.rs Widget struct |
| T-22.1-02 | Tampering | Command output | accept | Extensions are compiled-in (not dynamically loaded), so command output is trusted. ANSI escape sequences in widget content are expected and intentional (colored crate). Low risk since there is no dynamic plugin loading. |
| T-22.1-03 | Denial of Service | Extension panic | mitigate | Wrap all extension trait method calls in std::panic::catch_unwind() at dispatch boundary; log warning and skip faulting extension. Implemented in commands.rs dispatch_command |
</threat_model>
