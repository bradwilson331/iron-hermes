## What Was Built

Extended `crates/ironhermes-tools/src/hexapod_tcp.rs` with:

**New constants (5):**
- `ROTATE_MS_PER_DEGREE = 20` — calibration placeholder for rotate duration
- `HEAD_PAN_MAX = 90` — protocol max for head pan servo
- `HEAD_TILT_MAX = 90` — protocol max for head tilt servo
- `BUZZER_ON_CMD = "CMD_BUZZER#1\n"` — buzzer on wire string
- `BUZZER_OFF_CMD = "CMD_BUZZER#0\n"` — buzzer off wire string

**New pure helper function:**
- `build_rotate_wire(degrees: i64) -> String` — maps positive degrees → angle +10, negative → angle -10, formats `CMD_MOVE#1#0#0#5#{angle}\n`

**Updated DESCRIPTION const:** Lists all 10 actions including new rotate/head/buzzer actions; clarifies degrees (rotate only) and angle (head_pan/head_tilt only) params.

**Updated outer allowlist match arm:** Now covers all 10 actions — `"walk" | "stop" | "read_battery" | "read_distance" | "relax_servos" | "rotate" | "head_pan" | "head_tilt" | "buzzer_on" | "buzzer_off"`.

**New inner dispatch arms (5):**
- `rotate`: builds wire with `build_rotate_wire`, sleeps `degrees.unsigned_abs() * 20ms` via `tokio::time::sleep`, best-effort stop after
- `head_pan`: clamps angle to `[-90, 90]`, sends `CMD_HEAD#0#{angle}\n`
- `head_tilt`: clamps angle to `[-90, 90]`, sends `CMD_HEAD#1#{angle}\n`
- `buzzer_on`: sends `BUZZER_ON_CMD`
- `buzzer_off`: sends `BUZZER_OFF_CMD`

**Updated schema:** `action` enum now has 10 values; new `degrees` (integer) and `angle` (integer, min -90, max 90) properties declared.

**New tests (7):**
- Test 14: `test_rotate_wire_positive_degrees` — build_rotate_wire(90) → `CMD_MOVE#1#0#0#5#10\n`
- Test 15: `test_rotate_wire_negative_degrees` — build_rotate_wire(-45) → `CMD_MOVE#1#0#0#5#-10\n`
- Test 16: `test_rotate_wire_zero_degrees` — build_rotate_wire(0) → `CMD_MOVE#1#0#0#5#10\n` (zero is non-negative)
- Test 17: `test_head_pan_clamps_high` — 180 clamped to 90, wire = `CMD_HEAD#0#90\n`
- Test 18: `test_head_tilt_clamps_low` — -180 clamped to -90, wire = `CMD_HEAD#1#-90\n`
- Test 19: `test_buzzer_wire_strings` — both BUZZER_ON_CMD and BUZZER_OFF_CMD correct
- Test 20: `test_new_actions_not_blocked` — all 5 new actions pass outer allowlist (return env-var error, not blocked string)
