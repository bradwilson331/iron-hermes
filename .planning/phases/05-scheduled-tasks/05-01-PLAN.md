}

#[serde(rename_all = "snake_case")]
pub enum JobState { Scheduled, Paused, Completed }

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RepeatConfig {

}

pub struct JobOrigin {

Create `crates/ironhermes-cron/src/parser.rs` with (per D-01):

- `pub fn parse_schedule(input: &str) -> Result<ScheduleParsed>` — rule-based parser:
  1. If starts with "every " -> strip prefix, parse_duration on remainder -> Interval
  2. If 5+ whitespace-separated fields all matching `[\d\*\-,/]+` -> validate with cron crate Schedule::from_str (prepend "0 " for 5-field) -> Cron
  3. If contains 'T' or looks like ISO date (starts with 4 digits and len >= 10) -> parse RFC3339, fallback NaiveDateTime::parse_from_str with "%Y-%m-%dT%H:%M:%S" -> Once
  4. Otherwise try parse_duration -> Once with run_at = Utc::now() + duration

- `pub fn parse_duration(input: &str) -> Result<u32>` — regex `^(\d+)\s*(m|min|mins|minute|minutes|h|hr|hrs|hour|hours|d|day|days)$` -> minutes

- `pub fn compute_next_run(schedule: &ScheduleParsed, after: DateTime<Utc>) -> Result<Option<DateTime<Utc>>>` — replaces the old string-based compute_next_run:
  - Once: if run_at > after, return Some(run_at), else None
  - Interval: return Some(after + Duration::minutes(minutes))
  - Cron: normalize to 6-field, Schedule::from_str, .after(&after).next()

Add `regex` as a direct dependency to `crates/ironhermes-cron/Cargo.toml` (it is already a workspace dep).

Write tests in `crates/ironhermes-cron/src/parser.rs` in a `#[cfg(test)] mod tests` block covering all behavior items above.
