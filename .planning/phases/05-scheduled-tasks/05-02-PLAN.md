Implement `Tool` trait:
- `fn name(&self) -> &str { "cronjob" }`
- `fn toolset(&self) -> &str { "cronjob" }`
- `fn description(&self) -> &str` — return CRONJOB_DESCRIPTION const string: "Manage scheduled tasks. Actions: create, list, get, update, pause, resume, run, remove."
- `fn schema(&self) -> ToolSchema` — define JSON schema with required `action` parameter (enum of 8 actions), and optional parameters: `name` (string), `schedule` (string), `prompt` (string), `job_id` (string), `deliver` (string, default "local"), `skills` (array of strings). Match the Python reference schema from `cronjob_tools.py`.
- `async fn execute(&self, args: Value) -> Result<String>` — extract `action` string, match and dispatch:

Action handlers (all lock `self.store` via `.lock().unwrap()`):
- `handle_create(store, args)`: Extract name, schedule, prompt (required). Optional: deliver (default "local"), skills (default []), origin args. Call `parse_schedule(schedule_str)` to get ScheduleParsed. Call `scan_cron_prompt(&prompt)` — if Err, return error JSON. Call `store.add_job(...)`. Return `{"status":"created","job":{job_to_json(job)}}`.
- `handle_list(store)`: Call `store.list_jobs()`. Map each to JSON. Return `{"status":"ok","jobs":[...],"count":N}`.
- `handle_get(store, args)`: Extract job_id. Call `store.find_job(id)`. Return job JSON or error.
- `handle_update(store, args)`: Extract job_id (required). Build `JobUpdate` from optional fields (name, prompt, schedule, deliver, skills). If prompt is being updated, call `scan_cron_prompt()` first. Call `store.update_job(id, updates)`. Return `{"status":"updated","job":{...}}`.
- `handle_pause(store, args)`: Extract job_id. Call `store.toggle_job(id, false)`. Return `{"status":"paused","job_id":"..."}`.
- `handle_resume(store, args)`: Extract job_id. Call `store.toggle_job(id, true)`. Get updated job for next_run. Return `{"status":"resumed","job_id":"...","next_run":"ISO"}`.
- `handle_run(store, args)`: Extract job_id. Verify job exists. Return `{"status":"triggered","job_id":"..."}` (actual execution happens in tick runner, Plan 03).
- `handle_remove(store, args)`: Extract job_id. Call `store.remove_job(id)`. Return `{"status":"removed","job_id":"..."}`.

Helper `fn job_to_json(job: &CronJob) -> Value` — serialize job to JSON matching the response shapes from 05-UI-SPEC.md.

Create `crates/ironhermes-cron/src/scanner.rs` (per D-09):

Build a `CRON_THREAT_PATTERNS` using `regex::RegexSet` with these patterns (ported from Python `_CRON_THREAT_PATTERNS` + existing context_scanner.rs patterns):

```rust
static CRON_THREAT_PATTERNS: LazyLock<RegexSet> = LazyLock::new(|| {
    RegexSet::new(&[
        // Prompt injection
        r"(?i)ignore\s+(?:\w+\s+)*(?:previous|all|above|prior)\s+(?:\w+\s+)*instructions",
        r"(?i)do\s+not\s+tell\s+the\s+user",
        r"(?i)system\s+prompt\s+override",
        r"(?i)disregard\s+(your|all|any)\s+(instructions|rules|guidelines)",
        // Credential exfiltration
        r"(?i)curl\s+[^\n]*\$\{?\w*(KEY|TOKEN|SECRET|PASSWORD|CREDENTIAL|API)",
        r"(?i)wget\s+[^\n]*\$\{?\w*(KEY|TOKEN|SECRET|PASSWORD|CREDENTIAL|API)",
        r"(?i)cat\s+[^\n]*(\.env|credentials|\.netrc|\.pgpass)",
        // System tampering
        r"(?i)authorized_keys",
        r"(?i)/etc/sudoers|visudo",
        r"(?i)rm\s+-rf\s+/",
    ]).expect("CRON_THREAT_PATTERNS must compile")
});
```

Define `INVISIBLE_CHARS: &[char]` matching the context_scanner.rs invisible unicode set: `['\u{200B}', '\u{200C}', '\u{200D}', '\u{2060}', '\u{FEFF}', '\u{00AD}', '\u{180E}']` (read exact list from context_scanner.rs).

`pub fn scan_cron_prompt(prompt: &str) -> Result<(), String>`:
1. Check for invisible unicode: if `prompt.contains(INVISIBLE_CHARS)`, return `Err("Blocked: cron prompt contains restricted pattern -- invisible unicode characters".into())`
2. Check against `CRON_THREAT_PATTERNS`: if `patterns.is_match(prompt)`, identify which pattern matched (iterate `.matches(prompt)` to get index), return `Err(format!("Blocked: cron prompt contains restricted pattern -- {pattern_type}"))` where pattern_type describes the category (injection, exfiltration, system tampering).
3. Otherwise return `Ok(())`
