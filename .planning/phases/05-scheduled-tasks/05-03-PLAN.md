- Return the path written

`pub async fn run_tick(store: &Arc<Mutex<JobStore>>, send_fn: Option<&dyn Fn(&str, &str, Option<&str>) -> futures::future::BoxFuture<'_, Result<()>>>) -> Result<TickResult>`:

Actually, since the tick runner needs the agent loop to execute job prompts, and that is a complex integration, simplify for this plan:

`pub struct TickResult { pub jobs_checked: usize, pub jobs_run: usize, pub jobs_skipped: usize }`

`pub async fn run_tick_check(store: &Arc<Mutex<JobStore>>) -> Result<(Vec<CronJob>, TickResult)>`:
- Acquire tick lock via `acquire_tick_lock()`. If None (lock held), return empty vec + TickResult with 0s.
- Lock the store, call `get_due_jobs()`, clone the due jobs into a vec.
- Return the due jobs and a TickResult with jobs_checked = total enabled, jobs_run = due.len(), jobs_skipped = checked - run.
- The LockGuard is held for the duration (dropped when returned tuple goes out of scope at caller).

`pub async fn complete_job_run(store: &Arc<Mutex<JobStore>>, job: &CronJob, output: &str, success: bool) -> Result<Option<DeliveryTarget>>`:
- Save output to file via `save_job_output(&job.id, output)`
- Mark job run in store: `store.lock().unwrap().mark_job_run(&job.id, output, if success { "ok" } else { "error" })`
- If `is_silent(output)`, return None (suppress platform delivery)
- Otherwise return `resolve_delivery_target(job)`

```rust
// --- Cron tick task ---
if let Some(ref job_store) = self.job_store {
    let tick_cancel = self.cancel.clone();
    let job_store_tick = job_store.clone();
    let adapter_tick = adapter.clone();
    join_set.spawn(async move {
        let mut interval = tokio::time::interval(
            tokio::time::Duration::from_secs(60)
        );
        interval.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Skip);
        loop {
            tokio::select! {
                _ = tick_cancel.cancelled() => {
                    tracing::info!("Cron tick task shutting down");
                    break;
                }
                _ = interval.tick() => {
                    match ironhermes_cron::run_tick_check(&job_store_tick).await {
                        Ok((due_jobs, result)) => {
                            if result.jobs_run > 0 {
                                tracing::info!("Tick: {} jobs due", due_jobs.len());
                            }
                            // For each due job, spawn execution
                            // NOTE: Full agent loop execution will be wired
                            // when the tick runner has access to AgentLoop.
                            // For now, log that jobs are due and mark them.
                            for job in &due_jobs {
                                tracing::info!("Job due: {} ({})", job.name, job.id);
                                // Mark as run with placeholder output
                                // Full agent execution requires AgentLoop integration
                                // which involves building a fresh agent per job
                                if let Err(e) = ironhermes_cron::complete_job_run(
                                    &job_store_tick, job,
                                    "[Tick runner: agent execution pending full integration]",
                                    true
                                ).await {
                                    tracing::error!("Failed to complete job run: {}", e);
                                }
                            }
                        }
                        Err(e) => {
                            tracing::error!("Tick error: {}", e);
                        }
                    }
                }
            }
        }
    });
    tracing::info!("Cron tick task started (60s interval)");
}
```

The agent execution part can use a simplified approach initially (the `run` action in CronjobTool from Plan 02 marks the job as triggered for manual agent runs).
