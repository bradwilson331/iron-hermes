**Fix: replace `tokio::spawn` with `worker_join_set.spawn`** — the async move body is IDENTICAL, only the outer call changes:
```rust

                                // ... identical body ...
                            });
```

**worker_join_set declaration** — add before the dispatch_future loop (near where `join_set` is declared at the top of `start()`):
```rust
let mut worker_join_set: JoinSet<()> = JoinSet::new();
```

**Drain step to add in shutdown sequence** (runner.rs lines 667-674 — between `self.cancel.cancel()` and `drop(msg_tx)`):
```rust
        // Propagate cancellation to all subtasks
        self.cancel.cancel();

        // Plan 03: drain per-chat worker tasks with bounded timeout (D-11 / 5s cap).
        // Workers observe cancel_task.is_cancelled() after each turn; the 5s timeout
        // covers in-flight agent turns that haven't checked yet.
        let abort_deadline = tokio::time::Instant::now() + std::time::Duration::from_secs(5);
        while let Some(_) = tokio::time::timeout_at(
            abort_deadline,
            worker_join_set.join_next(),
        ).await.ok().flatten() {}
        // Any remaining tasks are aborted when worker_join_set is dropped.

        // Drop msg_tx to close the polling->dispatch channel
        drop(msg_tx);

        // Drain all JoinSet tasks (poll loop + session cleanup)
        while join_set.join_next().await.is_some() {}
```

**Ownership challenge (RESEARCH §5 — Open Question 3):** The dispatch loop (`dispatch_future`) owns `msg_rx` and runs as a `Future` in the `select!`. The `worker_join_set` must be accessible from INSIDE the dispatch closure (to call `worker_join_set.spawn(...)`) AND from OUTSIDE after the select exits (to drain). Use `RefCell<JoinSet<()>>` wrapped in a shared reference, or restructure the dispatch loop as a named async block that borrows `worker_join_set`. The planner must pick one; the simplest is to convert `dispatch_future` from a closure to an inline `async { }` block that borrows `worker_join_set` by `&mut`.
