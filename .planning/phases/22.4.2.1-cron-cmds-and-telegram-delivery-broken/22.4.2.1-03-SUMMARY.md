# Phase 22.4.2.1 Plan 03: Ctrl+C Gateway Shutdown Regression Fix Summary

**One-liner:** Restored Ctrl+C graceful shutdown by routing per-chat worker tasks through Arc<TokioMutex<JoinSet>> drained with a 5s timeout_at loop after cancel fires.

## What Was Done

This plan fixed the third regression in Phase 22.4.2.1: `hermes gateway` not exiting within a bounded time after Ctrl+C. Per RESEARCH §5/U7 the `tokio::signal::ctrl_c()` select arm was installed and fired correctly — the bug was structural: per-chat worker tasks at runner.rs:458 were spawned with bare `tokio::spawn` (detached), so they were not tracked. When `self.cancel.cancel()` fired, workers only observed it after completing their in-flight `handle_with_multimodal` agent turns (which can take 10–60s), keeping the tokio runtime alive.

### Fix Summary

**Task 1 (RED — Wave 0):**
- Created `crates/ironhermes-gateway/tests/gateway_shutdown.rs` with:
  - `gateway_drains_workers_within_timeout` — Wave 0 placeholder (GREEN), upgraded to real behavioral test in Task 2
  - `worker_join_set_drains_on_cancel` — source-grep test (RED until Task 2)
- Appended `invariant_22_4_2_1_03_worker_drain_present` to `invariants_22_4.rs` (RED until Task 2)
- Rule 3 auto-fix: two existing `execute_cron_job` call sites in runner.rs integration tests were missing the 8th argument (tg_client) added by Plan 02; fixed with `None` to unblock the build

**Task 2 (GREEN):**
- Declared `worker_join_set: Arc<TokioMutex<JoinSet<()>>>` in `start()` alongside existing `join_set`
- Cloned Arc into dispatch_future as `worker_join_set_dispatch`
- Replaced `tokio::spawn(async move { ... })` with `worker_join_set_dispatch.lock().await.spawn(async move { ... })` — body identical per D-02
- Inserted 5s-bounded drain block AFTER `self.cancel.cancel()` and BEFORE `drop(msg_tx)`:
  ```
  MCP shutdown_all → self.cancel.cancel() → worker_join_set drain (5s) → drop(msg_tx) → join_set drain
  ```
  Preserves Phase 21.2 Plan 11 ordering invariant
- Added `pub(crate) fn cancel_token(&self) -> CancellationToken` accessor
- Upgraded `gateway_drains_workers_within_timeout` to Path B behavioral test: spawns a JoinSet task that selects on CancellationToken, fires cancel, asserts drain completes within 5s
