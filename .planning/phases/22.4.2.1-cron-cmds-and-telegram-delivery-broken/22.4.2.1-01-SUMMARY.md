# Phase 22.4.2.1 Plan 01: /cron Slash UI Regression Fix Summary

**One-liner:** CronJobReader trait-object pattern wired from ironhermes-core through tui_rata with shared pure-text formatters in a new ironhermes-cron::display module.

## What Was Built

Restored the v1.1-validated `/cron` slash command UI that had been returning the stub `"/cron is not yet available. (No cron management UI)"` from `todo_stub` in handlers.rs.

### Task 1: Wave 0 — scaffolding

- Added `CronJobReader` trait to `ironhermes-core::commands::context` (7 sync methods: list_jobs_text, get_job_text, status_text, pause_job, resume_job, remove_job, queue_run)
- Added `cron_store: Option<Arc<dyn CronJobReader>>` field + `with_cron_store` builder to `CommandContext`
- Created `ironhermes-cron/src/display.rs` with three pure-text formatters (`format_job_list`, `format_job_detail`, `format_cron_status`) — no ANSI codes
- Added `pub mod display;` to `ironhermes-cron/src/lib.rs`
- Created `crates/ironhermes-core/tests/handlers_cron.rs` with `FakeCronJobReader` fixture + 6 test stubs
- Appended `invariant_22_4_2_1_01_no_cron_stub` to `invariants_22_4.rs` (RED at this point)

### Task 2: cmd_cron sub-dispatch

- Added `"cron" => cmd_cron(args, ctx)` arm to `dispatch()` match in handlers.rs
- Removed `"cron" => "No cron management UI"` from `todo_stub` (stub literal now absent)
- Added `fn cmd_cron` with full 7-subcommand sub-dispatch (list/status/get/pause/resume/run/remove)
- None guard returns `"/cron: cron store not configured."` per D-05
- `suggest_typo` wired for unknown subcommands (mirrors `cmd_agents`)
- All 6 behavioral tests in `handlers_cron.rs` turned GREEN
- INV-22.4.2.1-01 turned GREEN

### Task 3: CLI delegation + tui_rata wiring

- Added `use ironhermes_cron::display::{format_cron_status, format_job_detail, format_job_list}` to CLI cron.rs
- `cmd_list` now delegates to `format_job_list` (D-06 shared renderer)
- `render_job_details` now delegates to `format_job_detail`
- `cmd_status` now delegates to `format_cron_status`
- Added `CronJobReaderImpl` newtype adapter in `tui_rata/commands.rs` (Arc<Mutex<JobStore>> → CronJobReader)
- Added `with_cron_store` as 11th `with_*` call in `build_command_context`
- Added `cron_store: Option<Arc<Mutex<JobStore>>>` to `App` struct (defaults to None per D-02)
