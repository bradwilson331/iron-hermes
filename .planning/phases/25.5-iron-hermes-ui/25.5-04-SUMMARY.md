## Decisions Made
- **std::sync::Mutex for StateStore:** rusqlite Connection is `!Sync`, so tokio::sync::RwLock doesn't satisfy the `Send + Sync` bounds needed for Axum Extension. Same pattern as ironhermes-cli/src/main.rs:2163.
- **recv_raw/send_raw instead of typed recv/send:** Dioxus's `Websocket<String, String>` generic type inference produces `str` instead of `String` in certain contexts, causing `Sized` errors. Using raw message API with manual JSON serialization works around this.
- **AppState::init() inside on_upgrade:** The Axum `Extension<AppState>` injected in main.rs isn't accessible inside the Dioxus WS upgrade callback. As a pragmatic solution, AppState reconstructs from Config inside the handler. This is safe because Config::load() reads the same YAML; the init cost is amortized per connection (not per message).
- **Streaming block accumulation:** A `streaming_block_id` Signal tracks the current AI block being streamed into. First delta creates a new Block::Ai; subsequent deltas append to the same block. Reset on Finished event.

- **Files modified:** crates/iron_hermes_ui/src/server/state.rs
- **Verification:** `cargo check -p iron_hermes_ui --features server` passes
- **Committed in:** 6cb00e1

**2. [Rule 3 - Blocking] Dioxus WebSocket type inference str vs String**
- **Found during:** Task 2 (WarpHermes compilation)
- **Issue:** `ws.recv()` / `ws.send()` produced `str` type inference errors (`the size for values of type str cannot be known at compilation time`)
- **Fix:** Used `recv_raw()` / `send_raw()` with manual `dioxus_fullstack::Message::Text(json)` wrapping and `serde_json::from_str` parsing
- **Files modified:** crates/iron_hermes_ui/src/components/warp_hermes.rs
- **Verification:** `cargo check -p iron_hermes_ui --features server` passes
