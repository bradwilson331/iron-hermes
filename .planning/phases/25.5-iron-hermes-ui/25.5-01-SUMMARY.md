## Decisions Made
- **axum =0.8.4 pin:** axum 0.8.9 (latest) requires `axum-macros 0.5.1` which hasn't been published on crates.io yet. Pinned to 0.8.4 which resolves cleanly.
- **tokio always-present:** The existing `platform/timer.rs` needs `tokio::time::sleep` on native targets. Made tokio a regular dep with `time` feature; `server` feature adds `tokio/full` for the axum runtime. This avoids the "tokio not found" error when checking the web feature on native.
- **DioxusRouterExt from prelude:** Dioxus 0.7.7's server module is `dioxus::prelude::DioxusRouterExt` and `ServeConfig`, not a standalone `dioxus_server` module. Fixed to match actual API.
- **Backend crates via cfg(not(wasm32)):** ironhermes-core/agent/state/tools placed behind target cfg gate rather than feature flag. This is idiomatic тАФ they physically cannot compile to WASM (they pull in tokio-full, rusqlite, etc).
