## Decisions Made
- **Ungated server/api.rs:** Dioxus fullstack `#[get]`/`#[post]` macros generate both server endpoints AND client HTTP stubs — the types and function signatures must be compiled on both sides. Moved api.rs out of `#[cfg(feature = "server")]` gate while keeping ws.rs and state.rs server-only.
- **Inline type mapping:** Converted SlashCommandInfo→PaletteItem, SessionInfo→Tab, ConfigSummary→(model, provider) directly in WarpHermes component. No From impls — keeps types decoupled and conversion logic visible at the call site.
- **use_server_future over use_resource:** Per AGENTS.md hydration guidance, use_server_future runs the future on the server, serializes the result, and sends it to the client — ensuring hydration-safe initial render with no mismatch.
- **Empty palette during loading:** Rather than falling back to demo data while loading, the palette shows empty until real data arrives. Server functions return quickly and this avoids showing stale/incorrect mock commands.

- **Fix:** Removed cfg gate from `mod server` in main.rs; removed cfg gate from `pub mod api` in server/mod.rs (kept for ws.rs and state.rs)
