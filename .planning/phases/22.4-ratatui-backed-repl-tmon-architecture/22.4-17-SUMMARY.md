## Decisions Made

- **Color: Color::DarkGray + Modifier::DIM** (not Color::Yellow). DarkGray is unmistakably distinct from User=Cyan / Hermes=Green / Tool=Yellow; the DIM modifier visually demotes System rows as metadata. (DarkGray was the user's first suggestion in the locked Option B; we apply DIM on top to make the demotion explicit.)
- **Prefix: "System: "** via the existing `format!("{role_label}: ", ...)` Span machinery in transcript_text. role_label for Role::System is already the literal string `"System"` in role_style's match arm; only the color tuple changed.
- **Conditional DIM via `matches!(msg.role, Role::System)`** inside transcript_text (not via a new role_style return shape). Smallest diff (single conditional); zero impact on User/Hermes/Tool render paths.
- **Keep the `Option<Color>` return type and the let-else short-circuit.** The Option stays for future flexibility (a future Role variant may legitimately want to be hidden); the let-else stays as a defensive guard. It now passes for Role::System because role_style returns `Some(...)` instead of `None`.
- **No new render path for System.** The user's wording allowed an alternative ("change the role_style return type so System gets a distinct render path"). We chose the simpler path (return Some for System, gate DIM at the Style construction site) because the visual contract (dim gray, "System: " prefix) is sufficient and zero new variant handling is needed.
