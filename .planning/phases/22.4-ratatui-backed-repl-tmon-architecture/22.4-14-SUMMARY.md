# Plan 22.4-14: Input Prompt Block (UAT Gap 1 Closure)

## Outcome

UAT Gap 1 closed. The input area now wears a clearly defined bordered Block
with the title "Prompt" вҖ” users can no longer miss where to type. Render
cursor lands inside the bordered region (not on top of the border) thanks
to the +1/+1 offset bump.

## What shipped (Task 1)

**`crates/ironhermes-cli/src/tui_rata/app.rs`** (+13/-2 lines):
- New import: `use ratatui::widgets::{Block, Borders};`
- `App::new` (line 130): added
  `textarea.set_block(Block::default().borders(Borders::ALL).title("Prompt"));`
  immediately after the existing `set_cursor_line_style` call.
- `clear_textarea` (line 522): same `set_block(...)` reinstall after
  `set_cursor_line_style` вҖ” block survives every submit + Esc + slash dispatch
  reset cycle.
- `load_history_entry` (line 535): same `set_block(...)` on the fresh `ta`
  binding so arrow-key history recall preserves the frame.

**`crates/ironhermes-cli/src/tui_rata/ui.rs`** (+6/-3 lines):
- `render_cursor` (line 84-93): `.saturating_add(col as u16).saturating_add(1)`
  for `cursor_x`, `.saturating_add(row as u16).saturating_add(1)` for `cursor_y`
  to land the visible caret at `(area.x + 1, area.y + 1)` вҖ” inside the new
  top + left borders.
- Stale "Plan 22.4-05 Task 1 constructs textarea without a Block" comment
  removed; replaced with Plan 22.4-14 explanation referencing the
  `Borders::ALL` consumption of row 0 + column 0.
