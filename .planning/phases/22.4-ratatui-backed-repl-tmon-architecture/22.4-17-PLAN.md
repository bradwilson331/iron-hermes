User reported: "slash agent, help commands do not work."

Root cause (already diagnosed inline in 22.4-UAT.md Round 2 Gap 4 root_cause):
- `App::apply_slash_outcome` (app.rs:375-410) pushes slash output to `app.history` as a `Role::System` ChatMessage for FOUR variants — Handled, ClearSession, Unknown, Error.
- `transcript_text` (app.rs:570-587) iterates `app.history`, calls `role_style(msg)` for each, and does `let Some(color) = color else { continue };` at line 574.
- `role_style` (app.rs:637-644) returns `("System", None)` for Role::System (line 642).
- The `let Some else continue` short-circuit therefore SKIPS every System message — `/help`, `/clear` "Conversation cleared.", `/new` "New session started.", `/mouse on|off` "Mouse capture: on|off", and the "(tui_rata: /agent not yet wired ...)" typo-suggester / fallback hint are ALL silently dropped from the render. This is why the user observed `/mouse on|off` side-effects (capture state changed at the terminal level) but never saw the confirmation text — and why `/help`, which has no observable side-effect, appears completely dead.

User-locked decision (NON-NEGOTIABLE — Option B from the user response in CONTEXT prompt):
> Make System messages visible. Change `role_style` in app.rs:608-615 to return `Some(color)` for `Role::System` (suggested: `Color::DarkGray` or `Color::Yellow` with a dimmed Style). The 8 canonical insta snapshots in `crates/ironhermes-cli/tests/snapshots/` may need re-acceptance after the visual change — same pattern as Plan 22.4-14's snapshot acceptance step.

Locked implementation choices in this plan:
1. **Color: Color::DarkGray + Modifier::DIM.** DarkGray is unmistakably distinct from the existing palette (User=Cyan, Hermes=Green, Tool=Yellow); the DIM modifier visually demotes System rows so they read as metadata, not as conversation. 3. **No new render path** — the user's wording "OR change the role_style return type so System gets a distinct render path" is the alternative. We choose the simpler path (return Some(color) for System) because it requires zero additional variant handling in transcript_text and the visual contract (dim gray, "System: " prefix) is sufficient.
