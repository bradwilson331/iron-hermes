## Downstream Impact

**Contract for every Phase 22.4 Wave 1+ plan that imports the text-editing widget:**

- **Use** `use tui_textarea_2::TextArea;` — NOT `use tui_textarea::TextArea;`
- **Crate name in Cargo.toml:** `tui-textarea-2` (hyphen)
- **Import path:** `tui_textarea_2::*` (underscore — Rust's standard hyphen→underscore conversion for crate→module names)
- **API surface:** `tui-textarea-2 0.10.2` is a maintained fork of `tui-textarea 0.7.x` tracking ratatui head; the public API (`TextArea`, `Input`, `CursorMove`, cut/paste helpers, `widget()` render method) is source-compatible with the upstream 0.7 docs per RESEARCH-approved fallback row. Any method signature surprises should be treated as a Rule 1 bug and reported when encountered.

Later plans (22.4-01 through 22.4-10) that reference the original `tui_textarea::TextArea` import path in their PLAN.md snippets MUST be read with this rename substitution applied. If a downstream plan's acceptance-grep hard-codes `tui_textarea::` it needs a one-line planner-side correction before execution.
