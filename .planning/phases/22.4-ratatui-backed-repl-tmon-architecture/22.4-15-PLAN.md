<objective>
Close UAT Gap 2 from 22.4-UAT.md. User reported: "the web_search function does not work."

Diagnosis (from 22.4-UAT.md):
- `WebSearchTool` and `WebReadTool` are registered ONLY in the `rpc_registry` sub-registry (event_loop.rs:188-189) that is passed into `register_execute_code_tool_with_process_registry`. They are NEVER registered on the MAIN `registry` the LLM's AgentLoop sees, so the LLM has no top-level web_search tool — the model can only invoke web_search indirectly through execute_code, which is a guardrail-restricted path that the LLM does not naturally try.
- `register_delegate_task_tool` is missing from `build_app_deps` entirely (classic main.rs registers it at lines 500 and 978; ratatui omits it). Subagent dispatch is broken.
- The per-turn AgentLoop in `spawn_turn` does not chain `.with_fallback(fb_client)`, so PROV-07 provider fallback is silently disabled vs classic (main.rs:631-637).

User-locked decision (NON-NEGOTIABLE):
> Web tools must be top-level on the LLM's main registry, not just the rpc_registry sub-tree. The classic main.rs flow has the same gap structurally (lines 1011-1012) but full parity demands also adding `register_delegate_task_tool` (classic registers it at main.rs:500 + :978; ratatui does NOT) and wiring a fallback client into the per-turn AgentLoop in `spawn_turn` (PROV-07 parity).

Fix:
1. **Register WebSearchTool + WebReadTool on the MAIN registry** in build_app_deps, BEFORE the rpc_registry construction. The duplicate registration on rpc_registry stays — execute_code's safe-subset still needs them — but the LLM also sees them as top-level tools.
2. **Register delegate_task** on the MAIN registry by lifting the classic main.rs:487-507 block into build_app_deps. AgentSubagentRunner is already imported via ironhermes_agent (verified in main.rs:4); just construct the runner + semaphore and call `registry.register_delegate_task_tool(...)`.
3. **Wire with_fallback in spawn_turn** by lifting the classic main.rs:631-637 pattern. Resolver is currently only used in build_app_deps — pass it through into AppDeps so spawn_turn can resolve the fallback at per-turn time. (Alternative: resolve once at build_app_deps and store the fb_client option in AppDeps. Choose the latter — single resolve per session, mirrors classic.)
4. **Add INV-22.4-28** that grep-locks all 4 new wirings (web_search on main, web_read on main, register_delegate_task_tool on main, with_fallback on the spawn_turn builder chain).

Purpose: Achieve full D-18 item 1 + item 5 + item 10 parity with classic-tui so users on the ratatui default see identical tool behavior — including web_search, delegate_task, and provider fallback.

| Fallback provider → AgentLoop | with_fallback adds a second AnyClient that can be swapped in on 429/5xx/401. Same trust as the primary provider client (both come from `build_provider_client(&resolver, ...)`). |
