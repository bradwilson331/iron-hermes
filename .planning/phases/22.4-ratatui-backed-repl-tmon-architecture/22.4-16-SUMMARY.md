# Plan 22.4-16: /mouse on|off Slash Toggle (UAT Gap 3 Closure)

## Outcome

UAT Gap 3 closed per the user-locked decision. The ratatui REPL keeps mouse
capture ON by default (preserving scroll-wheel transcript scrolling), and
adds `/mouse on` and `/mouse off` slash commands so users can drop into
terminal-native text selection without leaving the REPL.

## Approach

**Approach A** — fast-path in `dispatch_slash` BEFORE the CommandRouter.
Approach B (adding `mouse` to the core CommandRouter) was rejected because
it would have touched ironhermes-core, which is out of scope for this plan
and would have widened the blast radius beyond the locked files_modified set.

The fast-path uses `input.strip_prefix("/mouse")` to detect the prefix; the
helper `handle_mouse_slash` parses the trailing arg and dispatches to one of
four arms: `on`, `off`, `""` (status query), or unknown.

    `SlashOutcome::Handled("Mouse capture: <on|off>. Use /mouse on or /mouse off to toggle.")`.
  - unknown arg: `SlashOutcome::Unknown` with usage hint.
- `render_help` updated to mention `/mouse on|off` and document the UX.

  end-to-end. Recommend the user launch `hermes chat`, type `/mouse off`,
  attempt Cmd+drag selection in the response area, then `/mouse on` and
  verify scroll-wheel scrolling is restored.
