# Plan 22.4-15: Registry Parity (UAT Gap 2 Closure)

## Outcome

UAT Gap 2 closed across all three sub-fixes. The ratatui REPL now has full
classic main.rs parity for tool registration and per-turn provider fallback.

## What shipped (Tasks 1+2)

**`crates/ironhermes-cli/src/tui_rata/event_loop.rs`** (+76/-13 lines):

1. Imports extended: `AgentSubagentRunner` added to the `ironhermes_agent`
   use line.
2. **Reordering:** `client` + `context_length` + `budget` block moved UP from
   its original position (just before `agent_loop_inst` construction) to right
   after the `memory_manager` build. Reason: the new `delegate_task` registration
   and `fallback_client` resolution both need `client` before the registry block.
3. **(a) + (b)** `WebSearchTool` and `WebReadTool` registered on the MAIN
   `registry` (in addition to the existing `rpc_registry` registration —
   both intentional). The LLM now sees them as top-level callable tools.
4. **(c)** `registry.register_delegate_task_tool(...)` called with a real
   `AgentSubagentRunner::new(client, resolver, Some(budget))
   .with_subagent_registry(...).with_transcript_scope(hermes_home, session_id)`,
   plus `tokio::sync::Semaphore::new(config.subagent.max_subagents)`,
   `memory_manager.map(...as SharedMemoryManager)`, `config.subagent.clone()`,
   `Some(cancel_parent.clone())`, and `None` for the progress callback.
   Mirrors classic main.rs:487-507.
5. **(d) build_app_deps half:** `fallback_client: Option<AnyClient>` pre-resolved
   once per session via
   `resolver.resolve_for_main().fallback_providers.first().and_then(|fb_name|
   build_provider_client(&resolver, fb_name, &main_endpoint.default_model).ok())`.
   Stored on `AppDeps`.
6. **(d) spawn_turn half:** clones `app.fallback_client`, then after the
   existing `if let Some(mm)` memory-manager chain block, adds
   `if let Some(fb) = fallback_client { agent = agent.with_fallback(fb); }`.
   Mirrors classic main.rs:631-637.

**`crates/ironhermes-cli/src/tui_rata/app.rs`** (+9/-0 lines, struct-field
expansion only):
- `AppDeps`: `pub fallback_client: Option<AnyClient>` field added.
- `App`: same field added.
- `App::new`: `fallback_client: deps.fallback_client,` initializer added.
- `test_deps()`: `fallback_client: None,` added so test-support constructors
  still compile.

> **Files-modified expansion:** Plan frontmatter listed only event_loop.rs
> + tests/invariants_22_4.rs. The actual change required app.rs (pure
> additive struct-field expansion). Justified — without the AppDeps field,
> event_loop.rs wouldn't compile. No logic added; only one new field
> surfaced through the dependency-injection bundle.
