### Gaps Summary

Two gaps block the phase goal declaration:

**Gap 1 — Missing `print_banner()` in ratatui path (D-03 explicit requirement)**

D-03 states: "print_banner() still fires BEFORE entering alt-screen so the banner lands in scrollback pre-TUI." The implementation calls `print_banner()` only inside the classic `run_chat()` function (line 758). The two ratatui dispatch sites in `main.rs` — `Commands::Chat` arm (lines 247-266) and bare-hermes arm (lines 305-324) — both call `run_chat_ratatui` without calling `print_banner()` first. The fix is a one-line addition of `print_banner()` call (and stdout flush) in each ratatui branch before `run_chat_ratatui(...)`. The invariant suite locks `print_yolo_banner_to_stderr` (INV-22.4-14) but not `print_banner`. A companion INV-22.4-xx should lock it.

**Gap 2 — ToolCall/ToolProgress/ToolResult never emitted from spawn_turn (D-17 partial, CR-02)**

`spawn_turn` in event_loop.rs emits 5 of 8 D-17 canonical StreamEvent variants. `ToolCall`, `ToolProgress`, and `ToolResult` are defined, handled, but never sent in production — AgentLoop's `with_tool_progress()` callback is never wired in `spawn_turn`. The snapshot test for `snap_tool_call_activity_row` passes because it directly injects the event, bypassing `spawn_turn`. Whether this blocks the phase goal depends on interpretation: the D-17 contract advertises all 8 variants as "live" for this phase; the CONTEXT D-18 item 1 says the AgentLoop is wired "with streaming callback (Phase 12, bridged via D-17)". If ToolProgress visibility during tool execution is a D-18 parity requirement (tools do display activity in classic-tui), this is a gap. If deferred acceptable, document explicitly.

**Non-blocking notes (code review findings — no phase goal impact):**

- CR-01: Duplicate home-dir loop in `build_app_deps` (lines 126-130 + `ensure_home_dirs` lines 301-307) — harmless no-op in practice; fix is deleting the inline loop.
- `tui-textarea-2 = "0.10.2"` instead of planned `tui-textarea = "0.7"` — user-approved via checkpoint Task 4; single ratatui compile unit confirmed. Not a gap.
- WR-01/WR-02/WR-05/WR-06 from review — correctness warnings for future cycles; do not block D-02 through D-19 goal.
