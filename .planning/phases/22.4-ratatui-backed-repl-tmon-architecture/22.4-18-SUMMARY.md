## Accomplishments

- **UAT Round 2 Gap 5 closed.** `/agents`, `/skills`, `/mcp`, `/sessions`, `/memory` now yield visible, informative output describing each command's contract, status (where readable from App state), and follow-up scope. `/agent` (singular) yields a visible "Did you mean `/agents`?" hint via the existing typo suggester (made visible by Plan 22.4-17). The "discoverable empty surface" problem is eliminated — typing any of these no longer hits the generic "(tui_rata: /X not yet wired in Phase 22.4 — see plan 22.4-07 §Handler Coverage)" fallback.
- **Dual routing strategy applied per command-set membership.** /agents and /skills are present in the core CommandRouter (`registry.rs:38` and `:107`) so they resolve as `ResolveResult::Exact` and reach `invoke_handler`; the clean fix is dedicated match arms returning `CommandResult::Output(...)`. /mcp, /sessions, /memory are NOT in the core 49-command set so the router would `NotFound` them; the clean fix is `strip_prefix` fast-path guards in `dispatch_slash` BEFORE the router call. Both strategies coexist without overlap; ironhermes-core stays untouched.
- **Trailing-char guard pattern.** Each of the three new fast-paths uses `rest.is_empty() || rest.starts_with(char::is_whitespace)` to prevent `/mcppanic` or `/mcp-list` from being routed to `handle_mcp_slash`. The /mouse fast-path from Plan 22.4-16 omitted this guard (no anticipated /mouse-* aliases); the new fast-paths add it because /mcp could collide with future /mcp-* hyphenated variants.
- **Sync stub helpers.** `handle_mcp_slash`, `handle_sessions_slash`, `handle_memory_slash` are `fn` (not `async fn`) and read O(1) Option fields (`app.mcp_manager.is_some()`, `app.memory_manager.is_some()`) plus plain data (`app.session_id`, `app.history_path`). No `RwLock` or `Mutex` is acquired — Phase 22.4 stub output is descriptive text, not live enumeration.
- **render_help expanded to 13-command structured listing.** `/help` now documents all currently-wired commands with `[stub]` markers on the 5 newly-wired (per the user-locked decision: "full functionality can be incremental, but visible 'this is what /X does' output is the bar"). Typo-suggester reference and /mouse documentation lines from Plan 22.4-16 are preserved.
- **INV-22.4-31 landed with 13+ sub-assertions.** Both routing strategies are grep-locked: Strategy 1 asserts `"agents" => CommandResult::Output(` and `"skills" => CommandResult::Output(` are present; Strategy 2 asserts `input.strip_prefix("/mcp")`, `("/sessions")`, `("/memory")` plus `fn handle_mcp_slash`, `fn handle_sessions_slash`, `fn handle_memory_slash` are present; sanity assertions cover render_help discoverability for all 5 names, the /mouse fast-path preservation, and the generic `not yet wired in Phase 22.4` fallback retention as the safety net.

## Decisions Made

- **Dual routing strategy in one plan.** /agents and /skills via invoke_handler match arms (in core CommandRouter); /mcp, /sessions, /memory via dispatch_slash fast-paths (out of core). Avoided cross-crate change to ironhermes-core for the three out-of-core names.
- **/agent (singular) handled implicitly by typo suggester.** No fast-path added, no dedicated match arm. The user's locked decision noted this would "already work once System messages are visible" (Plan 22.4-17 closure). End-to-end path: type `/agent` → router NotFound → `suggest_typo("agent", known)` returns `"agents"` → `SlashOutcome::Unknown { hint: "Did you mean `/agents`?" }` → `apply_slash_outcome` pushes Role::System → Plan 22.4-17 makes it visible.
- **Trailing-char guard `rest.is_empty() || rest.starts_with(char::is_whitespace)` on new fast-paths.** Stricter than /mouse from Plan 22.4-16 because /mcp could collide with future /mcp-* hyphenated aliases. /mouse has no anticipated /mouse-* variants so the guard wasn't added there. This inconsistency is documented as a candidate for a follow-up cleanup pass.
- **Sync helper fns, no async lock acquisition.** Reading `app.mcp_manager.is_some()` and `app.memory_manager.is_some()` are O(1) Option access. Acquiring async RwLock/Mutex would require making the helpers `async fn` and adding `.await` — out of scope for stub output.
- **No core-registry changes.** /mcp, /sessions, /memory stay OUT of `ironhermes-core::commands::registry::build_registry()`. Adding them would be a cross-crate change with no payoff for this gap closure — the fast-path is local to tui_rata/commands.rs.
