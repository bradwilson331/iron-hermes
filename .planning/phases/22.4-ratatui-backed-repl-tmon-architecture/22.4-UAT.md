- truth: "Mouse clicks inside the response/transcript pane are usable (text selection or focus), not silently consumed"
  status: resolved
  reason: "User reported: cannot click inside the response block"
  severity: major
  test: 3
  root_cause: |
    Two compounding causes:
    (a) `EnableMouseCapture` is unconditionally turned on in `run_chat_ratatui`
    (crates/ironhermes-cli/src/tui_rata/event_loop.rs:57). With mouse capture
    on, the terminal forwards every mouse event (including clicks) to the
    application instead of letting the terminal emulator handle native
    selection — so Cmd+drag/Shift+click for copy stops working. tmon's
    template (per CONTEXT) does NOT enable mouse capture by default.
    (b) `App::handle_mouse` (crates/ironhermes-cli/src/tui_rata/app.rs:299-313)
    only matches `MouseEventKind::ScrollUp` and `ScrollDown`. All other kinds
    — `Down(_)`, `Up(_)`, `Drag(_)`, `Moved` — fall through the catch-all
    `_ => {}` arm and are silently discarded. So even with mouse capture on,
    clicks have no observable effect.
  artifacts:
    - crates/ironhermes-cli/src/tui_rata/event_loop.rs:31-65 (mouse capture lifecycle)
    - crates/ironhermes-cli/src/tui_rata/app.rs:299-313 (handle_mouse)
    - crates/ironhermes-cli/tests/invariants_22_4.rs (INV-22.4-23 mouse_capture_paired)
  missing:
    - "USER DECISION (locked): mouse capture stays ON by default; add `/mouse off` and `/mouse on` slash commands to toggle EnableMouseCapture/DisableMouseCapture at runtime"
    - "Add MouseCaptureGuard.set(bool) or equivalent so the slash handler can flip the live state without exiting the alt-screen"
    - "Extend dispatch_slash + commands.rs to recognise /mouse {on|off}; status hint shows current state"
    - "Replace INV-22.4-23 with a paired invariant: (a) capture-on by default, (b) /mouse on|off slash route exists, (c) slash dispatch flips DisableMouseCapture at runtime"
    - "Optional polish: extend handle_mouse to handle MouseEventKind::Down(Left) for click-to-focus inside transcript pane (no-op outside) so users get visible feedback when capture is on"
