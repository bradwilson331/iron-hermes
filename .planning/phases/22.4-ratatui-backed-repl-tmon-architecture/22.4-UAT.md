- truth: "Mouse clicks inside the response/transcript pane are usable (text selection or focus), not silently consumed"
  status: resolved
  reason: "User reported: cannot click inside the response block"
  severity: major
  test: 3
  root_cause: |
    Two compounding causes:
    (a) `EnableMouseCapture` is unconditionally turned on in `run_chat_ratatui`
    (crates/ironhermes-cli/src/tui_rata/event_loop.rs:57). With mouse capture
    on, the terminal forwards every mouse event (including clicks) to the
    application instead of letting the terminal emulator handle native
    selection — so Cmd+drag/Shift+click for copy stops working. tmon's
    template (per CONTEXT) does NOT enable mouse capture by default.
    (b) `App::handle_mouse` (crates/ironhermes-cli/src/tui_rata/app.rs:299-313)
    only matches `MouseEventKind::ScrollUp` and `ScrollDown`. All other kinds
    — `Down(_)`, `Up(_)`, `Drag(_)`, `Moved` — fall through the catch-all
    `_ => {}` arm and are silently discarded. So even with mouse capture on,
    clicks have no observable effect.
  artifacts:
    - crates/ironhermes-cli/src/tui_rata/event_loop.rs:31-65 (mouse capture lifecycle)
    - crates/ironhermes-cli/src/tui_rata/app.rs:299-313 (handle_mouse)

## Gaps (Round 2)

- truth: "Slash command output (Handled, ClearSession, Unknown, Error variants — anything the user explicitly typed) is visible in the rendered transcript so the user gets feedback that their command did something"
  status: diagnosed
  reason: "User reported: /help and /agent appear to do nothing — no visible output despite the command running"
  severity: major
  test: 4
  root_cause: |
    `App::apply_slash_outcome` (crates/ironhermes-cli/src/tui_rata/app.rs:353-388)
    pushes the slash output to `app.history` as a `Role::System` ChatMessage.
    `transcript_text` (app.rs:541-573) iterates `app.history`, calls
    `role_style(msg)` for each, and does `let Some(color) = color else { continue };`
    — when the role is `System`, `role_style` returns `None` for the color,
    so the loop SKIPS the message. Slash command confirmations
    (/help text, /clear "Conversation cleared.", /new "New session started.",
    /mouse on|off "Mouse capture: on|off", /agent "(tui_rata: /agent not yet
    wired ...)") all push as System and are silently dropped from the render.
    This is why the user could observe /mouse on|off side-effects (terminal
    capture state changes) but never saw the confirmation text — and why
    /help, which has no observable side-effect, appears completely dead.
  artifacts:
    - crates/ironhermes-cli/src/tui_rata/app.rs:353-388 (apply_slash_outcome)
    - crates/ironhermes-cli/src/tui_rata/app.rs:541-573 (transcript_text loop)
    - crates/ironhermes-cli/src/tui_rata/app.rs:608-615 (role_style → System returns None)
  missing:
    - "USER DECISION (Option B locked): make Role::System messages visible — change role_style to return Some(color) for System (e.g. Color::DarkGray or Color::Yellow with dimmed Style) AND remove the `let Some(color) = color else { continue };` short-circuit OR change the role_style return type so System gets a distinct render path"
    - "Re-accept the 8 canonical snapshots in tui_rata_snapshots after the visual change (some snapshots include slash-output history)"
    - "Add an invariant grep-locking that role_style does NOT return None for any role variant (regression bite for the suppression bug)"
