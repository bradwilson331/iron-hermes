    Gap reason: runner.rs:120 unconditionally removes cancel sentinel, racing with cmd_cancel issued quickly after run starts. Also, lines 202-213 cancel check only runs at the dispatch gate — while semaphore is fully acquired the loop blocks on `acquire_owned()` and never re-checks cancel.

    **Fix 1 — Timestamp-guarded stale sentinel removal (line ~120):**
    Replace `let _ = std::fs::remove_file(&cancel_path);` with:
    ```rust
    let run_start = std::time::SystemTime::now();
    if cancel_path.exists() {
        if let Ok(meta) = std::fs::metadata(&cancel_path) {
            if let Ok(mtime) = meta.modified() {
                if mtime < run_start {
                    // Stale sentinel from a previous run — safe to remove
                    let _ = std::fs::remove_file(&cancel_path);
                }
                // else: sentinel is fresh (created after/at process start) — honor it
            } else {
                // Cannot read mtime (some platforms) — remove to avoid stuck state
                let _ = std::fs::remove_file(&cancel_path);
            }
        }
    }
    ```

    **Fix 2 — select!-based cancel polling during semaphore acquire (lines ~202-213):**
    Replace the blocking `semaphore.clone().acquire_owned().await?` with a `tokio::select!` that also polls for cancel:
    ```rust
    let permit = loop {
        tokio::select! {
            result = semaphore.clone().acquire_owned() => {
                break result?;
            }
            _ = tokio::time::sleep(std::time::Duration::from_millis(500)) => {
                if cancel_path.exists() {
                    println!("{} Cancel sentinel detected — stopping dispatch", "Batch:".yellow());
                    cancelled = true;
                    break 'dispatch; // break outer for-loop; use a label
                }
            }
        }
    };
    ```
    Add a `'dispatch:` label on the `for (hash, entry) in pending` loop so the inner `break 'dispatch` exits it. Add `use tokio::time` if not already imported.

    Move the existing `if cancel_path.exists()` check (lines 204-210) BEFORE the semaphore acquire so it still short-circuits when a permit is immediately available.

    **Tests (RED first):**
    Add to tests.rs:
    - `test_stale_sentinel_removed_at_startup`: Create a cancel file, backdate its mtime by 2 seconds using `filetime` crate or `std::fs::File::set_times` (Rust 1.75+), call the timestamp-guarded removal logic (extract as helper fn `clean_stale_sentinel`), assert file is removed.
    - `test_fresh_sentinel_preserved_at_startup`: Create a cancel file (current time), call `clean_stale_sentinel`, assert file still exists.

    Note: The select!-based cancel polling is best verified by the UAT re-test (manual), since it requires a running async runtime with actual semaphore contention. Unit tests cover the sentinel timestamp logic.

    Gap reason: filters.rs:49-58 filter_no_reasoning requires tool calls unconditionally. Text-only responses are always rejected regardless of quality. This was an overcorrection from the prior UAT fix that replaced the content heuristic with a structural requirement.

    **Fix — Restore content fallback in filter_no_reasoning (lines 49-58):**
    Replace current implementation with:
    ```rust
    pub fn filter_no_reasoning(result: &AgentResult) -> Option<String> {
        let has_tool_calls = result.messages.iter().any(|m| {
            m.tool_calls
                .as_ref()
                .is_some_and(|tc| !tc.is_empty())
        });
        if has_tool_calls {
            return None; // Tool usage = reasoning present
        }

        // Fallback: accept if assistant produced substantive text (>=100 chars)
        let has_substantive_text = result.messages.iter().any(|m| {
            m.role == ironhermes_core::Role::Assistant
                && m.content_text().is_some_and(|t| t.len() >= 100)
        });
        if has_substantive_text {
            return None;
        }

        // Also check final_response field
        if let Some(ref resp) = result.final_response {
            if resp.len() >= 100 {
                return None;
            }
        }

        Some("no_reasoning_steps".to_string())
    }
    ```
