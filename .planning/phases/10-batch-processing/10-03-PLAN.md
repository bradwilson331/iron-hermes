    **Gap 1 — Stale cancel sentinel (runner.rs):**
    At runner.rs line 118, immediately after `cancel_path` is defined, add:
    ```rust
    // Clear stale cancel sentinel from previous run (UAT gap: resume blocked by leftover file)
    let _ = std::fs::remove_file(&cancel_path);
    ```
    This ensures any leftover cancel file from a prior `batch cancel` is cleaned up before the
    dispatch loop begins. The `let _ =` ignores the error if the file doesn't exist.

    **Gap 2 — filter_no_reasoning too permissive (filters.rs:47-61):**
    Replace the filter_no_reasoning function body. The new logic: for batch agentic trajectories,
    tool use is the primary signal of reasoning. Reject if there are zero tool calls, regardless
    of text length. The function becomes:
    ```rust
    pub fn filter_no_reasoning(result: &AgentResult) -> Option<String> {
        let has_tool_calls = result.messages.iter().any(|m| {
            m.tool_calls
                .as_ref()
                .is_some_and(|tc| !tc.is_empty())
        });
        if !has_tool_calls {
            return Some("no_reasoning_steps".to_string());
        }
        None
    }
    ```
    This tightens the filter: any trajectory without tool calls is rejected. Text-only responses
    (like "How can I help?") no longer pass just because they exceed 10 characters.

    **Gap 3 — filter_secrets_in_output misses Role::Assistant (filters.rs:91-102):**
    Extend the iteration in filter_secrets_in_output to also check Role::Assistant messages:
    ```rust
    pub fn filter_secrets_in_output(result: &AgentResult) -> Option<String> {
        for msg in &result.messages {
            if msg.role == Role::Tool || msg.role == Role::Assistant {
                if let Some(text) = msg.content_text() {
                    if SECRET_PATTERNS.is_match(text) {
                        return Some("secrets_in_output".to_string());
                    }
                }
            }
        }
        None
    }
    ```
