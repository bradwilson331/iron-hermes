# Phase 21.8 Plan 06: Gap-Closure — Advisory Post-Install Hash Observation

Gap-closure plan that unblocks `hermes skills install <name>` against live skills.sh by reframing the post-install hash check from a strict equality failure into an advisory log. The previous implementation violated D-14 (opaque `snapshotHash`) by raising `HubErrorKind::ShaMismatch` whenever the server's hash algorithm diverged from our D-13 no-separator folder hash — which is every real install per UAT. Two code changes (installer.rs + error.rs) + four new tests (2 unit + 2 integration).

## What Shipped

### `crates/ironhermes-hub/src/installer.rs` (MODIFIED — Task 1)

**`install()` Step 7 rewritten as advisory observe-and-warn:**

Before (raises ShaMismatch, cleans final path):
```rust
// ── Step 7: Post-install hash verification (D-13/D-14) ─────────────────
let computed = compute_folder_hash(&final_path)?;
if let Some(server_hash) = &server_snapshot_hash {
    if !server_hash.is_empty() && &computed != server_hash {
        cleanup_final_path_safely(&final_path);
        return Err(typed(HubErrorKind::ShaMismatch, ...));
    }
}
```

After (log-only, install proceeds):
```rust
// ── Step 7: Post-install hash observation (D-13/D-14 — advisory) ────
let computed = compute_folder_hash(&final_path)?;
if let Some(server_hash) = &server_snapshot_hash {
    if !server_hash.is_empty() && &computed != server_hash {
        tracing::warn!(
            computed_hash = %computed,
            server_snapshot_hash = %server_hash,
            skill = %name,
            "server snapshotHash differs from local folder hash — advisory only, install proceeding (D-14 opaque contract)"
        );
    }
}
```

**`update()` post-rename verify mirrored with identical advisory semantics.**

**Module-level doc and PIPELINE ORDER LOCKED bullet (Step 7) updated** to reflect advisory posture.

**Two new unit tests** using an inline `FixedBundleSource` test double:
- `post_install_compare_is_advisory_when_hashes_diverge` — install() returns Ok with divergent server hash; D-14 verbatim round-trip and D-13 computed_hash both asserted; install dir survives
- `update_post_rename_compare_is_advisory_when_hashes_diverge` — update() returns Ok with divergent server hash; same three assertions; resolved_final survives

### `crates/ironhermes-hub/src/error.rs` (MODIFIED — Task 1)

`HubErrorKind::ShaMismatch` doc comment prepended with:
```rust
/// Local drift-detection sentinel: emitted by callers that compare a
/// fresh `compute_folder_hash(install_path)` against the stored
/// `SkillLockEntry.computed_hash`. NOT raised by the install or update
/// pipelines on server/client parity (D-14 is opaque). Retained for
/// tamper-detection call sites (`list --verify`, future strict mode).
```

### `crates/ironhermes-hub/tests/skills_sh_blob_adapter.rs` (MODIFIED — Task 2)

- Three-hop wiremock with server hash = `"server-side-opaque-value-that-does-not-match-d13"`
- Asserts: Ok install, D-14 verbatim round-trip, D-13 disk hash == entry.computed_hash, divergence precondition met (ne), install dir exists

### `crates/ironhermes-hub/tests/update_uninstall_test.rs` (MODIFIED — Task 2)
