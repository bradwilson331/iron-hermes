# Phase 21.8 Plan 01: Sanitize Module + Golden Vectors + HubErrorKind Extensions Summary

Wave 0 Rust port of the reference TypeScript `sanitize.ts` + `blob.ts::toSkillSlug` as 9 pure functions with 31 unit tests and a 24-vector golden-parity integration test, plus four new `HubErrorKind` variants (`ShaMismatch`, `ScanHit`, `PathTraversal`, `Audit`) per D-24 ‚Äî unblocking downstream plans 02-05.

## What Shipped

### `crates/ironhermes-hub/src/sanitize.rs` (NEW, ~340 lines)

Nine public pure functions implementing D-16/D-17/D-18/D-20 security primitives. Each is a verbatim port of the reference TypeScript semantics using Rust `regex::LazyLock<Regex>` statics and `std::path::Component` matching:

| Function | Purpose | D-Ref | Ported from |
|----------|---------|-------|-------------|
| `strip_terminal_escapes` | Strip CSI/OSC/DCS/PM/APC/simple-ESC/C1/controls | D-16 | `sanitize.ts:19-52` |
| `sanitize_metadata` | Run strip + collapse CR/LF to space + trim | D-16 | `sanitize.ts` |
| `sanitize_name` | Lowercase + `[^a-z0-9._]+` ‚Üí `-` + trim edges + 255 cap + unnamed-skill fallback | D-18 | `installer.ts:40-55` |
| `sanitize_subpath` | Reject `..`, NUL, absolute, drive-prefix, root; normalize `\` ‚Üí `/` | D-18 | `source-parser.ts:89-105` |
| `is_path_safe` | Canonicalize + `starts_with`, with lexical fallback for non-existent paths | D-18 | `installer.ts:63-68` |
| `is_contained_in` | Semantic alias for `is_path_safe` | D-18 | `installer.ts` |
| `assert_temp_contained` | Symlink-swap guard before `remove_dir_all` | D-20 | Pitfall 4 |
| `strict_yaml_delimiter` | Reject `---js`, `---javascript`, non-yaml; tolerate CRLF | D-17 | Pitfall 5 |
| `to_skill_slug` | ASCII-only slug derivation byte-for-byte against reference | D-22/slug | `blob.ts:55-62` |

12 `LazyLock<Regex>` statics back the 9 functions (6 for escape-stripping, 3 for slug, 2 for sanitize_name, 1 for CR/LF collapse). SP-1 private `typed()` helper copied verbatim from `tarball.rs:16-23`.

### `crates/ironhermes-hub/src/error.rs` (MODIFIED)

Added four new `HubErrorKind` variants per D-24 while preserving all 11 existing variants:

- `ShaMismatch` ‚Äî local folder hash drift vs `skills-lock.json::computedHash`
- `ScanHit` ‚Äî new 21.8 blob pipeline community-scan rejection (distinct from 19.1 `ScanBlocked` trust-gate)
- `PathTraversal` ‚Äî consumed by `sanitize_subpath` + `assert_temp_contained` + `is_path_safe`
- `Audit` ‚Äî reserved for audit-module internal plumbing (D-19 soft-fail)

All four auto-serialize to snake_case via the existing `#[serde(rename_all = "snake_case")]` attribute. 3 new unit tests (`tests_21_8`) verify serialization, `PartialEq`, and envelope shape.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 ‚Äî Bug] Rust string literal `\x85` out of range**
- **Found during:** Task 2 (strips_c1_control_codes test)
- **Issue:** Rust string literals only allow `\x` escapes in the ASCII range (`\x00-\x7f`). The original plan prescribed `"a\x85b"` as a C1 test input, which fails to compile with `error: out of range hex escape`.
- **Fix:** Replaced with `"a\u{85}b"` ‚Äî identical byte content, legal Rust syntax.
- **Files modified:** `crates/ironhermes-hub/src/sanitize.rs` test mod
- **Commit:** `b46595e`

**2. [Rule 1 ‚Äî Bug] `sanitize_name` test expectation wrong on underscore**
- **Found during:** Task 2 (sanitize_name_preserves_safe_chars test)
- **Issue:** My first test asserted `sanitize_name("foo.bar_baz")` ‚Üí `"foo.bar-baz"`, but the implementation's `NON_SAFE` regex is `[^a-z0-9._]+` ‚Äî underscore IS in the safe set (per reference `installer.ts:40-55`). Test expectation was wrong; implementation was correct.
- **Fix:** Updated test to assert `"foo.bar_baz"` ‚Üí `"foo.bar_baz"` (unchanged). Added a second assertion for `"Foo Bar"` ‚Üí `"foo-bar"` to lock in the contrast with `to_skill_slug` which DOES strip underscore.
- **Files modified:** `crates/ironhermes-hub/src/sanitize.rs` test mod
- **Commit:** `b46595e`

**3. [Rule 3 ‚Äî Blocking] `pub use sanitize::...` in Task 1 requires functions not yet written**
- **Found during:** Task 1 Step 2
- **Issue:** The plan's Task 1 Step 2 instructed adding the full `pub use sanitize::{...9 functions...}` line in lib.rs. But Task 1 Step 2 also says "create an empty sanitize.rs if needed" ‚Äî an empty file cannot satisfy the 9-item `pub use`, so `cargo check` fails at Task 1.
- **Fix:** Added only `pub mod sanitize;` in Task 1 (with empty stub); deferred the `pub use sanitize::{...}` line to Task 2 immediately after the functions are defined. Both Task 1 and Task 2 `cargo check` clean.
- **Files modified:** `crates/ironhermes-hub/src/lib.rs`
- **Commits:** `c902b66` (Task 1: `pub mod` only), `b46595e` (Task 2: `pub use` added)
