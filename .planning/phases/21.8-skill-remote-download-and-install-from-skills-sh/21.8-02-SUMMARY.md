# Phase 21.8 Plan 02: Wave 1 — blob.rs + lock.rs + paths::skills_lock_path Summary

Wave 1 Rust port of the reference TS `blob.ts` (three-hop HTTP adapter) and `local-lock.ts` (camelCase merge-clean lock schema) — plus threading `snapshot_hash: Option<String>` through `SkillBundle` as the locked surface for Wave 2 installer/CLI rewiring. Zero new workspace dependencies, 26 new unit tests, all 96 hub lib tests + 13 integration tests green.

## What Shipped

### `crates/ironhermes-hub/src/blob.rs` (NEW, ~580 lines)

`SkillsShBlobSource` — the replacement for the broken `SkillsShSource` — implementing the full three-hop pipeline with real (non-`unimplemented!()`) fetchers:

| Method | Hop | URL shape |
|--------|-----|-----------|
| `fetch_repo_tree` | 1: GitHub Trees API | `{github_api_base}/repos/{owner}/{repo}/git/trees/{branch}?recursive=1` |
| `fetch_skill_md_content` | 2: Raw content | `{raw_content_base}/{owner}/{repo}/{branch}/{path}` |
| `fetch_skill_download` | 3: skills.sh blob | `{download_base_url}/api/download/{owner}/{repo}/{slug}` (D-06 **path-based**) |
| `fetch_blob_skill` | orchestrator | 1 → 2 → parse frontmatter → slug → 3 → sanitize every file path |

Each HTTP call is wrapped in `with_one_retry` — exactly-once retry on 5xx/timeout/connect; never retries on 404/PathTraversal/ScanHit/Parse (D-24). Each hop uses branch fallback (`HEAD`/`main`/`master`) when no explicit ref is given — matches `blob.ts:89` behavior.

**Type surface (RESEARCH.md §Type Definitions — verbatim):**

```rust
pub struct TreeEntry { pub path: String, pub entry_type: String, pub sha: String, pub size: Option<u64> }
pub struct RepoTree { pub sha: String, pub branch: String, pub tree: Vec<TreeEntry> }
pub struct SkillSnapshotFile { pub path: String, pub contents: String }
pub struct SkillDownloadResponse { pub files: Vec<SkillSnapshotFile>, pub hash: String }
pub struct BlobSkill { pub name, description, raw_content, files, snapshot_hash, repo_path, metadata: Option<Value> }
pub struct SkillsShBlobSource { http, github: Arc<GitHubSource>, download_base_url, test_mode: bool,
                                 github_api_base: String, raw_content_base: String }
```

**HTTP client construction:**
- `https_only(true)` in prod (`SkillsShBlobSource::new`) — TLS-only, SSRF defense
- `user_agent("ironhermes/<CARGO_PKG_VERSION> (via openclaw)")` — D-22
- `timeout(Duration::from_secs(10))` per-call — D-08
- Test constructor `new_http_for_tests(..)` disables https_only for wiremock; `with_upstream_bases` overrides the GH/raw bases

**Security:**
- `sanitize::strict_yaml_delimiter(&raw_content)` rejects `---js`/`---javascript`/non-yaml frontmatter
- `sanitize::sanitize_metadata(name|description)` strips terminal escapes + CR/LF
- `sanitize::to_skill_slug(safe_name)` computes slug (byte-for-byte against reference)
- `sanitize::sanitize_subpath(file.path)` applied to every returned SkillSnapshotFile; returns `Err(PathTraversal)` on `..`/NUL/absolute/drive-prefix and does NOT retry (D-24)

**Regression guard (`compute_folder_hash_no_separator` test):**

```rust
// Hand-computed SHA-256 of: "a.txt" + "aa" + "b.txt" + "bb" (14 bytes, NO separators)
let mut h = Sha256::new();
h.update(b"a.txt"); h.update(b"aa"); h.update(b"b.txt"); h.update(b"bb");
let expected = hex::encode(h.finalize());
assert_eq!(actual, expected);
```

This test locks the algorithm in against accidental re-introduction of `\n` separators per CONTEXT.md's (now-superseded) D-13 language.

**1. [Rule 2 — Missing critical functionality] GitHubSource::auth() accessor not exposed**
- **Found during:** Task 2
- **Issue:** Plan's `fetch_repo_tree` sample calls `self.github.auth().authorization_header()` but `GitHubSource.auth` is private and has no accessor. Blob adapter cannot reuse Phase 19.1 auth machinery without it, which would force redundant env-probing + shelling out.
- **Fix:** Added `pub fn auth(&self) -> &GitHubAuth` to `GitHubSource`.
- **Files modified:** `crates/ironhermes-hub/src/github.rs`
- **Commit:** `a1ceda5`

**2. [Rule 1 — Bug] Unused `use crate::auth::GitHubAuth` import in blob.rs prod path**
- **Found during:** Task 2 clippy pass
- **Issue:** The prod path only needs `GitHubSource::auth()` return-type elaboration (via accessor) — the direct `GitHubAuth` type is only referenced in the `#[cfg(test)] fake_github()` helper, causing an unused-import warning in prod compile.
- **Fix:** Removed the top-level `use` and fully-qualified the test helper as `crate::auth::GitHubAuth::anonymous()`.
- **Files modified:** `crates/ironhermes-hub/src/blob.rs`
- **Commit:** `a1ceda5`

**3. [Rule 2 — Missing critical functionality] No test override for upstream GitHub/raw bases**
- **Found during:** Task 2 (designing the struct surface)
- **Issue:** Plan specified `new_http_for_tests(github, download_base_url)` for overriding the skills.sh base, but provided no mechanism to redirect the Trees API (hop 1) or raw.githubusercontent (hop 2) — plan 05 wiremock integration tests need all three.
- **Fix:** Added `github_api_base: String` + `raw_content_base: String` fields on `SkillsShBlobSource` with sensible prod defaults; added `#[doc(hidden)] pub fn with_upstream_bases(self, github_api_base, raw_content_base)` builder method. Both fields are read by `fetch_repo_tree` and `fetch_skill_md_content` respectively.
- **Files modified:** `crates/ironhermes-hub/src/blob.rs`
- **Commit:** `a1ceda5`

**4. [Rule 3 — Blocking] No URL-safe encoder in workspace**
- **Found during:** Task 2 (fetch_repo_tree implementation)
- **Issue:** GitHub branch refs occasionally contain `/` (e.g. `feature/foo`) — must percent-encode for the Trees API path segment. No `urlencoding` or `percent-encoding` crate in the workspace. Plan mandates zero new deps.
- **Fix:** Hand-rolled ~15-line `fn urlencoding(s: &str) -> String` in blob.rs, covers the unreserved-char set (RFC 3986 §2.3) and percent-encodes everything else. Covered by `urlencoding_preserves_unreserved_chars` test.
- **Files modified:** `crates/ironhermes-hub/src/blob.rs`
- **Commit:** `a1ceda5`
