### 3. Install emits D-21 5-line progress shape
expected: Running `ironhermes skills install <anything>` prints in order (even when the install fails partway): `Resolving skills.sh/...`, `Discovering skills in ...`, `Downloading N bytes...`, `Scanning for threats...`. note: Verified the 4 progress lines emit in correct order for `skills install excalidraw` and `skills install ascii-art`. 5th "Installed" line could not be observed because both attempts hit pre-existing ShaMismatch against live skills.sh (separate issue; server-computed snapshotHash algorithm doesn't match client D-13 folder hash).

### 4. D-23 restart message on successful install
expected: After a successful install, a separate stdout line reads `Installed. Restart the agent or start a new session to use <name>.` This ONLY shows on clean install — if every attempted install errors out (e.g., pre-existing ShaMismatch against live skills.sh), mark this test as blocked.
result: blocked
blocked_by: other
reason: "requires hash-parity fix — see Gaps entry on ShaMismatch blocker"

### 5. skills list reads from skills-lock.json
expected: Running `ironhermes skills list` runs without error regardless of whether any skills are installed. `ironhermes skills list --format json` outputs valid JSON. The data source is `~/.ironhermes/skills-lock.json` (NOT the old `~/.ironhermes/skills/.hub/lock.json`).
result: pass
note: `skills list --format json` → `[]` (empty but valid JSON); `skills list` → `No skills installed.` Both run clean with no installed skills (expected given ShaMismatch blocker on fresh installs).

### 6. skills uninstall alias works
expected: Running `ironhermes skills uninstall <some-name>` does NOT error with "unknown subcommand". Clap routes it to the same handler as `ironhermes skills remove <some-name>` (D-04 — one release cycle compatibility alias). If no skill is installed, an error like "skill not installed" is fine — the test is about the verb alias working, not the removal outcome.
result: pass
note: `skills uninstall excalidraw` and `skills remove excalidraw` produce identical `hub error (NotFound): skill 'excalidraw' is not installed` — proving the clap alias routes both verbs to the same handler.
