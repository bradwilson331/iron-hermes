## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] grep -A2 too shallow for Requirements line offset**
- **Found during:** Task 1 verification (INV-27.1-07 failing)
- **Issue:** The plan specified `grep -A2` to find `**Requirements:**` after a Phase heading, but the actual ROADMAP format has a blank line + Goal line between the heading and Requirements (offset = 4 lines)
- **Fix:** Changed to `grep -A4`
- **Files modified:** validate-import.sh
- **Commit:** c68f949c

**2. [Rule 1 - Bug] grep -P unavailable on macOS /usr/bin/grep**
- **Found during:** Task 1 verification (INV-27.1-07 and INV-27.1-10 failing with "invalid option -- P")
- **Issue:** The plan's script used `grep -qP` and `grep -nP` for Perl negative-lookbehind, but macOS system grep does not support `-P`
- **Fix:** Replaced with `perl -ne 'print if /(?<!HXP-).../'` for INV-27.1-07 and `perl -ne 'print "$.: $_" if /.../'` for INV-27.1-10
- **Files modified:** validate-import.sh
- **Commit:** c68f949c

**3. [Rule 1 - Bug] INV-27.1-08 stub check too broad**
- **Found during:** Task 1 verification (INV-27.1-08 failing)
- **Issue:** The plan checked `grep -qF "[Urgent work - to be planned]" "$ROADMAP"` which triggers on pre-existing stubs at ROADMAP lines 95, 130, and 751 (other phases, not Phase 27.1) — same class of bug as Plan 01's verification deviation
- **Fix:** Scoped to `grep -A3 "^### Phase 27\.1: " "$ROADMAP" | grep -qF "[Urgent work - to be planned]"` — checks only the Phase 27.1 block
- **Files modified:** validate-import.sh
- **Commit:** c68f949c

**4. [Rule 1 - Bug] grep -cF with pattern starting with dash**
- **Found during:** Task 1 verification (INV-27.1-12 failing with "invalid option -- ")
- **Issue:** `grep -cF "- Phase 27.1 inserted..."` — macOS grep treated the leading `- ` as a flag
- **Fix:** Added `--` separator: `grep -cF -- "- Phase 27.1 inserted..."`
- **Files modified:** validate-import.sh
- **Commit:** c68f949c

**5. [Rule 2 - Missing] INV-27.1-15 .DS_Store filter**
- **Found during:** Task 1 verification (INV-27.1-15 failing)
- **Issue:** macOS creates `.planning/.DS_Store` in the Hexapod source repo as a filesystem artifact, which appeared as an untracked file in `git status --porcelain`
- **Fix:** Added `| grep -v -E '\.DS_Store$'` to the MODS filter pipeline
- **Files modified:** validate-import.sh
- **Commit:** c68f949c
