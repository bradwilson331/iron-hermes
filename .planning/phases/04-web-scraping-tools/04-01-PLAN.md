1. **Firecrawl response types** (per D-01 research pattern):
```rust
#[derive(Debug, Deserialize)]
struct FirecrawlScrapeResponse {
    success: bool,
    #[serde(default)]
    data: Option<FirecrawlScrapeData>,
    #[serde(default)]
    error: Option<String>,
}

#[derive(Debug, Deserialize)]
struct FirecrawlScrapeData {
    #[serde(default)]
    markdown: Option<String>,
    #[serde(default)]
    metadata: Option<FirecrawlMetadata>,
}

#[derive(Debug, Deserialize)]
struct FirecrawlMetadata {
    #[serde(default)]
    title: Option<String>,
    #[serde(rename = "statusCode", default)]
    status_code: Option<u16>,
}
```

2. **Constants for selectors** (per D-01, D-08):
```rust
const BOILERPLATE_SELECTORS: &[&str] = &[
    "nav", "header", "footer", "aside",
    "[role=navigation]", "[role=banner]", "[role=contentinfo]",
    "script", "style", "noscript",
];

const CONTENT_SELECTORS: &[&str] = &[
    "article", "main", "[role=main]", "body",
];
```

3. **`async fn validate_url_async(url: &str) -> anyhow::Result<()>`** -- wraps `is_safe_url()` with `tokio::task::spawn_blocking` per D-16. If not safe, bail with "URL blocked by security policy (private IP)" per D-10.

4. **`fn truncate_content(content: &str, max_chars: usize) -> String`** -- per D-13, D-14:
   - If `content.len() <= max_chars`, return as-is
   - Use `content.char_indices()` to find the byte offset at the `max_chars`th character (UTF-8 safe -- addresses Pitfall 1 from research)
   - Within that range, search backward for `\n\n` (paragraph break), then `. ` (sentence break), then ` ` (word break), then hard cut
   - Append `\n\n[Content truncated at {displayed_chars} of {total_chars} characters]` where displayed_chars and total_chars are char counts (not byte counts)

5. **`fn extract_content_local(html: &str, url: &str) -> anyhow::Result<String>`** -- per D-01, D-05, D-06, D-08:
   - Parse with `scraper::Html::parse_document(html)`
   - Extract title from `<title>` element
   - Select main content by iterating `CONTENT_SELECTORS` in order, taking first match
   - Get inner HTML of selected element
   - Strip boilerplate: before passing to htmd, remove inner HTML of elements matching `BOILERPLATE_SELECTORS` from the content HTML string. Since scraper does not support DOM mutation, use a second parse of the content fragment and collect text only from non-boilerplate children, OR pass the full content HTML to htmd and strip boilerplate selectors from the selected content element's HTML before conversion. Simplest approach: select the content area, get its `.html()`, then for each boilerplate selector, find matches within a re-parsed fragment and replace their `.html()` with empty string in the content HTML string.
   - Convert cleaned HTML to markdown with `htmd::convert(&cleaned_html)?`
   - Prepend `# {title}\nSource: {url}\n\n` header per D-06
   - Return the assembled string

6. **`async fn fetch_with_firecrawl(url: &str) -> anyhow::Result<String>`**:
   - Read `FIRECRAWL_API_KEY` from env
   - POST to `https://api.firecrawl.dev/v1/scrape` with `bearer_auth`, body: `{"url": url, "formats": ["markdown"]}`
   - Timeout from `Config::load()?.web.timeout_secs` (as `Duration::from_secs`)
   - Parse `FirecrawlScrapeResponse`
   - If `!success`, return error with the error field
   - Extract `data.markdown` content, `data.metadata.title`
   - Prepend `# {title}\nSource: {url}\n\n` header per D-06
   - Return assembled markdown string

7. **`async fn fetch_local(url: &str, config: &ironhermes_core::config::WebConfig) -> anyhow::Result<String>`** -- per D-03, D-07, D-17:
   - Build `reqwest::Client` with `user_agent(&config.user_agent)`, `timeout(Duration::from_secs(config.timeout_secs))`, default redirect policy (follows redirects per D-03)
   - GET the URL
   - Check `response.url()` against original URL; if different, run `validate_url_async(response.url().as_str()).await?` for post-redirect SSRF re-validation per D-17
   - Check Content-Type header: extract from `response.headers()`, use `.to_str()` and `.starts_with("text/html")` (handles `text/html; charset=utf-8` per Pitfall 2). If not text/html, bail with `"web_read only supports HTML pages. Got: {content_type}"` per D-07, D-10
   - Read body as text: `response.text().await?`
   - Call `extract_content_local(&body, url)`

8. **`WebReadTool` struct** implementing `Tool` trait:
   - `name()` returns `"web_read"`
   - `toolset()` returns `"web"`
   - `description()` returns `"Fetch a web page and return its content as markdown."`
   - `schema()` returns ToolSchema with single required `url` string parameter per D-09
   - `is_available()` returns `true` per D-11
   - `execute()`:
     a. Extract `url` from args, bail with "Missing required parameter: url" if absent
     b. Call `validate_url_async(url).await?` per D-16
     c. Load config: `let config = Config::load().unwrap_or_default();`
     d. Try Firecrawl first if `FIRECRAWL_API_KEY` is set (per D-02):
        - If Firecrawl succeeds, use that content
        - If Firecrawl fails (any error), log with `tracing::warn!` and fall through to local
     e. If no API key or Firecrawl failed, call `fetch_local(url, &config.web).await?`
     f. Call `truncate_content(&content, config.web.max_content_chars)` per D-13, D-14, D-15
     g. Return the truncated content string

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user -> web_read tool | User-supplied URL is untrusted input |
| web_read -> external HTTP | Outbound HTTP to arbitrary URLs controlled by user input |
| web_read -> Firecrawl API | Trusted API but responses could contain unexpected content |
| redirect chain | Server-controlled redirects can target internal resources |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering / Information Disclosure | URL parameter | mitigate | `is_safe_url()` via `spawn_blocking` before every fetch (D-16) -- blocks private IPs, localhost, CGNAT, metadata endpoints |
| T-04-02 | Tampering | Redirect chain | mitigate | Post-redirect SSRF re-validation via `response.url()` check (D-03, D-17) |
| T-04-03 | Denial of Service | Large page response | mitigate | 30s timeout (D-04) + content truncation at 50K chars (D-15) |
| T-04-04 | Information Disclosure | Non-HTML content | mitigate | Content-Type check rejects non-text/html responses (D-07) |
| T-04-05 | Denial of Service | DNS rebinding | accept | Known limitation documented in ssrf.rs -- TOCTOU between DNS resolution and HTTP connection. Mitigation would require custom DNS resolver integration, which is out of scope for v1 |
</threat_model>
