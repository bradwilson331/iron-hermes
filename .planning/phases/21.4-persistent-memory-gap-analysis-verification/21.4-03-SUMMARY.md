## What Was Built

**Task 1 — on_session_end wiring (GAP-6)**

Added `on_session_end` calls to the two natural-exit paths in the CLI:

- **`run_single`**: After `agent.run(messages).await?` completes and before `state_store.end_session`, we acquire the memory manager lock and call `mgr_lock.on_session_end(&session_id, &entries).await`. Uses `MemoryEntries::default()` as best-effort — the file provider's `on_session_end` is a no-op, and external providers use their own internal state for fact extraction.

- **`run_chat` clean exit**: After the TUI shutdown and before `state_store.end_session`, added the same `on_session_end` call. This fires on EOF (Ctrl-D) and natural REPL loop end.

The ctrl-c exit path (`CtrlCDecision::ExitCleanly`) continues to skip `on_session_end` — it calls `std::process::exit(0)` directly, and the existing skip comment is preserved.

Added two static-grep regression tests (`INV-21.4-GAP6-01`, `INV-21.4-GAP6-02`) in the `tui_extension_wiring_tests` module to pin this invariant against future refactoring.

**Task 2 — memory status and off subcommands (GAP-5)**

Created `crates/ironhermes-cli/src/memory_cmd.rs` with two public async handlers:

- **`handle_memory_status`**: Loads `Config`, reads `MemoryConfig` for provider name and toggle state. If `memory_enabled`, opens `MemoryStore::new(hermes_home)`, calls `load_from_disk()`, then computes entry counts and char totals from `store.entries()` HashMap directly (no new MemoryStore methods needed). Displays MEMORY.md and USER.md sections with entry count, chars used/limit, and usage percentage. Follows `colored` crate output pattern from `cron.rs`/`memory_setup.rs`.

- **`handle_memory_off`**: Loads `Config`, checks if provider is already `"file"` with no mirror (prints "already using built-in" and returns). Otherwise sets `config.memory.provider = "file"` and `config.memory.mirror_provider = None`, then calls `config.save()`. Does NOT touch `memory_enabled` or `user_profile_enabled` — built-in MEMORY.md/USER.md remain active (D-10).

Updated `MemorySubcommand` enum with `Status` and `Off` variants, added dispatch in `main()`, registered `mod memory_cmd` in both `main.rs` and `lib.rs` (lib.rs so the module is available for integration tests).

Added unit tests verifying the off-logic round-trip with a temp config file.
