## Gaps Summary

No gaps. All 6 phase gaps (GAP-1 through GAP-6) are closed:

- **GAP-1 (Critical):** `queue_prefetch` now fires in CLI chat mode — `run_agent_turn` receives and wires `memory_manager` via `.with_memory_manager()`.
- **GAP-2 (Critical):** `on_pre_compress` now fires — `build_context_engine` and `attach_context_engine` accept `memory_manager` and apply `with_memory_manager()` on concrete engine types before `Arc::new()` wrapping.
- **GAP-3 (Critical):** `queue_prefetch` now fires in gateway mode — `handler.rs` wires `self.memory_manager` into `AgentLoop` via `.with_memory_manager()`.
- **GAP-4 (Major):** `memory_enabled` and `user_profile_enabled` config toggles added to `MemoryConfig` with serde defaults of `true`, `build_memory_manager` returns `Option<Arc<...>>` (None when disabled), and all consumers guard with `if let Some`.
- **GAP-5 (Major):** `hermes memory status` and `hermes memory off` subcommands implemented in `memory_cmd.rs`, wired into `MemorySubcommand` enum with dispatch in `main()`.
- **GAP-6 (Minor):** `on_session_end` wired in `run_single` (natural exit after `agent.run()`) and `run_chat` clean exit path. ctrl-c path intentionally skips (existing comment preserved; async context unsuitable).

**Pre-existing test failures (out of scope):** 2 ironhermes-core tests (`cmd_models_info_known_model`, `test_provider_resolver_populates_model_metadata`) fail due to a context-length mismatch for `claude-sonnet-4` (200K vs 1M).
