# Phase 21.4 Plan 02: Memory Gap Closure Summary

Close all four gaps identified in GAP-ANALYSIS.md: config toggles (GAP-4), context-engine wiring (GAP-2), CLI REPL wiring (GAP-1), and gateway wiring (GAP-3) so memory subsystem hooks fire end-to-end.

## Tasks Completed

| Task | Description | Commit |
|------|-------------|--------|
| 1 | Add memory_enabled/user_profile_enabled toggles; change factory return to Option | c57592a |
| 2 | Wire memory_manager into AgentLoop and ContextEngine across CLI and gateway | cc4bdc6 |

## What Was Built

**Task 1 — Config toggles and factory Option return (GAP-4)**

Added `memory_enabled: bool` and `user_profile_enabled: bool` to `MemoryConfig` in `config.rs` with `#[serde(default = "default_true")]` so they default to `true` and existing config files without these fields behave identically to before.

Changed `build_memory_manager` return type from `anyhow::Result<Arc<Mutex<MemoryManager>>>` to `anyhow::Result<Option<Arc<Mutex<MemoryManager>>>>`. Returns `Ok(None)` when `memory_enabled = false`. Added `set_user_profile_enabled(false)` call on the manager when `user_profile_enabled = false`.

Added `user_profile_enabled` guard in `MemoryManager::handle_tool_call` that returns `Err(String)` for `target = "user"` calls when disabled. Added matching guard in `PromptBuilder::load_memory` so user profile is not injected into system prompt when disabled.

Updated all CLI call sites to handle the `Option`: `register_memory_tool`, `register_delegate_task_tool` (with `m as SharedMemoryManager` coercion), `set_memory_manager`, `prompt_builder.set_user_profile_enabled`.

**Task 2 — End-to-end memory_manager wiring (GAP-1/2/3)**

Added `memory_manager: Option<Arc<TokioMutex<MemoryManager>>>` as the new last parameter to both `build_context_engine` (engine_factory.rs) and `attach_context_engine` (agent_wiring.rs). Applied `e.with_memory_manager(mgr)` on both `LocalPruningEngine` and `SummarizingEngine` inside `build_context_engine` before `Arc::new()` wrapping — this is the only location where the concrete type is visible so the method can be called.

Added `memory_manager` parameter to `run_agent_turn` in `main.rs`. Wired `agent.with_memory_manager(mgr)` at the top of `run_agent_turn` so `queue_prefetch` fires in the CLI REPL path (GAP-1). Updated both call sites in `run_chat` and the call in `run_single` to pass `memory_manager.clone()`.

Added `agent.with_memory_manager(mgr)` in `handler.rs` before `attach_context_engine` (GAP-3). Fixed the gateway unit test call site that was missing the 9th `memory_manager` argument.

Added five static-grep regression tests to `run_chat_invariants.rs` locking all three gap-closure invariants: `run_agent_turn` signature, `run_agent_turn` body wiring, `attach_context_engine` signature, `build_context_engine` signature + body, and gateway handler wiring.

Updated `chat_memory_persistence.rs` to unwrap the `Option` returned by `build_memory_manager` (all four call sites).
