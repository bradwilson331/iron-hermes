### WR-03: `load_memory` holds MemoryManager tokio::Mutex lock across multiple awaits

**Files modified:** `crates/ironhermes-agent/src/prompt_builder.rs`
**Commit:** 5b224e2
**Applied fix:** Restructured `load_memory` to acquire the tokio::Mutex lock separately for each async call (`format_for_system_prompt(Memory)`, `format_for_system_prompt(User)`, `system_prompt_block()`) using scoped blocks. The lock is now released between calls, reducing contention for concurrent memory writes (handle_tool_call, prefetch) during prompt construction. All existing logic (GAP-4 user_profile guard, dedup check) preserved.

### WR-04: `run_chat` ctrl-c ExitCleanly path bypasses `on_session_end` and memory flush

**Files modified:** `crates/ironhermes-cli/src/main.rs`
**Commit:** 75fe2c0
**Status:** fixed: requires human verification
**Applied fix:** Replaced `std::process::exit(0)` with controlled loop-break flow in both ExitCleanly paths. Added `exit_cleanly` flag before the outer REPL loop. The in-flight ExitCleanly (tokio::select ctrl-c) sets the flag and breaks from the `'turn` inner loop; a check after response processing breaks the outer loop. The at-prompt ExitCleanly (ReadlineError::Interrupted) now uses `break` directly. Both paths fall through to the existing `on_session_end` cleanup block at line 986+. The emergency 3x ctrl-c hard exit (process::exit(130)) is preserved unchanged.

### WR-05: Test helper `primary_file()` leaks tempdir via `std::mem::forget`

**Files modified:** `crates/ironhermes-agent/src/memory/manager.rs`
**Commit:** dd75f73
**Applied fix:** Changed `primary_file()` return type from `SharedProvider` to `(SharedProvider, tempfile::TempDir)`, removing `std::mem::forget(tmp)`. The TempDir is returned alongside the provider so callers keep it alive for the test duration via `let (primary, _tmp) = primary_file();`. The directory is cleaned up when `_tmp` drops at test end. Updated all 7 callers to destructure the tuple.
