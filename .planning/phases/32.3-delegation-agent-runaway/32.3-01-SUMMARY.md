RAII RegistrationGuard with Drop-on-future-drop closes the 6.7-hour ghost canonical bug at the type level — registry cleanup is now structurally guaranteed on every exit path (natural / timeout / panic / cancel).

## What Changed

### Task 1 — RegistrationGuard type + SubagentConfig field

- **`SubagentRegistry::register`** removed (no public API for raw registration).
- **`SubagentRegistry::register_guarded(info, weak) -> RegistrationGuard`** added — inserts the entry and returns a guard whose Drop calls `unregister_internal` synchronously.
- **`SubagentRegistry::unregister`** renamed **`unregister_internal`** with `pub(crate)` visibility — compiler-enforces D-01: external callers cannot bypass the guard.
- **`RegistrationGuard`** struct holds `Weak<RwLock<SubagentRegistry>>` + `SubagentId`. Drop uses the same `block_in_place + block_on` bridge as `SubagentRegistryHandle::kill` (subagent_registry.rs:187–190).
- **`SubagentRegistry.stale_warned: HashSet<SubagentId>`** dedup field added; cleared in `unregister_internal` so re-registered ids start fresh (D-06 prerequisite for Plan 02).
- **`SubagentInfo.activity_last: Option<Arc<Mutex<Instant>>>`** reservation slot added (Plan 02 wires the real Arc from `AgentLoop::activity_last`).
- **`SubagentConfig.stale_warn_seconds: u64`** field with `#[serde(default = "default_stale_warn_seconds")]` returning 120; D-07 ceiling lock (`child_timeout_seconds: 300`) preserved and asserted in `test_subagent_config_default` AND new `test_subagent_config_stale_warn_default`.
- **`AgentSubagentRunner::run_child`** migrated to `register_guarded`, binding `_guard` for the entire function body (D-02 — guard placement inside the `tokio::time::timeout` future).
- **Test helpers** in `subagent_registry_ops.rs`, `iron_hermes_ui/server/state.rs`, and `agents_list_live_mid_turn.rs` migrated to `register_guarded` with detached guards (`std::mem::forget`) where lifecycle is not the assertion target.

### Task 2 — Delete explicit unregister + regression suite

- **`subagent_runner.rs:331`** explicit `reg.write().await.unregister_internal(&subagent_id)` deleted. Replaced with `// D-01: ...` comment citing the 6.7-hour ghost bug site (`sub_20667cb71808` reported Active for 24,150s). The `_guard` binding above covers every exit path structurally.
- **`crates/ironhermes-agent/tests/registration_guard.rs`** (NEW, 5 tests, all `flavor = "multi_thread"`):
  - `test_guard_deregisters_on_natural_completion`
  - `test_guard_deregisters_on_timeout` — **THE canonical 6.7-hour ghost regression**
  - `test_guard_deregisters_on_panic` (FutureExt::catch_unwind + AssertUnwindSafe)
  - `test_guard_deregisters_on_cancel` (tokio::select + CancellationToken)
  - `test_registration_counter_balanced` (D-03 balance across all four scenarios)
- **`crates/ironhermes-tools/tests/delegate_task_runaway.rs`** (NEW, 2 tests):

  - `test_runaway_repro_natural_completion_also_deregisters` — belt-and-braces natural-success coverage.

## Runaway Repro Layer (D-2 Scope Clarification)

`delegate_task_runaway.rs` uses the **run_child harness shortcut** described in the plan's <action> step 11, not the full DelegateTaskTool → AgentLoop → AgentSubagentRunner pipeline. Justification (verified during implementation):

- `ironhermes-tools` cannot depend on `ironhermes-agent` (circular — agent depends on tools). The real `RegistrationGuard` lives in `ironhermes-agent::subagent_registry`, so the test cannot directly construct it from inside `ironhermes-tools`.
- The shortcut uses a synthetic `ActiveCounterGuard` that mirrors `RegistrationGuard`'s Drop contract exactly: an Arc<AtomicUsize> decremented on Drop. The test wraps it inside a `SubagentRunner` impl that "registers" by bumping the count and binding the guard for the lifetime of `run_child`.
- The test exercises the SAME structural call site that produced the bug — `DelegateTaskTool::execute()` calling `tokio::time::timeout(..., runner.run_child(...))`. The structural drop-on-future-drop contract that `RegistrationGuard` relies on is what gets locked in.
- Unit-level Drop coverage of the actual `RegistrationGuard` type (with real `block_in_place` bridge) lives in `crates/ironhermes-agent/tests/registration_guard.rs` and is exhaustive across all four exit paths.

The combination (unit coverage in ironhermes-agent + integration-boundary coverage in ironhermes-tools) closes the 6.7-hour ghost at both layers.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| RegistrationGuard holds `Weak<RwLock<SubagentRegistry>>` | Avoids Arc cycle; silent no-op when Arc dropped (session shutdown). Same pattern recommended in RESEARCH.md and consistent with `SubagentRegistryHandle` storage. |
| `block_in_place + block_on` bridge in Drop | Rust has no async Drop. This pattern is already established in `SubagentRegistryHandle::kill` (subagent_registry.rs:187–190) and is safe on multi-thread tokio runtime. |
| `unregister_internal` is `pub(crate)` not private | Internal callers (test helpers within the crate) need to construct guards via the dangling-Weak idiom for legacy tests; `pub(crate)` provides the needed surface while preventing external bypass. |
| End-to-end runaway test in `ironhermes-tools` uses synthetic guard | `ironhermes-tools` cannot depend on `ironhermes-agent` (circular). Synthetic `ActiveCounterGuard` mirrors `RegistrationGuard`'s Drop contract at the SAME integration boundary (DelegateTaskTool::execute + tokio::time::timeout). |
| Test helpers use `std::mem::forget(guard)` for non-lifecycle assertions | Legacy unit tests (tree-shape, JSON serialization) only assert structure, not lifecycle. Detaching the guard preserves their original "registered until end-of-test" semantics without changing test intent. Lifecycle exercise is exhaustive in `tests/registration_guard.rs`. |
| D-07 confirmation: `child_timeout_seconds` stays at 300 | The 6.7-hour ghost is the LEAK surviving timeout, not the timeout not firing. RegistrationGuard closes the leak structurally; raising/lowering the ceiling would not have fixed the bug. Asserted in 2 separate test functions to lock the no-op. |
