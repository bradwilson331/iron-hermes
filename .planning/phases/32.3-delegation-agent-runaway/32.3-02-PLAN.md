<objective>
Wire Phase 32.1's `AgentLoop::ActivityTracker` through to `SubagentInfo` so the registry can detect stale subagents in real time, then surface the `Stale` status in the `/agents` tree render with a once-per-id `tracing::warn!`.

Purpose: Operators need to see lagging subagents (no LLM call, no tool call, no stream token for `stale_warn_seconds`) before they become 6.7-hour ghosts. The activity_last Arc<Mutex<Instant>> is already populated by the existing four mark_activity sites in agent_loop.rs (lines 946 ApiCall, 1151 StreamToken, 1404 ToolCall before, 1412 ToolCall after — verified in RESEARCH.md). This plan exposes it and reads from it at registry-summary time. No new mark_activity bump point is needed.

**Wave 2 sequencing:** Plan 01 (Wave 1) lands `RegistrationGuard`, the `stale_warned` HashSet field, the `activity_last: None` placeholder on SubagentInfo, and the `SubagentConfig.stale_warn_seconds` config field. Plan 02 (Wave 2) builds on those: adds the per-call schema, the activity_last live wiring, the stale derivation in flatten_tree, and the render pill. This sequencing eliminates the prior file-collision on `subagent_registry.rs` (both plans were Wave 1 in the prior plan; checker flagged it as a merge-conflict bomb).

Output:
- `AgentLoop::activity_last_arc()` accessor returning a clonable `Arc<Mutex<Instant>>` (D-04 — confirms RESEARCH.md A1 assumption)
- `delegate_task.rs` single-task + per-task batch schema gains `stale_warn_seconds: Option<u64>` per-call override (D-05); SubagentInfo construction populates `activity_last` from `agent.activity_last_arc().clone()` before `agent.run(...)` is awaited
- `render_agent_tree` in handlers.rs reads `info.activity_last.elapsed()`, compares to the effective stale threshold (per-call > config), emits `[stale]` (or `⚠ Stale Ns idle`) status pill, and fires `tracing::warn!` gated by `SubagentRegistry::stale_warned` (added in Plan 01)
- Two new tests in `cmd_agents_and_stop.rs`: render shows stale pill when threshold crossed; warn fires exactly once per id
</objective>
