<objective>
Land the RAII registration guard that closes the 6.7-hour ghost canonical bug (live repro: `sub_20667cb71808` reported Active for 24,150s after natural completion because `tokio::time::timeout` dropped the `run_child` future before reaching the explicit `unregister` call on `subagent_runner.rs:331`).

Purpose: Make registry cleanup a Drop-guaranteed invariant rather than a code-path-dependent call. After this plan, no exit path (natural / error / timeout-drop / panic / cancel / abort) can leave a stale registry entry, because the `RegistrationGuard` returned by `register_guarded` runs its `Drop` impl on every future-drop and synchronously calls `unregister_internal` via the existing `block_in_place` bridge pattern.

Output:
- `RegistrationGuard` struct + `register_guarded` method on `SubagentRegistry` (D-01)
- `unregister` renamed to `unregister_internal` with `pub(crate)` visibility (D-01)

- `SubagentConfig.stale_warn_seconds: u64` field with default 120 (D-05 config side; per-call schema in Plan 02)
- D-07 confirmed no-op: `SubagentConfig.child_timeout_seconds` ceiling stays at 300 — asserted in tests
- Regression-test infra: `RegistrationCounter` helper + `tests/registration_guard.rs` covering all four exit paths (D-02 / D-03)
- End-to-end `delegate_task_runaway.rs` repro of the 6.7-hour ghost: spawn child via real delegate-task path, fire `tokio::time::timeout`, assert registry.list().len() == 0 within 1s (D-02)
