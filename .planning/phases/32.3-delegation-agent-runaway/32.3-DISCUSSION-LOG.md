## Registry Cleanup Contract

| Option | Description | Selected |
|--------|-------------|----------|
| RAII guard (Recommended) | `SubagentRegistry::register` returns a `RegistrationGuard` whose Drop impl calls `unregister`. Guard held by the spawn task. Survives panic, timeout-drop, early-return — cleanup is automatic at any exit path. | ✓ |
| Explicit unregister in finalize block | Add a `match result { Ok | Err => unregister(id) }` block wrapped around the agent run in delegate_task.rs. More visible in the code, but easy to forget on new exit paths. | |
| Both — guard + assertion | RAII guard for correctness + a tracing::error! assertion that fires if `register` count != `unregister` count at process shutdown. Belt and braces. | |

**User's choice:** RAII guard
**Notes:** D-01/D-02/D-03 in CONTEXT.md. Type-level fix at the registration site, no callers can forget to deregister.
