| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-32.3-01 | Tampering | Telegram spoofed-edit replay of `/agents kill <id>` triggers unintended kill | mitigate | D-09 `confirm` token enforced in `handler.rs::handle_slash_command` BEFORE dispatch — replay of original message (which lacks `confirm`) gets refused; replay of message with `confirm` is operator-intended; test_kill_requires_confirm asserts the refusal |
| T-32.3-13 | Tampering | Telegram spoofed `/agents prune confirm` sweeps active agents | mitigate | D-09 confirm covers prune; `prune` only removes entries WHERE elapsed > stale_secs, so confirm-replay still cannot prune fresh entries |
