<objective>
Land the "shrike service" — the four operator-facing termination/diagnostic operations (`kill`, `interrupt`, `prune`, `status`) as library code, with unit-test coverage. This plan delivers the BUSINESS LOGIC; per-surface adapters (TUI / web / gateway) ship in Plan 04.

Purpose: D-08 mandates a coherent termination surface with parity across three surfaces. Centralizing the four operations in a single `ShrikeService` module (and extending `SubagentListSnapshot` so the existing `cmd_agents` dispatch picks them up automatically) keeps the contract testable and avoids three divergent implementations. Per D-10 Claude's Discretion, RESEARCH.md §3 recommends dedicated `shrike.rs` module — adopted.

**D-12 carve-out (B3 fix):** The /help discoverability AUDIT itself is Plan 05 (audit-only). But the new subcommands added in 32.3 (kill already exists in build_registry — interrupt/prune/status are new) DO need help entries written as part of implementation per D-12 ("new /agents subcommands added in 32.3 DO need help entries — written as part of implementation, not the audit"). Plan 03 owns the implementation, so the help-entry additions live here.

**W3 fix:** The existing `/agents kill` arm in `commands/handlers.rs` currently calls `reg.kill(id)` (token cancel only, via `SubagentRegistry::kill` line 62-70). Under D-08, kill must ALSO `JoinHandle::abort` the spawned task. Plan 03 Task 2 migrates that arm to route through `ShrikeService::kill` (which performs both token cancel AND abort). This is the load-bearing functional change of the migration — without it, kill is broken (the canonical bug surface).

Output:
- `crates/ironhermes-agent/src/shrike.rs` with `pub struct ShrikeService` holding `Arc<RwLock<SubagentRegistry>>` and `Arc<Mutex<HashMap<SubagentId, JoinHandle<()>>>>` for batch-task aborts
- Four public methods: `kill(id) -> Option<KillResult>`, `interrupt(id) -> bool`, `prune(stale_secs) -> Vec<SubagentId>`, `status(id) -> Option<SubagentStatusInfo>`
- `SubagentListSnapshot` trait in `commands/context.rs` extended with `interrupt`, `prune(stale_secs)`, `status` methods (default impls returning sensible "not supported" defaults for non-Shrike implementors); `SubagentRegistryHandle` overrides them via block_in_place bridge
- `cmd_agents` extended with `interrupt | prune | status` arms following the existing `kill` arm shape (resolve_subagent_id + Ambiguous/None patterns); candidates list extended for typo suggestion
- Existing `kill` arm migrated from `reg.kill(id)` (bare-registry token cancel) to `reg.kill(id)` via the extended SubagentListSnapshot trait method that forwards to `ShrikeService::kill` — i.e. trait override on SubagentRegistryHandle routes through ShrikeService.
