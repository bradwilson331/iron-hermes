## FUTURE Follow-up Todos (verbatim)

Per `<output>` requirement, here are the 6 FUTURE-tagged entries verbatim from `32.3-HELP-AUDIT.md` §Follow-up Todos. A future doc-polish phase planner can adopt each as a plan task:

- **FUTURE-32.3-AUDIT-01** (`/agents kill` verb mismatch, all surfaces): `cmd_agents` kill arm at `handlers.rs:193` emits `"Cancelled subagent {}."` but the registry description starts with "List, **kill**, ..." (and `KillResult::Display` from `shrike.rs` says `Killed sub_xxxx (ran NNNs, NN turns)`). The handler should either route through `ShrikeService::kill -> KillResult::Display` to render `Killed sub_xxx (ran NNNs, NN turns)`, or update the registry description to explicitly say "Cancel a subagent". Surfaces affected: tui_rata, iron_hermes_ui, gateway.
- **FUTURE-32.3-AUDIT-02** (`/agents prune` threshold not surfaced, all surfaces): Plan 03 Deviation D-2 hard-coded `PRUNE_DEFAULT_STALE_SECS = 120` because `CommandContext` has no `delegation_stale_warn_seconds()` accessor yet. The /help description should mention the default (e.g. "Sweep entries idle > 120s") so operators know the threshold without reading source. Bonus: surface a `--stale-secs <N>` per-call override once a CommandContext accessor lands.
- **FUTURE-32.3-AUDIT-03** (`/agents kill <id>` Telegram confirm-token absent from `args_hint`): D-09 requires the literal `confirm` token as the LAST argument on Telegram for destructive ops. The bare `agents` `args_hint` enumerates `kill <id>` without the `[confirm]` suffix. Consider a Telegram-conditional `args_hint` (e.g. `kill <id> [confirm — Telegram only]`) so Telegram operators see the confirm requirement in /help. Documented in `docs/DELEGATION.md` Gateway Confirm Token section (Plan 05 Task 1), but /help does not surface it. Surface affected: gateway.
- **FUTURE-32.3-AUDIT-04** (`/agents prune` Telegram confirm-token absent from `args_hint`): The Plan 03 B3 `CommandDef` for `agents prune` has no `args_hint`. On Telegram, D-09 requires `/agents prune confirm`. Consider Telegram-conditional `args_hint` of `[confirm — Telegram only]` so operators discover the confirm requirement without reading docs. Surface affected: gateway.
- **FUTURE-32.3-AUDIT-05** (web surface alternative — REST endpoints): Per 32.3-RESEARCH §Pattern 5 Question 2, iron_hermes_ui's /help inherits the slash registry, but REST endpoint variants (e.g. `/api/agents/kill`, `/api/agents/prune`) would also be discoverable to web-only operators. This is forward-looking — Plan 04 Wave 4 (running in parallel) wires the surface adapters but does not add REST endpoint discoverability to /help. Track for future iron_hermes_ui doc-polish.
- **FUTURE-32.3-AUDIT-06** (web surface /agents kill verb mismatch — same as 01): Web inherits the same `"Cancelled subagent {}."` output as tui_rata via the shared handler. Closing FUTURE-32.3-AUDIT-01 also closes this. Listed separately so a future planner does not miss the surface count.
