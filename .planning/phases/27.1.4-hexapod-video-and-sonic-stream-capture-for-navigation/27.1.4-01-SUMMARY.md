- Added `camera_pan` and `camera_tilt` actions to hexapod_tcp using `CMD_CAMERA#x#y\n` wire format with silent i64 clamp and midpoint defaults for the unused axis
- Added `stream_distance` action that polls CMD_SONIC N times (clamped to [1,20]) with 200ms tokio sleep between polls, returning `"Distances: [...] cm | min=N max=N avg=N.N"` trend summary
- Added 7 named constants (CAMERA_PAN_MIN/MAX, CAMERA_TILT_MIN/MAX, CAMERA_PAN_DEFAULT, CAMERA_TILT_DEFAULT, STREAM_DISTANCE_MAX_SAMPLES) and updated outer allowlist, JSON schema, and DESCRIPTION

## Decisions Made

- **camera_pan/camera_tilt joint-axis default:** Used protocol midpoints (x=115, y=90) as defaults for the unused axis when calling CMD_CAMERA. Midpoints are within server-enforced ranges and produce a neutral camera position — clearer than requiring both params for single-axis calls.
- **stream_distance numeric parse:** Parsed directly from raw `CMD_SONIC#<dist>` response via `split('#')` and `parse::<i64>()` instead of calling `parse_distance_response` (which returns `"Distance: N cm"` — a formatted string, not a number). Per RESEARCH Pitfall 6.
- **TDD structure:** Task 1 added camera arms + stub stream_distance in allowlist (compiler-enforced); Task 2 added tests (RED) then replaced stub with real arm (GREEN).
