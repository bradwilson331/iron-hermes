Video frame protocol (Freenove Client.py receiving_video()): read 4 bytes → `struct.unpack('<L', ...)` → read that many bytes as JPEG. Rust: `read_exact(&mut [0u8;4])` → `u32::from_le_bytes(buf) as usize` → `read_exact(&mut vec![0u8; len])`. Server's `video_socket.listen(1)` is single-client → connect fails with `ConnectionRefused` when another client is attached.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent → tool args | LLM-supplied `action` string crosses into the tool (only `capture_frame` permitted; everything else hits the blocked arm) |
| robot → tool (port 8002 TCP) | the robot sends a 4-byte length prefix + JPEG bytes; the length value is attacker-influenceable if the robot/network were compromised |
| env → tool | `HEXAPOD_IP` env var supplies the connect target — used only in `TcpStream::connect(addr)` (no shell expansion, no format-string risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27.1.4-05 | Tampering / Denial of Service | 4-byte frame-length prefix → `vec![0u8; frame_len]` allocation | mitigate | bound-check `frame_len`: reject `0` and anything `> MAX_FRAME_BYTES` (2 MB) before allocating — prevents a malicious/buggy length prefix from triggering a multi-GB allocation; also wrapped in a 5s read timeout so a stalled stream cannot hang the tool |
| T-27.1.4-06 | Denial of Service | `read_exact` hanging on a stalled camera (Condition.wait() in camera.py) | mitigate | the entire frame read (both `read_exact` calls) is wrapped in `timeout(Duration::from_secs(5), ...)` — on elapse, return the "camera may not be streaming" error string; no indefinite hang |
| T-27.1.4-07 | Information Disclosure | base64 JPEG returned in the tool result (could be large, logged) | accept | the image is the intended output (consumed by Gemma 4); no PII concern beyond what the camera physically sees; tool result size is bounded by the 2 MB frame cap |
| T-27.1.4-08 | Spoofing / unauthorized access | any client on the local network can connect to port 8002 | accept | carry-forward of the Phase 27.1.1 trust model — the robot is on a trusted local network with no auth layer; `HEXAPOD_IP` env var acts as the implicit gate; out of scope for this phase |
| T-27.1.4-09 | Elevation of Privilege | agent sends a non-`capture_frame` action hoping for a side effect | mitigate | the `match action` in `execute()` only routes `"capture_frame"`; every other string returns the blocked-action string with no I/O — same compile-visible allowlist discipline as `hexapod_tcp` |
</threat_model>
