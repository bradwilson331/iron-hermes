# Phase 27.1.4 Plan 02: HexapodVideoTool — Capture Frame via Port 8002 Summary

HexapodVideoTool stateless JPEG capture over TCP port 8002 with 4-byte LE length-prefix protocol, returning base64 data URI, registered in the default tool set.

## What Was Built

**Task 1 — `hexapod_video.rs`:**

- `HexapodVideoTool` unit struct implementing the `Tool` trait (stateless, no persistent socket)
- Single `capture_frame` action: connects to `HEXAPOD_IP:8002`, reads 4-byte little-endian length prefix, reads JPEG frame bytes, returns `data:image/jpeg;base64,<...>` data URI
- Connection error handling: `ConnectionRefused` → "video port 8002 is busy" message; other errors → robot-unreachable message
- Missing `HEXAPOD_IP` env var → early return error string without any TCP attempt
- 5-second read timeout wrapping both `read_exact` calls (T-27.1.4-06 mitigation — prevents indefinite hang when camera not streaming)
- `MAX_FRAME_BYTES = 2MB` bounds check before buffer allocation (T-27.1.4-05 mitigation — prevents multi-GB allocation from malicious/buggy length prefix)
- `prerequisites()` returns single `HEXAPOD_IP` env_var Prerequisite (`required: true`) — default `is_available()` hides tool when env var unset
- `use base64::Engine as _` critical trait import (RESEARCH Pitfall 5 — without it, `STANDARD.encode()` fails to compile)
- 3 unit tests: `test_missing_env_var`, `test_unknown_action_blocked`, `test_capture_frame_passes_allowlist`

**Task 2 — `lib.rs` + `registry.rs`:**

- `lib.rs`: `pub mod hexapod_video;` added immediately after `pub mod hexapod_tcp;`
- `registry.rs`: `use crate::hexapod_video::HexapodVideoTool;` import added to `register_defaults_except()`
- `registry.rs`: `register_unless_skipped!(Box::new(HexapodVideoTool), "hexapod_video");` after hexapod_tcp registration
- `registry.rs`: docstring updated from Phase 27.1.1 to Phase 27.1.4 with `hexapod_video` in tool list
- New test `test_register_defaults_includes_hexapod_video` — asserts `register_defaults()` includes `hexapod_video`
- `test_register_defaults_except_terminal_skips_terminal_only` updated with `"hexapod_video"` in expected slice
