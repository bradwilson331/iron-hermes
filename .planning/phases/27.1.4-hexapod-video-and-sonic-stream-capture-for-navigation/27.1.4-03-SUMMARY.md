## What Was Built

**Task 1 — `skills/hexapod/SKILL.md`:**

- **Frontmatter:** `description` updated from "12 hexapod_tcp actions" to "15 hexapod_tcp actions" with companion hexapod_video mention; `video` tag added to tags list
- **Actions table:** Three new rows appended:
  - `stream_distance` — `CMD_SONIC\n` × N with 200ms sleep, returns `"Distances: [...] cm | min=M max=X avg=A.B"`, samples clamped to [1, 20]
  - `camera_pan` — `CMD_CAMERA#{x}#90\n`, x clamped 50–180, y defaults to midpoint 90
  - `camera_tilt` — `CMD_CAMERA#115#{y}\n`, y clamped 0–180, x defaults to midpoint 115
- **Camera Gimbal Joint-Axis Behavior subsection:** Explains that `CMD_CAMERA#x#y\n` sets both axes simultaneously; `camera_pan` sends y=90 midpoint, `camera_tilt` sends x=115 midpoint; camera gimbal is independent of head servos (`CMD_HEAD` vs `CMD_CAMERA`)
- **Parameter-ranges table:** Three new rows for `samples` (stream_distance, 1–20), `x` (camera_pan, 50–180, CAMERA_PAN_MIN/MAX constants), `y` (camera_tilt, 0–180, CAMERA_TILT_MIN/MAX constants)
- **hexapod_video Tool section:** New top-level `## hexapod_video Tool` section with:
  - Connection table: HEXAPOD_IP shared env var, port 8002, 4-byte LE length-prefix + JPEG protocol, one-connection-per-call pattern
  - Actions table: single `capture_frame` row returning `"data:image/jpeg;base64,<data>"`
  - Error Strings table: all four error conditions verbatim from Plan 02 implementation (HEXAPOD_IP missing, port busy 5s timeout, frame read error)
  - Usage Pattern block: `camera_pan(x=115)` → `camera_tilt(y=90)` → `capture_frame()` with note about ui_client.py occupying port 8002
