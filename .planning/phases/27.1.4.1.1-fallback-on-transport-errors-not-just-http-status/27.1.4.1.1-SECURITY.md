## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-27.1.4.1.1-01 | — (no STRIDE vector) | `classify_llm_error` transport-marker substring matching (`is_transport_failure`) | accept | Transport-error classification is not a security surface. No ASVS L1 category applies — V5 (Input Validation) is N/A because the haystack is an internal server-built error string, not user input; there is no spoofing / tampering / repudiation / info-disclosure / DoS / EoP vector in a `to_lowercase()` + `&str::contains` check over a server-generated string. Worst-case misclassification: a non-transport error is treated as transport → `should_fallback = true` → when a fallback IS configured the loop switches providers (a *correctness* concern, not a security one; mitigated by the conservative 7-needle allowlist and the `test_classify_sse_read_timeout_not_transport` Pitfall-1 regression guard); when no fallback is configured behavior is unchanged. `should_retry` stays `true`, `run()` is unchanged. | closed |
