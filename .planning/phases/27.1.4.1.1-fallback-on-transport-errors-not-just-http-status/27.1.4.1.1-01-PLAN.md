<objective>
Make `AgentLoop` provider fallback fire on transport-level failures (provider unreachable / request never delivered / no response), not just on extractable HTTP status codes. Extends PROV-07 coverage: the HTTP-status path was validated in Phase 12, the AgentLoop fallback wiring was closed in Phase 27.1.4.1 — this plan closes the classifier gap so that wiring actually activates on `Connection refused`, connect timeout, DNS failure, connection reset, and the reqwest `error sending request for url` marker.

Real-world repro this fixes: local-Ollama primary + OpenRouter fallback with Ollama not running → today the gateway retries the dead `localhost:11434` endpoint `MAX_RETRIES` (3) times and then errors instead of switching to OpenRouter, because none of those errors carry an HTTP status so `classify_llm_error` returns `(should_retry=true, should_fallback=false)`.

Purpose: a configured `fallback_providers` chain becomes usable in the most common failure mode (primary endpoint down).
Output: a private transport-failure detection helper + one edit site in `classify_llm_error` + repurposed and new unit tests + a new static-invariant grep test file.

Decision traceability (CONTEXT.md locked decisions):
- D-01: conservative transport-marker allowlist — not "any no-status error → fallback"; the helper matches only the validated transport/connectivity needles in `format!("{err:#}")` and anything unrecognized keeps `(true, false)`. Implemented in Task 1, step 1 (the validated lowercase needles inside `is_transport_failure`, incl. the rejected-needle exclusions from RESEARCH).
- D-01a: case-insensitive matching — `err_str.to_lowercase()` once, then `.contains(needle)` per lowercase needle; `extract_http_status` stays as-is. Task 1, step 1.
- D-02: repurpose `test_classify_other_error` (currently feeds `"Connection refused: failed to connect to LLM"` asserting `should_fallback == false`, which flips once the helper lands) to a clearly non-transport error string, keeping its intent (generic error → `(true, false)`); not deleted. Task 1, step 3 — and committed in the SAME atomic commit as adding the helper.
- D-02a: add new transport unit tests feeding connect-refused / connect-timeout / DNS-failure / connection-reset / `error sending request for url` strings, each asserting `(true, true)`. Task 1, step 3.
- D-03: detected transport markers return `(should_retry = true, should_fallback = true)` — `should_retry` stays `true` so the no-fallback path is behaviorally unchanged (still `MAX_RETRIES` retries with the existing backoff); `run()` is NOT modified. Task 1, step 2 (`return (true, Self::is_transport_failure(&err_str))`).
- D-04: new `crates/ironhermes-agent/tests/invariants_27_1_4_1_1.rs` (no dev-deps, `include_str!` pattern, modeled on `invariants_27_1_4_1.rs`) asserting BOTH that `agent_loop.rs` contains the transport-helper `fn` name AND the literal `error sending request for url`; failure messages name PROV-07 and cite phase 27.1.4.1.1. Task 2.
</objective>
