#### New helper implementation pattern (from RESEARCH.md validated allowlist)

```rust
/// Detect transport-level connectivity failures that carry no HTTP status code.
///
/// Called only when `extract_http_status` returns `None`. Matches a conservative
/// allowlist of reqwest/hyper-util/OS error substrings (case-insensitive).
/// Returns `true` → caller should return `(true, true)` (retry + fallback).
fn is_transport_failure(err_str: &str) -> bool {
    const TRANSPORT_MARKERS: &[&str] = &[
        "error sending request for url",
        "connection refused",
        "dns error",
        "failed to lookup address",
        "connection reset",
        "operation timed out",
        "tcp connect error",
    ];
    let lower = err_str.to_lowercase();
    TRANSPORT_MARKERS.iter().any(|m| lower.contains(m))
}
```

Key constraints:
- `to_lowercase()` called once on `err_str`; all needle comparisons use the lowercased string (D-01a)
- `"error sending request for url"` MUST be present verbatim — the static-invariant test (D-04) locks this literal
- Do NOT include bare `"timed out"` — it matches `"SSE stream read timed out after 60s"` (non-transport); use `"operation timed out"` instead (RESEARCH.md Pitfall 1)
