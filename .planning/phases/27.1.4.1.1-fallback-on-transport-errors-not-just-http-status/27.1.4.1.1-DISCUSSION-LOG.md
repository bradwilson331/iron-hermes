## Transport-error detection scope

| Option | Description | Selected |
|--------|-------------|----------|
| Conservative allowlist | Match a specific set of substrings (connection refused, connect-timeout, DNS errors, connection reset, reqwest `error sending request for url`); anything unrecognized stays (retry, no-fallback). Lowest blast radius; risks missing a marker variant on some platform/OS. | ✓ |
| Liberal: no status = fall back | If `extract_http_status` returns `None` at all, treat it as a connectivity failure → `should_fallback=true`. Catches every transport variant; risk is falling back on rare non-connectivity errors (e.g. serde error on a malformed response body). | |
| Allowlist + reqwest-source check | Conservative substring allowlist plus a downcast to `reqwest::Error` (`is_connect()`/`is_timeout()`/`is_request()`) as a structured signal. More robust, but only works if the `reqwest::Error` survives anyhow wrapping (unverified). | |

**User's choice:** Conservative allowlist
**Notes:** Captured as D-01 / D-01a in CONTEXT.md. Case-insensitive matching for the new transport helper; `extract_http_status` left as-is. The reqwest-source-check option was noted as a Deferred Idea.

## Retry behavior for transport errors

| Option | Description | Selected |
|--------|-------------|----------|
| Keep `should_retry = true` → `(true, true)` | Matches the ROADMAP wording. When fallback IS configured, fallback fires first anyway. When NOT configured, behavior unchanged: retries `MAX_RETRIES` (3) times with backoff then errors. ~3s wasted on a localhost refused, but zero behavior change to the no-fallback path. | ✓ |
| `should_retry = false` → `(false, true)` | Fail fast: a connection refused won't be cured by retrying the dead endpoint. When not configured, errors immediately instead of after ~3s. Slight behavior change to the no-fallback path. | |

**User's choice:** Keep `should_retry = true` → `(true, true)`
**Notes:** Captured as D-03. Fail-fast noted as a Deferred Idea.

## Claude's Discretion

- Exact `fn` name for the transport-detection helper (the D-04 invariant test locks whatever name ships).
- Exact final allowlist spellings — researcher verifies against current reqwest/hyper/std::io error formats; must include `error sending request for url`, `connection refused`, a connect-timeout marker, a DNS marker, and a connection-reset marker.
- Helper signature: `&str` (reusing the existing `format!("{err:#}")` string) vs `&anyhow::Error` — `&str` preferred.
- Placement of the new transport unit tests (existing `classify_*` test `mod` vs a small new `mod`).

## Deferred Ideas

- Structured `reqwest::Error` source-chain inspection (`is_connect()`/`is_timeout()`/`is_request()`) instead of/alongside substring matching.
- Fail-fast for transport errors when no fallback is configured (`(false, true)` instead of `(true, true)`).
- Honoring `fallback_providers` chains longer than one entry (`run()`'s fallback activation is currently one-shot).
