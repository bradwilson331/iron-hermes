## Shared Patterns

### ReadSignal prop coercion (applies to all new props in AgentPanel)
**Source:** `agent_panel.rs` line 21 receiving `ReadSignal<Vec<Message>>` while `warp_hermes.rs` line 809 passes `Signal<Vec<Message>>`
**Apply to:** All new `ReadSignal<T>` props — `active_side_tab`, `session_id`, `token_budget`

Dioxus 0.7 automatically coerces `Signal<T>` at the call site into `ReadSignal<T>` when the prop type is `ReadSignal<T>`. Do NOT call `.into()` or change prop types to `Signal<T>`. Pass the signal variable directly:
```rust
// warp_hermes.rs — pass Signal<T> directly; Dioxus coerces to ReadSignal<T>:
AgentPanel {
    session_id: session_id,       // Signal<String> → ReadSignal<String>
    token_budget: tokens,         // Signal<TokenBudget> → ReadSignal<TokenBudget>
    active_side_tab: active_side_tab,  // Signal<usize> → ReadSignal<usize>
}
```

### Signal read safety (applies to all RSX that calls `.read()`)
**Source:** `crates/iron_hermes_ui/CLAUDE.md` — "do NOT hold `GenerationalRef` across `.await` points"
**Apply to:** All `.read()` calls in AgentPanel RSX

All signal reads in this phase are synchronous in RSX expressions — no `.await` involved. `messages.read().len()` and `token_budget.read().used` inside `rsx!` are safe. No async paths are introduced.

### CSS class namespace isolation (applies to all new CSS classes)
**Source:** CONTEXT.md D-05; `warp-ih.css` lines 93-126
**Apply to:** Every new CSS class name

New classes use the `.wh-side-*` prefix family. Do NOT use `.wh-tab`, `.wh-tabs`, `.wh-tab-dot`, or any `.wh-tab-*` class name. These are the TitleBar tab strip classes and must remain isolated.

### Design token usage (applies to all new CSS)
**Source:** `warp-ih.css` lines 16-38
**Apply to:** All new CSS rule values

Never write raw hex color literals in new CSS rules. Always use `var(--token-name)`. The only exception is `background: transparent` and `border: none` which are CSS keywords, not hex values.
