# Phase 26.4: web-ui-side-tabs-panel - Context

**Gathered:** 2026-05-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a two-tab navigation strip to the `AgentPanel` (the right-side `.wh-side` panel) so it can show two views: "Agent" (the existing scrollable messages + tool calls list) and "Info" (session stats + config snapshot).

The tab strip sits **below** the existing `.wh-side-head` header — the Sigil, "HERMES" label, and personality pill are preserved unchanged. The new tabs use new `.wh-side-tab` CSS classes (not the TitleBar's `.wh-tab` classes). Active tab state is a single global signal in `WarpHermes`.

Out of scope:
- Memory tab (would need a new server function to read MEMORY.md)
- Sessions history tab (would need `list_sessions` data piped into the side panel)
- Per-session side tab state
- Any new server functions this phase — Info tab data comes entirely from existing signals

</domain>

<decisions>
## Implementation Decisions

### Tab Views
- **D-01:** Two tabs — "Agent" and "Info". No other tabs this phase.
- **D-02:** "Agent" tab shows the existing agent messages + tool calls list (current `AgentPanel` content, unchanged).
- **D-03:** "Info" tab shows two grouped card sections: a **Session card** (session id, message count, tokens used/max) and a **Config card** (model, provider, context window size, memory-enabled flag).

### Tab Strip Placement
- **D-04:** The tab strip is placed **below** `.wh-side-head` — the existing header (Sigil + "HERMES" + personality pill) is not replaced or modified.
- **D-05:** New `.wh-side-tab` and `.wh-side-tabs` CSS classes — do NOT reuse TitleBar's `.wh-tab` / `.wh-tabs`. Side panel tabs need full-width layout and panel-specific sizing distinct from the title bar tab strip.
- **D-06:** Active tab indicator: 2px bottom border in `var(--accent-primary)` (cyan `#39c5cf`). Matches the 2px left accent stripe used on block types throughout the shell.

### Info Tab Data Source
- **D-07:** Info tab reads entirely from existing `WarpHermes` signals and `ShellSettings` context — **no new server functions**:
  - `session_id: Signal<String>` → session id display
  - `tokens: Signal<TokenBudget>` → `used` and `max` token counts
  - `messages: Signal<Vec<Message>>` → `.read().len()` for message count
  - `ShellSettings` context → personality label (already available in AgentPanel via `use_context`)
  - Model/provider/memory fields: pass as new props from `WarpHermes` (string values from config available at root)
- **D-08:** Info tab renders as **styled info cards** — two sections with a section heading, then key-value rows. Not a raw key-value dump.

### Tab State
- **D-09:** `active_side_tab: Signal<usize>` is a single **global signal** in `WarpHermes` (index 0 = Agent, 1 = Info). Switching TitleBar session tabs does NOT reset or affect the side panel tab. One signal, app-wide.

### Brand / Design Reference
- **D-10:** Follow the existing brand/design system: Ioskeley Mono font, dark background (`#0d1117`), `var(--accent-primary)` cyan (`#39c5cf`), established CSS design token vocabulary from `warp-ih.css`.

### Claude's Discretion
- Exact `.wh-side-tab` CSS values (padding, font-size, height, border-width) — follow existing token vocabulary; match `.wh-side-head` height rhythm.
- Tab label casing ("Agent" / "Info" vs "AGENT" / "INFO") — follow existing side panel conventions (e.g., "HERMES" is all-caps; decide based on visual fit).
- Info card section header styling (subtle separator or bold label) — match the `wh-msg-meta` label pattern already in the panel.
- Whether personality pill stays in `.wh-side-head` or moves into the Agent tab's sub-header — keep in `.wh-side-head` (simpler, no layout change needed).
- `AgentPanel` component signature changes — planner decides which new props to add (`active_side_tab`, `session_id`, `token_budget`, `model_label`, `provider_label`, `memory_enabled`).

</decisions>

  ```
  +------------------+
  | ✦ HERMES /concise|   ← .wh-side-head unchanged
  | [Agent] [Info]   |   ← new .wh-side-tabs strip

  | messages...      |   ← .wh-side-scroll (or Info card content)
  ```

- Info tab has two card sections: "Session" and "Config" — not a flat key-value dump; each section has a styled heading
- No new server functions — all Info data from existing signals
