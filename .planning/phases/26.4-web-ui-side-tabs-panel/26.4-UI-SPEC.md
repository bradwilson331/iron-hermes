## Spacing Scale

Declared values aligned with the existing monospace cell scale (1 cell ≈ 8px).
The project uses `--space-N` tokens from `design-tokens.css`, all multiples of 4:

| Token      | Value | CSS Var       | Usage                                                     |
|------------|-------|---------------|-----------------------------------------------------------|
| xs         | 4px   | `--space-1`   | Icon gaps, inline icon padding                            |
| sm         | 8px   | `--space-2`   | Tab strip vertical padding, info row gaps                 |
| md         | 16px  | `--space-3`   | Info card internal padding, section spacing               |
| lg         | 24px  | `--space-4`   | Section gap between Session and Config cards              |
| xl         | 32px  | `--space-5`   | Not used in this phase                                    |
| 2xl        | 48px  | `--space-6`   | Not used in this phase                                    |
| 3xl        | 64px  | n/a           | Not used in this phase                                    |

**Phase-specific layout dimensions (from existing patterns in `warp-ih.css`):**

| Element                  | Value   | Rationale                                                      |
|--------------------------|---------|----------------------------------------------------------------|
| `.wh-side-tabs` height   | 32px    | Matches `.wh-titlebar` height (32px); sits below 36px `.wh-side-head` — one step tighter is appropriate for a secondary strip |
| `.wh-side-tab` padding   | `0 16px`| Design contract value (grid-aligned). Implementation note: executor may match legacy `.wh-tab { padding: 0 14px }` from existing CSS if visual parity with `.wh-tab` is required. |
| Tab active indicator     | 2px     | Bottom border — matches the 2px top stripe on `.wh-tab.is-active::before` and the 2px left stripe on `.wh-block::before` |
| Info card padding        | `8px 12px` | Design contract value (grid-aligned). Implementation note: executor may match legacy `.wh-toolcall { padding: 8px 10px }` from existing CSS if exact visual parity with tool call cards is required. |
| Info card row gap        | 4px     | Matches `.wh-msg { gap: 4px }` — consistent within-item spacing |
| Inter-card gap           | 8px     | Matches `.wh-side-scroll { gap: 10px }` rounded to 8px for slightly tighter info layout |

Exceptions: none beyond the phase-specific dimensions above.

Source: CONTEXT.md D-05, D-06; `warp-ih.css` measured values.

## Typography

All type is Ioskeley Mono. The project declares exactly 4 type sizes via `--type-*` tokens.
This phase uses 3 of the 4:

| Role               | Size   | CSS Var         | Weight | Line Height          | Usage                                                |
|--------------------|--------|-----------------|--------|----------------------|------------------------------------------------------|
| Body               | 14px   | `--type-body`   | 400    | 1.5 (`--leading-normal`) | Info card value text, message body in Agent tab   |
| Label / Meta       | 12px   | `--type-small`  | 700    | 1.2 (`--leading-tight`)  | Tab labels, info card section headings, key names |
| Micro / Hint       | 10px   | n/a (literal)   | 400    | 1.2                  | Personality pill, secondary metadata               |

**Tab label casing: UPPERCASE** — consistent with existing panel conventions. `.wh-side-head .wh-side-title` renders "HERMES" in all-caps with `letter-spacing: .04em`. Tab labels follow this pattern: `AGENT` and `INFO`.

Source: CONTEXT.md Claude's Discretion (tab casing); `design-tokens.css` type scale; `warp-ih.css` `.wh-side-head .wh-side-title` rule.

## Interaction Contract

### Tab Switching
- Click `.wh-side-tab` → calls `on_side_tab_click: EventHandler<usize>` prop (same pattern as `TitleBar.on_tab_click`)
- Active tab index stored in `active_side_tab: Signal<usize>` in `WarpHermes` (global, not per-session)
- Switching TitleBar session tabs does NOT reset `active_side_tab`
- No animation on tab switch — immediate content swap via RSX conditional
- Hover: `background: var(--w-bg-3)` on inactive tabs only
- `cursor: default` on all tab buttons (terminal aesthetic, no pointer)

### Active State
- `.wh-side-tab.is-active` class applied when `i == active_side_tab()`
- Active tab: `color: var(--fg-strong)` + 2px bottom border `var(--accent-primary)`
- Inactive tab: `color: var(--fg-dim)`, no border

### Info Card Data
- All data read-only — no editable fields, no actions within the Info tab
- Token display format: `"{used} / {max}"` — no progress bar (keep it textual, matches panel style)
- Session ID: full string as-is from `session_id: ReadOnlySignal<String>`
- Memory enabled: display as `"yes"` or `"no"` (lowercase, consistent with terminal aesthetic)
- Scroll: `.wh-side-info` has `overflow-y: auto` for small windows

Source: CONTEXT.md D-07, D-08, D-09; `warp-ih.css` cursor pattern.
