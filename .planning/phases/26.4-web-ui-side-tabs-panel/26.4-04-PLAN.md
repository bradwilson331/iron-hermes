## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WarpHermes (root composer) → AgentPanel (child) | Props (signals + values) cross from root scope to child via Dioxus prop coercion |
| Browser DOM event → on_side_tab_click closure | Mouse click event on a button element triggers the EventHandler callback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26.4-11 | Tampering | active_side_tab signal value | accept | The signal is internal to the component tree; it cannot be tampered with from outside the WASM boundary; index out-of-range (e.g., 999) gracefully falls into the `else` branch (renders INFO view) — no crash |
| T-26.4-12 | Spoofing | on_side_tab_click closure invocation | accept | The closure is captured by the AgentPanel only; it cannot be invoked from another origin; Dioxus's EventHandler model wraps it in a generationally-tracked handle |
| T-26.4-13 | Denial of Service | Rapid tab switching | accept | Each click sets a usize signal — O(1) work; no expensive recomputation; Dioxus reconciler diffs only the changed subtree (the conditional branch in AgentPanel) |
| T-26.4-14 | Information Disclosure | model_name / provider_name / context_length / memory_enabled passed via props | accept | All four values originate from `/api/config` (Plan 01 threat T-26.4-01 already analyzed and accepted); same exposure as already shipped via `StatusBar` (model + provider) — Plan 04 only widens `AgentPanel` to receive the same data |
| T-26.4-15 | Repudiation | No audit trail of tab clicks | accept | UI-only state change; not security-relevant; no persistence |
</threat_model>
