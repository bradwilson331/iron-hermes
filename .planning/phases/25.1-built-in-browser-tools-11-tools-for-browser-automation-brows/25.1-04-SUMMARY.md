## Key Implementation Notes

### chromiumoxide page.goto() return type

`page.goto(url).await` returns `anyhow::Result<()>` — it does NOT expose an HTTP status code. On success the method returns `Ok(())`, on failure (4xx/5xx, network error, navigation timeout) it returns `Err`. The `"status": 200` in the success envelope is therefore hardcoded. Plan 11 integration test authors should not assert on the numeric status value beyond `!response.contains("error")`.

### wait_for_navigation() semantics

`page.wait_for_navigation().await` in chromiumoxide 0.9 awaits the browser's next navigation-completed event. Since `goto()` already internally awaits DOMContentLoaded before returning, `wait_for_navigation()` is a redundant best-effort wait. Its result is intentionally ignored (`let _ = ...`). The `wait_until` argument is accepted and validated in the schema but is informational in v2.1 — "networkidle" is a future enhancement per plan spec.

### Structured rejection envelope routing

`BrowserSession::validate_navigation_url` (plan 02) serializes the rejection as JSON in the `anyhow::Error` message. `browser_navigate::execute` catches this as `Err(e)` and returns `Ok(e.to_string())` — the LLM receives valid JSON (`{"error":"domain_blocked",...}`) as the tool output rather than an Err propagation which would surface as an internal error.

### Scheme extraction edge cases

The D-16 scheme check handles three URL forms:
1. `https://example.com` — `split_once("://")` extracts `https`
2. `javascript:alert(1)` — no `://`; fallback `split_once(':')` extracts `javascript`
3. `data:text/html,...` — no `://`; fallback extracts `data`

All three are rejected by the default `["http", "https"]` allowlist.

### D-03 respawn contract

After `browser_close` executes, `*guard` is `None`. The `ensure_session()` helper in `browser_navigate` (and the same pattern in all other browser tools) checks `guard.is_none()` and calls `BrowserSession::spawn()` on the next invocation — satisfying the D-03 lazy-respawn contract without any explicit "respawn" call.
