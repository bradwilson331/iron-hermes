<root_cause_already_isolated>
The validator data structure backing the error string `"known toolsets: agent, code, memory, session, skills, web"` is a hardcoded `&[&str]` slice at the top of `crates/ironhermes-cli/src/toolset_cmd.rs`:

```rust
// Line 14-15
/// D-01: The six concrete toolsets shipped in v2.1.
const KNOWN_TOOLSETS: &[&str] = &["web", "code", "memory", "agent", "skills", "session"];
```

This slice is consumed by:
1. `check_known_toolset()` (line 64-75) — gates `enable`, `disable`, `show`.
2. `build_toolset_rows()` (line 244-249) — controls which toolsets appear in `hermes toolset list`.
3. `validate_toolset_name_accepts_known_d01_names` test (line 341-352) — iterates over the slice.

Adding `"browser"` to the slice fixes BOTH the `list` output (test 2) and the `enable` command (test 3) in a single edit, plus enables `disable` and `show` for the browser toolset.

The doc comment on line 14 (`/// D-01: The six concrete toolsets shipped in v2.1.`) is also stale and must be updated to reference D-04's seven-toolset extension.
</root_cause_already_isolated>
