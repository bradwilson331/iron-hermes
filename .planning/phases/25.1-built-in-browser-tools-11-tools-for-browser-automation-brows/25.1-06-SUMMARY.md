# Phase 25.1 Plan 06: browser_click + browser_type Summary

Two ref-consuming interaction tools that close the "discovery → action" loop. Both look up `ref` in `BrowserSession.ref_table` (populated by plan 05's OQ-1 fix with `[data-ironhermes-ref="N"]` CSS selectors), call `page.find_element(selector)` via chromiumoxide, and converge ALL failure modes onto the D-11 `element_stale` envelope.

## Key Implementation Notes

### chromiumoxide API verification

**`Element::type_str(&str)` accepts `&str` directly** — no `.into()` needed. `cargo check` passes with `el.type_str(&text).await?` where `text: String`. The method signature takes `impl AsRef<str>`, so both `&str` and `&String` work.

**`Element::focus()` exists as a native chromiumoxide method** — no JS workaround required. `el.focus().await?` compiles cleanly. The plan's action code was correct.

### element_stale envelope semantics

Both tools return `Ok(element_stale_json)` not `Err(...)` for all stale-ref conditions. This is intentional: an `Err` would propagate through the agent loop as a tool execution failure, while the LLM needs to receive the structured JSON to understand it should call `browser_snapshot` first. The envelope shape matches D-11 exactly:

```json
{"error":"element_stale","ref":N,"hint":"ref N not found in current snapshot — call browser_snapshot first"}
```

### T-25.1-03 mitigation coverage

Both tools handle the full stale-ref threat surface:
- **Cache miss** (ref not in ref_table): immediate element_stale return — ref was from an older snapshot
- **find_element error**: element_stale — DOM mutated between snapshot and click (T-25.1-03 attacker-controlled DOM mutation)
- **click/focus/type_str error**: element_stale — element became detached after find_element succeeded

### browser_type clear + type flow

The clear-before-type approach uses JS `el.value = ''` + `dispatchEvent(new Event('input'))` rather than keyboard select-all+delete. This is more robust across input types (contenteditable divs, React-controlled inputs that listen to the `input` event, native `<input>` elements). The type_str call then types char-by-char via CDP keyboard events — matching real user input dispatching.

### Note for plan 11 integration tests: canonical stale-ref reproduction

The canonical way to reproduce a stale ref for integration testing:
1. `browser_navigate` to page A → `browser_snapshot` → capture ref N
2. `browser_navigate` to page B (clears ref_table in browser_navigate per plan 04 implementation)
3. `browser_click` ref N → should return `element_stale` envelope

The ref_table clear on navigate is the reliable trigger — no need to mutate the DOM.
