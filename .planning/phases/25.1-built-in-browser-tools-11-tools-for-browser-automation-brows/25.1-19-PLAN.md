<objective>
Close GAP-8 from `25.1-UAT.md`: the local rata-based chat REPL (`ironhermes_cli::tui_rata::run_chat_ratatui`) does not register the 11 browser_* tools and does not wire `with_browser_session(...)` on its AgentLoop, so the model's tool schema in `ironhermes chat` does not contain `browser_navigate`/`browser_snapshot`/`browser_click`/etc. The result is the model says "I can't browse the web" when asked to navigate, which blocks UAT tests 7, 8, and 10 on the chat-REPL path.

Root cause (definitively confirmed by `grep -rn 'register_browser_tools\|with_browser_session\|browser_session' crates/ironhermes-cli/src/tui_rata/` returning ZERO matches): browser-tool wiring exists in three sibling paths in `crates/ironhermes-cli/src/main.rs`:

- `run_single` (main.rs:677) — browser tools wired
- `run_chat` (main.rs:1178) — browser tools wired
- `run_gateway` (main.rs:2139) — browser tools wired

But `Commands::Chat` dispatches to `tui_rata::run_chat_ratatui` (main.rs:355 + main.rs:437), NOT to `run_chat`. The rata path's `build_app_deps` builds a ToolRegistry with `register_cronjob_tool`, `register_skills_tool`, `register_memory_tool`, `register_delegate_task_tool`, web_search/web_read, and `register_execute_code_tool_with_process_registry` — but NEVER calls `register_browser_tools_with_vision`. Consequently the chat REPL's tool schema omits the 11 browser_* tools regardless of toolset enable/disable state. Test 5's earlier pass was on a non-rata path; the rata REPL became the default `Commands::Chat` handler in Phase 22.4.4 and was never re-verified for browser-tool exposure.

The fix is a single-file-plus-one-supporting-file mirror of `run_chat`'s wiring block (main.rs:1170-1184) into `build_app_deps`, plus a session-scoped browser_session field on `AppDeps` / `App` so the same Arc reaches BOTH the App-level AgentLoop builder (event_loop.rs:314-323) AND the per-turn AgentLoop builder (event_loop.rs:572-583 inside `spawn_turn`). The per-turn builder is the one that actually executes each user message, so without the wire on it, browser tool calls would race a fresh BrowserSession singleton per turn and break D-03's "one browser per AgentLoop" invariant.

Purpose: Restore the Phase 22 D-04 mirror-pattern invariant ("browser tools wired identically across all entry points") for the chat REPL. Unblock UAT tests 7, 8, and 10 — currently `blocked: gap-8` on the chat-REPL path — so GAP-3, GAP-4, and GAP-5 closure (already shipped via plans 13/14/15) can finally be verified end-to-end through `ironhermes chat`.

Output: One new field on `AppDeps` + one matching field on `App`, the 5-line wiring block lifted verbatim from main.rs:1173-1184 into `build_app_deps`, two `.with_browser_session(...)` chains on the two AgentLoop builders inside `tui_rata/event_loop.rs`, an INV regression test that locks the wiring against future deletion, and a unit test that asserts the registry built by the rata bootstrap actually contains `browser_navigate` in `get_definitions()`.
</objective>
