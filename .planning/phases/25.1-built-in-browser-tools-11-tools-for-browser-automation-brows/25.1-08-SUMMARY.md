# Phase 25.1 Plan 08: browser_vision Summary

One-liner: VisionClientHandle trait + BrowserVisionTool that screenshots full-page PNG, base64-encodes to data URL, and dispatches via Arc<dyn VisionClientHandle> (plan 09 wires the resolve_role("vision") cascade).

## What Was Built

`crates/ironhermes-tools/src/browser_vision.rs` (337 lines) implements:

1. **`VisionClientHandle` trait** — dependency-inversion trait that breaks the `ironhermes-agent` → `ironhermes-tools` circular dependency. Defined in `ironhermes-tools`; implemented in `ironhermes-agent` (plan 09).

2. **`BrowserVisionTool`** — full `Tool` trait implementation:
   - `name()` → `"browser_vision"`, `toolset()` → `"browser"`
   - `is_available()` → `find_chromium_binary(None).is_some() && self.vision_capable()`
   - `vision_capable()` → `resolve_role("vision").is_some() || main.capabilities.vision`
   - `prerequisites()` → two entries: `binary_present/chromium-or-chrome` + `config_field/auxiliary.vision OR multimodal-capable main provider`
   - `execute()` → full pipeline: lock session → `ScreenshotParams::builder().format(Png).full_page(true)` → base64 → `data:image/png;base64,...` URL → drop lock → `vision_client.vision_call(prompt, data_url)` → JSON envelope

3. **4 unit tests** (all passing):
   - `name_and_toolset_match_d04`
   - `prerequisites_declare_chromium_and_vision_role`
   - `schema_prompt_is_optional`
   - `vision_capable_with_default_resolver_reflects_main_provider_metadata`

### Architectural Deviation (OQ-5 — Plan-documented)

**VisionClientHandle trait instead of direct ironhermes-agent dep**
- **Reason:** `ironhermes-agent` already depends on `ironhermes-tools` (line 17 of ironhermes-agent/Cargo.toml). Adding `ironhermes-agent` to `ironhermes-tools` would create a cycle.
- **Chosen path:** Define `pub trait VisionClientHandle` in `ironhermes-tools/src/browser_vision.rs`, implement in `ironhermes-agent` plan 09.
- **Impact:** `BrowserVisionTool::new()` takes `Arc<dyn VisionClientHandle>` as third argument. Plan 09 (AgentLoop wiring) must construct the impl and pass it.
- **Cargo.toml:** No change needed — `ironhermes-core` (with `ProviderResolver`) was already a dep, sufficient for `is_available()` / `vision_capable()`.
