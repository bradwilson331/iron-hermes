## Architecture Note

The rata REPL requires browser-session wiring on **both** AgentLoop builders because they serve different roles:
- **App-level `agent_loop`** (line ~314): stored on `App.agent_loop`, used by integration code and slash-command handlers that call methods directly on it (e.g., checking state). It also participates in D-17 browser_close drop semantics.
- **Per-turn `AgentLoop` in `spawn_turn`** (~line 572): this is the load-bearing wire — every user message is executed through a fresh `AgentLoop` built here. Without `.with_browser_session(...)` on *this* builder, browser tool calls would never reach the BrowserSession singleton, breaking D-03's "one browser per AgentLoop" invariant.

Both builders share the **same Arc** (cloned at construction), so the lazy-spawn lifecycle (D-03) is preserved: the first `browser_navigate` call on any turn spawns chromium once and stores the `BrowserSession` in `Arc<Mutex<Option<...>>>`, subsequent turns reuse it through the same Arc, and `browser_close` re-sets the `Option` to `None` so the next nav spawns a fresh instance.
