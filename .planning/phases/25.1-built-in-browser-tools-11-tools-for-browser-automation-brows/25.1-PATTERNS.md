### `crates/ironhermes-tools/src/browser_session.rs` (shared-state — NEW pattern, no direct analog)

**Recommended new pattern** — modeled on the `todo_state` pattern in `agent_loop.rs` (lines 388-420) and `TerminalTool::with_process_registry` (terminal.rs lines 50-55), but for a stateful resource that is lazy-spawned:

```rust
// browser_session.rs — new file, no analog in codebase
use std::collections::HashMap;
use chromiumoxide::browser::Browser;
use chromiumoxide::page::Page;

/// Lazy-spawned CDP session. One per AgentLoop instance.
/// Held behind Arc<tokio::sync::Mutex<Option<BrowserSession>>> so
/// all 11 browser_* tools share the same browser process.
pub struct BrowserSession {
    pub browser: Browser,
    pub page: Page,
    /// Ref table: sequential integer → BackendNodeId string.
    /// Populated by browser_snapshot, cleared at the start of each snapshot call.
    pub ref_table: HashMap<u64, String>,
    /// CDP websocket pump task handle — dropped when BrowserSession is dropped.
    pub _handler: tokio::task::JoinHandle<()>,
}

/// D-05 binary discovery: walks env vars → PATH → platform paths.
/// Returns None if no chromium binary found (is_available() = false).
pub fn find_chromium_binary() -> Option<std::path::PathBuf> {
    // 1. BROWSER_PATH env var
    if let Ok(p) = std::env::var("BROWSER_PATH") {
        let path = std::path::PathBuf::from(&p);
        if path.is_file() { return Some(path); }
    }
    // 2. CHROMIUM_PATH env var
    if let Ok(p) = std::env::var("CHROMIUM_PATH") {
        let path = std::path::PathBuf::from(&p);
        if path.is_file() { return Some(path); }
    }
    // 3. Inline PATH search — no `which` crate (avoids new dep per RESEARCH.md OQ-4)
    for name in &["chromium-browser", "chromium", "google-chrome", "chrome"] {
        if let Some(p) = std::env::var("PATH").ok()
            .and_then(|path_var| {
                path_var.split(':').find_map(|dir| {
                    let p = std::path::PathBuf::from(dir).join(name);
                    p.is_file().then_some(p)
                })
            })
        {
            return Some(p);
        }
    }
    // 4. Platform-specific hard paths
    #[cfg(target_os = "macos")]
    for p in &[
        "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome",
        "/Applications/Chromium.app/Contents/MacOS/Chromium",
    ] {
        let path = std::path::PathBuf::from(p);
        if path.is_file() { return Some(path); }
    }
    #[cfg(target_os = "linux")]
    for p in &[
        "/usr/bin/chromium", "/usr/bin/chromium-browser",
        "/usr/bin/google-chrome", "/snap/bin/chromium",
    ] {
        let path = std::path::PathBuf::from(p);
        if path.is_file() { return Some(path); }
    }
    None
}
```

**Key guidance:** The `Arc<tokio::sync::Mutex<Option<BrowserSession>>>` type is the exact parallel of `Arc<Mutex<Vec<String>>>` for `todo_state` in agent_loop.rs. The Option wrapper enables lazy initialization: `None` until first browser_* call, `Some(session)` after first call, `None` again after `browser_close`.
