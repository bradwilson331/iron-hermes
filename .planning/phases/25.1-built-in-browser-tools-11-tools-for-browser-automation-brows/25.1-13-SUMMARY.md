# Phase 25.1 Plan 13: BROWSER_PATH/CHROMIUM_PATH Authoritative Override Summary

One-liner: Surgical 4-line fix makes BROWSER_PATH and CHROMIUM_PATH authoritative env-var overrides per D-05, preventing fall-through to system Chrome when operator sets an invalid path.

## What Was Done

Closed GAP-2 from `25.1-UAT.md`: `find_chromium_binary` in `crates/ironhermes-tools/src/browser_session.rs` was silently falling through from an invalid `BROWSER_PATH` to platform-specific paths (finding system Chrome at `/Applications/Google Chrome.app/...`) instead of returning `None` as D-05 specifies.

## Exact Diff of Env-Var Branches

**Before (broken — falls through on invalid path):**
```rust
// 1. BROWSER_PATH env var
if let Ok(p) = std::env::var("BROWSER_PATH") {
    let path = PathBuf::from(&p);
    if path.is_file() { return Some(path); }
}
// 2. CHROMIUM_PATH env var
if let Ok(p) = std::env::var("CHROMIUM_PATH") {
    let path = PathBuf::from(&p);
    if path.is_file() { return Some(path); }
}
```

**After (correct — authoritative early-return):**
```rust
// 1. BROWSER_PATH env var (D-05 step 1: authoritative when set).
//    Empty string treated as unset (POSIX convention: BROWSER_PATH= == not exported).
//    When set to a non-empty value: return Some(path) if valid file, else return None.
//    Do NOT fall through to subsequent sources — the operator's intent is explicit.
if let Some(p) = std::env::var("BROWSER_PATH").ok().filter(|s| !s.is_empty()) {
    let path = PathBuf::from(&p);
    return if path.is_file() { Some(path) } else { None };
}
// 2. CHROMIUM_PATH env var (D-05 step 2: authoritative when set).
//    Same authoritative semantics as BROWSER_PATH above.
if let Some(p) = std::env::var("CHROMIUM_PATH").ok().filter(|s| !s.is_empty()) {
    let path = PathBuf::from(&p);
    return if path.is_file() { Some(path) } else { None };
}
```

The key change: `if let Ok(...) { if valid { return Some } }` becomes `if let Some(...) { return if valid { Some } else { None }; }`. The `return` is inside the branch, so when the env var is set we commit to it — either `Some(path)` or `None`. Fall-through only occurs when the env var is unset or empty.

## Empty-String Env-Var Handling Decision

`BROWSER_PATH=` (set to empty string) is treated as **unset** via `.ok().filter(|s| !s.is_empty())`.

**Rationale:** In POSIX shell, `export BROWSER_PATH=` is semantically identical to `unset BROWSER_PATH` for tools checking "did the operator configure this?" — an empty value is not a meaningful path. Using `is_ok()` alone would capture empty-string exports and commit to returning `None` for `BROWSER_PATH=""`, which would be surprising. The filter ensures only a non-empty BROWSER_PATH triggers authoritative behavior, matching operator intent.
