#### 1. UAT Cycle 4 — End-to-End Chat REPL Browser Navigation

**Test:** Launch `ironhermes chat`, ensure `browser` toolset is enabled (`hermes toolset enable browser` or `tools.browser.enabled: true` in config). Ask: "Navigate to https://example.com and tell me the page title."

**Expected:** The model invokes `browser_navigate` (tool call visible in logs/output), Chromium spawns, page loads, and the model returns the page title from the snapshot. The pre-fix behavior was the canned "I can't browse the web or navigate to external websites directly."

**Why human:** Requires a live Chromium binary, running terminal TTY, and observable LLM tool-call behavior. The code fix (GAP-8) is fully verified programmatically — this test confirms the fix reaches the running binary.

#### 2. UAT Test 7 — URL Allowlist Enforcement (D-15 / T-25.1-01)

**Test:** Set `browser.allowed_domains: ["example.com"]` in the config file. In `ironhermes chat` with browser enabled, ask the agent to navigate to `https://wikipedia.org/` (should fail) and `https://example.com/` (should succeed).

**Expected:** wikipedia.org navigation returns a `domain_blocked` structured error envelope (not a successful response). example.com navigation succeeds. Verifies GAP-3 fix flows through to the chat REPL path (the allowlist is in the injected Arc<Config>).

**Why human:** Requires editing the config file, running Chromium, and observing the error envelope returned to the model. The code fix is verified; the runtime config-passthrough path must be exercised live.

#### 3. UAT Test 8 — browser_console eval Gating (D-13 / T-25.1-02)

**Test:** With `autonomous.yolo: false` in config (or yolo flag off), in `ironhermes chat`, ask the agent to evaluate `document.title` via browser_console. Then repeat with yolo=true. Also confirm `mode:log` works without any prompt.

**Expected:** yolo=false + mode=eval returns an `approval_needed` envelope; yolo=true + mode=eval returns the actual JS result; mode=log always returns console entries without any approval gate.

**Why human:** Requires toggling yolo config, running Chromium, and observing the approval-envelope vs. execution-result difference. The code fix (GAP-4) is verified; this test exercises the full runtime path.

#### 4. UAT Test 10 — browser_close Idempotency + Re-spawn (D-03)

**Test:** In `ironhermes chat` with browser toolset enabled, navigate to a page → close the browser → navigate again → confirm second navigation spawns a fresh Chromium session. Also close twice — confirm no error on the second close.

**Expected:** Two-close sequence returns `{"closed": true}` twice without error. Post-close navigation lazily re-spawns Chromium (D-03 invariant). Confirms plan 25.1-15 description fix and D-03 lazy-spawn lifecycle work end-to-end on the rata path.

**Why human:** Requires observing lazy-spawn lifecycle across multiple REPL turns and verifying browser_close description fix enables the model to actually invoke it.
