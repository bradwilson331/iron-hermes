## Decisions Made

- **Option (a) per UAT GAP-6 missing block** — fix scoped entirely to `toolset_cmd.rs`. No edits to `register_defaults`, `ToolRegistry`, `find_chromium_binary`, or any `browser_*` tool file. Confirmed via `git diff HEAD -- crates/ironhermes-tools/` returning 0 lines.
- **Single shared helper for the predicate** — `browser_chromium_unavailable()` consumed by both `build_toolset_rows` and `build_toolset_show_view`. Future drift between list/show is structurally impossible without deleting the helper.
- **Extract `build_toolset_show_view` as a pure function** — testable seam so the show-path contract can be unit-tested directly without spawning a subprocess or capturing stdout. Mirrors the seam pattern used elsewhere in this codebase (e.g., `run_memory_setup_with_io` in plan 20-03).
- **Green-arm test uses `std::env::current_exe()`** — `find_chromium_binary` uses `path.is_file()` only (no magic-byte / no executable-bit check); the test binary itself is always a real Mach-O/ELF, so the test passes regardless of executable-bit. Robust against any future tightening of the discovery function.
