<objective>
Close GAP-2 from `25.1-UAT.md`: `find_chromium_binary` in `crates/ironhermes-tools/src/browser_session.rs` does not honor D-05's "if set, must point to an executable file" authority for `BROWSER_PATH` (step 1) and `CHROMIUM_PATH` (step 2). When the operator sets `BROWSER_PATH=/dev/null/x` (intentionally invalid), the function falls through to the inline PATH search and platform paths and finds the system Chrome — silently overriding the operator's intent.

The contract from D-05 (CONTEXT.md, line 56-58):
> 1. `BROWSER_PATH` env var (if set, must point to an executable file)
> 2. `CHROMIUM_PATH` env var

"Must point to" is authoritative: when the var is SET, only that path is consulted. The current implementation treats env vars as "first try" rather than "if set, exclusive" — that violates D-05.

Purpose: Make BROWSER_PATH and CHROMIUM_PATH behave as authoritative overrides per D-05. Operators get a deterministic "no chromium found" behavior when they explicitly point at an invalid path, instead of mysteriously hitting their system browser.

Output: A 4-line surgical edit to `find_chromium_binary` plus one regression unit test that prevents this exact fall-through from recurring.
</objective>

The fix: when the env var is **set** (regardless of value), the function must commit to that source — return Some(path) on validity, return None on invalidity. Do NOT fall through.

Empty-string env var counts as "unset" per shell convention (e.g. `BROWSER_PATH=` exports empty, semantically identical to "not set"). Treat empty string as unset for ergonomics.

    ```rust
    // 1. BROWSER_PATH env var (D-05 step 1: authoritative when set).
    //    Empty string treated as unset (POSIX convention: BROWSER_PATH= == not exported).
    if let Some(p) = std::env::var("BROWSER_PATH").ok().filter(|s| !s.is_empty()) {
        let path = PathBuf::from(&p);
        return if path.is_file() { Some(path) } else { None };

    // 2. CHROMIUM_PATH env var (D-05 step 2: authoritative when set).
    if let Some(p) = std::env::var("CHROMIUM_PATH").ok().filter(|s| !s.is_empty()) {
        let path = PathBuf::from(&p);
        return if path.is_file() { Some(path) } else { None };
    }
    ```
