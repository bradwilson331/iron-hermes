# Phase 25.1 Plan 10: D-21 Schema-Exclusion Test + D-19 Default-Disabled Regression Gate Summary

**One-liner:** Deterministic schema-exclusion test suite (3 tests) guarding T-25.1-07 via IRONHERMES_BROWSER_TEST_DISABLE escape hatch and closed 11-tool enumeration constant.

## What Was Built

### `crates/ironhermes-tools/tests/browser_prereq.rs` (created, 183 lines)

Three integration tests:

1. **`browser_tools_excluded_when_chromium_missing`** (D-21 / T-25.1-07 mitigation): Sets `IRONHERMES_BROWSER_TEST_DISABLE=1`, constructs a registry with all 11 browser tools registered and the browser toolset explicitly enabled, calls `get_definitions(None)`, and asserts ALL 11 browser_* tool names are absent. Includes a closed-enumeration anchor: `assert_eq!(BROWSER_TOOL_NAMES.len(), 11)` which fails at runtime if a 12th tool is added without updating the constant.

2. **`browser_tools_excluded_when_toolset_disabled`** (D-04 toolset gate): Verifies the toolset-level gate independently — uses default `ToolsConfig` (browser disabled) and asserts all 11 names absent.

3. **`default_browser_toolset_disabled`** (D-19 cache-break-banner regression gate): Asserts `ToolsConfig::default().toolsets["browser"].enabled == false`. Fails fast if a config refactor silently flips the default, protecting the Phase 25 D-04 banner contract.

### `crates/ironhermes-tools/src/browser_session.rs` (modified)

Added `IRONHERMES_BROWSER_TEST_DISABLE` escape hatch at the top of `find_chromium_binary()`. When this env var is set, the function returns `None` immediately before any filesystem or PATH checks — making the tests deterministic on dev machines with Google Chrome installed at `/Applications/Google Chrome.app/...` (which would otherwise pass through the macOS platform-path fallback).

## Implementation Notes for Plan 11

- **`ProviderResolver::default()` does not exist**: `ProviderResolver` has no `Default` impl. Tests must use `ProviderResolver::build(&Config::default())`. This pattern is already established in `browser_vision.rs`'s internal `dummy_resolver()` helper and replicated here as `build_resolver()`.
- **`--test-threads=1`**: The `env_lock()` mutex serializes env mutations within the test binary, but `--test-threads=1` is required for `cargo test --test browser_prereq` to avoid races across the 3 tests.
- **D-22 SKIP guard**: Plan 11's live-integration tests should use the `chromium_available()` helper pattern from PATTERNS.md (checking `BROWSER_PATH`/`CHROMIUM_PATH`/`which chromium`) to skip gracefully in CI without a real Chromium binary — the D-21 deterministic tests in this plan are the CI-safe counterpart.
