## Decisions Made

- Used fully-qualified `ironhermes_state::StoredMessage` path in the helper signature rather than adding a `use` import — keeps the import block byte-stable and makes the server-only dep relationship explicit at the call site
- Query-param route style (`/api/session-messages?id=`) selected over path-param style — consistent with all existing `#[get]` routes in api.rs; no path-param routes exist anywhere in the file
- Filter before `.take(HISTORY_LIMIT)` is enforced by collecting the filtered iterator to `Vec<&StoredMessage>` first — Rust's `Filter` iterator does not implement `ExactSizeIterator`, making `.rev()` on it a compile error without the intermediate collect

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Iterator chain required intermediate collect for ExactSizeIterator**
- **Found during:** Task 2 (get_session_messages implementation)
- **Issue:** Plan's reference pattern used `.filter().rev().take(N).rev()` directly, but `Filter<Iter<_>, _>` does not implement `ExactSizeIterator`, so `.rev()` fails to compile (`Rev<Filter<...>>` cannot implement `DoubleEndedIterator` without `ExactSizeIterator`)
- **Fix:** Collect filtered results into `Vec<&ironhermes_state::StoredMessage>` first, then apply `.iter().rev().take(HISTORY_LIMIT).rev()` on the `Vec` which is `ExactSizeIterator`
- **Files modified:** crates/iron_hermes_ui/src/server/api.rs
- **Verification:** cargo check --features server exits 0
- **Committed in:** `1ebe61cc` (Task 2 commit)
