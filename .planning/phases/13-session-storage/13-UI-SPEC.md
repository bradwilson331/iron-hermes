## Phase Classification: Backend-Only

**This phase has no user-facing UI surface.**

Phase 13 implements `ironhermes-state` — a Rust crate providing SQLite-backed session persistence with FTS5 search, write-through caching, lineage tracking, export, pruning, and WAL contention handling. Every deliverable is a Rust public API method. There are no screens, no components, no visual states, and no web interface.

- Confirmed by REQUIREMENTS.md Out of Scope: "Web UI: CLI, Telegram, and ACP cover primary use cases"
- Confirmed by SESS-01 through SESS-11: all requirements describe data storage, query, and retry behavior
- Confirmed by RESEARCH.md: standard stack is `rusqlite`, `tokio`, `serde_json` — no UI libraries
- Confirmed by codebase scan: no `components.json`, `tailwind.config.*`, `postcss.config.*`, or `src/` web directory

**The "interface" this phase defines is a Rust public API contract, not a visual contract.**

### Snippet Format Contract

FTS5 search results use `snippet(messages_fts, 0, '<<', '>>', '...', 32)`.

Match markers: `<<matched text>>` — downstream consumers (Phase 17 session_search tool) must parse this format when rendering search results to users.

### Export JSON Format Contract

Single session export:
```
{ "session": { ...Session fields... }, "messages": [ ...StoredMessage array... ] }
```

Bulk export: array of the above structure. This format is stable — Phase 17's session_search tool and any future export commands depend on it.

### Error Behavior Contract

| Condition | Behavior | Source |
|-----------|----------|--------|
| SQLITE_BUSY after 5000ms timeout | Retry with jitter: 50ms, 125ms, then propagate `StateError::Sqlite` | CONTEXT.md D-09 |
| FTS5 malformed query (user input) | `sanitize_fts_query()` strips operators before execution; raw mode bypasses this | CONTEXT.md D-05 |
| Prune of active session | Only sessions with `ended_at IS NOT NULL` are eligible — active sessions never pruned | CONTEXT.md D-08 |
| Migration on existing schema | Idempotent — v7 migration guarded by `current_version < 7` check | CONTEXT.md D-09, RESEARCH.md Pitfall 4 |

## Copywriting Contract

Phase 13 produces no user-visible copy directly. However it defines the data structures consumed by Phase 17 (session_search tool) which will surface copy to users. The following strings are embedded in the API layer and will appear in tool output.

| Element | Value | Consumer |
|---------|-------|---------|
| Snippet match open marker | `<<` | Phase 17 session_search tool renders this as bold/highlight |
| Snippet match close marker | `>>` | Phase 17 session_search tool renders this as bold/highlight |
| Snippet ellipsis | `...` | Shown between non-adjacent fragments |
| Export structure key (session) | `"session"` | Stable key — any parser depending on export format uses this |
| Export structure key (messages) | `"messages"` | Stable key — any parser depending on export format uses this |
| Error on SQLITE_BUSY (propagated) | `StateError::Sqlite(...)` — callers convert to user message | Callers in gateway/CLI convert to user-facing error text in later phases |

No empty states, primary CTAs, or destructive confirmation copy are defined in this phase — those belong to the tool and CLI phases (17, 20, 22) that consume this API.
