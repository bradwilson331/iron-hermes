**`execute()` signature and `tokio::time::timeout` pattern** (`terminal.rs` lines 118–244):
```rust
async fn execute(&self, args: serde_json::Value) -> anyhow::Result<String> {
    let command = args["command"]
        .as_str()
        .ok_or_else(|| anyhow::anyhow!("Missing required parameter: command"))?;
    // ...
    let fut = async {
        // ... async work ...
        Ok::<String, anyhow::Error>(combined)
    };

    let result = timeout(Duration::from_secs(timeout_secs), fut)
        .await
        .map_err(|_| anyhow::anyhow!("Command timed out after {}s", timeout_secs))??;

    Ok(result)
}
```

`hexapod_tcp.rs` uses `timeout(Duration::from_secs(3), read_fut).await.map_err(|_| anyhow::anyhow!("Error: read timed out after 3s waiting for robot response"))??` for sensor-response reads (D-18). Fire-and-forget commands (walk, stop, relax) do NOT wrap with timeout after the write succeeds — they return `Ok("OK".to_string())` immediately (D-19).

**Error feedback as `Ok(...)` not `Err(...)`** — every user-visible error (missing env var, blocked command, connection failure, read timeout) returns `Ok("Error: ...")`. Only true programming errors (bugs) propagate as `Err`. This is the `anyhow::Result<String>` convention used by every existing tool.

**Current Tool trait definition** (`registry.rs` lines 24–71):
```rust
#[async_trait]
pub trait Tool: Send + Sync {
    fn name(&self) -> &str;
    fn toolset(&self) -> &str;
    fn description(&self) -> &str;
    fn schema(&self) -> ToolSchema;

    fn is_available(&self) -> bool {
        self.prerequisites()
            .iter()
            .filter(|p| p.required)
            .all(|p| match p.kind.as_str() {
                "env_var" => std::env::var(&p.name).is_ok(),
                "config_field" => true,
                _ => true,
            })
    }

    fn prerequisites(&self) -> Vec<Prerequisite> {
        vec![]
    }

    fn redact_args(&self, raw: &serde_json::Value) -> serde_json::Value {
        raw.clone()
    }

    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<String>;
}
```

**Splice preview — insert after `redact_args`, before `execute`** (`registry.rs` line 70):
```rust
    /// Called once when the agent session ends (D-13, Phase 27.1.1).
    /// Default is a no-op — existing tools get this for free with no breaking changes.
    /// Override to run fire-and-forget cleanup (e.g., HexapodTcpTool sends stop + relax).
    fn on_session_end(&self) {}

    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<String>;
```

Note: `on_session_end` is a synchronous `fn`, not `async fn`. `HexapodTcpTool` uses `tokio::spawn(...)` internally to fire-and-forget without blocking the shutdown path (D-14). This avoids the `async_trait` overhead on a no-op path for the 18 tools that use the default.

## Cross-Cutting Conventions

### Conventional Commit Prefix Style

From `git log --oneline -- crates/ironhermes-tools/`:
```
fix(26.3): cargo fmt normalization ...
fix(26.3): WR-03 remove duplicate test ...
feat(26.3-01): wire user_data_dir into BrowserSession::spawn()
feat(25.3-05): WebExtractTool::redact_args override ...
test(25.3-05): add failing tests for WebExtractTool redact_args override
feat(25.3-05): add Tool::redact_args default method to trait
chore: merge executor worktree ...
```

Pattern: `<type>(<phase>): <description>` where `<type>` is `feat` / `fix` / `test` / `chore`, `<phase>` is the phase number (e.g., `27.1.1`). Plan-numbered commits use `<type>(<phase>-<plan-num>):`. Description is imperative, lowercase.

### Test Naming Conventions

From `terminal.rs` lines 251–365 and `registry.rs` test module:
- Snake case, descriptive: `test_terminal_new_no_cwd`, `background_without_registry_errors`, `background_true_spawns_into_registry`
- No rigid `test_` prefix required — some tests omit it (`background_false_keeps_foreground_path`)
- Test names encode the scenario + expected outcome
- All async tests: `#[tokio::test]`
- Platform-conditional tests: `#[cfg(unix)]` attribute on individual `#[tokio::test]` items

### Module File Organization

Every tool is a free file (not `mod.rs`), named after the tool: `terminal.rs`, `cronjob_tool.rs`, `file_tools.rs`. `lib.rs` declares `pub mod <name>` entries in alphabetical order. No subdirectories per tool. `hexapod_tcp.rs` follows this convention: flat file, alphabetically placed `pub mod hexapod_tcp;` in `lib.rs`.

### `pub use` Re-export Pattern

`lib.rs` only re-exports items that are consumed by name from outside the `ironhermes-tools` crate (e.g., `MemoryManagerHandle`, `ToolRegistry`, `Tool` trait). Individual tool structs (`TerminalTool`, `CronjobTool`, `ReadFileTool`) are NOT re-exported from `lib.rs` — they are imported at registration call sites via `use crate::<module>::<Struct>` inside the registering function body (see `registry.rs` lines 477–507). `HexapodTcpTool` follows the same pattern: no `pub use` in `lib.rs`, local `use crate::hexapod_tcp::HexapodTcpTool` inside `register_hexapod_tool()`.

## Anti-Patterns to Avoid

| Anti-pattern | Correct approach | Source |
|---|---|---|
| Define a custom `HexapodError` enum | Use `anyhow::Result<String>` return; user-visible errors are `Ok("Error: ...")` strings | `terminal.rs:118`, `registry.rs:70` |
| Return `Err(...)` for robot connection failures | Return `Ok("Error: cannot connect to robot at ...")` so the LLM sees it inline (D-17) | CONTEXT.md D-16/D-17 |
| Return `Err(...)` for blocked commands | Return `Ok("Action '...' is blocked — not permitted via hexapod_tcp.")` (D-16) | CONTEXT.md D-16 |
| Persistent TCP socket as struct field | One fresh `TcpStream` per `execute()` call — stateless (D-11) | CONTEXT.md D-11, `terminal.rs` pattern |
| `async fn on_session_end` in the trait | Keep it `fn on_session_end(&self) {}` (synchronous); use `tokio::spawn(...)` inside the override for fire-and-forget (D-14) | `registry.rs:24-71` |
| Add `HexapodTcpTool` to `register_defaults()` | Add a dedicated `register_hexapod_tool()` method in `registry.rs` (mirrors `register_cronjob_tool` shape) | `registry.rs:502-507` |
| Import the tool struct via `pub use` in `lib.rs` | Import locally inside the registering function body (`use crate::hexapod_tcp::HexapodTcpTool`) | `registry.rs:477-507` |
| `#[test]` for async tests | Use `#[tokio::test]` for all async tests | `terminal.rs:251-365` |
| Invent a new `toolset()` string | Use `"agent"` (matches `CronjobTool` for hardware/orchestration tools) or verify against existing toolset strings | `cronjob_tool.rs:346-348` |
