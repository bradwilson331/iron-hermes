## Task 2 — PENDING USER ACTION

**Task 2 is a `checkpoint:human-verify` gate. It cannot be automated.**

The developer must:

1. Power on the Freenove hexapod and confirm it has joined the same Wi-Fi network as the dev machine
2. Set `HEXAPOD_IP` to the robot's LAN IP (e.g., `export HEXAPOD_IP=192.168.1.42`)
3. Confirm `nc -z $HEXAPOD_IP 5002` returns within 1 second
4. Build a current binary: `cargo build -p ironhermes-cli`
5. Execute the full UAT script at `.planning/phases/27.1.1-safe-foundation/27.1.1-UAT.md` — walk through every criterion (1, 2, 3, 4, 5, 6, 6b, 6c, 7, 8), check each box

Pay particular attention to Criteria 6/6b/6c (auto-halt on session end) — these verify HXP-TOOL-06 end-to-end across all three IronHermes invocation paths (run_chat, run_single, ratatui TUI). If the robot does NOT auto-halt on session end, the gap is most likely in Plan 03 (shutdown wiring) — open `27.1.1-gap-NN-PLAN.md` via `/gsd-plan-phase 27.1.1 --gaps`.
