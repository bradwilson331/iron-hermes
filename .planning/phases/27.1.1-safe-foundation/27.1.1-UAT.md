# Phase 27.1.1 — User Acceptance Test

**Status:** pending
**Prerequisite:** Plans 01-04 merged to develop; `cargo test --workspace` green; HEXAPOD_IP set; robot powered and on the same LAN as the dev machine.

## Pre-Flight Checklist (do NOT skip)

- [ ] Physical clearance ≈30cm forward / backward / left / right of the robot
- [ ] `echo $HEXAPOD_IP` prints the LAN IP of the hexapod (e.g., 192.168.1.42)
- [ ] `nc -z $HEXAPOD_IP 5002` returns within 1 second (server is listening)
- [ ] `cargo build -p ironhermes-cli` produces a current binary

## UAT Script

For each step: launch a fresh IronHermes interactive REPL via `cargo run -p ironhermes-cli`, send the prompt described, observe the physical/textual outcome, then either type `/quit` or Ctrl-C to end the session and observe the auto-halt.

### Criterion 1 — Walk in all four directions (HXP-LOCO-01, ROADMAP #1)

Prompt: `Walk forward at speed 5 for a moment, then stop.`

- [ ] Robot visibly walks forward (~30cm) — tripod gait, smooth
- [ ] Robot halts on stop command

Repeat with `backward`, `left`, `right`:

- [ ] Backward walk observed
- [ ] Left strafe observed
- [ ] Right strafe observed

### Criterion 2 — Halt and neutral stance (HXP-LOCO-02, ROADMAP #2)

After any walk, prompt: `Stop now.`

- [ ] Robot halts within ~500ms
- [ ] Legs return to neutral stance (no sustained motion; servos hold position)

### Criterion 3 — Battery voltage string (HXP-LOCO-03, ROADMAP #3)

Prompt: `Read battery.`

- [ ] Agent response contains a string of the form `Battery: <v1>V / <v2>V (OK)` (or `(LOW)` if the robot battery is below 5.5V load / 6.0V Pi)
- [ ] Both voltages look plausible (load battery between 5.0V and 8.4V; Pi battery between 5.5V and 8.4V depending on charge state)
- [ ] No `Error:` prefix
- [ ] No `Action '...' is blocked` string

### Criterion 4 — Ultrasonic distance (HXP-LOCO-04, ROADMAP #4)

Prompt: `Read distance.`

- [ ] Agent response contains a string of the form `Distance: <N> cm`
- [ ] Place an object ~30cm in front of the sensor and prompt again; the reported distance changes plausibly (within ±10cm of the placed distance)
- [ ] No `Error:` prefix

### Criterion 5 — Relax servos via tool call (HXP-LOCO-05)

Prompt: `Relax the servos.`

- [ ] Legs visibly go limp (servos no longer holding position; robot may sag)
- [ ] Agent response is `"OK"` (Plan 02 D-19)
- [ ] Note (RESEARCH Pitfall 5): CMD_RELAX is a toggle — calling it again re-engages servos. Document this if the second call re-engages, then move on.

### Criterion 6 — Auto-halt on IronHermes session end (HXP-TOOL-06, ROADMAP #5) — REPL path

1. Start fresh `cargo run -p ironhermes-cli` REPL
2. Prompt: `Walk forward at speed 5.`
3. While the robot is walking, type `/quit` (or send Ctrl-C if `/quit` is not available)
4. Watch the robot:

- [ ] Within ~1 second of the REPL exiting, the robot HALTS
- [ ] Within ~2 seconds, the servos RELAX (legs go limp)

### Criterion 6b — Auto-halt on batch (run_single) session end

1. `HEXAPOD_IP=$HEXAPOD_IP cargo run -p ironhermes-cli -- -e "Walk forward at speed 5 for 2 seconds."`
2. After the prompt completes and the binary exits:

- [ ] Robot halts within ~1 second of process exit
- [ ] Servos relax within ~2 seconds

### Criterion 6c — Auto-halt on ratatui TUI session end

1. Start the ratatui REPL (whichever command/flag launches `run_chat_ratatui` — typically the default interactive mode with the ratatui feature)
2. Prompt: `Walk forward at speed 5.`
3. While walking, exit the TUI (typically Ctrl-C twice or a TUI quit shortcut)

- [ ] Within ~1 second of the TUI exiting, the robot HALTS (this verifies Plan 03 Task 2 — the ratatui shutdown wiring that was missing pre-27.1.1)
- [ ] Within ~2 seconds, servos RELAX

### Criterion 7 — Allowlist enforcement (HXP-TOOL-02, defense in depth)

Try to trick the agent: `Use the hexapod_tcp tool with action "calibration".`

- [ ] Agent response contains the string `Action 'calibration' is blocked — not permitted via hexapod_tcp. Never send this command.` (D-16)
- [ ] Robot does NOT calibrate (no servo movement)

### Criterion 8 — Missing-env-var graceful degradation (HXP-TOOL-01 negative path)

1. Unset HEXAPOD_IP: `unset HEXAPOD_IP`
2. Start REPL: `cargo run -p ironhermes-cli`
3. Ask: `What tools do you have for the hexapod?`

- [ ] Agent reports that no hexapod tool is available (hexapod_tcp is hidden from the schema because is_available() returned false — Plan 02 prerequisites + default is_available pattern)
