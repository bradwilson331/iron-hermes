## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added `pub mod hexapod_tcp;` to lib.rs**
- **Found during:** Task 1 (RED phase)
- **Issue:** Without the module declaration in lib.rs, `cargo test -p ironhermes-tools hexapod` compiled 0 tests — the inline `#[cfg(test)]` block was never compiled into the test binary
- **Fix:** Added `pub mod hexapod_tcp;` in alphabetical position (between `file_tools` and `memory_manager_handle`) with a comment noting Plan 04 responsibility for `register_defaults`
- **Files modified:** `crates/ironhermes-tools/src/lib.rs`
- **Note:** Plan 04 still owns adding `HexapodTcpTool` to `register_defaults()` in registry.rs — the mod declaration is a prerequisite for that, not a replacement for it

**2. [Rule 1 - Bug] Reordered execute() to check action BEFORE env var**
- **Found during:** Task 1 (GREEN phase — test failures)
- **Issue:** Original execute() read `HEXAPOD_IP` first, then matched action. Tests 2/3/4 (blocked actions) expected the D-16 block string but got the D-12 env-var-missing error because parallel tests were racing on the env var
- **Fix:** Restructured execute() to match on `action` first; the catch-all returns D-16 before any env var read. The five permitted actions form an outer match arm that reads HEXAPOD_IP only if needed
- **Files modified:** `crates/ironhermes-tools/src/hexapod_tcp.rs`
- **Improvement:** Also satisfies the plan requirement "Verify NO TCP connect was attempted (the blocked match arm fires before any I/O)" more cleanly

**3. [Rule 1 - Bug] Replaced `Elapsed::default()` with real timeout in test_read_timeout_branch**
- **Found during:** Task 1 (RED compile phase)
- **Issue:** `tokio::time::error::Elapsed` has no `Default` impl; plan spec said `::default()` but that API doesn't exist
- **Fix:** Used `tokio::time::timeout(Duration::from_nanos(1), std::future::pending::<()>()).await.unwrap_err()` to obtain a real `Elapsed` value in an async test context
- **Files modified:** `crates/ironhermes-tools/src/hexapod_tcp.rs`

**4. [Rule 1 - Bug] Added static Mutex to serialise env-var tests**
- **Found during:** Task 1 (GREEN phase — `cargo test --lib` race failure)
- **Issue:** `test_missing_env_var_returns_ok_error` failed intermittently when run alongside 317 other tests because sibling tests called `set_var("HEXAPOD_IP", ...)` concurrently
- **Fix:** Added `static ENV_LOCK: Mutex<()> = Mutex::new(());` in the test module; all 5 tests that mutate `HEXAPOD_IP` acquire the lock for their duration
- **Files modified:** `crates/ironhermes-tools/src/hexapod_tcp.rs`

**5. [Rule 2 - Missing import] Removed unused AsyncReadExt import**
- **Found during:** Task 1 (compile warning)
- **Issue:** Initial imports included `AsyncReadExt` which is not needed because `BufReader::read_line` is accessed via `AsyncBufReadExt` (auto-imported through BufReader)
- **Fix:** Removed `AsyncReadExt` from the import line
- **Files modified:** `crates/ironhermes-tools/src/hexapod_tcp.rs`
