# Phase 27.1.1 Plan 03: Wire call_session_end_hooks into All Three CLI Shutdown Paths

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] registry Arc moved into AgentLoop::new before shutdown hook call site**

- **Found during:** Task 1 — cargo check after inserting the hook in run_single
- **Issue:** `AgentLoop::new(client, registry, max_turns)` consumed the `registry` Arc by move. The `call_session_end_hooks()` call at line 920 then tried to borrow the moved value, causing E0382 borrow-of-moved-value.
- **Fix:** Changed `AgentLoop::new(client, registry, max_turns)` → `AgentLoop::new(client, registry.clone(), max_turns)` at line 814 — the compiler's own suggestion. Preserves Arc reference count; zero behavioral change.
- **Files modified:** `crates/ironhermes-cli/src/main.rs`
- **Commit:** 9be6c2b1

**2. [Rule 1 - Bug] RESEARCH documented app.deps.registry but live App struct has registry flattened**

- **Found during:** Task 2 — cargo check after inserting hook using `app.deps.registry`
- **Issue:** RESEARCH §Path 3 said the field path was `app.deps.registry` (AppDeps → App delegation). In the live code, `App::new(deps)` copies all `AppDeps` fields directly onto `App` (flattened). There is no `deps` field on `App`; `app.registry` is the correct path.
- **Fix:** Changed `app.deps.registry.clone()` → `app.registry.clone()`.
- **Files modified:** `crates/ironhermes-cli/src/tui_rata/event_loop.rs`
- **Commit:** 151d5af9 (amended)
