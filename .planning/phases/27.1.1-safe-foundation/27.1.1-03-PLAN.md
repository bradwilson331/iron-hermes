<objective>
Wire `ToolRegistry::call_session_end_hooks()` (created in Plan 01) into the three IronHermes shutdown paths: `run_single` (batch `-e` mode) and `run_chat` (classic REPL) in `crates/ironhermes-cli/src/main.rs`, and `run_with_deps` (ratatui TUI) in `crates/ironhermes-cli/src/tui_rata/event_loop.rs`. After this plan, any tool registered in the IronHermes registry — present or future — that overrides `on_session_end` will fire when the agent session ends, regardless of which CLI/TUI path the user invoked.

Purpose: HXP-TOOL-06 requires that the robot halt + relax when the IronHermes session ends. Plan 02 implements `HexapodTcpTool::on_session_end`; Plan 04 registers the tool in the registry. This plan delivers the third half of the contract — the call sites that trigger the hook. RESEARCH §Shutdown Sequence Discovery flagged that the ratatui path currently has no shutdown hook at all, so this plan adds one for the first time (closing the gap noted in CONTEXT D-15 wording "REPL/CLI").

Output: Three additive splices — none of which modify existing logic. Each insertion site already has a memory_manager `on_session_end` block (or in the TUI case, an `Ok(())` return) as the architectural template.
</objective>
