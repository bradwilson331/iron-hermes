**Task 1 ŌĆö All CLI wiring in `crates/ironhermes-cli/src/skills_cmd.rs` (09353802)**

**Sub-step A: Import**
Added `LocalDirSource` to the `use ironhermes_hub::{...}` group.

**Sub-step B: build_sources()**
Added `Box::new(LocalDirSource)` as the 4th entry in the returned `vec!` (after `SharedGitHubSource`, `WellKnownSkillSource`, `SkillsShBlobSource`). Stateless unit struct ŌĆö no config needed.

**Sub-step C: expand_path helper**
New module-scope function `fn expand_path(raw: &str) -> anyhow::Result<std::path::PathBuf>` implementing D-A2:
- `~/rest` ŌåÆ `dirs::home_dir().join(rest)`, error if HOME unavailable
- `~` alone ŌåÆ `dirs::home_dir()`
- relative paths ŌåÆ `std::env::current_dir().join(raw)`
- absolute paths ŌåÆ `PathBuf::from(raw)`

**Sub-step D: local: dispatch arm in cmd_install**
Position: BETWEEN `skills-sh:` arm (L305) and `contains('/')` GitHub arm (L353). Exact dispatch order: well-known (298) ŌåÆ skills-sh (305) ŌåÆ local (311) ŌåÆ contains('/') (353) ŌåÆ bare name.

The arm:
1. Strips `local:` prefix, calls `expand_path(raw_path)`
2. Calls `std::fs::canonicalize(&expanded)`, hard-fails with `strip_terminal_escapes`-wrapped error on failure (D-A2, T-21.8.1-04)
3. Checks `canonical.is_dir()`, hard-fails if not
4. Emits `"Resolving local:<canonical>..."` (RULE 5 option b ŌĆö corrected D-21 line 1)
5. Stashes canonical path in `canonical_local_identifier: Option<String>`
6. Returns `LocalDirSource` from `sources.iter().find(|s| s.source_id() == "local-dir")`

Pre-dispatch D-21 line 1 (`"Resolving skills.sh/..."`) guarded with `if !identifier.starts_with("local:")` to prevent the `"Resolving skills.sh/local:..."` double-print (Pitfall 6 fix).

`hub_install` call site changed to use `canonical_local_identifier.as_deref().unwrap_or(identifier)` so the adapter receives the canonical absolute path, not the raw `"local:<path>"` string (RULE 3).

**Sub-step E: cmd_list_impl [local] annotation**
Added `source_tag` match arm inside the text-format iterator:
```rust
let source_tag = match r["source"].as_str().unwrap_or("") {
    "local-dir" => " [local]",
    _ => "",
};
format!("{} [{}]{} ({}) ŌĆö hash: {}", name, trust_level, source_tag, source, snap_short)
```
JSON output already carries `"source":"local-dir"` ŌĆö no additional change needed.

**Sub-step F: cmd_update generalizes automatically (RULE 2)**
No code change. The existing `sources.iter().find(|s| s.source_id() == entry.source.as_str())` dispatch at the cmd_update loop finds `LocalDirSource` now that it is in `build_sources()`. The failure-isolation `Err(e)` arm catches `LocalSourceMissing` propagated from `LocalDirSource::fetch()`.

**Sub-step G: cmd_remove unchanged (RULE 6)**
No code change. Source dir is never touched ŌĆö only `skills_root` and lock entry are affected. T-21.8.1-07 mitigated structurally.
