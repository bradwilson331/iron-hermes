## Decisions Made

1. **`parse_skill_name_from_frontmatter` hand-rolled (not reused from existing helpers):** `blob.rs::parse_frontmatter_fields` is a module-private function requiring `serde_yaml` full parse; `github.rs::GitHubSource::parse_frontmatter_name` is the closer analog (same hand-rolled line-scan approach). Implemented as a standalone module-level function mirroring that pattern. No new imports required.

2. **`target/` skip applied (YES):** Claude's discretion per CONTEXT.md. Developers pointing `local:` at a Rust crate they are building will have large `target/` directories full of binaries. Skipping avoids bundling gigabytes of build artifacts. Documented as a divergence from `lock.rs::walk` (which runs on already-installed skill dirs that never contain `target/`).

3. **No doc-test added to lib.rs:** The plan's Sub-step C requested a doc-test using `use ironhermes_hub::LocalDirSource; use ironhermes_hub::HubSource;`. The `HubSource` trait is not in the crate's public re-exports (it is, via `pub use source::...` but without re-exporting the trait by name in a way that doc-tests can easily reference). The unit tests prove the re-export works end-to-end. Deferred to plan's acceptance note: "From any consumer crate... `use ironhermes_hub::LocalDirSource;` compiles" — verified by build success.
