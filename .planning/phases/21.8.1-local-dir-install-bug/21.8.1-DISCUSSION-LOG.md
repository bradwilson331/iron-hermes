## Input format detection

| Option | Description | Selected |
|--------|-------------|----------|
| Explicit prefix only (`local:`) | Add `local:<path>` mirroring `skills-sh:` and `well-known:`. Predictable, no surprises. Users learn one verb but it composes with shell tab-completion. | âś“ |
| Auto-detect by disk existence | Before falling into the GitHub branch, check if `Path::new(identifier).is_dir()` and contains SKILL.md. Magical for users; introduces ambiguity (could the user have meant a GitHub repo with a colliding name?). | |
| Both (prefix wins, then auto-detect) | Try `local:` first. If no prefix, before routing to GitHub check disk existence. Best UX but most code paths to test. | |
| Path-shape heuristics (`.`, `/`, `~`) | Treat any identifier starting with `/`, `./`, `../`, or `~/` as local. Simple, no I/O probe. `local:` could still be supported as an alias. | |

**User's choice:** Explicit prefix only (`local:`).
**Notes:** Recommended option chosen â€” predictable dispatch, sibling pattern to existing `skills-sh:` and `well-known:`. No silent routing.

## Path forms accepted by `local:`

| Option | Description | Selected |
|--------|-------------|----------|
| Absolute, relative, and ~ | Accept `local:/abs/path`, `local:./rel/path`, `local:../up/path`, `local:~/skills/foo`. Tilde via dirs::home_dir(); relative against CWD. Canonicalize before any I/O. | âś“ |
| Absolute + ~ only | Reject relative without `./` (e.g., `local:my-skill` is ambiguous). Forces unambiguous input. Slightly more typing; cleaner errors. | |
| Absolute paths only | Require canonical absolute path. No ~ expansion, no relative paths. Easiest to reason about, smallest attack surface, but worst UX. | |

**User's choice:** Absolute, relative, and ~ all accepted.
**Notes:** Most ergonomic; matches shell intuition. Canonicalize via `std::fs::canonicalize` before any further I/O â€” fail fast if path missing.

| Option | Description | Selected |
|--------|-------------|----------|

| Symlink to source dir | Symlink the install dir to the source. Live edits visible at next session. Faster dev loop. But: scanner can't validate once-and-trust; D-18/D-20 guards need rework; lockfile drift becomes 'always drifted'. | |

**User's choice:** Copy into skills/ (parity with remote).
**Notes:** Cleanest path; reuses 21.8 install pipeline. One code path. Atomic move from quarantine.

## Error UX when misrouted

| Option | Description | Selected |
|--------|-------------|----------|
| Pre-dispatch hint when it looks like a path | Before routing to GitHub, check (a) starts with `/`, `./`, `../`, `~/`, OR (b) resolves to existing dir. If yes, emit 'looks like a local path â€” did you mean local:<path>?' and exit 1. Cheap probe. | âś“ |
| Always wrap GitHub failures with the local-prefix hint | Don't pre-check. After GitHub fetch fails with NotFound, append a generic suffix. Less precise (fires for legitimate typos) but zero new code paths. | |
| No special handling â€” user reads the docs | Document `local:` and let GitHub's existing error stand. Smallest scope; worst UX for the bug that triggered this phase. | |

**User's choice:** Pre-dispatch hint when it looks like a path.
**Notes:** Recommended option. Cheap probe (one fs::metadata call); near-zero false positives. The hint line is the headline UX deliverable for the bug fix.
