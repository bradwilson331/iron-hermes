# Phase 21.8.1 Plan 01 Summary

Wave 1 pure-additive surface: `HubErrorKind::LocalSourceMissing` enum variant, `recompute_trust_str` `"local-dir" => "trusted"` arm, and D-D1 pre-dispatch path-shape hint in `cmd_install` with testable `looks_like_local_path_with_probe` helper — plus lock-file forward-compat round-trip test.

## What Was Built

Three production source files modified, one new helper function, seven new unit tests, one new lock integration test.

**Task 1 — `HubErrorKind::LocalSourceMissing` variant (37af97d2)**

Added `LocalSourceMissing` variant after `Audit` in `HubErrorKind` enum in `crates/ironhermes-hub/src/error.rs`. Includes the D-C2 doc-comment verbatim. Extended `new_variants_serialize_snake_case` to assert serialization as `"local_source_missing"`. Extended `new_variants_partial_eq` to assert identity and inequality with `Audit`. **Task 2 — `recompute_trust_str` local-dir arm + D-D1 hint (d63e1541)**

Added `else if source == "local-dir" { "trusted" }` arm to `recompute_trust_str` (between `well-known/skills-sh` and the `else` fallback), implementing D-B2.

Inserted D-D1 pre-dispatch hint block in `cmd_install` after `build_sources()` and before the `let source:` dispatch chain. Uses a `has_known_prefix` guard (`well-known:`, `skills-sh:`, `local:`) to avoid false-firing. The hint calls `looks_like_local_path_with_probe` — a pure testable helper with a closure parameter — then emits `error: ... looks like a local path — did you mean 'hermes skills install local:...'?` via `eprintln!` with both interpolations passing through `strip_terminal_escapes` (T-21.8.1-04 mitigation). Returns `Ok(1)`.

Added 7 unit tests in new `#[cfg(test)] mod tests` block at the bottom of `skills_cmd.rs`:
- `recompute_trust_str_returns_trusted_for_local_dir` (with github-community regression guard)
- `local_dir_hint_absolute_path` (`/tmp/foo`)
- `local_dir_hint_relative_dot_slash` (`./skill`)
- `local_dir_hint_dotdot` (`../skill`)
- `local_dir_hint_tilde` (`~/skill`)
- `local_dir_hint_existing_dir_via_probe` (simulates metadata probe = true)
- `local_dir_hint_no_false_positive_github_id` (probe = false, no hint)

**Task 3 — Lock-file forward-compat round-trip test (9422835d)**

Added `lock_entry_with_local_dir_source_round_trips` test immediately after `extras_passthrough_round_trip` in `crates/ironhermes-hub/src/lock.rs`. Proves `source:"local-dir"`, `snapshotHash:""`, and an absolute-path identifier round-trip through serde without error. Asserts `e.source == "local-dir"`, `e.snapshot_hash == ""`, `e.identifier`, `e.repo_path`, `e.computed_hash`, and that re-serialization contains `"source":"local-dir"` and `"snapshotHash":""`.

## Decisions Made

1. **Extract `looks_like_local_path_with_probe` helper (YES):** Extracted as a standalone `fn looks_like_local_path_with_probe<F: Fn(&str) -> bool>(identifier: &str, dir_probe: F) -> bool`. This eliminates `std::env::set_current_dir()` from tests and any need for `ENV_LOCK` mutex. Pure-testable. Consistent with Phase 21.3 decision: "pure parse functions take serde_json::Value for testability."

2. **Unit tests in `skills_cmd::tests` mod (not integration file):** The D-D1 path-shape tests are pure function calls needing no subprocess or binary. Placing them in an integration test file would require cargo to rebuild and spawn a subprocess for what is effectively a unit test. Tests 1-6 are unit tests of the helper directly.

3. **`has_known_prefix` guard includes `local:`:** The plan specifies guarding against future `local:` prefix to avoid false-positive hints once Plan 03 wires `LocalDirSource`. The guard is forward-compatible.
