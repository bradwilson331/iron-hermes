# Phase 21.8.1 Plan 05 Summary

## One-Liner

Two-level-aware SkillRegistry::load_with_paths closes gap-01: skills installed at `<skills_root>/<category>/<name>/SKILL.md` (Phase 21.8 installer layout) now appear in chat /skills, gateway-driven prompts, and Telegram surfaces — proven by 9 unit + 3 CLI integration + 2 gateway integration tests.

## What Was Built

Three files modified; 14 new tests total across three test tiers.

### Task 1 — crates/ironhermes-core/src/skills.rs (9edaa419)

**Before (one-level only):**
The `load_with_paths` scan loop walked `<root>/<dir>/SKILL.md` only. When the Phase 21.8 installer wrote `creative/ascii-art/SKILL.md`, `load_with_paths` scanned `creative/` (a directory), checked `creative/SKILL.md` (absent), and silently skipped — the skill was invisible to every surface.

**After (two-level aware):**
The inline processing body was extracted into a private helper `try_register_skill_from_dir(subdir, seen_names, skills)` — byte-for-byte equivalent to the pre-21.8.1-05 inline body (same scan, same D-15 trust gate, same SKILL-07 dir-name-match warn, same platform filter, same dedup). The outer loop now:
1. Skips `.hub` and any hidden dir (`starts_with('.')`) at the outer level.
2. Tries level-1: if `<root>/<dir>/SKILL.md` exists, calls `try_register_skill_from_dir`.
3. Always tries level-2: iterates `<root>/<dir>/` entries, skipping hidden dirs and `node_modules`, and calls `try_register_skill_from_dir` on any `<root>/<dir>/<sub>/SKILL.md`.
4. Never recurses a third time — depth is hard-capped at 2 by construction.

## Gap-01 Closure Readiness

`21.8.1-VERIFICATION.md` gap-01 entry (`gap-01-chat-tool-registry-visibility`) is ready to flip from `status: failed` to `status: verified`. The verifier in the post-execute pass owns the actual flip; this plan delivers the readiness signal.

## Decisions Made

1. **try_register_skill_from_dir as free function:** Keeps visibility minimal; `SkillRegistry` method would have exposed it as pub(crate) with a wider surface.

2. **Two-level scan always runs (not only when SKILL.md absent at level 1):** Handles the edge case (Test 9) where a directory is both a one-level skill AND a category containing sub-skills. The dedup-by-name guard handles same-name collisions.

3. **Depth bounded at 2 structurally:** No `if depth < 2 { recurse }` state — the inner loop simply has no recursion call. Test 4 would catch any future accidental introduction of deeper recursion.

4. **PromptBuilder::new without skip_context_files:** The skills slot in `build_split` is populated via the fallback path at line 536 (`!self.skip_context_files && !slots.contains_key(&PromptSlot::Skills)`) — no explicit `load_skills()` call needed in tests.

## Deviations from Plan

**[Rule 1 - Bug] GAP01_SKILL_MD em dash as byte escape**
- **Found during:** Task 2 compile attempt
- **Issue:** `\xe2\x80\x94` (UTF-8 em dash) is invalid in Rust string byte escape syntax for bytes > 0x7f
- **Fix:** Replaced with ASCII hyphen `-` in both the constant and assertion
- **Files modified:** `crates/ironhermes-cli/tests/skills_cmd_integration.rs`
- **Impact:** Functionally equivalent — the description still proves gap-01 closure
