## Fixed Issues

### CR-01: Command Injection via IRONHERMES_UID/IRONHERMES_GID in entrypoint.sh

**Files modified:** `docker/entrypoint.sh`
**Commit:** 6a3cb8f
**Applied fix:** Replaced one-liner `[ -n "$VAR" ] && command` pattern with explicit `if` blocks that validate IRONHERMES_UID and IRONHERMES_GID are strictly numeric (regex `^[0-9]+$`) before passing to `usermod`/`groupmod`. Non-numeric values now cause an immediate error exit with a descriptive message to stderr.

### WR-01: entrypoint.sh groupmod error silently swallowed

**Files modified:** `docker/entrypoint.sh`
**Commit:** 6a3cb8f
**Applied fix:** Resolved as part of CR-01. The `[ -n ... ] && groupmod ... || true` pattern was replaced with an explicit `if` block, so `|| true` now applies only to the `groupmod` command itself, not to the entire conditional expression. This prevents masking of genuine `groupmod` failures caused by the shell operator precedence issue.

### WR-02: install.sh trap uses single quotes inside double quotes

**Files modified:** `install.sh`
**Commit:** 8a234d9
**Applied fix:** Replaced `trap "rm -rf '$tmpdir'" EXIT` with a named `cleanup()` function using properly double-quoted `"$tmpdir"`, invoked via `trap cleanup EXIT`. This eliminates the fragile single-quote-inside-double-quote pattern.

### WR-03: install.sh trap set inside download_binary is never cleared on early return

**Files modified:** `install.sh`
**Commit:** 8a234d9
**Applied fix:** Wrapped the download logic in a subshell `( ... )` so the EXIT trap is scoped to the subshell and does not leak to the caller. Changed `return 0`/`return 1` to `exit 0`/`exit 1` inside the subshell (exits the subshell, returning its exit code to the caller). The parent function's `return` semantics are preserved by the subshell's exit code.

### WR-04: Shell scripts lack `set -u` (nounset) for undefined variable protection

**Files modified:** `docker/entrypoint.sh`, `install.sh`, `setup-ironhermes.sh`
**Commit:** b6e7b2b
**Applied fix:** Changed `set -e` to `set -euo pipefail` in all three shell scripts. Updated `$IRONHERMES_UID` and `$IRONHERMES_GID` references in `docker/entrypoint.sh` to use `${IRONHERMES_UID:-}` and `${IRONHERMES_GID:-}` syntax for `set -u` compatibility (these env vars are optional and may not be set). All other variable references in the three scripts already use `:-` defaults or are explicitly assigned, so no further changes were needed.
