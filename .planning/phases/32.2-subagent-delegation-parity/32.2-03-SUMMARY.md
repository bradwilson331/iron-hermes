The 4th through 7th parameters are the additions from this plan. All existing call sites were updated to pass `ChildRole::Leaf, 0, &SubagentConfig::default(), None`.

## Depth Threading Wiring Path

**Chosen path: field-on-runner (RESEARCH Pitfall 6 recommendation)**

```
AgentSubagentRunner.current_depth
    ↓ (set via with_current_depth() builder at construction time)
DelegateTaskTool::execute() passes child_depth=0 to build_child_registry
    ↓ (for orchestrators, runner_for_orchestrator carries a runner with current_depth already set)
orchestrator arm registers DelegateTaskTool::new(runner_for_orchestrator, ...)
    ↓ (that runner's current_depth is the depth for the next spawn level)
```

**Key constraint:** `DelegateTaskTool::execute()` always passes `child_depth=0` to `build_child_registry`. Full end-to-end depth propagation requires the caller to supply a runner whose `current_depth` matches the current spawn depth. The `AgentSubagentRunner` struct now carries this state; callers constructing runners for orchestrator children should call `.with_current_depth(parent_depth + 1)` before registering the DelegateTaskTool.

The `SubagentRunner` trait's `run_child` method signature is NOT modified — depth state travels exclusively via `AgentSubagentRunner` struct fields.

## Accomplishments

### Task 1: ChildRole + build_child_registry expansion (both delegate_task.rs copies)

- Added `pub enum ChildRole { Leaf, Orchestrator }` with `#[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Deserialize)]`, `#[serde(rename_all = "lowercase")]`, and `impl Default for ChildRole { fn default() -> Self { Self::Leaf } }`

- Split the `"delegate_task"` arm: `"delegate_task" if matches!(role, ChildRole::Leaf) => { /* AGENT-05 */ }` + `"delegate_task" => { /* Orchestrator arm: register DelegateTaskTool */ }`
- Added `tracing::warn!` downgrade message when orchestrator conditions are not met
- Added role resolver in `execute()` and per-task role resolver in `execute_batch()` — both default Leaf on parse failure
- Added `"role"` property to JSON schema at both top-level and `tasks.items.properties` levels
- Updated all 13 existing `build_child_registry(` call sites in tests
- Applied byte-identically to both `ironhermes-tools` and `ironagent-tools-api`
