## Example: 3-node nested tree ASCII output

Registry contains: `sub_root1234` (root) → `sub_child_aa` (child 1) + `sub_child_bb` (child 2).

```
Active subagents:
├── sub_root  orchestrate pipeline  42s  [running]
│   ├── sub_chil  fetch data        30s  [running]
│   └── sub_chil  write output      25s  [killed]
```

- `├── ` for non-last children, `└── ` for last child
- `│   ` continuation prefix for non-last ancestors, `    ` for last ancestors
- id prefix: 8 chars; summary: 80-char clip with `…`; status pill: `[running]` or `[killed]`

## JSON shape produced by `build_subagent_tree_json` for the same 3-node tree

```json
[
  {
    "id": "sub_root1234abcd",
    "task_summary": "orchestrate pipeline",
    "uptime_secs": 42,
    "started_at_unix_ms": null,
    "children": [
      {
        "id": "sub_child_aa1234",
        "task_summary": "fetch data",
        "uptime_secs": 30,
        "started_at_unix_ms": null,
        "children": []
      },
      {
        "id": "sub_child_bb5678",
        "task_summary": "write output",
        "uptime_secs": 25,
        "started_at_unix_ms": null,
        "children": []
      }
    ]
  }
]
```

`started_at_unix_ms` is `null` because `std::time::Instant` carries no wall-clock epoch. A future plan can thread a session-start `SystemTime` to compute absolute timestamps.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] AppRuntimeBundle does not expose subagent registry post-init**
- **Found during:** Task 2 (`AppState::subagent_tree_json`)
- **Issue:** Plan assumed `AppState` could reach `delegate_task.runner.subagent_registry()` via `runtime_bundle.delegate_task`. But `AppRuntimeBundle` does not have a `delegate_task` field — only `AppRuntimeFactoryInput` does. The runner is transferred into `DelegateTaskWiring` during `build_app_runtime_bundle` and not re-exposed.
- **Fix:** Changed `subagent_tree_json` from an async method that tries to reach the registry through the bundle, to a synchronous method that accepts `&SubagentRegistry` as a parameter. `build_subagent_tree_json` is a pure free function — directly testable. The test covers the JSON shape without needing a full `AppState`.
- **Files modified:** `crates/iron_hermes_ui/src/server/state.rs`
- **Committed in:** `daea8a8f`

**2. [Rule 3 - Blocking] tokio-util not a direct dep of iron_hermes_ui**
- **Found during:** Task 2 (test compilation)
- **Issue:** Test module uses `CancellationToken` from `tokio_util::sync`, but `tokio-util` was not in `iron_hermes_ui`'s `[dev-dependencies]`.
- **Fix:** Added `tokio-util = { workspace = true }` to `[dev-dependencies]` in `iron_hermes_ui/Cargo.toml`.
- **Files modified:** `crates/iron_hermes_ui/Cargo.toml`
- **Committed in:** `daea8a8f`
