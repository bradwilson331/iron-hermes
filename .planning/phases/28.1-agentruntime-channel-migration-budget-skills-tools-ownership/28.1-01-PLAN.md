<objective>
Add the `AgentRuntime`-level budget-reset regression test that the whole phase
depends on for proof. CONTEXT success criterion #2 requires a regression test "at
the AgentRuntime/budget level" proving that a second top-level turn after a
budget-exhausting first turn starts with a full budget. This is the single
automated guarantee that the `Stop100` latch class of bug is fixed for every
channel that routes through `run_turn`.

The `AgentRuntime` type and `run_turn`'s `self.budget.reset()` boundary already
exist (commit `0f8807c6`). What is missing is the test that locks the behavior
in. `agent_runtime.rs` currently has no `#[cfg(test)]` module.

This plan adds NO production behavior change. It is foundational: it lands first
so every subsequent channel migration can lean on it.

Purpose: Make the budget-reset guarantee an executable, regression-protected
contract before any channel is rewired to depend on it.
Output: A `#[cfg(test)]` module in `agent_runtime.rs` with the budget-reset test.
</objective>

    - Test `budget_resets_between_turns`: build (or directly construct) an
      AgentRuntime whose shared BudgetHandle is observable via `runtime.budget()`.
      Drain the budget to exhaustion by calling `consume()` in a loop until it
      returns `None` (simulating a budget-exhausting first turn). Assert
      `runtime.budget().remaining() == 0`. Then invoke the reset boundary that
      `run_turn` uses — assert that after the boundary fires the budget is back to
      full (`remaining() == max_iterations`).
    - The cleanest deterministic form: assert the INVARIANT that `run_turn` resets
      before building the loop. Since `run_turn` against a live model is not
      hermetic, prove the reset boundary directly: construct the runtime, drain
      `runtime.budget()` to 0, call `runtime.budget().reset()` (the exact call
      `run_turn` makes at line 198), and assert `remaining()` is full again.
      Additionally add a static assertion (string include of the source) that
      `run_turn`'s body contains `self.budget.reset()` BEFORE the `AgentLoop::new`
      call, so a future refactor that moves or drops the reset fails this test.
    - Test `runner_shares_budget_arc`: assert that the budget the runtime exposes
      and the budget cloned into the subagent runner are the SAME underlying Arc

      `from_config` passes `Some(budget.clone())` to `AgentSubagentRunner::new` and
      that the same `budget` is stored on `Self`. Prefer the behavioral Arc-identity
      check if a lightweight `from_config` is feasible with a localhost:0 client;
      fall back to the source-include guard otherwise. Document which form was used
      and why in the test doc comment.
