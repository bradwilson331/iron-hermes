# Phase 28.1 Plan 05: TUI AgentRuntime Migration Summary

TUI ratatui REPL migrated from per-turn `AgentLoop` builder to `AgentRuntime::run_turn`, fixing the latent Stop100 latch (T-28.1-11) and the `max_turns`/`max_iterations` config drift (T-28.1-12).

## What Was Built

### Task 1: Build one AgentRuntime in build_app_deps; reshape AppDeps/App

- **`build_app_deps` (event_loop.rs)**: Removed `BudgetHandle::new(...)` and the App-level `AgentLoop::new(...).with_budget(...)...` construction (~199, ~440-455). Added `AgentRuntime::from_config(AgentRuntimeInput { ... })` call. Budget is now sized from `config.agent.max_iterations` inside `from_config`, fixing the `max_turns` config drift.
- **`AppDeps`/`App` structs (app.rs)**: Replaced `agent_loop: Arc<AgentLoop>`, `budget: BudgetHandle`, `context_length: usize`, `config_compression: f64`, `max_turns: usize`, `fallback_client: Option<AnyClient>` with `agent_runtime: Arc<AgentRuntime>`. Kept `client` (mutated by /model and /fast) and `registry` (session-end hooks + slash dispatch).
- **`test_deps()` fixture (app.rs)**: Updated to use `AgentRuntime::for_tests()` instead of `Arc::new(AgentLoop::for_tests()) + BudgetHandle::new(10)`.
- **`AgentRuntime::for_tests()` (agent_runtime.rs)**: Added `#[cfg(any(test, feature = "test-support"))]` constructor that builds a minimal runtime with localhost:0 client, `load_with_paths(&[])` skill registry, and temp-dir JobStore.
- **merged_tools**: Previously extracted from the now-removed `initial_runtime_bundle` call; computed inline via `config.tools.clone().with_default_toolsets_merged()`.
- **subagent_runner/semaphore**: The production runner moved into `AgentRuntime::from_config`. A lightweight `tui_subagent_runner` is still built for the TUI's local slash-dispatch registry so `delegate_task` appears in `/tools list`.

### Task 2: spawn_turn → run_turn; re-point AgentLoopHandle integration

- **`spawn_turn` (event_loop.rs)**: Replaced the ~50-LOC `AgentLoop::new(...).with_budget(...)....with_fallback(...)` chain with a `TurnRequest` struct build + `runtime.run_turn(request).await`. All 3 callback types preserved: `stream: Some(streaming_cb)`, `tool_progress: Some(tool_progress_cb)`, `tool_result: Some(tool_result_cb)`. `browser_session`, `memory_manager`, `compression`, and `fallback` are now DURABLE (runtime owns them — `run_turn` wires them internally).
- **`commands.rs`**: Replaced `AgentLoopAdapter(Arc<AgentLoop>)` with `AgentRuntimeAdapter(Arc<AgentRuntime>)` implementing `AgentLoopHandle`. `is_running()` returns `false` (conservative, matches prior behaviour).
- **Integration surface**: `AgentLoopHandle` only exposes `is_running() -> bool`; the adapter satisfies it via the runtime handle. All slash commands that read agent state continue to function.

### Task 3: Reconcile invariant tests + grep gate

- **INV-22.4-24**: Translated from `AgentLoop::new` ordering to `AgentRuntime::from_config(AgentRuntimeInput` ordering. Uses the call-site form (not plain `AgentRuntime::from_config`) to avoid matching comment prose as the first occurrence.
- **INV-22.4-04**: Extended to accept `run_turn(` as valid per-turn execution form alongside `agent.run` and `run_agent_turn`.
- **INV-22.4-26**: Extended to accept TurnRequest field form (`tool_progress: Some(` / `tool_result: Some(`) alongside AgentLoop chain form.
- **INV-22.4-28(d)**: Translated PROV-07 fallback check from `.with_fallback(fb_client)` per-turn to `AgentRuntime::from_config(AgentRuntimeInput` durable path.
- **INV-22.4-30 (new)**: Grep-gate asserting (a) zero `BudgetHandle::new(` occurrences, (b) zero `with_budget(` occurrences, (c) `run_turn(` present in production event_loop.rs.
- **`cli_web_runtime_parity.rs`**: Updated `ratatui_build_app_deps_uses_shared_runtime_factory` from asserting `build_app_runtime_bundle(` to asserting `AgentRuntime::from_config(` + `AgentRuntimeInput {`.
