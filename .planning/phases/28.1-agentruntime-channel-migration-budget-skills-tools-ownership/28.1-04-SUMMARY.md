# Phase 28.1 Plan 04: CLI Entry Points → AgentRuntime::run_turn Summary

Migrated both CLI agent entry points (`run_single`, `run_chat`) from manual
`BudgetHandle::new` + `AgentSubagentRunner` + `build_app_runtime_bundle` + per-turn
`AgentLoop::with_budget` assembly to a single `AgentRuntime::from_config` owner whose
`run_turn` resets the budget at every turn boundary — fixing the latent multi-turn
Stop100 latch in `run_chat`.

## What Changed

- **Task 1 — `run_single`:** Deleted the manual `BudgetHandle::new`, subagent-runner
  block, and `build_app_runtime_bundle` call. Built one `AgentRuntime::from_config`
  (`subagent_progress_callback: None`, `subagent_cancel_token: None`, one-shot). The
  `AgentLoop::new(...).with_budget(...)` chain became a `TurnRequest`
  (`compression_count: 0`, `trajectory_writer` now wired, dead todo intercept dropped)
  followed by `runtime.run_turn(request).await`. Post-run drains and session-end hooks
  re-sourced from `runtime.*` accessors.
- **Task 2 — `run_chat` + `run_agent_turn`:** Built one `Arc<AgentRuntime>` per session
  (`subagent_progress_callback: Some(..)`, `subagent_cancel_token: Some(child_token)`).
  `run_agent_turn` rewritten to take `&Arc<AgentRuntime>` and build a `TurnRequest`
  threading `cancel_token`, `pressure_tracker` (`Some(..)`), `state_store`,
  `trajectory_writer`, and `compression_count: starting_count`. After the turn it
  stores `result.compression_count_after` back into the shared `Arc<AtomicUsize>`
  (full round-trip). A separate `subagent_semaphore` was added for `build_cmd_ctx`
  slash-command CommandContext construction (3 call sites now use `runtime.budget()`).
- **Task 3 — grep gate + test alignment:** Added the `agent_runtime_migration_gate_28_1_04`
  module in main.rs (forbids `BudgetHandle::new(` and bare `.with_budget(budget`,
  asserts `>=2 run_turn(` call sites). Realigned the static-grep invariant tests to
  the real post-migration code (see below).
