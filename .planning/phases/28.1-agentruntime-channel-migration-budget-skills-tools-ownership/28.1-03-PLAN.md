<objective>
Migrate the web UI to `AgentRuntime::run_turn` (design §5 stage 3). The web
server state ctor (state.rs ~80-187) currently builds a `BudgetHandle` (~95), a
`subagent_runner` with a clone of it (~109-120), and a `runtime_bundle` via
`build_app_runtime_bundle` (~148-167). Crucially, the web UI's per-turn loop
(`build_agent_loop` ~273-306, called from `run_web_turn` ~210-271) NEVER installs
the budget on the top-level loop — only its subagents share a never-reset handle.
That is the latent top-level-loop budget gap this plan closes.

After this plan: the state ctor builds ONE `AgentRuntime::from_config`;
`run_web_turn` builds a `TurnRequest` (carrying the `state_store` for the
`session_search` intercept and the per-turn callbacks) and calls
`self.runtime.run_turn`. `build_agent_loop` is deleted. The `BudgetHandle::new` +
the (absent) `.with_budget` gap both disappear; the top-level loop now gets a
budget that resets each turn.

This plan depends on Plan 02 only to keep the `AgentRuntime` API contract
stable across the parallel Wave-1 gateway work landing first; it touches a
disjoint crate (`iron_hermes_ui`) and can be implemented independently.

Purpose: Close the web top-level-loop budget gap; one owner of the web agent.
Output: `run_web_turn` routes through `AgentRuntime::run_turn`.
</objective>
