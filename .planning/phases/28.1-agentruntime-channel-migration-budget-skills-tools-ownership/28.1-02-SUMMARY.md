# Phase 28.1 Plan 02: Telegram Gateway AgentRuntime Migration Summary

Gateway migrated to `AgentRuntime::run_turn`: one runtime built at startup via `from_config`, every top-level turn delegated via `TurnRequest`; the `367eaa79` per-turn `reset()` band-aid is removed and budget lifecycle is now owned by `run_turn`.

## What Changed

### Task 1 ÔÇö AgentRuntime in run_gateway

`run_gateway` (main.rs ~2524-2573) previously hand-built:
- `BudgetHandle::new(config.agent.max_iterations)` (~2540)
- `AgentSubagentRunner::new(..., Some(budget.clone()))` (~2547)
- `build_app_runtime_bundle(AppRuntimeFactoryInput {...})` (~2554)

Replaced with a single `AgentRuntime::from_config(AgentRuntimeInput {...})` call. The runtime is wrapped in `Arc<AgentRuntime>` and the shared Arcs (`registry`, `hook_registry`, `skill_registry`, `active_skills`, `browser_session`, `job_store`) are now sourced from runtime accessors so all paths operate on the same instances.

`GatewayRunner.budget_handle: Option<BudgetHandle>` field + `set_budget_handle` setter replaced with `agent_runtime: Option<Arc<AgentRuntime>>` + `set_agent_runtime`. `build_gateway_handler` passes the runtime clone into `GatewayMessageHandler`.

Two new accessors added to `AgentRuntime`:
- `mcp_manager()` ÔÇö for run_gateway MCP shutdown wiring
- `merged_tools()` ÔÇö for ToolsetSession construction from the same merged baseline

### Task 2 ÔÇö run_agent via run_turn

`GatewayMessageHandler.budget_handle: Option<BudgetHandle>` + `set_budget_handle` replaced with `agent_runtime: Option<Arc<AgentRuntime>>` + `set_agent_runtime`.

`run_agent` (handler.rs ~985-1082) previously hand-assembled an `AgentLoop` with `.with_streaming/.with_tool_progress/.with_active_skills/.with_budget` (+ the `handle.reset()` band-aid) `/.with_hook_registry/.with_memory_manager/.with_browser_session/.with_trajectory_writer` + `attach_context_engine`.

Replaced with:
1. Build `stream_callback` + `tool_callback` (identical)
2. Resolve per-session trajectory writer from `SessionStore` (identical)
3. Snapshot `messages_for_nudge` BEFORE moving messages into `TurnRequest`
4. Build `TurnRequest { messages, session_id, stream, tool_progress, trajectory_writer, pressure_tracker: None, state_store: None, compression_count: 0, cancel_token: None }`
5. `self.agent_runtime.run_turn(request).await`

Preserved verbatim: scrubber tail flush, channel/typing teardown, ResponseSent hook, LEARN-01 nudge block (with `nudge_client` now sourced from `runtime.client()`), `result.appended` write-through, process/subagent drains, memory `on_session_end`.
