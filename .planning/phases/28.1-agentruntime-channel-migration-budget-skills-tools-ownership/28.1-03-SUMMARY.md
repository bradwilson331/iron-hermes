# Phase 28.1 Plan 03: Web UI AgentRuntime Migration Summary

Web server state migrated from manual BudgetHandle+AgentLoop construction to `AgentRuntime::from_config` + `run_turn`, closing the latent top-level-loop budget gap (T-28.1-06).

## What Was Built

- `AppState::init` now builds a single `AgentRuntime::from_config` with the existing subagent_progress_callback threaded in (preserving Phase 26.7.1 ws SubagentEvent forwarding)
- `AppState.runtime: Arc<AgentRuntime>` replaces `AppState.runtime_bundle: Arc<AppRuntimeBundle>`
- `run_web_turn` assembles a `TurnRequest` and calls `self.runtime.run_turn(request)` — budget reset happens inside `run_turn` at the correct boundary
- `build_agent_loop` method deleted
- `build_messages_for_turn` updated to use `self.runtime.skill_registry()` and `self.runtime.merged_tools()` accessors
- Static grep regression gates added to `server_runtime_parity.rs` integration tests (3 new tests)
