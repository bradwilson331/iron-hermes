## What Was Built

Added `#[cfg(test)] mod tests` to `crates/ironhermes-agent/src/agent_runtime.rs` with two regression tests:

### `budget_resets_between_turns`

Proves the budget-reset guarantee that CONTEXT success criterion #2 requires:

1. **Behavioral assertion:** Constructs a `BudgetHandle`, drains it to 0 via repeated `consume()` calls (simulating a budget-exhausting first turn), then calls `budget.reset()` — the exact call `run_turn` makes — and asserts `remaining() == max_iterations`.
2. **Source-include guard:** `assert!(SOURCE.contains("self.budget.reset()"))` — fails if the reset is removed or renamed.
3. **Byte-offset position guard:** Finds both `"self.budget.reset()"` and `"AgentLoop::new("` in the source via `.find()`, then asserts the reset byte offset is strictly less than the loop construction offset. Mirrors the pattern in `crates/ironhermes-cli/tests/invariants_22_4.rs` (INV-22.4-24). Fails if the reset is relocated after loop construction.

### `runner_shares_budget_arc`

Locks PROV-10 (parent/subagent budget sharing):

1. **Source-include guard:** Asserts `Some(budget.clone())` is passed to `AgentSubagentRunner::new`.
2. **Struct field guard:** Asserts `budget,` is present in the `Self` initializer.
3. **Byte-offset position guard:** Asserts the clone-pass site appears before `Ok(Self {` in the source.

Form rationale: building a full `AgentRuntime` via `from_config` in a unit test is impractical (requires a reachable model endpoint and assembles MCP/tool bundle). The Arc-identity sharing behavioral invariant is already covered by `budget.rs::reset_is_visible_through_shared_clone`. Source guards are sufficient for PROV-10 structural assurance.
