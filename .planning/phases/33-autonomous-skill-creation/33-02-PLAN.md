## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-02-A | Tampering | write_file file_path parameter | mitigate | Reject paths containing ".." or starting with "/"; resolve_skill_file_path enforces boundary before any fs call |
| T-33-02-B | Tampering | create/edit content (prompt injection in SKILL.md body) | mitigate | scan_skill_content() called before every write; returns "[BLOCKED:" prefix on detection; file never written |
| T-33-02-C | Elevation of Privilege | allowed-tools escalation embedded in skill content | mitigate | scan_skill_content covers allowed-tools privilege escalation via existing SKILL_THREAT_PATTERNS |
| T-33-02-D | Tampering | category/slug path traversal via two-segment join | mitigate | validate_skill_name applied to slug (rejects "/" and ".."); category validated same way |
| T-33-02-E | Tampering | delete action removing non-skill files | mitigate | delete checks dir.exists() inside get_hermes_home()/skills/ path; remove_dir_all on verified scope only |
| T-33-02-SC | Tampering | cargo workspace dependencies | accept | No new external packages in this plan — all workspace deps. No slopcheck gate needed. |
