# Phase 21.1 — UI Design Contract: Slash Command Text Output

> Visual and interaction contract for text-based output surfaces.
> This phase produces NO frontend components. All output is formatted plain text
> rendered in a terminal (CLI) or a Telegram chat (gateway). The contract below
> governs text layout, spacing conventions (column alignment), color/ANSI usage,
> copywriting, and interaction states.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Rust CLI + Telegram text output |
| Preset | not applicable |
| Component library | none — raw String formatting via Rust `format!` / `push_str` |
| Icon library | none — Unicode symbols only (see Symbol Inventory below) |
| Font | terminal monospace (CLI); Telegram default sans-serif (gateway) |

**shadcn gate:** Not applicable. Tech stack is Rust/terminal, not React/Next.js/Vite.

**Registry safety gate:** Not applicable. No third-party component registries.

## Spacing Scale (Column Alignment)

Terminal output uses character-count spacing, not pixel values. The following
column widths govern all formatted output. All values are in character columns.

| Token | Chars | Usage |
|-------|-------|-------|
| indent | 2 | Left indent for all command/keybinding list items |
| cmd-col | 14 | Command name column width (left-padded with spaces to align descriptions) |
| arg-col | 16 | Args hint column width (right of command name, before description) |
| section-gap | 1 blank line | Between major sections in /help output |
| category-gap | 1 blank line | Between command categories in /help |

**Exceptions:** The `/commands` paginated output for gateway uses single-line
entries with no indent (Telegram renders monospace blocks only with backtick
fencing). Page separator is a line of 20 dashes `--------------------`.

---

## Typography (Text Formatting Conventions)

Since output is plain text, "typography" maps to formatting rules.

| Role | Format | Usage |
|------|--------|-------|
| Section heading | ALL CAPS, no decoration | Category names in /help (e.g. `SESSION`, `CONFIGURATION`) |
| Command name | lowercase, `/` prefix | All command references inline (e.g. `/help`, `/model`) |
| Args hint | `<required>` / `[optional]` | Angle brackets = required, square = optional |
| Status label | UPPERCASE word | State indicators (e.g. `RUNNING`, `IDLE`, `TODO`) |
| Body / description | sentence case, no trailing period | All command descriptions in help and status output |

**Line length:** Wrap at 80 characters for CLI output. No wrapping constraint
for gateway (Telegram handles it).

**Monospace block (gateway only):** Wrap multi-line command output in triple
backticks for Telegram rendering. Single-line confirmations are plain text.

---

## Color / ANSI

CLI terminal output may use ANSI codes via the `colored` crate (already a
workspace dependency from Phase 21).

| Role | ANSI | Usage |
|------|------|-------|
| Dominant (60%) | none — terminal default | All body text, command descriptions |
| Secondary (30%) | dimmed | Args hints, category separators, TODO stub notes |
| Accent (10%) | bold | Command names in /help listings only |
| Error | red (bright) | Error messages only (unknown command, ambiguous prefix, guard block) |
| Success / confirmation | green | Positive confirmations only (/new success, /clear success, /stop fired) |
| Warning / TODO stub | yellow | TODO stub responses informing user the feature is not yet implemented |

**Accent reserved for:** Command names (the `/name` portion) in help listings.
Never applied to descriptions, args hints, or body text.

**Gateway constraint:** Telegram does not render ANSI codes. All gateway
`CommandResult` strings must be plain text only — no ANSI escape sequences.
The `CommandResult` formatter must strip ANSI before sending to gateway adapter.

## Copywriting Contract

All strings are sourced from upstream artifacts (CONTEXT.md, RESEARCH.md,
existing code in tui/commands.rs and handler.rs). Nothing re-asked.

### /help Output Structure (CLI)

```
Available commands:

SESSION
  /new           [name]    Start a new session
  /clear                   Clear screen and start a new session
  /stop                    Stop the running agent
  ...

CONFIGURATION
  /model         [provider:model] [--global]    Switch model for this session
  /personality   [name]    Apply a session-level personality preset
  ...

INFO
  /help                    Show this help message
  /status                  Show current session status
  /profile                 Show active profile and HERMES_HOME
  /commands      [page]    List available commands (messaging)

EXIT
  /quit                    Exit the CLI

Keybindings:
  {key:<12}  {description}  ({context})
```

**Source:** Existing `format_help()` pattern in tui/commands.rs (header
`"Available commands:\n"`, 2-space indent, 2-space gap between columns).
Extended with category grouping (RESEARCH.md Pattern 1: CommandCategory).

### /help Output Structure (Gateway — /commands)

```
Commands (page 1/3):

/new — Start a new session
/stop — Stop the running agent
/status — Show current session status
/help — Show this help message

Use /commands 2 for next page
--------------------
```

**Source:** RESEARCH.md command inventory, hermes-agent gateway pattern.
Page size: 10 commands per page.

## Interaction States

These govern what the router returns in each command-dispatch scenario.

| Scenario | Behavior | Output |
|----------|----------|--------|
| Exact match, platform OK | Dispatch to handler | Handler's CommandResult |
| Alias match, platform OK | Dispatch to handler (same as exact) | Handler's CommandResult |
| Unique prefix match, platform OK | Dispatch to handler | Handler's CommandResult |
| Ambiguous prefix | Return error, list all candidates | Error string (red CLI / plain gateway) |
| No match, no skill | Return error | Unknown command error string |
| No match, skill found | Return skill invocation result | Skill output |
| Command exists, wrong platform | Return platform-unavailable error | Platform error string |
| Command exists, agent running, no bypass | Return guard error | Running-guard error string |
| Command exists, agent running, bypass list | Dispatch immediately | Handler's CommandResult |
| TODO stub command | Return stub message | Yellow stub string (CLI) / plain (gateway) |

**Running-agent bypass list (locked from RESEARCH.md Pattern 3):**
`stop`, `new`, `reset`, `status`, `queue`, `q`
