   - `Quit` -- exit the application (CLI only)
   - `ClearSession` -- clear session history
   - `NewSession { message: String }` -- start new session, display message
   - `PassThrough` -- not a command; pass input to agent as normal message

6. **CommandRouter struct** with fields:
   - `commands: Vec<CommandDef>`
   - `by_name: HashMap<&'static str, usize>` -- index into commands
   - `by_alias: HashMap<&'static str, usize>` -- alias -> index

   Methods:
   - `CommandRouter::new(commands: Vec<CommandDef>) -> Self` -- builds name and alias lookup maps. Panics on duplicate name or alias (programming error in registry).
   - `pub fn resolve<'a>(&'a self, input: &str, platform: &Platform) -> ResolveResult<'a>` -- three-stage resolution per RESEARCH.md Pattern 2:
     1. Extract command name: `input.trim_start_matches('/').split_whitespace().next().unwrap_or("").to_lowercase()`
     2. Stage 1 (exact): `by_name.get(name)` -> if found and `is_available_on(platform)` -> `Exact`
     3. Stage 2 (alias): `by_alias.get(name)` -> if found and `is_available_on(platform)` -> `Exact`
     4. Stage 3 (prefix): filter `commands` where `is_available_on(platform)` AND (`name.starts_with(&input_name)` OR any alias `starts_with(&input_name)`). If 1 candidate -> `PrefixMatch`. If 2+ candidates -> check shortest-match preference (shortest name length wins if unique). If still ambiguous -> `Ambiguous(sorted candidate names)`. If 0 -> `NotFound`.
   - `pub fn commands_for_platform(&self, platform: &Platform) -> Vec<&CommandDef>` -- all commands available on platform, ordered by category then name.
   - `pub fn commands_by_category(&self, platform: &Platform) -> Vec<(CommandCategory, Vec<&CommandDef>)>` -- grouped by category, for /help output.

   Re-export: `pub mod registry; pub mod context; pub mod handlers;`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> resolve_command | Untrusted slash command string from CLI readline or gateway message |
| command handler -> system state | Handler output may trigger session mutations (clear, new, quit) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21.1-01 | Tampering | resolve_command input | mitigate | Lowercase normalize input, trim whitespace, split on whitespace to extract command name. No shell injection possible -- input is matched against static string literals. |
| T-21.1-02 | Information Disclosure | CommandContext | mitigate | CommandContext holds Arc references only, no raw secrets. Config access is read-only. No API keys exposed in command output. |
| T-21.1-03 | Denial of Service | prefix matching | accept | Linear scan over ~45 commands is O(n) with n<50. No regex or unbounded allocation. Acceptable. |
| T-21.1-04 | Elevation of Privilege | platform filtering bypass | mitigate | Platform is set by the caller (CLI/gateway), not by user input. resolve_command filters by platform before returning. Gateway-only commands unreachable from CLI. |
| T-21.1-05 | Spoofing | agent_running guard | mitigate | AtomicBool set by the agent loop, not by user input. Guard check happens before dispatch. Cannot be spoofed via command input. |
