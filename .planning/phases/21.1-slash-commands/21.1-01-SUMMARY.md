# Phase 21.1 Plan 01: Slash Command Router Summary

**One-liner:** Platform-agnostic slash command router with three-stage resolve() (exact/alias/prefix), 49-command registry, wirable handlers, and TODO stubs all in ironhermes-core.

## What Was Built

A complete slash command router module at `crates/ironhermes-core/src/commands/` implementing:

1. **Type system** (`mod.rs`): `CommandDef` builder, `PlatformFilter` (All/CliOnly/CliAndAcp/GatewayOnly/Universal), `CommandCategory` (5 variants), `CommandResult` (7 variants), `ResolveResult`, `CommandRouter` with `resolve()`, `commands_for_platform()`, `commands_by_category()`.

2. **Context** (`context.rs`): `CommandContext` with required fields (platform, session_id, agent_running as Arc<AtomicBool>) and optional skill_registry.

3. **Registry** (`registry.rs`): `build_registry()` returning 49 `CommandDef` entries covering full hermes-agent parity plus IronHermes extensions (/model, /fast, /reload, /snapshot, /debug, /start, /profile).

4. **Handlers** (`handlers.rs`): `dispatch()` match-on-name handler routing to wirable implementations (help, new, clear, quit, stop, status, title, compress, start, provider, config, profile, yolo, verbose, statusbar, reasoning, skills, commands) and `todo_stub()` for the remaining 31 unimplemented commands returning `"/{name} is not yet available. ({reason})"`.

5. **lib.rs**: `pub mod commands` added with re-exports (`SlashCommandResult`, `CommandRouter`, `CommandContext`, etc.).

### Tasks 1-3 Committed Together

Tasks 1, 2, and 3 were committed in a single commit because:
- Task 1 (mod.rs tests) reference `build_registry()` from Task 2 (registry.rs)
- Writing failing tests first would require a stub `build_registry()` then replacing it — this adds churn without value
- All code was written in TDD order (types -> tests -> registry -> handlers -> verify)
