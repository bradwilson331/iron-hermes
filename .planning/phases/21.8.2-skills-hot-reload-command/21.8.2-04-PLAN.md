pub fn submit(&mut self) {
    // ... elided: reads textarea, slash-precheck, pushes user message ...
    self.history_store.push(text.clone());
    self.history_store.reset_cursor();
    self.history.push(user_message(text));

    let (tx, rx) = tokio::sync::mpsc::unbounded_channel::<StreamEvent>();

The agent-turn re-engagement is `self.auto_follow = true;` only — it does NOT also reset `transcript_scroll`. This is because `reconcile_scroll` (called next render tick from `ui.rs`) clamps `transcript_scroll` to `max` whenever `auto_follow == true`. So setting `auto_follow = true` alone is sufficient at the slash-outcome layer too. We additionally set `transcript_scroll = 0` for symmetry with `scroll_to_top` (which sets both fields) — the next-tick `reconcile_scroll` will overwrite it with `max` anyway.
