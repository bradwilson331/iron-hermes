## Decisions Made

1. **Wider scope than minimum G-01 fix (intentional, per plan).** UAT G-01 only flagged SkillsReload + SkillActivated. The plan correctly identified that Handled, ClearSession, Unknown, and Error share the exact same root-cause class — they all `history.push(...)` without re-engaging `auto_follow`. Fixed all 6 in one pass; this is a UX upgrade for `/help`, `/clear`, unknown commands, and errors as a side benefit.
2. **Helper sets both `auto_follow=true` and `transcript_scroll=0`.** `submit()` at app.rs:718 sets only `auto_follow=true` (relying on `reconcile_scroll` to clamp `transcript_scroll` to `max` next tick because `auto_follow==true`). The new helper additionally sets `transcript_scroll=0` for symmetry with `scroll_to_top` and so the test assertions can be explicit about both fields. The `transcript_scroll=0` write is harmless because `reconcile_scroll` overwrites it with `max` on the next render tick.
3. **`pub(super)` widening kept unconditional.** Putting it behind `#[cfg(test)]` would require `#[cfg_attr(test, …)]` gymnastics and break IDE tooling for the test module. Since `pub(super)` only widens visibility within the `tui_rata` parent mod (still crate-private to outside callers), the change is benign.
4. **Tests assert on `role` + `history.len()`, not on `.content`.** `ChatMessage.content` is `Option<MessageContent>`, not `String` — calling `.content.contains(...)` would not compile. The actual subject under test is the scroll-state mutation, not the message body, so the chosen assertions are sufficient and stable.
