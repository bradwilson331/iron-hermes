## What Was Built

### Task 1: Normalization in parse_skill_md (D-10/D-11/Pitfall 4)

Inserted a normalization block in `parse_skill_md` (skills.rs, before `validate_skill_name`):

```rust
let normalized_name = frontmatter
    .name
    .to_lowercase()
    .replace(' ', "-")
    .split('-')
    .filter(|s| !s.is_empty())
    .collect::<Vec<_>>()
    .join("-");
let mut frontmatter = frontmatter;
frontmatter.name = normalized_name;
```

This handles:
- Title Case: `Command Development` → `command-development`
- Spaces: `my skill name` → `my-skill-name`
- Consecutive spaces (Pitfall 4): `Code  Review` → `code-review` (NOT `code--review`)
- Mixed hyphen+case: `MCP-Integration` → `mcp-integration`
- Already-valid names: `ascii-art` → `ascii-art` (no-op)

### Task 2: D-12 Warn + extract_raw_name_from_yaml

Added private helper `extract_raw_name_from_yaml(content: &str) -> Option<String>` that peeks the `name:` field from YAML frontmatter without full deserialization. Added D-12 warn in `try_register_skill_from_dir` immediately after `parse_skill_md` returns:

```rust
if let Some(raw_name) = extract_raw_name_from_yaml(&content) {
    if raw_name != frontmatter.name {
        warn!(
            "SkillRegistry: normalized skill name {:?} -> {:?} at path {:?}",
            raw_name, frontmatter.name, skill_md_path
        );
    }
}
```
