   use std::path::{Path, PathBuf};
   use ironhermes_core::{scan_context_content, truncate_content, CONTEXT_FILE_MAX_CHARS};

2. `pub struct SubdirDiscovery` with a single field: `visited: HashSet<PathBuf>`.

3. `impl SubdirDiscovery`:
   - `pub fn new() -> Self` — empty HashSet
   - `pub fn check_path(&mut self, file_path: &Path) -> Option<String>`:
     - Determine start_dir: if file_path is a directory, use it; otherwise use file_path.parent().
     - Canonicalize start_dir with `std::fs::canonicalize()`, falling back to the original path if canonicalization fails (per Pitfall 1 from RESEARCH.md).
     - Walk upward from start_dir, checking up to 5 parent directories (depth counter starts at 0, increments on each parent() call, breaks at depth >= 5). Per D-06, CTX-04.
     - For each directory: if already in `self.visited`, `break` (already checked this dir and all above it). Insert canonicalized dir into visited set.
     - Check CONTEXT_CANDIDATES in order (`.hermes.md`, `AGENTS.md`, `CLAUDE.md`, `.cursorrules`). First match wins per D-07.
     - On match: read file content. If empty (trim), skip and continue to next candidate.
     - If filename is `.hermes.md`, call `strip_yaml_frontmatter(&content)` first.
     - Then `scan_context_content(content, filename)`.
     - Then `truncate_content(&scanned, filename, CONTEXT_FILE_MAX_CHARS)`.
     - Format result: `"\n\n[Context: {dir_display}/{filename}]\n{truncated}"`.
     - Return `Some(formatted)`. Only inject one context file per check_path call (first match in first unvisited dir wins).
     - If no match found after walking all parents, return None.
   - The walk stops early if the current directory is already in visited (meaning we've checked it and all its parents in a prior call).

   const FILE_ACCESS_TOOLS: &[&str] = &["read_file", "write_file", "patch", "search_files"];
   if FILE_ACCESS_TOOLS.contains(&name.as_str()) {

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| subdirectory filesystem -> tool result | Untrusted context files from subdirectories are appended to tool results visible to the LLM |
| tool argument (path) -> filesystem access | Tool path argument controls which directory is scanned for context files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-06 | Tampering | Subdirectory .hermes.md injection | mitigate | `scan_context_content()` applied to all discovered files before appending to tool result |
| T-14-07 | Tampering | Re-scan of same directory | mitigate | `HashSet<PathBuf>` with canonicalized paths prevents duplicate injection; each dir checked at most once per session |
| T-14-08 | Denial of Service | Deep directory tree walk | mitigate | Depth capped at 5 parent directories per D-06/CTX-04 |
| T-14-09 | Tampering | Non-file tool triggering discovery | accept | FILE_ACCESS_TOOLS allowlist restricts to read_file/write_file/patch/search_files per D-05; other tools cannot trigger discovery |
| T-14-10 | Information Disclosure | Context from unexpected directories | mitigate | Walk direction is upward only (toward project root), not downward; canonicalization prevents path traversal tricks |
</threat_model>
