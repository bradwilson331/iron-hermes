# Phase 22.4.1 Plan 00: Core Registry — Register mouse / mcp / sessions / memory + INV-22.4-32 Summary

Promoted the four `strip_prefix` fast-path commands (`/mouse`, `/mcp`, `/sessions`, `/memory`) into the canonical `ironhermes-core` `CommandRouter` registry so `app.command_router.resolve(...)` now returns `ResolveResult::Exact` for each, and locked the change with INV-22.4-32 (`router_membership`) — a pure source-grep regression test that fails if any of the four names is silently removed.

## Behavior-Neutrality Note

tui_rata user-visible behavior is unchanged because the four `strip_prefix` fast-paths in `dispatch_slash` (added by Plans 22.4-16 and 22.4-18) still fire BEFORE the router. They are scheduled for removal by Plan 22.4.1-01. With this plan landed, the router resolves the four names as `ResolveResult::Exact` so 22.4.1-01 can safely retire the fast-paths without losing handler coverage.
