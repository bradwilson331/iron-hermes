### Task 1: CommandContext extension (context.rs)

Added 8 new trait objects to `CommandContext`, all `Option<Arc<dyn ...>>` defaulted to `None`:

| Field | Trait | Used by |
|-------|-------|---------|
| `mcp_manager` | `McpManagerHandle` | `/mcp` server enumeration |
| `memory_manager` | `MemoryManagerHandle` | `/memory` status |
| `state_store` | `StateStoreHandle` | `/sessions` `/resume` `/save` `/history` `/title` |

| `context_compressor` | `ContextCompressorHandle` | `/compress` `/rollback` |
| `personality_overlay` | `PersonalityHandle` | `/personality` |
| `history` | `Arc<RwLock<Vec<ChatMessage>>>` | `/history` `/retry` `/undo` |
| `agent_loop` | `AgentLoopHandle` | Tier D session control |

Eight corresponding `with_<name>()` builder methods added.

**Type corrections from CONTEXT.md applied:**

- `history` uses snapshot clone pattern — App.history stays `Vec<ChatMessage>`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] StateStore wrapper type correction**
- **Found during:** Task 1 (RESEARCH OQ-1)
- **Issue:** CONTEXT.md D-04 proposed `Arc<RwLock<StateStore>>` but all existing call sites (AgentLoop, gateway) use `Arc<std::sync::Mutex<StateStore>>` (std, not tokio). Using RwLock would create an incompatible wrapper type.
- **Fix:** Used `Arc<std::sync::Mutex<StateStore>>` in CommandContext StateStoreHandle trait and App field.
- **Files modified:** context.rs, app.rs

**2. [Rule 1 - Bug] PersonalityOverlay type name correction**
- **Found during:** Task 1 (RESEARCH OQ-2)
- **Issue:** `PersonalityOverlay` does not exist in the workspace. The actual type is `PersonalityRegistry` in `ironhermes_agent::personality`.
- **Fix:** Used `PersonalityRegistry` as the concrete type. Field name `personality_overlay` preserved for naming consistency.
- **Files modified:** app.rs

**3. [Rule 1 - Bug] ProviderResolver not in AppDeps**
- **Found during:** Task 2 (RESEARCH OQ-3)
- **Issue:** Plan 00 adds `provider_resolver` to CommandContext but `build_command_context` reads it from `app.resolver` — which did not exist. ProviderResolver was built locally in `build_app_deps` and discarded.
- **Fix:** Added `resolver: ProviderResolver` to AppDeps and App. Propagated from the existing `let resolver = ProviderResolver::build(&config)?` in build_app_deps.
- **Files modified:** app.rs, event_loop.rs

**4. [Rule 2 - Missing functionality] Trait objects required for cross-crate types**
- **Found during:** Task 1
- **Issue:** `ironhermes-core` has no dependency on `ironhermes-state`, `ironhermes-agent`, or `ironhermes-mcp`. Adding direct type references would create circular dependencies. CONTEXT.md D-04 assumed direct types could be used.
- **Fix:** Defined 8 new handle traits in context.rs. All new CommandContext fields use `dyn Trait` objects, following the established `McpReloader`, `SubagentListSnapshot`, `ProcessRegistrySnapshotHandle` pattern.
- **Files modified:** context.rs, commands.rs

**5. [Rule 1 - Bug] StateStoreAdapter list_sessions API correction**
- **Found during:** Task 2 compile
- **Issue:** `StateStore::list_sessions` takes 2 args (source, limit), not 3 (source, limit, offset) as assumed from RESEARCH.
- **Fix:** Updated call to `guard.list_sessions(None, limit)`.
- **Files modified:** commands.rs
