## Implementation Decisions

### Architecture: dispatch routing

- **D-01:** **Delegate `tui_rata::invoke_handler` to `core::handlers::dispatch`.** The 30-arm match table from Phase 22.4.1 collapses to:
  ```rust
  async fn invoke_handler(
      name: &str,
      ctx: &CommandContext,
      router: &CommandRouter,
      args: &[&str],
  ) -> Result<CommandResult, anyhow::Error> {
      let def = router
          .find(name)
          .ok_or_else(|| anyhow::anyhow!("unknown command: {name}"))?;
      Ok(ironhermes_core::commands::handlers::dispatch(def, args, ctx, router))
  }
  ```
  Single source of truth across gateway + classic-tui + tui_rata. The 28 wire-up arms from 22.4.1 D-08 are deleted. The standalone helper functions `handle_mcp_slash` / `handle_sessions_slash` / `handle_memory_slash` (already deleted in 22.4.1) stay deleted; `handle_mouse_slash` survives because it is invoked from the post-router hook (D-02 below).

- **D-02:** **Generalize the `/mouse` post-router hook (D-10/D-11/D-12 from 22.4.1) to multi-name.** `dispatch_slash`'s post-router branch on `def.name` expands from 1 case (`mouse`) to ~13 cases via four helper functions for diff legibility:
  ```rust
  match def.name {
      // Toggles (existing /mouse + new /yolo /verbose /statusbar /skin /fast /debug)
      "mouse" | "yolo" | "verbose" | "statusbar" | "skin" | "fast" | "debug"
          => handle_toggle(app, def.name, args),
      // App-handle inspectors (read mcp_manager / memory_manager / history / state_store handles)
      "memory" | "mcp" | "sessions" | "history"
          => handle_app_inspector(app, def.name, args).await,
      // Tier D session control (cancel_child token, AgentLoop integration, message manipulation)
      "stop" | "retry" | "undo" | "rollback" | "background" | "btw" | "queue"
          => handle_session_control(app, def.name, args).await,
      // Subsystem mutators
      "model" | "personality" | "compress"
          => handle_subsystem_mutator(app, def.name, args).await,
      // Default: trust core dispatch result
      _ => map_core_to_slash_outcome(result),
  }
  ```
  Args extraction continues to use `input.strip_prefix(&format!("/{}", def.name))` (D-11 from 22.4.1) so INV-22.4-34 grep stays clean. Whether to use the four-helper split or one big match is Claude's Discretion (planner picks; INV grep is insensitive).

### Architecture: where do real handler bodies live

- **D-03:** **All real handler bodies in `core::commands::handlers`.** For the ~15 commands currently returning `todo_stub("…")` (memory, mcp, sessions, history, personality, model, fast, debug, skin, resume, save, retry, undo, rollback, background, btw, queue), write real handler bodies in `crates/ironhermes-core/src/commands/handlers.rs`. Bodies read shared handles from `CommandContext` — same pattern as `cmd_skills` (reads `ctx.skill_registry`), `cmd_stop` (reads `ctx.process_registry`), `cmd_agents` (reads `ctx.subagent_registry`), `cmd_reload_mcp` (reads `ctx.mcp_reloader`). Gateway gains all wired commands automatically as a side-effect (handler.rs:246 already calls `core::handlers::dispatch`). App-state mutators (toggle bools, App field writes, ratatui state) stay in tui_rata's post-router hook because they touch ratatui-only state that doesn't belong in core.

- **D-04:** **Extend `CommandContext` with handles tui_rata + gateway both hold.** New fields, all `Option<Arc<...>>` and defaulted to None for backwards compat:
  - `mcp_manager: Option<Arc<McpManager>>` — for `/mcp` full server enumeration (separate from existing `mcp_reloader: Option<Arc<dyn McpReloader>>`)
  - `memory_manager: Option<Arc<Mutex<MemoryManager>>>` — for `/memory` inspection
  - `state_store: Option<Arc<RwLock<StateStore>>>` — for `/sessions` `/resume` `/save` `/history` `/title` (Phase 13 SESS-01)
  - `provider_resolver: Option<Arc<ProviderResolver>>` — for `/model` `/provider` `/fast` (Phase 12 PROV-01)
  - `context_compressor: Option<Arc<dyn ContextEngine>>` — for `/compress` `/rollback` (Phase 18 PRMT-11)
  - `personality_overlay: Option<Arc<RwLock<PersonalityOverlay>>>` — for `/personality` (Phase 15 PRMT-06/PRMT-07)
  - `history: Option<Arc<RwLock<Vec<ChatMessage>>>>` — for `/history` `/retry` `/undo`
  - `agent_loop: Option<Arc<AgentLoop>>` — for Tier D session control (`/background` `/btw` `/queue`) spawn paths
  
  Mirrors the existing pattern: `mcp_reloader`, `subagent_registry`, `process_registry`, `skill_registry`, `agent_running`, `platform`, `session_id`. Each new field has a `with_<name>(self, handle: …) -> Self` builder. Existing `CommandContext::new(...)` constructor signature is unchanged.

- **D-05:** **`ctx.<subsystem>.is_some()` guard pattern is mandatory.** Every new handler body that reads a `ctx` handle handles the `None` case gracefully — returns informational `CommandResult::Output("…not configured.")` instead of panicking. Required for backwards-compat with consumers (gateway, classic-tui, future binaries) that don't yet thread the handle. Pattern source: `cmd_stop` (handlers.rs:106-124), `cmd_skills` (421-426), `cmd_reload_mcp` (643-648).
