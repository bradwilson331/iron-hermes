## Accomplishments

### Task 1 (commit 86f6dde): Thread ProcessRegistry + add 6 new core handlers

**ProcessRegistry and SubagentRegistry threaded into `build_command_context`:**

Both `ProcessRegistryHandle` (from `ironhermes-exec`) and `SubagentRegistryHandle` (from `ironhermes-agent`) are now wrapped and installed on the `CommandContext` via `with_process_registry()` and `with_subagent_registry()`. `/stop` now uses the real `ProcessRegistry.drain_and_kill()` path (not the "not yet wired" advisory fallback); `/agents` now uses the real `SubagentRegistry`.

**6 new handler functions in `core::commands::handlers`:**

| Handler | Description | ctx Handle |
|---------|-------------|------------|
| `cmd_retry(args, ctx)` | Finds last User message in `ctx.history` snapshot; signals post-router to re-queue | `ctx.history: Option<Arc<RwLock<Vec<ChatMessage>>>>` |
| `cmd_undo(args, ctx)` | Validates history non-empty; signals post-router to truncate last pair | `ctx.history` |
| `cmd_rollback(args, ctx)` | Parses N (default 1); signals post-router to truncate N pairs | `ctx.history` |
| `cmd_background(args, ctx)` | Guards `ctx.agent_loop`; returns queued-message signal | `ctx.agent_loop: Option<Arc<dyn AgentLoopHandle>>` |
| `cmd_btw(args, ctx)` | Guards `ctx.agent_loop`; returns aside signal | `ctx.agent_loop` |
| `cmd_queue(args, ctx)` | Guards `ctx.agent_loop`; returns queued-message signal | `ctx.agent_loop` |

All 6 wired into `dispatch()` match table. Removed `retry/undo/rollback/background/btw/queue` from `todo_stub`; updated test list accordingly.

### Task 2 (commit e7a4251): Fill handle_session_control bodies

Plan 03's pass-through stub replaced with real per-command bodies:

| Command | App Mutation |
|---------|-------------|
| `/stop` | Pass-through to `map_core_to_slash_outcome` (ProcessRegistry drain happens in core) |
| `/retry` | Pop trailing assistant msgs from `app.history`; set `pending_tx`/`pending_rx` to re-queue last user msg |
| `/undo` | Pop last assistant + last user from `app.history` |
| `/rollback [n]` | Pop N (user, assistant) pairs from `app.history`; report count removed |
| `/background <msg>` | Push `ChatMessage::user(msg)` to `app.history`; set `pending_tx`/`pending_rx` to queue agent spawn |
| `/btw <msg>` | Push `ChatMessage::user("[btw] msg")` to `app.history` (picked up by next spawn_turn) |
| `/queue <msg>` | Pre-populate `app.textarea` with message for user review before submit |

## Deviations from Plan

### Deviation 1: SubagentRegistryHandle also threaded in build_command_context (not just ProcessRegistryHandle)

**Found during:** Task 1
**Issue:** The plan specified threading ProcessRegistry for `/stop`. Reviewing `build_command_context`, `subagent_registry` was also not threaded — `/agents` was working only because `cmd_agents` in core reads `ctx.subagent_registry`, but it was never being populated from App state in the tui_rata path.
**Fix (Rule 2 — missing critical functionality):** Added `SubagentRegistryHandle` wrapping alongside `ProcessRegistryHandle`. Both are already on App; both now flow to CommandContext.
**Files modified:** `crates/ironhermes-cli/src/tui_rata/commands.rs`

### Deviation 2: ChatMessage struct literal avoided; used ChatMessage::user() constructor

**Found during:** Task 2 (build error)
**Issue:** Initial implementation used struct literal `ChatMessage { role: User, content: ..., tool_use_id: None, ... }` which failed because the actual fields are `tool_calls`, `tool_call_id`, `name` (not `tool_use_id`/`tool_use_name`/`tool_input`).
**Fix (Rule 1 — auto-fix bug):** Replaced struct literals with `ChatMessage::user(text)` constructor which is field-signature stable.
**Files modified:** `crates/ironhermes-cli/src/tui_rata/commands.rs`
