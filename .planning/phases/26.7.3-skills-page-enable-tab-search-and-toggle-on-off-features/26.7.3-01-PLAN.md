    ```rust
    /// Phase 26.7.3 D-01/D-03/Pitfall 6 — pure predicate evaluated by the
    /// Skills screen `use_memo` to decide whether a skill should appear under
    /// the active tab. The `enabled` argument is the server-sourced
    /// `SkillInfo.enabled`, NOT the optimistic UI state.
    pub fn tab_predicate(category: &str, enabled: bool, tab: &str) -> bool {
        match tab {
            "bundled" => category == "bundled",
            "installed" => category != "bundled",
            "enabled" => enabled,
            // "all" and any unknown tab → no filtering
            _ => true,
        }
    }

    /// Phase 26.7.3 D-04/D-05 — case-insensitive substring match against the
    /// skill name OR description. Category is intentionally excluded (tabs
    /// already filter on category). Empty query returns true.
    pub fn search_matches(name: &str, description: &str, query: &str) -> bool {
        if query.is_empty() {
            return true;
        }
        let q = query.to_lowercase();
        name.to_lowercase().contains(&q) || description.to_lowercase().contains(&q)
    }
    ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26.7.3-01-01 | Tampering | SkillsConfig YAML parse | accept | The `disabled` field is a `Vec<String>` of skill names. Names that do not match a known skill produce no behavior (cosmetic-only). No filesystem path, command, or interpolation derives from this list in this plan. Plan 02 adds an existence check before mutating `active_skills`. |
| T-26.7.3-01-02 | Denial of Service | serde parse on extremely large `disabled` vec | accept | A user manually crafting a 10MB `disabled` list in their own config has already broken their own machine — same risk surface as any other Vec field on the struct. No mitigation needed for single-operator deployment. |
| T-26.7.3-01-03 | Tampering | search_matches query input | accept | The query is lowercased and substring-matched against name + description; no regex, no eval, no interpolation. Pathological queries (e.g., 10MB string) are bounded by browser input limits and the O(N) substring scan is per-render; same risk surface as any text-input filter. |
| T-26.7.3-01-04 | Tampering | tab_predicate tab input | accept | Tab string is matched against a fixed allowlist (`all` / `bundled` / `installed` / `enabled`); anything else defaults to `true` (show all). No state mutation, no I/O. |
</threat_model>
