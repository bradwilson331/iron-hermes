## Toggle persistence

### Persistence scope

| Option | Description | Selected |
|--------|-------------|----------|
| Session-only | Mutate active_skills in memory; resets on restart | |
| Persist to config + cross-surface | Write disabled: Vec<String> to config YAML; hot-reload in-process | ✓ |

**User's choice:** Persist to config (cross-surface architecture)
**Notes:** User asked about TUI/gateway consistency first: "How is this handled by the gateway and TUI system? Let's keep the system in sync." After investigating, both surfaces have no UI-level toggle today — active_skills is set at startup and never mutated via UI. User decided to implement config-persisted toggle, establishing the pattern for all surfaces. Full cross-surface hot-reload (TUI + gateway) is deferred to a follow-on phase; this phase covers web UI + config write + in-process reload only.
