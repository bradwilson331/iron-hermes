## Decisions Made

- Used `apply_disable_toggle` private fn extraction (DRY + testability) rather than inlining the toggle logic directly in `toggle_skill` — tests can call it without constructing a live `AppState`
- Scoped `{ }` block around the mutex lock ensures the `MutexGuard` drops before `Ok(())` is returned — no `.await` inside
- `was_disabled: bool` return value from `apply_disable_toggle` determines push vs retain direction in the `active_skills` mutation
