## Component Inventory

Components modified or introduced this phase. No new CSS classes are needed тАФ all states are expressed via existing classes and inline styles.

### ScreenSkills (modified)

New signals added (declared before `rsx!` per signal-borrow discipline):

| Signal | Type | Initial value | Purpose |
|--------|------|---------------|---------|
| `active_tab` | `Signal<&'static str>` | `"all"` | Active tab key; drives tab `is-active` class |
| `search_query` | `Signal<String>` | `String::new()` | Live search input value |
| `toggle_errors` | `Signal<HashMap<String, String>>` | empty | Per-skill inline error message map |
| `toggle_states` | `Signal<HashMap<String, bool>>` | populated from `skills_list` | Optimistic enabled state per skill name |

Derived memo:

| Memo | Dependencies | Output |
|------|-------------|--------|
| `filtered_skills` | `active_tab`, `search_query`, `skills_list` | `Vec<SkillInfo>` тАФ AND-combined tab + search filter |

Tab counts are computed from `skills_list` (the unfiltered server response):
- ALL: `skills_list.len()`
- BUNDLED: `skills_list.iter().filter(|s| s.category == "bundled").count()`
- INSTALLED: `skills_list.iter().filter(|s| s.category != "bundled").count()`
- ENABLED: `skills_list.iter().filter(|s| s.enabled).count()` тАФ uses server-sourced `enabled`, not optimistic state
- UPDATES: hardcoded `0`

### SkillCard (modified)

Receives optimistic `enabled` state and error message from parent signals (not from `skill.enabled` directly). Prop additions:

| Prop | Type | Description |
|------|------|-------------|
| `enabled` | `Signal<bool>` | Optimistic toggle state owned by parent `ScreenSkills` |
| `error_msg` | `Option<String>` | Inline error text; `None` = no error |
| `on_toggle` | `EventHandler<()>` | Fires when `.tgl` div is clicked; parent handles spawn |

## Interaction Contracts

### Tab filtering

- Clicking a tab sets `active_tab` signal to that tab's key string: `"all"`, `"bundled"`, `"installed"`, `"enabled"`.
- Active tab button gets class `"tab is-active"`. All others get `"tab"`.
- UPDATES tab: rendered with `class: "tab"`, inline `style: "opacity:0.5; pointer-events:none;"`, and count `"UPDATES ┬╖ 0"`. Not interactive. Source: CONTEXT.md D-02.
- Tab switch does NOT reset `search_query`. Source: CONTEXT.md D-09.

### Search

- The `input` in the `.search` bar binds `oninput` to update `search_query` signal with `event.value()`.
- Filter is client-side, no server round-trip. Source: CONTEXT.md D-04.
- Match: case-insensitive substring on `skill.name` OR `skill.description`. Source: CONTEXT.md D-05.
- Search does NOT reset `active_tab`. Source: CONTEXT.md D-09.
- Placeholder text: `"Search skills, tags, capabilitiesтАж"` тАФ unchanged from current markup.

### Filter composition

- `filtered_skills` memo: first apply tab predicate, then apply search predicate on the tab-filtered result.
- Tab predicate per key:
  - `"all"`: pass all
  - `"bundled"`: `skill.category == "bundled"`
  - `"installed"`: `skill.category != "bundled"`
  - `"enabled"`: `skill.enabled == true` (uses server-sourced value, not optimistic)
- Search predicate: `skill.name.to_lowercase().contains(&query)` OR `skill.description.to_lowercase().contains(&query)` where `query = search_query.to_lowercase()`.
- If `search_query` is empty, search predicate passes all. Source: CONTEXT.md D-04, D-09.

### Toggle (optimistic pattern)

Mirrors `agents.rs` kill/interrupt optimistic pattern. Sequence:

1. User clicks `.tgl` div on a card.
2. `ScreenSkills` captures `current_enabled = toggle_states.read()[&skill.name]`.
3. Flip: `toggle_states.write().insert(skill.name.clone(), !current_enabled)`.
4. Clear any prior error: `toggle_errors.write().remove(&skill.name)`.
5. `spawn(async move { ... })` calls the `toggle_skill(name)` server fn.
6. On `Ok(_)`: no further state change needed (optimistic state is already correct).
7. On `Err(e)`: revert `toggle_states.write().insert(skill.name.clone(), current_enabled)` and set `toggle_errors.write().insert(skill.name.clone(), "Toggle failed тАФ try again.".to_string())`.

Source: CONTEXT.md D-08; agents.rs lines 498тАУ554 pattern.

### Toggle server fn

- `#[server]` attribute (not `#[get]` or `#[post]`). Source: CONTEXT.md Claude's Discretion, Phase 26.7 D-03/D-04 locked pattern for writes.
- Signature: `async fn toggle_skill(name: String) -> Result<(), ServerFnError>`
- Located in: `crates/iron_hermes_ui/src/server/api.rs`
- Steps: (1) `global_app_state()`, (2) read config path, (3) read + mutate `SkillsConfig.disabled: Vec<String>` (add name if currently enabled, remove if disabled), (4) write YAML back to disk, (5) lock `runtime_bundle.active_skills`, (6) recompute from updated config.

Source: CONTEXT.md D-07.

## Accessibility

| Element | Contract |
|---------|---------|
| Tab buttons | Already `<button>` elements тАФ no change needed; active tab should carry `aria-selected="true"` for screen reader parity with `.ih-tabs` pattern in `components.css` |
| UPDATES tab | `disabled` attribute on the `<button>` element тАФ preferred over `pointer-events:none` alone |

| Search input | Existing `<input placeholder="...">` pattern тАФ no change needed |
| Error message | Rendered as a sibling `div` inside the card; no `role="alert"` required (optimistic revert is synchronous) |

Source: `components.css` `.ih-toggle` uses `aria-checked`; project parity with that pattern.
