## Enforcement semantics

### No active skills behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, all allowed | Empty active_skills = no restrictions. Backward compat preserved. | ✓ |
| No, block everything | Require at least one active skill for any tool call | |

**User's choice:** Yes, all allowed

### Multi-skill combination

| Option | Description | Selected |
|--------|-------------|----------|
| Union | Tool allowed if ANY active skill permits it | ✓ |
| Intersection | Tool allowed only if ALL active skills permit it | |

**User's choice:** Union

### Skills without allowed_tools field

| Option | Description | Selected |

**User's choice:** Skip them
