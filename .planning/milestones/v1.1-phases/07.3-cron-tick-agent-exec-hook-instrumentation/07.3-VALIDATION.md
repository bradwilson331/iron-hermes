## Wave 0 Requirements

The following test scaffolding does NOT exist and MUST be created in Wave 0 before any implementation tasks run:

- [ ] **`test_placeholder_string_absent`** — `#[test]` in `crates/ironhermes-gateway/src/runner.rs` `#[cfg(test)]` module. Reads `runner.rs` source via `include_str!("runner.rs")` and asserts the placeholder string `"[Tick runner: agent execution pending full integration]"` is absent. (D-17 grep-verifiable criterion.)
- [ ] **`test_cron_hook_registry_receives_events`** — `#[tokio::test]` in same module. Constructs an `Arc<HookRegistry>` with a capture listener (`Arc<Mutex<Vec<HookEvent>>>` pattern from `registry.rs` tests), invokes the per-job execution helper directly (does NOT spin up a 60s timer or real LLM), asserts captured events include `MessageReceived` and `ResponseSent` with `platform: "cron"` and a non-empty `chat_id` derived from the Job record. (D-16 minimum.)
- [ ] **`test_complete_job_run_receives_real_output`** — `#[tokio::test]` that wraps a fake `JobStore`, runs one tick iteration with a stubbed agent path, and asserts the `output` argument passed to `complete_job_run` is NOT the placeholder string. (D-14 + SCHED-04.)
- [ ] **`test_cron_skill_reaches_llm`** — `#[tokio::test]` `#[ignore]` gated on `IRONHERMES_TEST_LLM=1`. Creates a scheduled job with a known skill attached (skill content instructs the agent to emit a specific token), runs one tick iteration end-to-end against a real LLM endpoint, asserts the stored output contains content derived from the skill. (D-15 / SCHED-03.)
- [ ] **Per-job execution helper extraction** — Extract the per-job execution body of the tick closure into `pub(crate) async fn execute_cron_job(...)` (or equivalent name) in `runner.rs`. The four Wave 0 tests above call this helper directly; the tick closure also calls it. (Open Question #1 in RESEARCH.md — recommendation accepted.)

*No new test framework install required — Rust test harness is built in. No new files required — all tests live in `runner.rs` `#[cfg(test)]` module.*
