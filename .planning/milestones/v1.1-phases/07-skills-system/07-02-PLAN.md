Create `crates/ironhermes-tools/src/skills_tool.rs`:

1. **SkillsTool** struct:
   - `registry: Arc<SkillRegistry>` (NOT Arc<Mutex> — registry is read-only per research)
   - `pub fn new(registry: Arc<SkillRegistry>) -> Self`

2. **Tool trait impl** (mirror CronjobTool exactly):
   - `fn name(&self) -> &str` returns `"skills"`
   - `fn toolset(&self) -> &str` returns `"skills"`
   - `fn description(&self) -> &str` returns `"Browse and activate skill documents. Actions: list, view, activate."`
   - `fn schema(&self) -> ToolSchema` returns JSON schema with:
     - `action`: string enum ["list", "view", "activate"], required
     - `name`: string, description "Skill name. Required for view and activate."
   - `async fn execute(&self, args: Value) -> anyhow::Result<String>`:
     - Extract `action` string from args, error if missing
     - Match on action:
       - `"list"` -> `handle_list`: iterate `self.registry.list()`, build JSON array of `{"name": r.name, "description": r.description}`, return `{"status":"ok","skills":[...],"count":N}`
       - `"view"` -> `handle_view`: extract `name` from args (required), call `self.registry.find(name)`, if found read full SKILL.md from disk via `std::fs::read_to_string(record.path)`, return `{"status":"ok","name":"...","content":"full file content"}`. If not found: `{"status":"error","message":"Skill not found: {name}"}`
       - `"activate"` -> `handle_activate`: extract `name` from args (required), call `self.registry.read_content(name)` (returns body only, no frontmatter per D-07), return `{"status":"ok","name":"...","content":"body only"}`. If not found: `{"status":"error","message":"Skill not found: {name}"}`
       - unknown -> `{"status":"error","message":"Unknown action '{action}'. Valid: list, view, activate"}`
     - Serialize result with `serde_json::to_string(&result)?`

3. Update `crates/ironhermes-tools/src/lib.rs`:
   - Add `pub mod skills_tool;`

   - Add import: `use ironhermes_core::SkillRegistry;`
   - Add method on ToolRegistry:

     ```rust
     /// Register the skills tool with a shared SkillRegistry.

         use crate::skills_tool::SkillsTool;

     }
     ```
