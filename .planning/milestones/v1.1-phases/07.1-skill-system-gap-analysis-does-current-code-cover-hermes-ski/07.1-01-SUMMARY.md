# Phase 07.1 Plan 01: Skills System Gap Analysis Summary

**One-liner:** 14-capability audit against agentskills.io spec and Python hermes-agent (6,136 LOC) found 10 gaps — 4 HIGH, 4 MEDIUM, 2 LOW — with 9 new SKILL-05..SKILL-13 requirements appended to REQUIREMENTS.md.

## What Was Built

This plan executed a full gap analysis of the ironhermes Phase 7 skills system against three sources of truth:
1. The agentskills.io open standard

## Decisions Made

1. **D-09 applied:** Extended frontmatter fields (`metadata.hermes.*`) are treated as "parse leniently" — store but don't enforce hermes-specific behavior as a compliance blocker.
2. **S-effort quick wins identified:** SKILL-05, SKILL-06, SKILL-07, SKILL-08 are all S-effort items that should be bundled into a single Phase 07.2 plan.
3. **Directory depth scanner gap:** Phase 7's scanner only walks one level (`read_dir` + `is_dir`) — category-nested skills like `category/skill-name/SKILL.md` are missed. Not classified as a separate gap; SKILL-08's `extra_paths` is the workaround; noted in GAP-REPORT summary appendix.
4. **`skills_guard.py` can reuse existing ironhermes infrastructure:** `scan_context_content()` already exists in `ironhermes-core/src/security.rs` — SKILL-12 implementation should extend it rather than building from scratch.
