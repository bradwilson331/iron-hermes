## Decisions Made

1. **Length check before regex:** `name.len() < SKILL_NAME_MIN_LEN` runs first — O(1) byte count catches empty string before calling the regex engine.
2. **Consecutive-hyphen guard via `str::contains`:** The `regex` crate's NFA engine does not support negative lookaheads, so `--` detection is done separately with `name.contains("--")`. This is checked after length but before the main regex call so the regex is never reached for names with `--`.
3. **`chars().count()` for description length:** The spec says "1-1024 chars" which means Unicode scalar values, not bytes. `.chars().count()` is correct; `.len()` would incorrectly count multi-byte UTF-8 sequences.
4. **Dir-name-match before platform filter:** Placed before `skill_matches_current_platform` so a warn is emitted even for skills that will subsequently be filtered. This gives useful debugging output when a user's macOS-only skill in a misnamed directory doesn't load on other OSes.
5. **Regex source:** `regex` crate already in `ironhermes-core/Cargo.toml` — no new dep added.
