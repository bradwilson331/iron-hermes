# Phase 07.2 Plan 02: SKILL-05 Platforms Filter Summary

**One-liner:** Added `platforms: Option<Vec<String>>` to `SkillFrontmatter` and `SkillRecord` with strict no-alias D-05 platform matching (`skill_matches_current_platform`) wired into `load_with_paths` BEFORE the `seen_names` dedup check (D-06), with 10 new tests covering all spec behaviors.

## What Was Built

Extended `crates/ironhermes-core/src/skills.rs` to implement the `platforms` frontmatter field per SKILL-05. Skills can now declare which OSes they support; skills listing only non-matching platforms are silently skipped at discovery time without reserving their name slot.

## D-06 Ordering — Exact Placement Confirmation

The filter was inserted between `parse_skill_md` and the `seen_names` check. The execution order in `load_with_paths` is:

```
parse_skill_md(&content)          ← parse YAML
  → platform filter (SKILL-05)   ← NEW: skips non-matching skills
    → seen_names dedup check      ← dedup (name slot NOT reserved for filtered skills)
      → seen_names.insert(...)    ← register name
        → skills.push(SkillRecord { platforms, ... })  ← store with platforms mirrored
```

This pins D-06: a filtered-out skill in `path_first` (e.g. `platforms: ["nonexistent-os"]`) does NOT insert into `seen_names`, allowing a same-named skill in `path_second` with no platforms restriction to load successfully.

## D-05 Compliance Confirmation

The `skill_matches_current_platform` function contains exactly three match arms — no alias strings:

```rust
"macos"   => cfg!(target_os = "macos"),
"linux"   => cfg!(target_os = "linux"),
"windows" => cfg!(target_os = "windows"),
_ => false,  // unknown / alias → non-match
```

`grep '"darwin"\|"osx"\|"win32"' crates/ironhermes-core/src/skills.rs` returns matches only in the test code (lines 665–670, where `"darwin"` appears as test input to verify it is REJECTED). No aliases appear in the helper function.

## Decisions Made

1. **Field placement:** `platforms` inserted BEFORE `compatibility` in `SkillFrontmatter` to group agentskills.io-standard fields together (per plan spec shape).
2. **Empty list = load:** `Some(vec![])` returns `true` — treats empty list as "no restriction" matching the spec default, avoiding surprising behavior where an accidentally-empty list silences a skill on every OS.
3. **`debug!` not `warn!`:** Platform-filtered skips log at debug level — this is spec-normal operation, not a validation failure, so warn-level would be misleading.
4. **`serde(default)` on platforms:** Ensures all existing SKILL.md files without the `platforms` key continue to parse without error — backward compat with `Option<T> + serde(default)`.
